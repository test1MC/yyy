Flare Project: New File Checklist

Whenever Flare adds support for a new file type, verify that the new file type passes every item in this checklist.

 

  • Add File dialog
    • First, use CTRL-T to open the Add File dialog and verify the new file type appears in the file type drop-down
    • In the Add File dialog, if you choose New From Existing and click the browse button, the Open File dialog offers the new file type's correct file extension(s).
    • Bottom section of Add File dialog updates to display new file type's name
    • If this is a Project file (not a Content file), the file is added to the appropriate Project folder (most new files will go in the Advanced folder; new import files in Imports)
  • Add File Menus (if necessary; not necessary if new file type is a new type of a file we already support, ie image or multimedia file type)
    • New file type is added to !Project | New (Ribbon interface)
    • New file type is added to !Project menu (Toolstrip interface)
    • Content files: New file type is added to Content Explorer right-click menu: !New | [File Type]
    • Project files: New file type is added to Project Organizer, Project folder right-click menu: !Add | [File Type]
    • Project files: New file type is added to Project Organizer subfolder right-click menu (most new files will go in Advanced): Add [File Type]
  • Templates
    •  In most cases, we must provide a template. (The only reason we would NOT provide a template would be if the new file type falls into an existing category such as images or multimedia) If we do, verify new files can be added to a Flare project when Flare is run in all localized UI languages.
      • English
      • German
      • French
      • Japanese
    • Verify that the new file type can be saved as a template (!File | Save as Template)
    • Verify that the new file type can be added to an existing template folder via the Template Manager. In the Template Manager dialog, click the Add Template File button. Verify the extension for the new file type appears in the Template Files filter.
  • Condition Tag can be applied to the file
    • If the file is a MadCap proprietary file (fltar, flglo, etc), we should be able to write the condition to the file
    • If the file is not a MadCap proprietary file (css, htm, png, swv, etc), we may or may not be able to write the condition to the file. In cases like image or multimedia files, we must write the condition to a props file.
  • File Tag can be applied to the file
    • If the file is a MadCap proprietary file (fltar, flglo, etc), we should be able to write the File Tag to the file
    • If the file is not a MadCap proprietary file (css, htm, png, swv, etc), we may or may not be able to write the File Tag to the file. In cases like image or multimedia files, we must write the File Tag to a props file.
  • Import Flare Project: Add a Flare Project Import file. For "Include Files" and "Exclude Files" click the Edit... button. In the Import File Filter dialog, click Add...
    • In the Import File Filter Designer, the new file type appears and can be added
    • Flare project imports with the new file type work
    • Flare project imports using conditions work with the new file type
  • Content Files Only
    • File extension added to the file filter in the Content Explorer
    • File extension added to the file filter in the File List
    • File extension added to the file filter in the Pending Check-Ins pane
  • External Resource: Place new file type in a designated External Resources folder
    • In the External Resources pane, the file appears and can be copied to project. Verify the file is copied to the proper location in the project (Content versus specific Project folder)
  • Scanning - Dependencies/Reports
    • Basic dependencies: What files link to the new file type? To which files does the new file type link?
      • If new file type links to other files, does new file type update when linked files are moved or renamed
      • If linked to, do files update when new file type is moved or renamed
    • Link Viewer
      • Displays Linked From and Linked To for this file.
      • This file displays in the Linked From and Linked To when appropriate
      • Associated Targets display as appropriate
    • Does the new file type appear in appropriate reports? (Go through all reports and consider if the file is an appropriate candidate.)
      • Condition reports (Used Condition Tags, Files with Conditions, if a Condition is used solely on the new file type, make sure that condition tag does not appear in the Unused Condition Tags report)
      • File Tag reports (Used File Tags, Files with File Tags, etc as above)
      • Other reports (Used Variables if new file type supports variables, etc)
  • Analyzer
    • Is this file allowed to be opened in Analyzer?
      • If yes, test that all UI works in Analyzer
      • If no, verify that Analyzer offers to open file in Flare
    • File extension appears in the file filter in Analyzer's File List (Content or Project file)
