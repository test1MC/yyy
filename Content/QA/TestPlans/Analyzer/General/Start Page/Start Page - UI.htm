Start Page - UI

Opening the Start Page

The Start Page should be open and docked in the MDI using Flare's default layout. If the Start Page is closed or not currently in focus, it can be reopened or given focus by:

!View | Start Page

Click the Open Start Page button in the Standard Toolbar

Start Page - UI Test Cases

By default, the Start Page will appear open in the MDI. It is still, essentially, a pane that can be floated and docked anywhere. Right-click the Start Page tab and select Floating to dock it somewhere else and test the following:

Open an Existing Project

 

Open an Existing Project: Lists recently opened Flare projects. Clicking on any listed project will open that project in the current instance of Flare

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • Files are listed with the Flare or Blaze icon, “FileName” then “Path”
  • Only Flare or Blaze project files should appear in this list
  • Tooltips with full paths will display when hovering over the links
  • Clicking the link will open the project and turn the Title bar orange for a few seconds.
  • The most recently used project should be listed on top
  • Projects removed from !File | Recent Project | Manage Recent Projects will be removed from the list
  • If a project is clicked that no longer exists, Flare will present an error.

Getting Started

Getting Started: Lists recently opened Flare projects. Clicking on any listed project will open that project in the current instance of Flare 

 

UI Controls (Tooltips in parentheses):

  • Launch the Getting Started Wizard (Launch the Getting Started Wizard): Launches the Getting Started Wizard. Test that the Wizard is functional.
  • View the Getting Started Guide (Open the Getting Started Guide PDF file (in Adobe Acrobat)): Launches the PDF. Verify that it’s the correct one.
  • Create a New Project (Start the Create New Project Wizard): Opens the New Project Wizard. Test that the Wizard is functional.
  • Browse the Help System (Open the Help System TOC Pane): Opens the Help’s TOC embedded on the left pane. Test that you can click and open help topics.
  • Search Help System (Open the Help System Search Pane): Opens the Help’s Search embedded on the left pane. Test that you can search and open help topics.
  • Browse Video Tutorials (Browse available video tutorials): Opens the “Quick Start Tutorials” Help topic in the MDI. Test that the links open the correct movies.
  • Visit the Help Community (Point the web browser to the MadCap Help Community): Opens the Flare Forums in a new browser window.

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • Icon is correct
  • Tooltips with full paths will display when hovering over the links
  • Clicking the link will open the project and turn the Title bar orange for a few seconds.

News and Updates

UI Controls

  • Welcome: This should contain a brief blurb and a list of new features. Clicking the text should launch our homepage in a new window.
  • MadCap Events: Should contain a blurb about future events. Clicking the text should launch our ‘Events’ page in a new window.
  • Technical Support: Should have a blurb about our Technical Support and other support options. Clicking the text should launch our ‘Support’ page in a new window. – Verify that it’s the correct product.
  • Feature Requests: Should have blurb about bugs and feature requests. Clicking the text should launch our ‘Bug Submission’ page in a new window.

Test Cases

  • The correct pages open for all sections
  • The correct product and version should be populated in the fields for the bug reporting link (including "Beta" or "Early Experience").
  • If there is no Internet access, the News and Updates Section should display a message indicating such.
