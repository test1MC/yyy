Setting up a 3rd party SMTP service on Windows Vista

To send mail via Outlook Express:

  1. Image a Vista machine
  2. Install and launch the First Mail Server application from \\dc2\Software\First Mail Server
    Note: DO NOT REGISTER First Mail Server. It should offer a 30-day trial. This is a specific 3rd party SMTP service. Others should work as well.
  3. Open Outlook Express
  4. Create an account using Localhost as the SMTP server. Put anything in the POP3 field as that doesn’t concern us for Sending mail. All other fields can be junk (ie bob@adfasd.com), even the password.
  5. Create mail and send. Should send it just fine. If you are on an internal machine, do not send to a madcapsoftware.com email address, as it won’t work. Any external email will work fine.
  6. You may receive an error if you click the send/receive button. Receive is not set up for this example. Just try to execute a Send command and see if the error persists.

 

For sending mail via Feedback Server:

  1. Image a Vista machine. Add it to the domain (necessary for Feedback, not SMTP)
  2. Install and launch the First Mail Server application from \\dc2\Software\First Mail Server
    Note: DO NOT REGISTER First Mail Server. It should offer a 30-day trial. This is a specific 3rd party SMTP service. Others should work as well. FreeSMTP is also in the software folder but limits you to 10 emails per day.
  3. Install Feedback Server
  4. Launch Feedback Server (This assumes you met all the prereq’s for Feedback Server installation)
  5. Install the Feedback Service. Local or remote SQL is ok.
  6. When the service is installed, the admin UI should launch.
  7. In the admin, under the SMTP Server area; enter localhost (or the IP of the local machine)
  8. You can leave user name and password blank (this is because the First Mail Server application has no authentication)
  9. Click on Test…
  10. Fill out an email address (not a madcap one, since we can’t email those from within our own network)
  11. Check your email. Usually ends up in my yahoo junk mail folder, just an FYI.

NOTE: If you close FreeSMTP, you will not be able to send email from Feedback Server Admin. FreeSMTP needs to be running for email to be sent.

Things to find out…?

  1. Domain/Workgroup (Workgroup is fine for Outlook testing. Domain is needed for Feedback testing)
  2. Local User/ Domain user (Again, local is ok for Outlook. Feedback requires a domain user logged in)
  3. UAC on/off (Doesn’t matter. Feedback will need it launched by Right Click | Run as administrator, if UAC is left on)
  4. Firewall on/off/exceptions (Didn’t matter for Outlook or Feedback)
