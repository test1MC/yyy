Feedback Server Install Matrix

Use this matrix to make sure you test all configurations of a Feedback Server installation. SQL can be installed using all default settings or it can be installed using a Named Instance or Specified Authentication ("custom" install).

 

Print this matrix and mark off the box for each configuration that is tested.

Indicate testing was completed using a default SQL install with a single *

Indicate testing was completed with a custom SQL install with two **

RECOMMENDED: For each install configuration that is tested, generate at least one output from Flare using that Feedback Server. Use the Feedback Testing Matrix to test that all data is properly tracked with that server/database configuration. This will provide you with a head start in completing that matrix and save you time setting up unique configurations later.

Follow Up Testing for Test Cases 2 and 4: Active Help Systems. Test that a published Help system that is using a "previous version" Feedback Server continues to run smoothly following an upgrade to the new version.

 

Manifest: This needs to done anytime the server is installed.

The following additional steps need to be performed after the server is installed to make sure Manifest reports are sent out.

  1. Click on the start menu and search for ‘services’ or go to control panel | Administrative tools | Services
  2. Find the MadCap Service that will be running after you install Feedback Server
  3. Right-click on the service
    1. Go to properties
    2. Log in tab
    3. Make sure the log in information matches the log in information for the user / machine that Feedback Server is installed on.
  4. Restart MadCap Service.
