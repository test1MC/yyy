Installing ASP.NET Manually

Released versions of Feedback Server Admin should provide functionality in the Required Windows Components dialog to install the necessary versions of ASP.NET that are required for Feedback/Pulse to function properly. However, if new features eventually require newer versions of ASP.NET to be installed, you may find need to register it manually during development (until that version of ASP.NET is ultimately included).

 

It is also worth noting that there is a bug which causes the Required Windows Components dialog to incorrectly identify that ASP.NET is installed which it is in fact not. The only workaround for this is to install ASP.NET manually.

 

To install ASP.NET manually:

  1. Launch the Command Prompt (you may need to launch it in Administrator mode)
  2. Navigate to C:\Windows\Microsoft.NET\Framework\

  3. Navigate down one more directory for the version of ASP.NET you want to install:

    Example: C:\Windows\Microsoft.NET\Framework\v4.0.30319

  4. Type "aspnet_regiis -i" and press Enter
  5. You should see two prompts indicating Start and Finish of the install
