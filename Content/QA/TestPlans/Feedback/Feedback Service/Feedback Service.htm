Feedback Service

Bug Base Product Area:

  • Output - (Various)

 

Description:

This test plan focuses on the feedback functionality that exists in Flare output. In lieu of standard test plans, use the Feedback Testing Matrix to verify that all data that is tracked by Feedback Server is indeed tracked in all outputs and, in the case of WebHelp and WebHelp Plus, all browsers and protocols (http, https, local). Test Cases are still collected per output and should also be tested after completing the matrix.

 

Test Plans:

  • Feedback Testing Matrix
  • Output General - Test Cases
  • Output CHM - Test Cases
  • Output DotNet - Test Cases
  • Output Web - Test Cases

 

Dependencies:

  • Feedback enabled CHM
  • Feedback enabled WebHelp output
  • Feedback enabled WebHelp Plus output
  • Feedback enabled mchelp (DotNet output)
  • Feedback Explorer (to verify that data is being tracked)
