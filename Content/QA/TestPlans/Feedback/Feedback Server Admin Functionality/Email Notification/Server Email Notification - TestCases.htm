Server Email Notification - Test Cases

Create administrators, reviewers and Help system users with a variety of email addresses.

  • Test very long email addresses
  • Test email addresses with various characters

Baseline

13933 If your Feedback is configured such that comments do not require review, but you do enter Reviewer e-mail addresses, the reviewers are still sent e-mails with the option to Accept or Discard the comment.

25562 A user registers to receive an e-mail when another user replies to his/her comments. The e-mail that user receives with the reply does not contain the original comment.

25564 The emails which the end user receives to alert them to comments being added to the Help system (all 3 notification options) displays the Topic as a filename; the end user should see the title of the Topic.

25876 Reviewer/Admin e-mails are not going out for top level comments (e-mails are sent for reply comments).

35330 If "Enable anonymous comments" is turned on, there is no means of being notified (as a Reviewer or Admin) whenever a comment is added to the Help system.

36165 After clearing the database of all statistical data, reviewers and admins cannot accept/hide/delete comments from the emails they receive.
