Feedback Configuration - UI

Creating a new configuration file

From Flare

  1. Open a Target file that has MadCap Feedback enabled
  2. In the Feedback Service tab, click the "Configure with MadCap Feedback Explorer" button
  3. Feedback Explorer will launch and a Feedback Configuration file will automatically be created using the Feedback Server URL and Feedback Service Key set in the Flare Target file

From Feedback Explorer

!File | New

UI Controls

  • Feedback Server URL: Enter the URL (the name of the machine) running MadCap Feedback Server Admin, or enter "madcap" to test the MadCap-hosted feedback service.
  • Feedback Key: Enter the Feedback Service key. (If connecting to Feedback Server Admin, the key must begin with a 'W'. If connecting to the "madcap" hosted service, the key must begin with 'LH'.)

Opening a configuration file

!File | Open

Only one configuration file can be open at any time. If a different configuration file is opened, the previous configuration file will close.

Editing the configuration file

General Tab

UI Test Cases

  • Feedback Server URL: Displays the URL (the name of the machine) running MadCap Feedback Server Admin, or displays "madcap" if using the MadCap-hosted feedback service.
  • Feedback Key: Displays the Feedback Service key.
  • Published Output URL: If the deployed Help system using this feedback key is WebHelp or WebHelp Plus, you can enter the published output URL in this field. Doing so will allow the "Live View" button in the Topic View to function.
  • Options
    • Enable anonymous comments: Enable this option to allow users to add comments to the Help system without needing to create a user profile. This option is disabled by default.
    • Require new comments to be reviewed: Enable this option to require that all comments be accepted by a reviewer (or accepted in the Feedback Explorer app) prior to being made visible by all users of the Help system. This option is enabled by default.

Expired Comments:

Comments will become hidden after the set number of days

Test that after the set number of days comments that have been in the help system that long are not hidden. Their status has been changed to hidden.

Administrators Tab

UI Test Cases

  • (Administrators): You can enter a list of e-mail addresses (one address per line) where notifications will be sent for certain events. The specific events for which Administrators will receive an e-mail notification are set in the Notifications tab.

Administrators Properties Dialog

Dialog allows you to change the email address of the admin, the subscription status to the server manifest, and all admin email notifications.

 

Test Cases:

  • Edit email address and test that email notifications go to the new address.

  • Edit email address and test that email notifications do not go to the old address.

  • Toggle the Subscribe to server manifest on. Advance your computer time to 11:49PM and wait until 11:50PM. Make sure you recieve the project manifest.

  • Toggle the Subscribe to server manifes off. Repeat previous test case and verify you no longer get the manifest. You need to delete and re-add a new manifest as manifest will only go out once a day.

  • Toggle Search is performed on. Perform a search in your feedback enabled help system and verify the correct email notification is sent to the admin.

  • Toggle Search is performed off. Perform a search in your feedback enabled help system and verify no email notification is sent to the admin.

  • Toggle Topic is rated notification on. Rate a topic in your feedback enabled help system and verify the correct email notification is sent to the admin.

  • Toggle Topic is rated notification off. Rate a topic in your feedback enabled help system and verify no Topic rated email notification is sent to the admin.

  • Toggle CSH call notification on. Perform a CSH call in your feedback enabled help system and verify the correct email notification is sent to the admin.

  • Toggle CSH call notification off. Perform a CSH call in your feedback enabled help system and verify no CSH email notification is sent to the admin.

  • Toggle the Topic comment submitted by user notification on. Submit a comment as a user that is different that the admin on your feedback enabled help system and verify the correct email notification is sent to the admin.

  • Toggle the Topic comment submitted by user notification off. Submit a comment as a user that is different that the admin on your feedback enabled help system and verify no Topic comment submitted by user email notification is sent to the admin.

  • Toggle the Topic comment status changed notification on. Change the status of a comment and verify the correct email notification is sent to the admin.

  • Make the status change from the comment report local toolbar drop down. Verify Status change email notification is sent to the admin.

  • Make the status change from the right click menu on the comment grid. Verify Status change email notification is sent to the admin.

  • Make the status change from the edit comment dialog. Verify Status change email notification is sent to the admin.

  • Make the status change from the admin notification email. Verify Status change email notification is sent to the admin.

  • Toggle the Topic comment status changed notification off. Change the status of a comment and verify no topic comment status change email notification is sent to the admin.

Excluded IPs Tab

UI Test Cases

  • (Excluded IPs): You can enter a list of IP addresses (one address per line) that should be excluded from collecting feedback. Test specific IP addresses as well as addresses that use wildcards (the * character).

Notifications Tab

UI Test Cases

  • Search is performed which returns no results: Select this option to send an e-mail notification to all administrators whenever a search is performed which returns no results.
  • Topic is rated: Select this option to send an e-mail notification to all administrators whenever a topic is rated.
  • CSH call is made: Select this option to send an e-mail notification to all administrators whenever a CSH call is made.
  • Topic comment submitted by user: Select this option to send an e-mail notification to all administrators whenever a topic comment is submitted.
  • Topic comment status changed by reviewer: Select this option to send an e-mail notification to all administrators whenever a comment's status is changed.

Manifest Tab Added V4

 

Manifest are statistical reports that can be sent to administrators.

 

Frequency:

Create a new schedule to run the manifest: Opens the new task dialog

Test cases:

  • Add daily, weekly, and monthly manifest schedules and verify that each type of report is sent out.

 

Modify the scheduled running of the manifest: Opens the New task dialog for editing.

Delete the selected schedule: Deletes the selected schedule.

Date Filter: Allows you to schedule the date from which the reports should collect data from.

Range: From the date specified to now.

Period of days: From now back the number of days specified.
