Topic View - UI

Opening the Topic View

  • Double-click a topic listed in the Topics pane
  • Select a topic listed in the Topics pane and press <F4>
  • Double-click a topic listed in the Context-Sensitive Help Calls pane
  • Select a topic listed in the Context-Sensitive Help Calls pane and press <F4>

Local Toolbar and Columns

 

Local Toolbar Buttons (Tooltips in parentheses):

  • (Refresh): Refreshes the data in the Report Tab.
  • (Date filter): Invokes the Date Filter dialog which lets you limit the visible data to data collected within a particular date range.
  • (Open Live Topic in the default web browser): If the Published Output URL of the WebHelp Help system is set in the General tab of the configuration file, this button will open a live view of the topic. The topic will open outside the context of the Help system and will therefore not register a topic view.
  • (Select columns to show/chart): Opens the Options dialog which lets you set which columns are visible and which columns should be included when charting data.
  • Chart Options: Open the Chart Options dialog which allows you customize the row colors of the grid.

    • Add a custom chart color button: tooltip: Add a custom chart color: Opens the color picker dialog.

    • Delete the custom chart color button: Tooltip: Delete the custom chart color: removes the selected custom chart color.

  • (Select the chart type): Lets you select the type of chart which displays for this Report Tab. This option should be grayed out when the Comments tab is selected.
  • (Select the data for the chart): Lets you select which rows in the grid view should be considered in the generated chart. This option should be grayed out when the Comments tab is selected.
  • (Page Setup): Invokes the Page Setup dialog which lets you select paper size, orientation and margins. This option should be grayed out when the Comments tab is selected.
  • (Print Preview): Invokes the Print Preview dialog which displays a preview of the chart which can be printed from the Report Tab. This option should be grayed out when the Comments tab is selected.
  • (Print): Prints the chart from the Report Tab.
  • (Discard selected comment(s)): Deletes the comment from the Feedback Server. Clicking this button should prompt for confirmation.
  • (Set comment status): Clicking here invokes a drop-down which lets the reviewer or administrator change the comment status to Accepted, Pending Review or Hidden.

 

Topic View | Comments grid

  • User: Displays the username of the user who added the comment.
  • Subject: Displays the subject of the comment.
  • Comment: Displays the comment. Most likely just the beginning of the comment followed by "...". Hovering over the cell will display the full comment in a tooltip.
  • Date: Shows the date and time at which the comment was added to the topic.
  • Status: Displays the review status of the comment.
  • CommentID: Displays the Feedback Server comment ID number for the comment.
  • ParentID: If the comment is a reply to another comment, displays the Feedback Server comment ID number for the parent comment (the comment being replied to).
  • #: Displays a count for every comment in the list.

Right-Click Menu

Menu items

  • Properties: Invokes the (Comment) Properties dialog.
  • Accept Comment: Changes the status of all selected comments to Accepted.
  • Pending Review: Changes the status of all selected comments to Pending Review.
  • Hide Comment: Changes the status of all selected comments to Hidden.

 

Options dialog - UI Test Cases

UI Controls

  • Move Up: Moves the selected row(s) up in the sequence which in turn moves the corresponding column to the left in the Search Phrases view.
  • Move Down: Moves the selected row(s) down in the sequence which in turn moves the corresponding column to the right in the Search Phrases view.
  • Reset to Factory Settings... : Removes all custom column settings and restores factory defaults.

Columns dialog grid

Columns

  • Show: Lets user show/hide the associated column in the Search Phrases view.
  • Chart: Lets user determine if the column should be included in charting. Only one column can be selected to chart at any one time.
  • Column: Displays the name of each column that can be displayed in the Search Phrases view.
  • Width: Lets user set the width in pixels of each column in the Search Phrases view.
  • Color: Lets user open the Color Picker dialog to set the font color for each column in the Search Phrases view.
  • Background: Lets user open the Color Picker dialog to set a background color for each column in the Search Phrases view.

Grid Test Cases

  • Columns dialog columns should not be sortable
  • Columns dialog columns should not be resizable
  • Columns dialog columns cannot be moved by clicking on a column heading and dragging
  • Horizontal and vertical scrollbars should be working correctly
  • Columns dialog rows should not be able to be multi-selected with Shift- or CTRL-click

(Comment) Properties Dialog - UI Test Cases

To open the dialog:

  • Double-click an item in the Comments tab of the Topic View
  • Select an item in the Comments tab of the Topic View and press <F4>

UI Controls

  • Subject: Displays the subject of the comment.
  • User: Displays the username of the user who added the comment.
  • Date: Shows the date and time at which the comment was added to the topic.
  • Comment: Displays the comment. This field should be scrollable.
  • In reply to: If viewing Properties of a comment that is a reply to another comment, the lower fields will be populated as well. If the comment is not a reply, these lower fields should appear empty.
    • User: Displays the username of the user who added the parent comment.
    • Comment: Displays the parent comment. This field should be scrollable.

(Rating) Properties Dialog - UI Test Cases

To open the dialog:

  • Double-click an item in the Rating tab of the Topic View
  • Select an item in the Rating tab of the Topic View and press <F4>

UI Controls

  • Rating: Displays the numerical value of each rating.
  • Date: Shows the date and time at which the rating was added to the topic.
  • Feedback: Displays any author-only feedback provided with the rating, if the user provided any. This field should be scrollable.

Topic View | (both tabs) - Keyboard Navigation

<Up Arrow> moves the selection to the row above

<Down Arrow> moves the selection to the row below

<Enter> moves the focus to the row below

<Page Up> will move the selection one page at a time upward

<Page Down> will move the selection one page at a time downward

<Left Arrow> and <Right Arrow> do nothing

<Home> and <End> do nothing

<Insert> does nothing

<Delete> does nothing
