Comments - UI

Opening the Comments view

  • !View | Comments
  • Press <CTRL>-E on the keyboard
  • Click the Comments button on the Standard Toolbar

Comments - UI Test Cases

Local Toolbar and Columns

 

Local Toolbar Buttons (Tooltips in parentheses):

  • (Hide/Show the Info Bar): Displays or hides the Info Bar.
  • (Refresh): Refreshes the data in the Report Tab.
  • (Date filter): Invokes the Date Filter dialog which lets you limit the visible data to data collected within a particular date range.
  • Export the data to a csv file: Exports the data to a csv file that can be opened in Microsoft Excel.
  • (Filter comment status): Lets you .
  • (Discard selected comment(s)): Deletes the comment from the Feedback Server. Clicking this button should prompt for confirmation.
  • (Set comment status): Clicking here invokes a drop-down which lets the reviewer or administrator change the comment status to Accepted, Pending Review or Hidden.

 

Comments grid

  • Topic: Displays the path and filename of the topic file that received the comment.
  • User: Displays the username of the user who added the comment.
  • Subject: Displays the subject of the comment.
  • Comment: Displays the comment. Most likely just the beginning of the comment followed by "...". Hovering over the cell will display the full comment in a tooltip.
  • Date: Shows the date and time at which the comment was added to the topic.
  • CommentID: Displays the Feedback Server comment ID number for the comment.
  • ParentID: If the comment is a reply to another comment, displays the Feedback Server comment ID number for the parent comment (the comment being replied to).

 

Right-Click Menu

Menu items

  • Info Bar: Displays or hides the Info Bar.
  • Properties: Invokes the (Comment) Properties dialog.
  • Accept Comment: Changes the status of all selected comments to Accepted.
  • Pending Review: Changes the status of all selected comments to Pending Review.
  • Hide Comment: Changes the status of all selected comments to Hidden.

 

 

(Comment) Properties Dialog - UI Test Cases

To open the dialog:

  • Double-click an item in the Comments View
  • Select an item in the Comments View and press <F4>
  • Right-click an item in the Comments View and select Properties

UI Controls

  • Subject: Displays the subject of the comment.
  • User: Displays the username of the user who added the comment.
  • Date: Shows the date and time at which the comment was added to the topic.
  • Comment: Displays the comment. This field should be scrollable.
  • In reply to: If viewing Properties of a comment that is a reply to another comment, the lower fields will be populated as well. If the comment is not a reply, these lower fields should appear empty.
    • User: Displays the username of the user who added the parent comment.
    • Comment: Displays the parent comment. This field should be scrollable.

Date Filter dialog - UI Test Cases

UI Controls

  • Enable Filter: Enables/disables the date filter for the data set.
  • Filter Settings: Lets you select between a user-defined date range or a preset date range.
    • Range:  Select this option to set a specific date and time for the filter. Click on the down arrow in the "From" and "To" fields to set the specific month and date. You can also click inside the "From" and "To" fields and type to specify a particular time.
    • Preset: Lets you set the Date filter to one of a set of factory presets. These include: Today, Yesterday, Last 7 days, and Last 30 days

Keyboard Navigation

<Up Arrow> moves the selection to the row above

<Down Arrow> moves the selection to the row below

<Enter> moves the focus to the row below

<Page Up> will move the selection one page at a time upward

<Page Down> will move the selection one page at a time downward

<Left Arrow> and <Right Arrow> do nothing

<Home> and <End> do nothing

<Insert> does nothing

<Delete> does nothing

 

EDIT COMMENT DIALOG ADDED V4 (Double click a comment in the comment window)

 

 

You can Edit the actual content of a comment as well as change the Status of the comment.

Test that setting the status from this dialog sends the right email notification to the registered admins.

When status is changed to pending review, the admin receives email notification saying the status of that comment was changed to pending review.

When status is changed to hidden, the admin receives email notification saying the status of that comment was changed to hidden.

When status is changed to accepted, the admin receives email notification saying the status of that comment was changed to accepted.

If the status is changed to the same status such as Accepted to Accepted, the administrator should not receive a notification because the status was never changed.
