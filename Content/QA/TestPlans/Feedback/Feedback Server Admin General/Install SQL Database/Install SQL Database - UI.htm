Install SQL Database - UI

This test plan will cover the UI associated with installing the IIS Server and setting up the SQL Database when setting up a Feedback Server. Should you encounter any difficulty setting this up when testing, refer to the Feedback Server Admin Help for more information. If you are testing a remote SQL server setup, refer to this document for assistance: "Connecting a Feedback Server to a remote SQL machine"

Installing Feedback Server

Upon launching Feedback Server Admin for the first time, you will need to install the IIS Server and set up your SQL Database. Feedback Server Admin will initiate this process automatically when it is launched. If all of the required Windows components are installed on the IIS machine, you will immediately presented with the Feedback Server Setup dialog. If not, the Required Windows Components dialog will appear until all required components or installed/registered.

Required Windows Components dialog

UI Controls

  • Internet Information Services (IIS): Displays whether or not IIS is installed on the machine. Needs to indicate "Installed" in green text to proceed.
  • ASP.NET: Displays whether or not ASP.NET is registered on the machine. Needs to indicate "Registered" in green text to proceed.
    • Register: Use this button to register ASP.NET. After clicking this button, a progress bar should appear. This button will be inactive if IIS is not installed. (It will also be inactive once ASP.NET gets registered.)
  • IIS 6 Metabase Compatibility: (Vista only)Displays whether or not IIS 6 Metabase Compatibility is installed on the machine. Needs to indicate "Installed" in green text to proceed. (Verify that this field does not appear on OS for which it is not applicable.)
  • Try Again: If any of the items above are not installed or registered, use this button to attempt to proceed once you've set up all the components.
  • Cancel: Cancels the attempt to install the Feedback Server and returns you to Feedback Server Admin.

Feedback Server Setup dialog

UI Controls

  • SQL Server: Displays a list of all potential SQL Servers on the domain. When initially opened, this field should display "(Retrieving list of SQL Servers...)." If the SQL instance that you would like to use does not appear in the list, you should also be able type the IP address or the name of the machine in this field. (The drop-down should display all instances of supported SQL and SQL Express on the network)
  • Database Location: Displays the path to the SQL database you will be installing. This field will be grayed out if the SQL Server designated above is on a different machine. If a remote SQL database is used, the default location will always be used.
    • Browse: Opens a browse dialog that lets you browse for a folder for the SQL database. This button will only be active if the SQL Server designated above is not a remote machine.
  • Please select the website... : Select the website where you want MadCap Feedback Server installed.

  • Manually set IIS authentication setting: 

    - Remote SQL Server: This option will be grayed out, but selected. You will be required to fill out the "Authentication" section below.

    - Local SQL Server: This setting is optional. In most cases it can be left unchecked. Enabling the option should enable the "Authentication" UI below.

  • Authentication:
    • Domain: Enter the domain name of the IIS and SQL machines for authentication purposes
    • Username: Enter the username for authentication purposes. The user must have admin rights on the IIS machine and sysadmin rights on the SQL machine.
    • Password: Enter the password for authentication purposes.
    • Verify Credentials: Ensures
  • Import data from the previous Feedback Server installation: If a previous Feedback Server installation is detected on the IIS machine, this option will become active (and be selected by default). Leave the box checked to import the data from that Feedback Server database to the new version. If there is no previous installation, this option will be grayed out. (It is also possible to import the SQL database from another machine. This is covered in the test plan "Server Install - UI".)
  • OK: In this dialog, if you are using a remote SQL machine, the OK button will not allow you to proceed until you have successfully verified authentication credentials. Test clicking OK with invalid and unverified authentication data and verify that the messages that appear are appropriate.

Setting up Feedback service for Manifest notifications

This needs to done anytime the server is installed.

The following additional steps need to be performed after the server is installed to make sure Manifest reports are sent out.

  1. Click on the start menu and search for ‘services’ or go to Control Panel | Administrative Tools | Services
  2. Find the MadCap Service that will be running after you install Feedback Server
  3. Right-click on the service
    1. Go to properties
    2. Log in tab
    3. Make sure the log in information matches the log in information for the user / machine that Feedback Server is installed on.
  4. Restart MadCap Service.

Feedback Server Configuration

After successful creation of the Feedback Server SQL Database, Feedback Server Admin will open the Feedback Server Configuration tab. Proceed to "Server Install - UI" for additional testing.
