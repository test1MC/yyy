Resx Projects - Export

Opening the UI

Select File>Export to Resx… (Tooltip should read: Export translated files back to Resx)

UI Test Cases:

Export Finished Translation dialog:

 

UI Controls

  • Project to Export: Displays the name and path of the current Lingo project.
  • Destination File name: Displays the name you provided for the Lingo project. You can keep the same name or enter a different one for the exported project.
  • Destination Folder: Either accept the folder location entered, or click Browse to find and select a different one. Typically, you want to export the project to a location where the author can retrieve it.
  • Browse: Opens the Browse For Folder dialog, which lets you select a different folder to store the project files.
  • Export: Click this button to export the project.

 

 

'Progress' dialog window:

 

 

  • Cancel should stop the export.

 

Successful export dialog window:

 

  • ‘Yes’ closes dialog and should open translated Resx project (as long as you have a Resx reader program such as Visual Studios)
  • ‘No’ closes dialog

 

 

Trial Mode

When Lingo is used in Trial Mode, it will prompt you with the following dialog window:

 

 

  • Clicking “OK” should still open the translated Resx project (as long as you have a Resx reader program such as Visual Studios)
  • Open the exported Resx project in trial mode and make sure its contents are scrambled.
  • Enter a valid key, export and verify that the contents are not longer scrambled. 

 

 

You should always be able to create a new project using Resx files. You don't need to have a Resx reader installed in order to create a new project.
