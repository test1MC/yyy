!Edit | Find and Replace | Find and Replace

Press <CTRL>-F on the keyboard

Opening the Find in Files pane

!Edit | Find and Replace | Find in Files

Press <CTRL>-<Alt>-F on the keyboard

Load Find Selection

Load Find Selection functionality allows you to select text in the XML Editor and load that selection into either the Find and Replace pane or the Find in Files pane. If the Find in Files pane is open If neither pane is open and currently visible (not hidden behind other accordion panes), Load Find Selection will load the selection into the Find in Files pane. Otherwise, Load Find Selection will always load the selected content into the Find and Replace pane.

To perform Load Find Selection, select some text in the XML Editor and:

  • Press <CTRL>+<Shift>+<G> on the keyword
  • !Edit | Find and Replace | Load Find Selection
  • Click the Load Find Selection button in either the Find In Files pane or the Find and Replace pane (see below)

Opening the Find Results panes

!Edit | Find and Replace | Find Results 1 or Find Results 2

Select "Output in Find Results 1" or "Output in Find Results 2" in the Find in Files pane when performing a find (see below).

Find and Replace Pane - UI Test Cases

Default view docked in right pane:

UI Controls

Find

  • Find What: Type the text that you want to find.
  • (Load selection into current Find window): Copies the selected text in the XML Editor to the "Find what" field in the Find and Replace window.
  • Find Next: Locates the next occurrence of the text. The text is highlighted in the topic where it is found, or in the Internal Text Editor if you are searching within style sheets.
  • Replace With: Type the text that will replace the found text.
  • Replace: Replaces only the current instance of found text.
  • Replace All: Replaces all instances of the found text in all topics or style sheets in the project.
  • Replace in All Files: Replaces all instances of the found text in the current topic or style sheet.

Options

  • Find in: Click the down arrow to select the parameters of the search:

    • (current document): Flare looks for the text in the active topic (or style related style sheet).
    • (whole project): Flare looks for the text in every topic (or related style sheet) in the project.
    • (all open documents): Flare looks for the text in all topics (or related style sheets) currently open in the interface.
    • (documents in same folder): Flare looks for the text in all topics (or related style sheets) located in the same folder as the current one in the Content Explorer.
  • File types: Select the kind of files that you want to include in the search (topics or CSS style sheets).
  • [ >> ] : Skips searching in the current topic or style sheet and moves on to the next file.
  • Start: Initiates the search.
  • Match case: Finds only occurrences of the text that match the case (e.g., uppercase, lowercase) of the text entered in the "Find what" field.
  • Use: An advanced feature for users who are familiar with wildcard and regular expressions in searches. If you select this check box, you can specify whether the search will include wildcards or regular expressions.
  • Find in source code: Includes the source code of the topics or style sheets in the search.

Find in Files - UI Test Cases

Default view docked in right pane:

UI Controls

Find

  • Find What: Type the text that you want to find.
  • (Load selection into current Find window): Copies the selected text in the XML Editor to the "Find what" field in the Find in Files window.
  • Find All: Initiates the search.

  • Find in: Click the down arrow to select the parameters of the search:

    • (whole project): Flare looks for the text in every topic (or related style sheet) in the project.
    • (all open documents): Flare looks for the text in all topics (or related style sheets) currently open in the interface.
    • (documents in same folder): Flare looks for the text in all topics (or related style sheets) located in the same folder as the current one in the Content Explorer.
  • File types: Select the kind of files that you want to include in the search (topics or CSS style sheets).

Options

  • Match case: Finds only occurrences of the text that match the case (e.g., uppercase, lowercase) of the text entered in the "Find what" field.
  • Use: An advanced feature for users who are familiar with wildcard and regular expressions in searches. If you select this check box, you can specify whether the search will include wildcards or regular expressions.
  • Find in source code: Includes the source code of the topics or style sheets in the search.
  • Output in Find Results 1: Displays the search results (list of files) in the Find Results 1 window.
  • Output in Find Results 2: Displays the search results (list of files) in the Find Results 2 window. This is useful if you have already performed a search using the Find Results 1 window and want to keep those results open while you do another search.

Find Results 1 and 2 - UI Test Cases

Default view docked in bottom pane:

 

UI Controls

Local Toolbar Buttons (Tooltips in parentheses):

  • (Open selected): Opens the selected file in the Find Results pane and highlights the found text.
  • (Open previous): Opens the file previous to (above) the selected file in the Find Results pane and highlights the found text.
  • (Open next): Opens the file following (below) the selected file in the Find Results pane and highlights the found text.
  • (Clear table): Clears the Find Results pane.

Find Results grid

Columns

  • File: Shows the name of the file.
  • Context: Shows the context of the found text within the file.
  • Folder: Shows the location of the file within the project.
  • Title: Shows the Topic Title as set in the Topic Properties dialog.
