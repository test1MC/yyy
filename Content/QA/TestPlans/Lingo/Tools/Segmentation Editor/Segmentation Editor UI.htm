Segmentation Rules Editor

Controls:

This dialog lets you work add, edit, and remove custom segmentation rules. You can perform any of the following tasks:

  • Quick Help icon: Provides some quick help regarding the use of spaces in the SRX Editor. Includes a book icon that when clicked, opens the CSH Help topic - "Segmentation Rules and Spaces"
  • Add rules: Creates a new rule by which Lingo will segment files the next time it creates a new project or a does a project update
  • Remove rules: Any custom rule can be removed by highlighting the rule and then pressing the DELETE key or pressing on the respective "Delete Rule" icon.
  • Edit/Modify Rules: A rule can be edited by placing the cursor into the field
  • Import SRX files: Any SRX file that was created by Lingo can be imported into the SRX Editor
    • If there are no custom rules, and the editor is empty, clicking on "Import" will invoke the standard Windows Open Files Dialog with the filter set to Segmentation Rules eXchange File (*.srx) .

    • If there is at least one rule, then the a warning dialog should appear. See Import SRX File Warning.

  • Export SRX files: Any custom rules currently in the editor and all default rules can be exported. Clicking on 'Export' will invoke the standard Windows Save Files dialog with the filter "Save As Type" set to Segmentation Rules eXchange File (*.srx) .
  • Restore Default: Restores the editor to factory state without any custom rules. Clicking on this will invoke a warning dialog. See Restore SRX File Warning

Import SRX File Warning

Controls:

  • Yes: Invokes the standard Windows Open Files Dialog with the filter set to Segmentation Rules eXchange File (*.srx) .
  • No: Returns user to SRX Editor without any changes to custom rules

Restore SRX File Warning

Controls:

  • OK: Removes all customer / user-defined rules and returns user to Segmentation Rules Editor.
  • Cancel: Returns user to SRX Editor without any changes to custom rules
