UI

Start New Project Wizard-Select source project to start translating…

 

Controls:

 

  • Add…: Will open the ‘Select file(s) or project to translate’ dialog window.
  • Remove: Will remove the selected file(s) or project.
  • Back: This button is disabled in this window.
  • Next: Opens the next page of the wizard.
  • Finish: This button is disabled in this window.
  • Cancel: Closes the wizard.
  • Reference Translation Memory:
    • Browse for TM... (See SQL Express-UI.htm--Connect to Lingo Translation Memory Dialog.) 
    • Install New TM (see SQL Express-UI.htm)

 

Test Cases:

  • Type invalid text, numbers, character, etc into fields that will take user input
  • Type extra long input into fields
  • Type characters and symbols into fields
  • In the dialog that opens, navigate to the project or file that you want to translate, and double-click the appropriate file. The main extensions for the various types of files that you can translate are: .flprj (Flare projects), .blprj (Blaze projects), .miprj (Mimic projects), .mimov (Mimic movies), .mdcoct or .mcdoc (MadCap documents), .dita (DITA files), .doc (Microsoft Word files), ppt/pptx (PowerPoint files), various image file types (Capture files). Files with other extensions are also supported (e.g., .flprjzip, .ditacon, .docx, .htm, .xml, *.rtf).

 

Start New Project Wizard-Select source project to start translating…

 

Controls:

 

  • Source language: Select the original language that is used in the project you are translating.
  • Target language: Select the language that you want to use for the translation.
  • Hide unsupported languages: Hides languages that are not yet supported by the spell check feature (i.e., only the languages in bold will be shown).
  • Back: Opens the previous page of the wizard.
  • Next: Opens the next page of the wizard.
  • Finish: This button is disabled in this window.
  • Cancel: Closes the wizard.

 

 

Start New Project Wizard-Enter name and folder for the translated project…

 

 

Controls:

 

  • Project name: Type an appropriate name for your project.
    • EXAMPLE: If you are translating a Flare project called "DoohickeyPro," and the localized language is German, you might name your project something like "DooHickeyPro German."
  • Generate: Adds a number to the end of the name already entered.
  • Project folder: By default, a path to the My Documents\My Translated Projects folder on your hard drive is entered in the Project folder field. (Lingo creates the My Translated Projects folder for you when you install the program.) All files related to your project will be placed in this folder as you work on your project. If you want to have the project files placed in a different folder, use the "Browse" button.
  • Note: In Vista, the folder is called "Documents" instead of "My Documents."
  • Browse: Opens the Browse For Folder dialog, which lets you select a different folder to store the sample project files.
  • Back: Opens the previous page of the wizard.
  • Next: Opens the next page of the wizard.
  • Finish: Creates the project and loads the files into Lingo.
  • Cancel: Closes the wizard.

 

Test:

  • Clicking ’Browse’ invokes the Browse For Folder dialog window.

Open Project

 

Opening the Open Project Dialog:

  1. Click on in standard toolbar ! File | Open or...
  2. Click <CTRL><O>

  3. These will invoke the standard Windows Open Files Dialog with the filter set for *.liprj and *.lialign – options in the drop-down are “Lingo Packages (*.liprjzip; *.lialignzip; *.lidoc)”

  4. Double-clicking the *.liprj or *.lialign files in Windows Explorer

Note: Let's say you have already created a Lingo project for an earlier version of a particular Flare or Blaze project. You have already translated it, exported it to a new Flare or Blaze project, and given it to the author. Sometime after this, the author makes many changes to the Flare or Blaze project (in the source language). Now the author needs you to translate those changes into the target language. When this occurs, you can create a "difference" project to translate the new version of the Flare or Blaze project. This allows you to easily locate the differences between the old and new versions, rather than looking through each piece of text. If a file or string has been modified since the previous version, an icon displays next to that content (the type of icon depends on the type of change made). In addition, the "Difference" column in an editor's grid provides an indication of the change. You can then compare the two versions in order to determine what was modified and translate the changes as necessary. If the author has deleted content from the new Flare or Blaze project, that content is automatically deleted from the new Lingo difference project when you create it.
