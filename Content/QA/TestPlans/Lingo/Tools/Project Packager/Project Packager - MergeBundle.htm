Test Cases:

  • Clicking "Cancel" should stop merging the bundle and close dialog with no changes
  • In the event of an error, the "Cancel" button should change to "Close" and the "Save Log" button should be enabled

 

 

  • Clicking the "Save Log" button should save the errors and make the log accessible via the project properties

 

Statistics report:

This report should come up after merging the completed bundle and show a description of which files were translated:

 

 

Test Cases for Merging:

  • Merge after completing statistics.
  • Merge a bundle that was just created and has not yet been translated. File status should return to untranslated.
  • Merge a bundle with a random zip file. You should get no relevant files found.
  • Update the project then Merge a translated bundle.
    • To translate a bundle without other translation software you can use notepad. Just extract the files form the Zip file and open them in notepad. You can find the translated text and change it or even add new lines to the text.
  • Send out a bundle. Update the project. It should repackage the bundles. Merge with the first bundle sent out, then merge the second.
  • Merge a bundle from a previous version i.e. a V3 bundle if using V4 Lingo.
