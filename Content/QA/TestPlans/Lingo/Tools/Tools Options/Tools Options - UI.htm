General tab

UI Controls

 

  • Minimize to System Tray: By default, when you minimize Flare, a shortcut button on the main part of the taskbar lets you reopen the application. However, you can select this check box if you want to minimize Flare to the system tray instead.
  • Auto-Reload Documents: Select this check box if you want Flare to reload documents automatically. In other words, if you have topics or editor pages open and then close the project, this option ensures that those same documents are reopened the next time you open the project.
  • Auto-Save Documents: Select this check box if you want Flare to periodically auto-save any open files.
    • "#" minutes: You can specify the number of minutes that should elapse between each auto-save.
  • Open files with these extensions in the Document Dock pane: If you want certain types of files to open automatically in the Document Dock pane, enter those file extensions in this field, separating each one with a semi-colon (e.g., xml;text;asp).
  • Close others when opening new document: An option which tells Flare to close other documents as more documents are opened based on the behavior set here:
    • No action: The default option. Opening new files will have no affect on files already open. All files will remain open.
    • Close others: This option automatically closes all other open documents. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with no changes: This option automatically closes all other open documents that do not have unsaved changes. If you have files open where changes have been made but not saved, those files will remain open until you save and close them.
    • Close others with same extension: This option automatically closes all other open documents that share the same file extension. For example, let's say you have a topic, a style sheet, and a target open. If you open another topic, the topic file that is currently open will be closed, but the style sheet and target will remain open. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with same extension and no changes: This option is the same as the previous option, except that files with unsaved changes will not automatically be closed. Those files remain open until you save and close them.
  • Show Select UI Language Dialog on Startup: Select this box to show the Select UI Language dialog the next time you launch Flare.
  • Reset "Show Once" Dialogs…: There are some dialogs in Flare that open just once to give you a message or warning when something changes or you switch modes. An example of this is the dialog that opens when you first switch between layout modes in the XML Editor. These dialogs have a check box labeled "Show this dialog." If you deselect that checkbox, you will not see the dialog again. However, you can use this reset feature to display that dialog (and all other "Show Once" dialogs) again the next time.

Suggestions Tab UI

 

Use this tab to specify settings matching behavior for translation suggestions. This includes minimum matches and fuzzy matches. You can also use this tab to instantly apply translated strings to any duplicates within the current file. After entering the first translation of a term, the other instances of that same term will also be translated.

 

Button/Section Description:

 

  • Retrieve suggestions with a minimum match of:
    • Enter a percentage in the field. Only translations with a rank greater than or equal to the specified percentage will be displayed as suggestions in the interface.
    • Example:
      • Let's say that you set this percentage to 50%. When using translation suggestions, you might have one translation in your database that matches the current string exactly (100%), another translation that matches it 75%, and a third translation that matches it only 30%. In this case, only the first two translations will be available for you to select in the interface.

       

  • :Fill-in fuzzy matches of at least
    • Enter the percentage at which you can insert matching translations automatically. By default, this percentage is set at 100%, meaning you can use automatic inserts only when text strings match existing translations exactly. However, you can reduce this percentage if you wish. Any matching translation that falls below that specified percentage is considered "fuzzy," because it does not match the string closely enough to warrant an automatic insert.
    • Example:
      • Let's say that you open a topic to translate the text in it. You press ALT+S to open the Suggestions window pane. In that pane, a few matching translations from your translation memory database are shown. Perhaps two of the translations match segments exactly (100%) and there are other partial matches that are below 100%. Perhaps one of those is a 98% match. All of the partial matches, including the one at 98%, are considered "fuzzy."
      • When you first opened the Suggestions window pane, a message displayed. It might say something like this: "Two segments had matches of at least 100% and there were three other partial matches. Do you want to use the top matches automatically?" If you select Yes, the two translations that match text 100% will replace the corresponding strings in the topic automatically. If you want to use any of the partial matches, you must do so manually.
      • Now suppose you want to be able to automatically use matching translations if they match 97% of the string text. Therefore, you set the fuzzy matching option to 97%. If you then return to the same topic and press ALT+S, this time the message will say the following: "3 segments had matches of at least 97% and there were 2 other partial matches. Do you want to use the top matches automatically?" If you select Yes, both of the 100% matches are automatically used, as is the 98% match (because it is no longer considered a "fuzzy match").

       

  • Automatically apply translations to the rest of the file:
    • Click the check box if you want to instantly apply translated strings to any duplicates within the current file. After entering the first translation of a term, the other instances of that same term will also be translated.
