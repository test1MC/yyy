UI and Local Toolbar

Local Toolbar:

(Open) Opens the selected file in the editor.

(Run Properties Dialog for this item) Opens the Properties dialog for the selected item.

(Add new file(s) to project) Lets you add additional files to a translation project after the project has been created. You can add additional files to any project created from other individual files, such as individual Word documents or XML files.

(Create a bundle to send to translators) Lets you create a translation bundle. If you are working with a translator who is using a third-party tool—such as Trados or memoQ—you first need to "package" your Lingo project in order to separate the necessary files for the translator. You can then send the "bundle" (which is a ZIP file) to the translator. This ZIP file consists of the content files requiring translation.

(Merge a bundle received from translators) After the translator returns the completed ZIP file, you can use this button to merge its contents into your original Lingo project.

(Conditional indicators) Lets you show or hide the small squares next to each folder and file, which are used to show colors representing condition tags that have been applied. Hiding condition tags can greatly speed the population of the window pane.

(Show Folders) Toggles between showing or hiding the folder structure.

(Browse and replace images) Lets you replace the selected file with another file of the same type. You might use this button, for example, if you have used a different application to translate the callout in an image file. You can select the image file you want to replace, click this button, and find the new image file on your computer to replace it. If you select a file with a different name, the name will be changed when it is added to the project so that it matches the old file name. This feature does not work for replacing .htm files in this version of Lingo.

 

Lets you limit the files that are displayed by file format:

 

Select an option from the drop-down:

  • Type a file type into the drop-down | Press <Enter> to filter
  • Notice that the typed file type is added to the drop-down (a scroll bar appears if the list gets too long).

 

  • Target:Displays only the files associated with a specific Flare target
  • Translation Candidates:Displays all files that are likely to require translation.
  • All Files: Displays all content files. (*.*)
  • Topic Files: Displays only the topic (HTM and HTML) files.
  • Page Layout Files: Displays only the page layout (FLPGL) files
  • Snippet Files: Displays only the snippet (FLSNP) files.
  • Stylesheet Files: Displays only the style sheet (CSS) files.
  • MasterPage Files: Displays only the master page (FLMSP) files.
  • Image Files: Displays only the image files (BMP, EMF, EXPS, GIF, HDP, JPG, JPEG, PNG, SWF, TIF, TIFF, WDP, WMF, XAML, XPS).

 

“No Files” appears in the list if there are no files of the type in the application or the file type was mistyped

MDI View (Docked in the MDI area of the framework)

 

 

Columns:

 

Name: Displays the file name for the item.

Type: Displays the type of file (e.g., Topic, Target, Variable Set).

# Segments: Displays the number of text segments in the file.

Status:Displays the translated status of the file (e.g., In Progress, Completed, Reviewed). You can change the status for a file manually to help you keep track of your work. To do this, select the file in the list, click the Properties button in the local toolbar, and select the status on the Translation tab of the dialog.

  • The status also changes automatically after sending files for translation or Updating a project. Every file you send for translation should have the file status set to Sent to translation. When you merge a completed bundle the file status should change and not remain Sent for translation.
  • When you update a project the file status will change to in progress if some of the segments are translated.

Difference: If you have created a "difference" project and the text from previous project is not the same as the new version, the differences are indicated in this cell.

Title: Displays the title for the topic.

Path: Displays the location in the project where the file is stored.

Date Modified:Displays the most recent date and time when the file was changed.

File Tags:If the author creates file tags in the source files, these will be displayed in columns with the tag names associated with each file.

Editing in the File List

File List Right-Click Menu

 

Open:opens the selected topic in the Lingo editor.

     Open with:opens the selected topic in the program of your choosing such as notepad.

Save File(s) for Review:Save files to send to a third-party reviewer for comments and revisions.

Prepare Translation Bundle: Lets you create a translation bundle. If you are working with a translator who is using a third-party tool—such as Trados or memoQ—you first need to "package" your Lingo project in order to separate the necessary files for the translator. You can then send the "bundle" (which is a ZIP file) to the translator. This ZIP file consists of the content files requiring translation.

Compute Statistics:Generates a statistical report for the selected file(s). It displays the results in an HTML file and lets you see information about your translation work.

Upload Selected:Upload all translated segments in the selected file(s) to the Translation Memory

Properties:Opens the Properties dialog window for the selected files

 

Right Clicking on the horizontal and vertical scrollbars:

Scroll Here, Left Edge, Right Edge, Page Left, Page Right, Scroll Left, Scroll Right

Scroll Here, Top, Bottom, Page Up, Page Down, Scroll Up, Scroll Down

Properties dialog window/ Translation Tab

Use this tab to set the translation status of a particular file. The status that you select affects the "Translated" column in the File Map window pane.

 

 

Button/Selection:

 

Translation Status Select the translation status for the selected file:

  • In Progress
  • Untranslated
  • Completed
  • Reviewed
  • Approved
  • Sent for translation

 

Properties dialog window/ File Tags Tab

If you are translating a MadCap Flare project, use this tab to view and edit file tag values.

 

 

 

Button/Selection:

 

Tag Type: Select a tag type, based on any file tag sets that have been created.

File Tag Sets: Based on the tag type you selected, the appropriate file tag sets are listed.

File Tags: Based on the file tag set you select from the list on the left, the file tags in it are listed. Click next to each one that you want to associate with the file.

Comment: Displays comments (if any) next to each file tag.

Unselect All: Removes check marks from all condition tags above.

Select All: Adds check marks to all condition tags above.

 

Properties dialog Translation/ General Tab

Use this tab to mark the files as "Read Only" so that they cannot be modified. Other information is also shown on this tab, although the word "multiple" will often display in many of the fields (rather than the actual data) if more than one file has been selected.

 

 

Button/Selection:

 

Name:Displays the name of the file.

Location:Displays the location of the file.

Open Folder:Opens the Windows folder holding the file.

Read Only:If you do not want the file to be edited, click the check box to add a check mark.

Size:Displays the size of the file.

Created:Displays the date and time that the file was created

Modified:Displays the date and time that the file was last modified.

 

 

 

Keyboard Navigation / Multi-Select:

<Up Arrow>: Moves the selection “Up”

<Up Arrow><Shift>: Moves the selection “Up” and continues the selection

<Up Arrow><CTRL>: Moves the selection “Up” to the top of the Files Map

<Up Arrow><CTRL><Shift>: Moves the selection “Up” to the top of the files map, selecting all topics from the initial selection to the top

<Down Arrow>: Moves the selection “Down”

<Down Arrow><Shift>: Moves the selection “Down” and continues the selection

<Down Arrow><CTRL>: Moves the selection “Down” to the bottom of the Files Map

<Down Arrow><CTRL><Shift>: Moves the selection “Down” to the bottom of the files map, selecting all topics from the initial selection

<Page Up>: Moves the selection to the top of the next full screen

<Page Up><Shift>: Moves the selection to the top of the next full screen and selects all of the topics in that area.

<Page Up><CTRL>: Same as<Page Up><Shift>

<Page Down>: Moves the selection to the bottom of the next full screen

<Page Down><Shift>: Moves the selection to the bottom of the next full screen and selects all of the topics in that area.

<Page Down><CTRL>: Same as<Page Down><Shift>

<Home>: Moves the list to the top

<End>: Moves the list to the bottom

<Shift> + Mouse Click: Selects all topics from the original selection to where the mouse was clicked

<CTRL> + Mouse Click: Selects or de-Selects only the topics that you click in the multiple selection

 

Right Click: brings up a menu with Open, Open With, and Properties.
