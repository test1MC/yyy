Topic Editor - UI

Opening the UI

1. Open the File List window pane (View>File List).

2. At the top of the window pane, next to the Filter field, click the down arrow and select Translation Candidates.

3. In the list, locate a row with "Topic," "Snippet," or "Master Page" displayed in the Type column and double-click it.

Grid Mode (UI)

 

Grid mode allows you to work with the file content in text strings organized in columns, similar to the way other content (e.g., tables of contents, variables, glossaries) is displayed in the respective editors. The grid mode not only lets you translate the text in small segments, and it is also necessary to use this mode for translating hidden text strings (e.g., hyperlink screen tips, alternative image text that is used for Section 508 compliance). These hidden strings can be translated only in grid mode.

 

Background colors: White=Not Translated; Gray=Translated/Accepted; Red=Match Exists From TM (darker the shade, higher the match percentage).

Although index keywords and concepts may appear in topics, they are not translated in those same files. For example, to translate all index keywords, open the Index.liindexmap file from the File List window pane.

 

Columns:

  • Segment number:Displays the segment number.
  • Source: Displays the text in the source language.
  • Translation: Displays a copy of the text, which needs to be translated into the target language. You can click on the row and, in the local toolbar, select Translate>Suggest. In the dialog that opens, you can provide the translation, and when you are finished, that translated text will display in this cell.
  • All columns should be sortable and resizeable.

Controls:

  • UP/Down arrows: moves to the previous/next segment
  • Enter: moves to the next segment and changes the color to grey(accepted), however the translation is not actually uploaded to the TM if the source is the same as the target.
  • Alt+Down/Up arrow: Accepts the current segments translation, uploads it to the TM, and moves to the next UNTRANSLATED segment.

 

  • Right Click Menu: Right clicking any row should invoke sub-menu:
  • Revert to Source: Reverts the content in the target language column back to the content that is in the source language column.
  • Add Tags From Source: Adds all the tags that are in the content of the source language column to the target language side.
  • Add New Term: Opens the Add to Termbase dialogue window.

    • Upload Selected: Uploads the selected segment with a new translation to the translation memory

    Upload File: Uploads all new translated segments within the select file(s) to the translation memory.

    The visual mode displays two window panes. The text in the source language is shown in the left pane, and the text in the translated (or soon-to-be translated) language is shown in the right pane. The visual mode displays the file in the same way that the author edits it in Flare or Blaze, with content shown in complete headings, paragraphs, lists, and so on.

    Note You might notice light green or dark green rectangles in the topic with text inside. These are index and Concept keywords, which are not translated using the Topic Editor, but rather their own editors. You also might notice places in the topic where you cannot select text. This is likely a snippet or variable, which is inserted into a topic but not edited from there. Like index and Concept keywords, snippets and variables can be translated separately from topics.

    Note The bars that you see to the left of content are called "structure bars." They allow you to see the exact tag structure of the file as you work.

    Controls:

    • Source Topic This is the left window pane, which displays the file in the source language.
    • Translated Topic This is the right window pane, which displays the file in the target language.

     

    Translation Area

     

    The source(left) side is read only in the translation area.

    The translation(right) side is the only content that is editable.

     

    Controls:

    • Confirm Translation: this button accepts the current segments translation, uploads it to the TM, and moves to the next UNTRANSLATED segment
    • Down/Up Arrows: Move to the previous/next segment.
    • CTRL + UP/Down arrows: selects content from the previous/next segment without leaving the translation area.
    • Enter:accepts the translation and moves to the next untranslated segment and changes the color of the segment in the grid to grey(accepted).
    • Alt+Down/Up arrow: Accepts the current segments translation, uploads it to the TM, and moves to the next UNTRANSLATED segment.

    As you make changes in the translation area the content in the grid area should update as well.

    Test case: If the tags in the translation area match the source then you should be able to move to the previous or next segment. If they do not match the user should get an message saying the tags do not match.

     

    Right Click Menu: Right clicking any row should invoke sub-menu: 

     

     

    • Cut: Cuts out the selected content and copies it to the clipboard
    • Copy: Copies the selected content to the clipboard
    • Copy Xml as Text: copies the Xml code behind the content to the clipboard
    • Paste: Paste the stored content of the clipboard into editor.

    • Revert to Source: Reverts the content in the target language column back to the content that is in the source language column.

    • Add Tags From Source: Adds all the tags that are in the content of the source language column to the target language side.
    • Add New Term: Opens the Add to Termbase dialogue window.
    • Show Space: Toggle show space on/off. Shows the spaces in the content represented by a dot for each space.

     

    Suggestion Area

     

    The suggestion area shows matches based on segments in the translation memory. If it is an exact match it will show up at a %100 match. If it is a 'fuzzy' match such as 92% or 81% it will show up as well. This means that most of the segment matches but some of the words are different.

     

    You can select a match in the suggestion area and the content will appear in the translation area. You can use the shortcuts of Alt+# to quickly select a suggestion overwriting the content in the translation area with the suggestion content. Alt+CTRL+# will add the content of the suggestion to the content that is in the translation area vs overwriting it.

     

    If no translation memory is select it will say No translation memory in the suggestion area.

    Conflict with Translation Memory dialog

    This dialog opens if you have already uploaded translated text to a translation memory (TM) database, later modify that text, and then attempt to upload the new translation to TM. Lingo recognizes this conflict and gives you different options for handling it.

     

    To open:

    • Open an editor and modify the translation for text that has already been uploaded to your TM database.
    • Attempt to upload your changes to the TM database

     

     

     

    UI Controls

    • Source Segment: Displays the text in the source language.

    • Your Translation:Displays the translation that you have provided. You can click in this field and modify the translation, if necessary.
    • Translation Memory Version:Displays the matching translation that is already uploaded into the TM database.
    • Change Nothing:Select this option if you do not want to make any changes to the text you have modified or to the TM database. Your translation changes will be kept, and the old translation will continue to exist in the TM database.
    • Use your translation and overwrite translation memory:Select this option if you want to keep your modified translation and upload it to the TM database, replacing the old translation.
    • Use translation from TM:Select this option if you prefer to use the old translation that is already contained in the TM database. The text that you modified in the editor will revert back to the translation already contained in the TM database.
    • Apply to all conflicts:Select this check box if you want future conflicts such as this to be treated according to the option that you have selected.
