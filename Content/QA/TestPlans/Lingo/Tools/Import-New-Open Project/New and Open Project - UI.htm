UI

Start New Project Wizard-Enter name and folder for the translated project...

 

 

Controls:

 

  • Name: Type an appropriate name for your project.
  • Folder: By default, a path to the Documents\My Translated Projects folder on your hard drive is entered in the Folder field. (Lingo creates the My Translated Projects folder when you install the program.) All subfolders and files related to the translated project will be placed in this folder as you work on the project. If you want to have the project files placed in a different folder, use the "Browse" button.
  • Domain: You can type a subject category for your project, or select one from the drop-down. If you enter your own custom category in the Domain field, it will appear in the drop-down the next time you create a new project. Domain metadata appears throughout the project, such as when you select a suggestion in the translation memory.
  • Client:You can type the name of the client you are creating a translation project for. When you enter a client name in the field, it will appear in the Client drop-down the next time you create a new project. Client metadata appears throughout the project, such as when you select a suggestion in the translation memory.
  • Source language: Select the original language that is used in the project you are translating.
  • Target language: Select the language(s) that you want to use for the translation and click the right arrow to move it to the Selected Languages grid. If you need to remove a language from the Selected Languages grid, click the left arrow.
  • Back: This button is disabled in this window.
  • Next: Opens the next page of the wizard.
  • Finish:This button is disabled in this window.
  • Cancel: Closes the wizard.

 

Test:

  • Make sure that ‘Generate’ generates project name with the format ProjectName{n+}
  • Clicking ’Browse’ invokes the Browse For Folder dialog window.

 

 

Start New Project Wizard-Select files or project to translate...

 

Controls:

 

  • Add Files:Will open the ‘Select file(s) or project to translate’ dialog window.

    • If you need to select only one file, you can double-click it. If you need to select multiple files (e.g., multiple Word documents) in the same folder, you can hold your SHIFT or CTRL key and select either a range or individual files; then click Open.

    • If you want to add more files from another location, you can repeat the previous steps. Keep in mind that if you select MadCap project type files, you cannot also add other types of files (e.g. Word documents). However, if you have not selected MadCap project files, you can add any other types of files to the same Lingo project.

    • If you want to filter the translated files, you can create a file type filter. Use the Filter drop-down to select an existing filter, or select Create New to create a new one. You can add file type filters to XML, Word, or text files. See Lingo's Help for more information.

  • Add Folder:Will open the ‘Add Folder’ dialog window.
    • In the dialog that opens, navigate to the folder that contains the files that you want to translate and click Select Folder.
    • Note: If you select a folder that contains a MadCap project, such as a Flare or Mimic project, only translatable file types will display in the Translatable File Types dialog. For example, a Flare project may display Capture images (PNG or JPG files) and Flare topic files (HTM files), but not Flare glossary files (FLGLO files).
  • Remove: Will remove the selected file(s) or project.
  • Back: Opens the previous page of the wizard.
  • Next: Opens the next page of the wizard.
  • Finish: This button is disabled in this window.
  • Cancel: Closes the wizard.

Start New Project Wizard-Select Translation Memories to use for this project...

 

Controls:

 

  • Add:Click Add and from the drop-down select one of the following:
    • Local:Select this option if you want to select a TM that resides on your computer or network. Next to the Select existing Translation Memory field, click Browse. A dialog opens where you can locate the TM. Click Open.
    • Server:Select this option if you want to use a TM that resides on your SQL Server. This opens a dialog with fields for the name, server, and security information related to the database.
  • Create: Click Create and from the drop-down select one of the following:
    • Local:Select this option if you want to use SQLite for a local TM database. This option opens a simple dialog that lets you give the new database a name and choose where to store it. By default, the location for SQLite databases is in Documents\Lingo Translation Memories on your computer. However, you can choose a different location if you like.
    • Server:Select this option if you want to use SQL Server Express for a TM database to be stored on a server. This opens a dialog with fields for the name, server, and security information related to the database.
    • Note: When you are finished, the new database is shown in the grid and the Use check box is automatically selected.
  • Remove:Allows you to remove a particular TM from the grid.
  • Import:Allows you to import a TMX file with translations into a new TM database.
  • Access: Allows you to specify the access rights for each database.

    • Read & Write:By default all databases are set to Read & Write, which means they can be altered once they are loaded into the Lingo project.

    • Read Only:You can change any of the TMs to Read Only access if you want. In order to switch to Read Only access, the Use check box must first be selected for that TM.

  • Select Machine Translation Provider: You can select a machine translation service from the Select Machine Translation Provider drop-down, then enter your authentication credentials in the Settings area. Each provider requires different credentials.

    • Google: Enter your API Key in the field.

    • Microsoft: Enter your Client ID and Client secret in the fields.

    • MyMemory: Enter your Key in the field.

  • Pre-translate options: Automatically apply matching translations from a TM database

    • No pre-translation: Do not pre-translate the project

    • Pre-translate with TM: Apply matches from the translation memory to segments in the project

    • Pre-translate with TM then MT: Apply matches from the translation memory to segments in the project, then apply machine translation to any segments that are not translated

  • Back: Opens the previous page of the wizard.
  • Next: Opens the next page of the wizard.
  • Finish: Creates the project and loads the files into Lingo.
  • Cancel: Closes the wizard.

Open Project

 

Opening the Open Project Dialog:

  1. Click on in standard toolbar ! File | Open or...
  2. Click <CTRL><O>

  3. These will invoke the standard Windows Open Files Dialog with the filter set for *.liprj and *.lialign – options in the drop-down are “Lingo Packages (*.liprjzip; *.lialignzip; *.lidoc)”

  4. Double-clicking the *.liprj or *.lialign files in Windows Explorer

Note: Let's say you have already created a Lingo project for an earlier version of a particular Flare project. You have already translated it, exported it to a new Flare project, and given it to the author. Sometime after this, the author makes many changes to the Flare project (in the source language). Now the author needs you to translate those changes into the target language. When this occurs, you can create a "difference" project to translate the new version of the Flare project. This allows you to easily locate the differences between the old and new versions, rather than looking through each piece of text. If a file or string has been modified since the previous version, an icon displays next to that content (the type of icon depends on the type of change made). In addition, the "Difference" column in an editor's grid provides an indication of the change. You can then compare the two versions in order to determine what was modified and translate the changes as necessary. If the author has deleted content from the new Flare project, that content is automatically deleted from the new Lingo difference project when you create it.
