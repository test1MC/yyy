UI

New Database

 

Controls:

  • Name of new database: Type a name for the TM database.
  • Database server: Click Browse next to this field. The Connect to SQL Server dialog opens, displaying a list of usable servers after Lingo scans your network. Lingo supports SQL Server 2005 Standard and SQL Server 2005 Express. You can download SQL Server 2005 Express free from msdn.microsoft.com.
  • Location: If you want to select a different folder to store the database, click the Browse button next to this field and choose a location. The default location is C:\Databases.
  • Connection timeout: You can adjust the number of seconds to allow a connection to the server to be successful.
  • Authentication: In this section, you can use Windows Integrated Security (the default), or you can enter a specific user name and password for the server (if necessary).

 

Note: If you elect to use Windows Integrated Security, leave the user name and password fields blank. Otherwise, if your server requires specific credentials, enter them in the fields provided.

 

Installing a Translation Memory Database

 

Before you can take advantage of translation memory, you need to install a database—either the one provided by Lingo or a third-party system. Following are steps for installing the built-in Lingo TM database. For details on installing a third-party TM database (e.g., across) on your server, refer to the provider for instructions.

 

How to install the built-in Lingo TM database:

 

  1. Select Translate>Install Lingo TM. The New Lingo Translation Memory Database dialog opens.
  2. In the Name of new database field, type a name for the TM database.
  3. Next to the Database server field, click Browse.
  4. The Connect to SQL Server dialog opens(see next image), displaying a list of usable servers after Lingo scans your network.
  5. Select the server and click OK.
  6. (Optional) In the New Lingo Translation Memory Database dialog, you can click the Browse button next to the Location field and select a different folder to store the database (the default is C:\Databases).
  7. (Optional) In the Connection timeout field, you can adjust the number of seconds to allow a connection to the server to be successful.
  8. In the Authentication section, you can use Windows Integrated Security (the default), or you can enter a specific user name and password for the server (if necessary).

Current Translation Memory Provider Dialog

 

Drop Down: Selecting the drop down will show all recently used Translation Memories. Verify the list is accurate.

Selecting None will set Lingo to not use any TM database. (Suggestions will not be available)

Change…: Selecting Change… will bring up the Select Translation Memory dialog (shown below)

 

Select Translation Memory Dialog

This dialog will show you three options:

  • Lingo Built-in Translation Memory
  • Across crossAPI
  • (No Translation Memory)

Selecting None will take you back to the previous screen with “none” shown

Selecting Lingo Built-in Translation Memory will bring up the Connect to Lingo Translation Memory dialog (shown below)

Selecting across crossAPI will bring up the Across Translation Memory Settings (shown and described in the Across test plan)

 

Lingo Built-in Translation Memory Dialog

 

Controls:

  • Server: Click Browse next to this field. The Select Database dialog opens; displaying a list of usable servers and databases after Lingo scans your network. Lingo supports SQL Server 2005 Express, which you can download free from msdn.microsoft.com.
  • Database: Displays the name of the TM database that you select.
  • Authentication: In this section, you can use Windows Integrated Security (the default), or you can enter a specific user name and password for the server (if necessary).
    • Note: If you elect to use Windows Integrated Security, leave the user name and password fields blank. Otherwise, if your server requires specific credentials, enter them in the fields provided.
  • Install New Lingo Database: Opens the New Database dialog, which lets you create a new Lingo TM database.
  • Browse:Selecting Browse will bring up the Select Database dialog (shown below)

 

Select Database dialog

 

Controls: 

  • Server: Your local server and all servers on the local network should be shown here
  • Database:All the TM databases will be shown here for the selected server
  • Progress area: The blank bar at the bottom, will show connection information when attempting to connect to a Server
    • Click OK after selecting a Database, will populate the Server and Database field of the Connect to Lingo Translation Memory dialog
    • Clicking OK again at the Connect to Lingo Translation Memory dialog, will populate the server information in the Current Translation Memory Provider
    • Clicking OK again, will return you to Lingo. The selected TM database will now be used for uploading and suggestions with lingo
