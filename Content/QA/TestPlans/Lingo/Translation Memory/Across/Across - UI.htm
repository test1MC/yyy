Across - UI

Installing Across Client and API

  1. Get the installer from Craig or Jay, run the setup.exe
  2. Choose to install the Client, and the acrossAPI. You can uncheck everything else.
  3. During installation, you will be asked for a .CAP file. Grab the one in the link above and copy it locally. Point Across to that file.
    1. Password is: simon
    2. All other defaults are fine for installation.
    3. Across may patch itself during installation. That’s fine.

Opening the UI

The server is already set up, there is no Installing the TM. The Across server will be the TM and all Lingo needs to do is point to it.

UI

 

Picking a TM:Opens the Current Translation Memory Provider dialog (Choose Translation Memory provider)

 

  • Verify that the Current Translation Memory Provider dialog launches after selecting this option
  • Verify that this option is available with a project closed or opened.

 

 

 

 

 

 

 

Current Translation Memory Provider Dialog

 

Drop Down: Selecting the drop down will show all recently used Translation Memories. Verify the list is accurate.

Selecting None will set Lingo to not use any TM database. (Suggestions will not be available)

Change…: Selecting Change… will bring up the Select Translation Memory dialog (shown below)

 

Select Translation Memory Dialog

This dialog will show you three options:

  • Lingo Built-in Translation Memory
  • Across crossAPI
  • (No Translation Memory)

Selecting None will take you back to the previous screen with “none” shown

Selecting Lingo Built-in Translation Memory will bring up the Connect to Lingo Translation Memory dialog(Shown and described in the SQL Express test plan)

Selecting across crossAPI will bring up the Across Translation Memory Settings (shown below) 

Lingo Built-in Translation Memory Dialog

 

Controls:

  • Server: The across servers will be listed in this drop down (The .CAP file specified during across installation determines what is shown here)
  • User: The users for the server selected will be shown in this drop down. (The .CAP file specified during across installation determines what is shown here)
  • Password:Enter the password here that is associated with the User selected
    • After all is selected and a password is entered, hit OK. You should be returned to the Current Translation Memory Provider dialog 
    • Hit OK again to return to Lingo. The Across TM will be used for suggestions, dictionaries, and uploads.

Test Cases:

  • Verify when closing and reopening Lingo, the selected TM database is still selected
  • Verify when closing and reopening Lingo, the selected TM database still provides suggestions and allows uploads
  • Verify changes made on the Across server (definitions, suggestions, etc…) are reflected in Lingo
  • Verify when using Across as the TM the follow works in lingo:
    • Suggestions from the Across server are given
    • Dictionary items are shown (Items from Across as well as installed dictionaries)
    • Uploads work (and can overwrite!)
