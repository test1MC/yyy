Translation Memory Editor - UI

Opening the UI

1. Select Translate> Edit Lingo TM…

UI

Local Toolbar:

Buttons (Tooltips in Parentheses):

  • New Item: A new row is added to the bottom of the tab and is highlighted.
  • Delete: Deletes the selected row(s).
  • Filter: Filter for fast search in the TM. Use wild cards as necessary.
  • Language Filter: If more than one language exist it allows the user to select specific languages so only part of the TM is displayed thus improving performance. The language chosen upon creation of a new project is what the language filter in the translation memory will default too.
    • Test case: Test to make sure the language of the project is the default language selected in the TM.
  • Import TMX...(Import translation memory from TMX):Opens Import TMX File dialogue. *see below for more testing details
  • Export to TMX...(Export translation memory to TMX):Opens Export to TMX dialogue. *see below for more testing details.

How to edit a translation memory database:

  • Select Translate>Edit Lingo TM. The Translation Memory Editor opens.
  • You can do any of the following:

    • Add a segment:

      1. In the local toolbar, click the new item button. A new row is added to the bottom of the tab and is highlighted.
      2. Double-click on the new row. The Database Segment Editor opens at the bottom of the interface.
      3. Enter the text for the source language on the left, and enter the text for the target language on the right.
      4. Press CTRL+S or click to save your work.
      5. When asked if you want to accept the translation. Click Yes.

    • Edit a segment:

      1. Double-click the row that you want to edit or click the button. The Database Segment Editor opens at the bottom of the interface.
      2. Edit the text for the source language on the left, and edit the text for the target language on the right.
      3. Press CTRL+S or click to save your work.
      4. When asked if you want to accept the translation. Click Yes.

       

      Delete a segment:

      1. Click the row that you want to delete. A line appears through the middle of the text.
      2. Press CTRL+S or click to save your work.

       

      Import a Translation Memory eXchange (TMX) file:

       

      1. In the local toolbar, click Import TMX.

        The Import TMX File dialog opens.

      2. Click Browse.
      3. In the dialog that opens, navigate to the TMX file that you want to import, and double-click it.

      4. (Optional) By default, the current database is shown in the Translation Memory field. If you are working with other databases and want to use a different one instead, select it from this field.

        You might use this field, for example, if you forgot to change to the database you want before starting the import process.

      5. Click Start Import.

        A message lets you know the results of the import.

      6. Click OK.

      Export segments to a TMX file:

      1. In the local toolbar, click Export to TMX.

        The Export to TMX Dialog opens.

      2. Next to the TMX File field, click Browse.
      3. In the dialog that opens, navigate to the folder where you want to export the file. Also in the File name field, enter a name for the new TMX file. Then click Save.

      4. In the Languages to export field, all of the languages involved in your translation memory database are listed, showing both the source and target languages. Select the languages that you want to include in the exported TMX file.

      5. In the Source language field, select the language(s) that you want to be available as the source in the exported TMX file. You can select an individual language or you can select "Any," which allows tools to use translations in any direction in the TMX file.

      6. Click Start Export.

        A message lets you know the results of the export.

      7. Click OK.

     

    Translation Editor

    UI Controls

    • Columns

      • Source Language: Shows the source language represented by the first 2 letters in of the language in lowercase (i.e. English=en)
      • Source: Source text
      • Target Language: Shows the target language represented by the first 2 letters in of the language in lowercase in it's native spelling ( i.e. Spanish=es for Espanol)
      • Target:Target text(translated text)

  • Right Click Menu: Right clicking any row should invoke sub-menu:

  • Revert to Source: Reverts the content in the target language column back to the content that is in the source language column.
  • Add Tags From Source: Adds all the tags that are in the content of the source language column to the target language side.
  • Add New Term: Opens the Add to Termbase dialogue window.

    Translation Area

     

    Language picker: This allows you to change the source or the target language of an entry. You can not have the same source and target language.

    Test case: Test that you can change the language and that it is saved when you click to a different segment. Save the TM, close and reopen, and make sure the change was applied.

     

    Right Click Menu:

     

    • Cut: Cuts out the selected content and copies it to the clipboard
    • Copy: Copies the selected content to the clipboard
    • Copy Xml as Text: copies the Xml code behind the content to the clipboard
    • Paste: Paste the stored content of the clipboard into editor.

    • Revert to Source: Reverts the content in the target language column back to the content that is in the source language column.

    • Add Tags From Source: Adds all the tags that are in the content of the source language column to the target language side.
    • Add New Term: Opens the Add to Termbase dialogue window.
    • Show Space: Toggle show space on/off. Shows the spaces in the content represented by a dot for each space.

     

    Controls:

    • Up arrow: Move to previous segment (CTRL+Up arrow).

    • Down arrow: Moves to the next segment(CTRL+Down arrow).

      Note*: any changes in the TM are accepted automatically since the user is editing the database. Hence there is no accept button or accept shortcuts.

    • If the tags in the translation area match the source then you should be able to move to the previous or next segment. If they do not match the user should get an message saying the tags do not match.
