SQL Express

Bug Base Product Area:

  • TM SQL

 

Description:

One of Lingos method of using its Translation Memory, is to use SQL Express. The Translation Memory will install as an SQL database for use within Lingo. This test plan will show how to install and use the Translation Memory via SQL Express.

The purpose of Translation Memory is to assist the translator and make their job easier. The translator can upload translated segments to the TM and in turn the TM will offer suggestions for future translations.

 

Test Plans:

  • SQL Express - UI
  • SQL Express - Baseline

 

Dependencies:

  • SQL Express

 

Testing Resources (Copy all files to local Machine before importing):

  • Go to \\dc2\Software\Microsoft\SQL Express 2005\ and install the SQL2005EXPR32.exe file. Or Install SQL Express 2008.
  • You can install locally to your machine, or the test machine you are testing on.
  • Accept all defaults during installation.
  • Optional: Install SQL management express from \\dc2\Software\Microsoft\SQL MISC\ and install the SQLServer2005_SSMSEE.msi
  • SQL Management express is a utility that allows you to open up tables inside the Translation Memory database. It is not necessary for testing Lingo.
