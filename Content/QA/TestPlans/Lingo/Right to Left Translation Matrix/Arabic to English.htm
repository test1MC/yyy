Arabic to English

Instructions:

  • Prepare the source file/project in such way that it will make sense to translate (No gibberish, no non-sense)
  • Make sure to include duplicate segments in different parts of the document and in different files
  • Make sure to include similar segments in different parts of the document and in different files
  • Make sure to include images (when possible)
  • Create a Translation Memory in Lingo

 

 

Source Language: Arabic

Target Language: English

Create Project:

  • Make sure you can successfully create a Lingo project without crashes/errors

 

Translate:

  • The Source language should have its contents shifted to the right.
  • The contents in the Translation Area (Target Language) should look normal, all shifted to the left.
  • Make sure you can translate and accept content correctly. The grids should display the translated contents as you type
  • If you come across a segment which you've already translated before, you should get a 100% suggestion match.
  • If you come across a segment similar to one you've already translated before, you should get a fuzzy match.
  • The "Status" column in the 'File List' should update to "In Progress" after you translate and save

 

Update Project:

  • Add Content to the Source files
  • Delete Content in the Source files
  • Modify Content in the Source files
  • Add a new file (When possible)
  • Get the latest changes in Lingo by doing a project update
    • If you do a Project Update while a bundle is out for translation, you should get the following message:

      • "Some of the updated files are currently out for translation. A bundle with the updated versions will be shown momentarily. Please send this to the translators as soon as possible." (*)

  • Make sure all of the changes above show/work correctly

 

Prepare Bundle:

  • Select the changed/added files from above, right-click them and select "Prepare Translation Bundle"
  • Unzip the ZIP file and make sure the file(s) you selected are present
  • Open the files and translate them using Notepad. Translate the contents between the following tags:

    • state="needs-translation">TEXT TO TRANSLATE</target></trans-unit></body>

    • MadCap:diff="New">TEXT TO TRANSLATE</target></trans-unit></body>

  • Save and ZIP the files back up
  • Merge the ZIP files (Translated Bundle) back into Lingo
  • The 'Status' column should change to:
    • 'In Progress': If you partially translated the file
    • 'Completed': If you translated the entire file
    • 'Untranslated': If you didn't translate the file
  • Make sure to open the files in Lingo and make sure they're translated
    • Segments that are completely translated should show up as 100% matches in the 'Suggestions' area
  • Make sure to prepare a bundle and then do a project update. You should get the message above (*)

 

Compute Statistics:

  • Select some files from the 'File List', right click them and select "Compute Statistics"
  • Make sure the results are accurate

 

Save for Review:

  • Only HTM/HTML files should work with this feature
  • All other files should get the following message: "The current project contains no HTML files."
  • For HTM/HTML files, make sure to select Word/PDF options and verify that the contents display correctly

 

Upload Selected:

  • After translating all files, create a new Translation Memory
  • Right-click all files from the 'File List' and select "Upload Selected"
  • Open each translated file and/or the Translation Memory Editor
  • The translated content should be there

 

Apply TM to Project:

  • After you translated all the files, create a new Lingo project from the source files making sure to select the same Source/Target languages
  • Once the project is created, click Translate> Apply TM to Project
  • Open each file(s) and the contents should be translated

 

Export:

  • Click File> Export
  • Make sure the tooltip reads the correct application (i.e., if you are exporting a Flare project it should say: "Export translated files back to Flare")
  • Fill in the required fields and click "Export"
    • Make sure to test using a trial key
    • You should get a message if you try to export using a trial key
  • Once done, it should ask you if you want to open the exported project
    • Clicking "Yes" should open the project in its respective program (If available) (i.e., it should open the exported files in Flare, Word, PowerPoint, etc.)
    • Clicking "No" should close the message and leave the "Exporting" dialog open
  • Open the translated contents and make sure they're translated correctly
    • If you're using a trial key, the contents should be scrambled
    • Make sure to generate outputs (Word, PDF, HTML5, WebHelp)
      • The outputs should compile fine, no crashes
      • Verify that the translations are preserved (Contents should be translated and display all the way to the left margin)
