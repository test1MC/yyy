Crash Reports - UI

In order to open the UI, you will need to crash the application. Search the BugBase for an active crash bug.

UI Controls

  • Other actions…: Invokes a drop-down which lets you select the following options:
    • Provide additional information: Invokes the User Information dialog which lets users add personal information and a description of what they were doing when the exception occurred.
    • Report Details: Invokes the View Exception Log dialog which lets the user see what is being sent.
    • Save Log: Invokes the Save Exception Log dialog which lets you save the report locally as an XML file.
  • Send Report: Submits the report to MadCap and provides a confirmation message. The dialog is then dismissed and the application will continue to crash.
  • Don't Send: Dismisses the dialog and the application will continue to crash.

User Information dialog - UI Test Cases

UI Controls

  • First Name: Lets the user enter their first name if they want to include it in the report sent to MadCap
  • Last Name: Lets the user enter their last name if they want to include it in the report sent to MadCap
  • Email: Lets the user enter their email address if they want to include it in the report sent to MadCap
  • Description: Lets the user enter up to 1,024 characters to describe what was happening when the exception occurred

View Exception Log dialog: Exception Tab - UI Test Cases

UI Controls (Read Only)

  • Time: Displays the time the crash occurred to be included in the report sent to MadCap
  • Source: Displays the source of the crash to be included in the report sent to MadCap
  • Type: Displays the type of crash to be included in the report sent to MadCap
  • Message: Displays the message the user would have seen when the crash occurred to be included in the report sent to MadCap
  • Stack Trace: Displays the stack trace for the crash to be included in the report sent to MadCap

View Exception Log dialog: System Tab - UI Test Cases

UI Controls (Read Only)

  • Exe: Displays the application in which the crash occurred to be included in the report sent to MadCap
  • Exe Version: Displays the version number of the application in which the crash occurred to be included in the report sent to MadCap
  • Framework Version: Displays the Framework version to be included in the report sent to MadCap
  • OS Architecture: Displays the architecture of the user's OS (x64 or x32) to be included in the report sent to MadCap
  • OS Version: Displays the version of the user's OS to be included in the report sent to MadCap
