Performance - Testing

 

While testing Performance you must have the Task manager open to keep an eye on CPU and MEM usage.

Lingo, like any application, will have an upper limit of what it can do while keeping solid performance. This is mostly dependant on the end users machine, but Lingo should always be able to handle projects even if it has to slow down. This test plan will establish some thresholds while also stress testing the app with some very large projects.

 

Task Manager

  • 1) Right click the taskbar
  • 2) Choose Task Manager
  • 3) The performance tab shows CPU usage
  • 4) The Processes tab shows MEM usage

The Performance tab                                                               The Processes tab

                    

 

You may see the CPU spike to 100% during some functions. This is normal and OK.

  • The problem you want to look for is EXTENDED 100% CPU usage
  • Look for CPU usage when Lingo is apparently not doing anything. CPU activity should be accounted for. (i.e. If you’re importing a project, CPU should spike. If you’re just looking at a topic, CPU usage should be clear.)
  • MEM usage will fluctuate as well, but not nearly as much as the CPU. Any large spikes in MEM usage should be noted, and any large spikes in MEM usage that does NOT get freed up should be bugged. (i.e. MEM jumps up to 150k from 80k, but stays there and doesn’t drop back down to 80k)
  • As MEM usage goes up, the more severe the performance hit in theory. If Lingo’s performance is not impacted, note that in the bug
  • And of course if at any time Lingo becomes unresponsive, freezes, or crashes it’s a bad thing
  • Verify minimizing Lingo drops the MEM usage significantly

 

General Test Areas

Testers should always be keeping an eye on performance and stability. Below are some areas that are known for causing some issues with either sluggishness, or instability. Verify each of the following when working with the projects below:

  • Moving window panes around constantly
  • Massive amount of topics opened at once
  • Updating links in a massive project
  • Large content in a topic (like a very large table or picture)

Feel free to set your own thresholds. The above suggestions are arbitrary depending on the machine the tester is using. Find a large project where you actually DO break Lingo. Your machine may only need 50 topics or as many as 5000. At some point you will get Lingo to behave poorly and will have to decide if the cause is something to be concerned about.
