Getting Started Wizard - UI

Opening the Getting Started Wizard

By default, when Lingo is first launched, the Getting Started Wizard should open. There is an option in the Getting Started Wizard to show it on launch. If that option is turned off, the Getting Started Wizard can be opened by:

  • !File | Getting Started Wizard
  • In the Start Page, click the "Launch the Getting Started Wizard" option in the "Getting Started" section

Getting Started Wizard - UI Test Cases

  • Dialog title is correct
  • Spelling and grammar are correct
  • No duplicate accelerators
  • All accelerators function
  • Tab order is correct (forward and reverse orders)
  • Keyboard can navigate and change settings
  • Dialog is easy to use and understand
  • F1 and “?” function and the content is correct
  • Default options are displayed in the screenshot
  • Hovering over (New, Open, Align and Tutorials) get's them highlighted.
  • Hovering over the recent projects get's them underlined and blue colored.
  • Cancel, Esc, Red ‘X’ and ALT+F4 close the dialog without change
  • OK and Enter close dialog with changes
  • Dialog is resizable and redraws correctly
  • Dialog remembers setting and sizes from previous session
  • Dialog opens correctly on a dual monitor system
  • Clicking the icon in the upper left corner displays options:
    • Restore, Move, Size, Min., Max., Close – All work
    • Restore, Min., Max are disabled

UI Controls

  • New: This is the first from the left and takes the user to the "Start New Project Wizard" dialog.
  • Open: This is the 2nd option from the left and takes the user to the "Open" dialog to open an existing Lingo project.
  • Align: This is the 3rd option from the left and it takes the user to "Madcap Align" editor to create an alignment project.
  • Tutorials: This is the 4th option from the left and it takes the user to "Movies" page of Lingo's help and it displays as a tab within Lingo.
  • Recent Projects Which consists of a list of the recent projects created or opened by lingo sorted by order starting from the very recent opened or created project.
    • When minimizing the "Getting started wizard" there should be at least 4 projects listed with their names on the left and path on the right.
    • When clicing a project that's no longer exists in the defined path, a message below should appear notifying the user that the project cannot be found




  • Show this dialog on launch: If there is a check mark in the box, this wizard will open each time you open Lingo in the future. If you do not want to see the Getting Started Wizard each time you open Lingo, remove the check mark from the box.
  • Cancel: Closes the Wizard.
