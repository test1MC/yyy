Menu Options

 

  • New: Opens the Start New Project Wizard, which you can use to create a new Lingo project.
  • Update: Update Project allows you to compare a project that you have translated previously with a new version of the project.
  • Open: Opens a dialog, which lets you search for and select a project to open.
  • Reload: Returns an open file to its previous state, discarding any unsaved changes if you instruct it to.
  • Startup: File Opens the Startup File dialog, which lets you change the file that is automatically opened when you launch the application.
  • Project Properties: Opens the Project Editor, which displays basic information about the project (e.g., project path, source language, target language).
  • Close: Closes the active file.
  • Close Project: Closes the current project, but the application itself does not close.
  • Save: Saves changes to the active file.
  • Save All: Saves changes to all of the open files.
  • Backups: Opens a submenu, which lets you select options related to creating temporary backups of your files.
    • EXAMPLE: You might use this feature, for example, if you want to make some changes to a file but are not certain if you will want to keep them when you are finished. Therefore, you can make a backup of the file in question. When you are finished making your changes, you might decide that you prefer the file the way it was. If so, you can quickly roll back to the backup file, which overwrites the changes that you made.
      • Send Copy to "Backups Folder": Creates a temporary copy of the file that is open and active in the interface. The file is placed in the Backups window pane (File>Backups>Backups).
      • Clear the "Backups Folder": Removes all temporary backup files from the Backups window pane.
      • Backups Opens: the Backups window pane so that you can see the files for which you have created backups and restore (roll back) any of those files if necessary.
  • Export to Flare: Opens the Export Translated Project dialog, which lets you create a copy of your translated project in Flare or Blaze. This enables the author to open the translated project and build output for it.

  • Send To: Opens a submenu, which lets you open the active file in another application (e.g., Dreamweaver, Word, Notepad), attach it to an email, or open the Windows folder where it is stored.

     

  • Recent Files: Opens a submenu, which lets you select and open one of your most recently opened files.

     

  • Manage Recent Files: Opens the Recent Files dialog, which lets you remove files from the list of recently opened files. This allows you to keep the list shorter and more manageable. When you use this feature, the file is merely removed from the list, but the actual file is not deleted.

     

  • Recent Projects: Opens a submenu, which lets you select and open one of your most recently opened projects.

     

  • Manage Recent Projects: Opens the Recent Projects dialog, which lets you remove projects from the list of recently opened projects. This allows you to keep the list shorter and more manageable. When you use this feature, the project is merely removed from the list, but the actual project is not deleted.

     

  • Exit: Closes the project as well as the application itself.

Menu Options:

 

  • Undo: Lets you reverse the most recent action in a file. The undo and redo functions are enhanced in the following ways:
    • Unlimited: You can undo or redo an unlimited number of times, back to the last time you opened the project, even after you have saved the document.
    • Multiple buffers: Each document in the interface has its own Undo/Redo buffer. This means that if you undo actions in one document and then switch to another document and undo actions in that document,
    • User-friendly interface: The name of the action is displayed on the Undo or Redo button. In addition, you can use the drop-down arrow next to the button to select multiple steps to undo or redo at once, even resizing the drop-down box as necessary.

     

  • Redo: Lets you repeat the most recent action that you reversed in a file. Cut Cuts the selection and sends it to the clipboard. Copy Copies the selection and sends it to the clipboard. Paste Pastes the selection from the clipboard to the place where the cursor is inserted. Delete Deletes the selection. Select All Selects all content in the file. Find and Replace: Opens a submenu, which lets you choose one of several "Find and Replace" options.
    • Find and Replace: Opens the Find and Replace window pane, which lets you search for and replace text in your documents.
    • Find in Files: Opens the Find in Files window pane, which lets you display a list of files containing text that you search for.
    • Load: Find Selection Copies the selected text in the editor to the "Find what" field in the Find and Replace window pane or the Find in Files window pane.
      • Example: If you highlight the text "Please find this" in a document and then select this option, the words "Please find this" will be loaded into the "Find what" field so that you can perform a search for it
        • Find Next: Initiates the search for the next instance of the search term.
        • Find Results 1: Opens the Find Results 1 window pane, which is used to display results from a search in the Find in Files window pane.
        • Find Results 2: Opens the Find Results 2 window pane, which is used to display results from a search in the Find in Files window pane.

         

  • Properties: Opens the Properties dialog for the selected item. Insert Translation Replaces selected content with a matching translation in the Suggestions window pane (if you are using translation memory).

Menu Options:

 

  • Toolbars: Opens a submenu, which lets you select a toolbar to open or close.
  • Standard: Opens or closes the Standard toolbar. Tools in the Standard toolbar let you perform basic functions, such as Save, Cut, Copy, Paste, Undo, and Redo. There are also shortcut buttons for performing source control functions, as well as sending documents to other applications (e.g., opening a topic's true code in Notepad). To see this toolbar, select View>Toolbars>Standard.
  • Switch to Visual/Grid Mode: Toggles between grid mode and visual mode if you have a file open in the Topic Editor.
  • File List: Opens the File List window pane, which lets you view and open files in the project so that you can translate the content.
  • Backups: Opens the Backups window pane so that you can see the files for which you have created backups and restore (roll back) any of those files if necessary.
  • Suggestions: Opens the Suggestions window pane, which displays matching translations that you have already created and uploaded to a translation memory database.
  • Dictionary: Opens the Dictionary window pane, which displays matching translations from a dictionary that you have installed on your computer.
  • Start Page: Opens the Start Page, which is a handy window pane that displays when you first launch MadCap Lingo and remains open unless you choose to close it. This window pane consists of three sections (Recent Projects, Getting Started, and News and Updates), which contain links to high-level tasks and information.
  • Messages: Opens the Messages window pane, which displays important information about your project as you work. For example, if a necessary file is missing, this window pane lets you know.
  • Info Bar: Hides or shows special information pertaining to the editor or window pane. Status Bar Opens or closes the status bar at the bottom of the program window.
  • Status Bar: Hides or shows the status bar.

Controls:

 

  • Minimize to System Tray: By default, when you minimize Lingo, a shortcut button on the main part of the taskbar lets you reopen the application. However, you can select this check box if you want to minimize Lingo to the system tray instead. With this option selected, a shortcut icon is placed in the system tray (i.e., notification area) of the taskbar when Lingo is minimized. You can click on this icon and select Restore to reopen the application.
  • Auto-Reload Documents: Select this check box if you want Lingo to reload documents automatically. In other words, if you have topics or editor pages open and then close the project, this option ensures that those same documents are reopened the next time you open the project.
  • Auto-Save Documents: Select this check box if you want Lingo to periodically auto-save any open files. You can specify the number of minutes that should elapse between each auto-save.
  • Open files with these extensions…: If you want certain types of files to open automatically in the Document Dock pane, enter those file extensions in this field, separating each one with a semi-colon (e.g., xml;text;asp). The Document Dock pane is an empty window pane that is docked by default on the left side of the Lingo interface. You can use this pane to display any type of document in that window pane, instead of displaying the document in its current location.
  • Close others when opening new document: You can have multiple files of all types open in the interface at the same. By default, when you open a new file, other documents that are already open remain open. However, you can change this behavior so that files already open in the interface are automatically closed when you open a new document.
  • You can use any of the following auto-close options:
  • Close others: This option automatically closes all other open documents.
  • Close others: with no changes This option automatically closes all other open documents that do not have unsaved changes. If you have files open where changes have been made but not saved, those files will remain open until you save and close them.
  • Close others: with same extension This option automatically closes all other open documents that share the same file extension.
  • Close others: with same extension and no changes This option is the same as the previous option, except that files with unsaved changes will not automatically be closed. Those files remain open until you save and close them.
  • Reset "Show Once" Dialogs:There are some dialogs in Lingo that open just once to give you a message or warning when something changes or you switch modes. These dialogs have a check box labeled "Show this dialog." If you deselect that checkbox, you will not see the dialog again. However, you can use this reset feature to display that dialog (and all other "Show Once" dialogs) again the next time.

Controls:

 

  • Retrieve suggestions with a minimum match of: Enter a percentage in the field. Only translations with a rank greater than or equal to the specified percentage will be displayed as suggestions in the interface.
    • EXAMPLE: Let's say that you set this percentage to 50%. When using translation suggestions, you might have one translation in your database that matches the current string exactly (100%), another translation that matches it 75%, and a third translation that matches it only 30%. In this case, only the first two translations will be available for you to select in the interface.
  • Fill-in fuzzy matches of at least: Enter the percentage at which you can insert matching translations automatically. By default, this percentage is set at 100%, meaning you can use automatic inserts only when text strings match existing translations exactly. However, you can reduce this percentage if you wish. Any matching translation that falls below that specified percentage is considered "fuzzy," because it does not match the string closely enough to warrant an automatic insert.
    • EXAMPLE: Let's say that you open a topic to translate the text in it. You press ALT+S to open the Suggestions window pane. In that pane, a few matching translations from your translation memory database are shown. Perhaps two of the translations match segments exactly (100%) and there are other partial matches that are below 100%. Perhaps one of those is a 98% match. All of the partial matches, including the one at 98%, are considered "fuzzy." When you first opened the Suggestions window pane, a message displayed. It might say something like this: "2 segments had matches of at least 100% and there were 3 other partial matches. Do you want to use the top matches automatically?" If you select Yes, the two translations that match text 100% will replace the corresponding strings in the topic automatically. If you want to use any of the partial matches, you must do so manually. Now suppose you want to be able to automatically use matching translations if they match 97% of the string text. Therefore, you set the fuzzy matching option to 97%. If you then return to the same topic and press ALT+S, this time the message will say the following: "3 segments had matches of at least 97% and there were 2 other partial matches. Do you want to use the top matches automatically?" If you select Yes, both of the 100% matches are automatically used, as is the 98% match (because it is no longer considered a "fuzzy match").
  • Automatically apply translations to the rest of the file: Click the check box if you want to instantly apply translated strings to any duplicates within the current file. After entering the first translation of a term, the other instances of that same term will also be translated.

Menu Options:

 

Close All Documents: Closes all files that you have opened in the workspace.

Close All Documents Except This One: Closes all files that you have opened in the workspace, with the exception of the document that is currently active.

Dockable: If you select the "Dockable" option when a window pane is active, the Drag Pane area is shown or hidden in the window pane accordingly. The Drag Pane area lets you move the window pane and dock it elsewhere in the program window.

Auto Hide: "Pins" the active window pane or editor to the side of the interface so that you have more room for your workspace. You can show the window pane by hovering over the thin bar on the side of the interface. If you select the "pin" button at the top-right of the window pane, the window pane returns to its previous state (no longer "pinned" to the side of the interface).

Floating: Converts the active window pane or editor to a "floating" window, which means it is no longer "attached" to the interface. This is a good option to use if you want to work on two topics side-by-side. If you do not want a window pane or editor to be "floating" anymore, you can right-click in the title bar and select Floating.

Layouts: Opens a submenu, which lets you select options for creating and selecting layouts. A layout is a particular configuration of the workspace.

 

 

Auto-save Window Layout: Automatically saves the layout of the workspace when you exit Lingo. The next time you launch Lingo, that same layout will be displayed.

Lock Window Layout: Freezes the layout so that the window panes and editors cannot be moved floated or docked to different areas of the interface.

Save Window Layout: (Default) If you are working in a particular layout and have opened different interface elements or moved interface elements around, you can select this option to save the changes to the layout.

Save Window Layout: As Opens the Rename Window Layout dialog, which lets you save the current interface configuration as a layout.

Reload Window Layout: If you are working in a particular layout and have opened different interface elements or moved interface elements around, you can select this option to return to the saved configuration of the layout.

Reset Window Layout: Returns the current configuration of the interface to the factory layout settings (the original layout when you first installed and launched Lingo).

Layouts: Opens the Manage Window Layouts dialog, which lets you select or delete an existing layout.

Select Layout Opens a submenu that lists the layouts in your project. Select a layout to display it in the interface.

Windows: Opens the Manage Windows dialog, which lists the files that are currently open in your project and lets you activate (i.e., switch to), save, or close a particular file.

Document Dock: Opens the Document Dock pane. This is an empty window pane that is docked by default on the left side of the Lingo interface. You can use this pane to display any type of document in that window pane, instead of displaying the document in its current location.

Send To Other Dock After you open the Document Dock pane, you can move the active document to that window pane from its current location.

Menu Options:

 

  • Contents: Opens the Help TOC window pane, which lets you locate and access topics in Lingo's online Help. Browse Sequences Opens the Help Browse Sequences window pane, which lets you browse through related groups of topics in Lingo's online Help.
  • Index: Opens the Help Index window pane, which lets you search for information using the index in Lingo's online Help.
  • Search: Opens the Help Search window pane, which lets you search for information in Lingo's online Help. Favorites Opens the Help Favorites window pane, which lets you quickly access your favorite Lingo Help topics.
  • Glossary: Opens the Help Glossary window pane, which lets you view glossary terms and definitions in Lingo's online Help.
  • Help: Topic Opens the Help topic related to the active element in the Lingo interface.
  • New Help Topic Window: Opens a new instance of a Help topic in another window.
  • Dynamic Help: Opens the Dynamic Help window pane, which automatically displays the Help topic related to the active element in the interface.
  • Recent Comments: Opens the Recent Comments window pane, which lets you view the most recent comments submitted by readers of the Lingo online Help.
  • Topic Comments Opens the Topic Comments window pane, which lets you submit community-wide comments on any aspect of the Lingo online Help. You can also reply to comments from other readers.
  • Locate in TOC: Lets you manually synchronize the current Help topic with the Help TOC. In other words, the TOC book containing that topic also opens to show you where the topic resides in the TOC hierarchy.
  • Auto-Sync TOC: Automatically synchronizes the current Help topic with the Help TOC. In other words, when a particular Help topic is displayed, the TOC book containing that topic also opens to show you where the topic resides in the TOC hierarchy.
  • Index Results: Opens the Index Results window pane, which displays links to related topics after you perform a search in the Help index.
  • Help Community: Opens the MadCap Forums website, where you can obtain additional information about Lingo and communicate with other Lingo users.
  • Guides: Opens a submenu, which lets you access one of the printed guides related to Lingo.
  • Video Tutorials: Opens a topic that provides access to video tutorials in the online Help. This option is not available in some MadCap products.
  • Register Product: Lets you register your version of Lingo if you have not done so already.
  • Check for Updates: Opens the New Updates Available dialog, which lets you download Lingo updates from MadCap.
  • About: Displays a popup that shows information about the current version of Lingo. Click the popup to make it disappear.
