Uploading to YouTube and Vimeo

Starting with Mimic v8, there is a new feature in Mimic which allows a user to upload/publish their movies onto their Youtube/Vimeo account. It’s a pane called “Video Services” (!Tools | Video Services).

 

Process

  1. Login – the login process uses OAuth (open authentication) protocol. Users should go through the instructions to authorize their accounts for access.
    1. Test Accounts for QA to use that has already been set up with allowing Mimic to integrate with their API's
      1.  For YouTube:

        2. Password - R3m3mb3r!
      2. For Vimeo:
        1. Username - test1@madcapsoftware.com
        2. Password - R3m3mb3r!
    2. Instructions for setting up new accounts - t/c
    3. After logging in once, we store the refresh tokens that allow for auto login later on. The login dialog also has a “Manual Access Token Entry” which allow for users to manually enter in their Access Token/Token Secret. (This is applicable for Vimeo uploading)
  2. Basic API Calls
    1. Retrieve username – the user name should be displayed on the pane once the user has authenticated
    2. Retrieve uploaded videos – the video pane should display a list of previously uploaded videos (title, thumbnail, uploaded date)
      1. This can be called manually with the “Refresh” button at the bottom of the pane
  3. Uploading
    1. The metadata dialog should display giving users parameters and data to attach to their video
    2. The video pane should display the video and automatically update the progress until the video is finished uploading
      1. If it does not display right away, try hitting “Refresh”
    3. Once the video is finished, the video should be “viewable”, the context menu opens an option to view the video. (The actual video may still be processing on the Youtube/Vimeo servers)
    4. Uploads should be resumable. To test this, disconnect the machine from the network in the middle of a download. Wait until the progress displays, failed. Then reconnect. Then right click the failed download and hit “Retry/Resume Upload”. If the progress is gets stuck for a while, try using this option as well.
    5. Check to see that the metadata got properly applied in YouTube/Vimeo
  4. Drag/Drop – drag and dropping videos into the Videos window should automatically pull up the metadata dialog with the filepath pre-filled to allow for quick upload queuing
  5. Integration with “Publish To Html5’”. The Publish to Html5 dialog should have Youtube and Vimeo as destinations now. Selecting those will change the options on the bottom to Metadata. Hitting “Start publish” will automatically open up the Video service pane, login, and queue up the Mimic html5 movie (single file format).
