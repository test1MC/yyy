Audio - Output Specifics

Silverlight

Supporting WAV sounds in Silverlight output requires Windows Media Encoder (WME) to be installed. We package this with Mimic, and the user will be prompted when compiling a Silverlight movie that contains a WAV file. Below are the conditions that relate to this feature.

  1. If user compiles non-Silverlight output OR .wav sounds are NOT used, the compilation proceeds normally.
  2. If user compiles Silverlight output AND .wav sounds are used:l
    1. If users machine already has Windows Media Encoder installed, compilation will proceed as norma
    2. If users machine does not have WME installed:
    3. If user does not want to install WME, compilation proceeds and .wav sounds are ignored
    4. If user wants to install WME:
    5. If user is running Vista, both WME and a Vista Hotfix must be installed
    6. If Mimic has the installers, user is guided through the installation processes
    7. If mimics installers are missing or corrupt, user is given the web addresses to download the installers.
    8. If user is not running Vista, only the WME must be installed
    9. If Mimic has the installers, user is guided through the installation processes
    10. If mimics installers are missing or corrupt, user is given the web address to download the installer.

Flash

Our Flash output has a couple of areas the tester should be aware of.

wav to mp3 conversion

If a Movie has a .wav file and outputs to flash, the compiler needs to convert the sound to .mp3. The Flash player in Mimic versions 1 through 3, could not do.mp3 conversions at all on Vista (and subsequently Win7) machines. This is due to Windows Vista/7 no longer containing a built in mp3 converter. Mimic 4 introduced the ability for Mimic to lean on any 3rd party mp3 converters to do this job. See Required Third Party Installs for information on installing a recommended codec. Below are related test cases:

 

  • Test compiling a movie in flash, with a .wav and an .mp3 sound file , on clean Vista / Win 7 machine. (No mp3 codec)
    • Our compiler should prompt accordingly
  • Test compiling a movie in flash, with a .wav and an .mp3 sound file, on a Vista / Win 7 machine that has an mp3 codec.

Frequency

Our flash compiler needs standard frequency rates on the mp3 and wav files it uses. That does not mean it wont work with other frequencyies, we just say Mimic only supports the standard ones.

  • Test using wav & mp3 files that 44.100khz
  • Test using wav & mp3 files that 22.050khz
  • Test using wav & mp3 files that 11.025Khz
  • Test using wav & mp3 files that use other frequencies. If we dont convert correctly, we should prompt correctly during the compile.
