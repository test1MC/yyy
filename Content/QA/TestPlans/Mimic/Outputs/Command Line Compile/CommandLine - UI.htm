Command Line - UI

There is no UI for Command Line Compile within the mimic application. This Test Plan will focus on the process of Command Line Compile from windows.

Initial Setup

Initial Setup is a 2 step process:

  1. Open the command window
  2. Change directory to the mimic.app folder

 

To access the command prompt:

  1. Windows Key + R or !Start | Programs | Accessories | Command Prompt
  2. Type CMDin the open field

Note: Typing Command instead will open the 16-bit command prompt. You don't want that, unless you like pain.

 

To navigate to the mimic.app folder:

  1. Type: cd “c:\program files\madcap software\madcap mimic V4\mimic.app”

 

Syntax

Console.mimiccompiler.exe -file <project or movie> -output-type <mcmovie, flash, silverlight, xps, pdf> -variables <variable file>

  • -file: This flag is required. Following this flag, should be the path to the mimic movie or project to compile
  • -output-type: This flag is optional. If the flag is not used, the output type that is set in the mimic movie or project will be used. Any output type mimic supports can be specified here.
  • -variables: This flag is optional. IF the flag is not used, the variables used in the mimic movie or project will be used. If this flag is used, the variables in the mimic movie or project will be overwritten. To use this flag, the path of the variable file you wish to use needs to be specified.
  • -target: This flag is used to specify a specific mimic target for compiling
  • -recompileThis flag is used to force a recompile of a movie that has not changed since its last compile.Note: When this flag is not specified, Mimic will NOT recompile a movie it thinks has not changed since its last compile.
  • -lazy [true/false]This flag was the original-recompile. We do not advertise this exists and we don not put in the help. True or False needs to be supplied. By default,-lazy trueis the behavior a command line compile. This will keep movies from recompiling if there has not been a change. Supplying-lazy false, will force a recompile. This option only exists for legacy support for any user that may have used it, which is unlikely.

Examples

-File

Compiling a movie named Test Movie:

Type: Console.mimiccompiler.exe –file “c:\documents and settings\jwulkowicz\my documents\my mimic movies\testmovie.mimovf\testmovie.mimov”

Hit enter, and the movie will compile

 

 Verify the movie compiled by locating the output folder in the Movie or project folder.

-Output-type

Adding to the above step

Using the -output-type flag to compile TestMovie in silverlight format:

Type: Console.mimiccompiler.exe –file “c:\documents and settings\jwulkowicz\my documents\my mimic movies\testmovie.mimovf\testmovie.mimov” –output-type silveright

Output Types:

  • Silverlight
  • mcmovie
  • Flash
  • PDF
  • XPS

-variables

The variable flag is used to specify an external variable file

Step one(a): Create an .mcvar file for a project

  • Open notepad
  • Add the following:

<?xml version="1.0" encoding="utf-8"?>

<VariableSet>

<Variable Name="Project.Bob">Cat</Variable>

</VariableSet>

 

  • Bob is the variable name. Cat is the definition. Change to whatever variable you are using in your mimic project
  • Save as an .mcvar file
  Step one(b): Create an .mcvar file for a movie
  • Open notepad
  • Add the following:

<?xml version="1.0" encoding="utf-8"?>

<VariableSet>

<Variable Name="movie.Bob">Cat</Variable>

</VariableSet>

 

  • Bob is the variable name. Cat is the definition. Change to whatever variable you are using in your mimic movie
  • Save as an .mcvar file
Step two: Run the command line compiler

Type: Console.mimiccompiler.exe –file “c:\documents and settings\jwulkowicz\my documents\my mimic movies\testmovie.mimovf\testmovie.mimov” –variables “c:\mimictest.mcvar”

  • Check the output to verify the variable file was used

-target

The target flag is used to specify a movie or projects particular target to be compiled.

Assuming you have a target named "TargetFlash":

Type: Console.mimiccompiler.exe –file “c:\documents and settings\jwulkowicz\my documents\my mimic movies\testmovie.mimovf\testmovie.mimov” –target TargetFlash

Multiple targets can be compiled at once
