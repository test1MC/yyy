General

Configure general settings.

 

  • Site URL: Enter the site URL. Changing this doesn't work.
  • Restricted Word List: click Show or Hide to see the list.  Type restricted words, separate with commas.
  • Customer Group: Enabled customer access to Pulse.
  • Group Sync: Enabled group sync.
  • SMTP Host: Enter the name or IP address of the SMTP Host.
  • SMTP Port: Enter the SMTP port.
  • SMTP SSL: Enable SMTP SSL.
  • SMTP Username: Enter the SMTP username.
  • SMTP Password: Enter the SMTP password.
  • Incoming Mail Server Type: Select from the dropdown menu: Pop3, Exchange Web Services or Poll the Local SMTP Drop Folder.

  • Incoming Bad Mail Folder: Enter the path of the local server folder to hold raw incoming messages that have failed processing.

  • POP3 Host: Enter the name or IP address of the POP3 Host.

  • POP3 Username: Enter the POP3 username.

  • POP3 Password: Enter the POP3 password.

  • POP3 Poll Task Frequency in Minutes: Enter the POP3 poll task frequency in minutes (0=off, 1=1 minute).
  • POP3 Enable SSL: Check to enable SSL.
  • Notification Email Type: Select from the dropdown menu: Static, Domain or Plussing. Static, for a single email address that doesn't change, and no support for posting to a group via email. Domain, if you have a whole domain you'd like Pulse to control. Plussing, if you have one email address you'd like Pulse to manage but would like to be able to post to groups via email. The last two require support from your email server.

  • Notification Email Address: Type a single email address from which notifications and registration emails are sent.

  • Notification Email Display Name: The name that you want to appear as the sender in email notifications sent by Pulse.

  • Activity Purge Task Frequency in Days: The frequency at which Pulse will run message archive task. When messages reach the age defined in the Message Archive Task Max Message Age in Day field, Pulse archives the messages and they will not display any longer in the user's task pane. 0=off, 1=1 day, 2=2days, etc.

  • Activity Purge Task Max Message Age in Days: The maximum age a message can be until it is archived. After the established days, all messages will be archived and will not display any longer in the user's messages pane.

  • Account Sync Period: The number of minutes the authentication providers will sync with Pulse.

  • Always Use SSL: Place a checkmark in this box to redirect calls for http:// to https://.

  • Windows Auto Login IP Ranges: Define the range of IP addresses that will be permitted to automatically login to Pulse using windows credentials. If a user attempts to login from an IP address outside of the defined range, Pulse will prompt for credentials.

  • Mobile Client Persistent Login: Place a checkmark in this box to enable mobile client persistent login.

  • Video Encoding: Place a checkmark in this box if you have setup an encoding server. When this feature is enabled Pulse can support additional video formats (.mov, .3gp, .wmv, .avi) in addition to MP4. Requires a licensed copy of Microsoft Expression Encoder or Handbrake.

  • Log Retention: Specify the number of days to retain the error message log on the server.

  • Content Purging: Place a checkmark in this box to enable content purging.

  • Direct Messaging: Select where direct messaging should be enabled in Pulse.

  • Files: Select where files should be enabled in Pulse.

Test Cases

  • Test restricted words by typing them in a comment/discussion/status, etc.  You should get the message "You may not use the word 'xxxx'".
  • Deselect "Customer Groups" and all customers should have Pulse access revoked.
  • Change Log Retention to 1, you should see the logged error messages removed after 1 day.
  • Turn off Direct Messaging and Files, you should not be able to direct message or post a file.

Groups

Add new groups or modify settings for existing groups. Group Types are Administrators, Customer, Employees.

Manage Groups

In the Select a group to manage box, type part or all of a group name. Place a checkmark next to the group that you want to manage, then click Manage Group.

  • Basic:
    • Name: The name of the group.
    • Description: A description to help identify the group.
    • Group Type: Sets access permission for group member.
  • Membership: Will list all members currently in this group. 
    • To remove a user from a group, deselect their name.  
    • To add a user to a group, type their name, select it.
  • Advanced:
    • Host Site Url: type the URL of the site that hosts Pulse. (Our documentation says it is strongly recommended not to change the default setting)
  • Save: Saves changes and returns to the Select a group to manage page.
  • Cancel: Cancels any changes and returns to the Select a group to manage page.
Add a Group

  • Name: The name of the group you would like to create.
  • Description: A description to help identify the group.
  • Create Group: Closes the dialog and creates the group.
  • Cancel: Closes the dialog without creating a group and returns to the Select a group to manage page.

You can delete a group that you have made (not Administrators, Customer or Employees) by selecting the group to manage, and clicking the remove button.

Test Cases

  • Pulse should only allow one Administrators group.
  • Only Administrators can create and delete groups.
  • Add a user to multiple groups - the level of system access corresponds to group with the highest permissions. For example, if a user belongs to both the Employees and Customers group, the user will be granted all of the permissions for the Employees group.
  • Remove a user from all groups - the level of system access should be the same as the Customer group

Suggested Feeds

In the Select a Suggestions List to Manage box, place a checkmark next to the Suggestions List that you want to manage to enable the Configure and Remove buttons.

  • Configure: Opens the selected feed to configure.
  • Remove: Removes the selected feed.
  • Add a Suggestion List: Opens a blank configuration page.

 

Configure Suggested Feeds

  • Name: Type a name for the feed.
  • Automatic Follow: Select whether users should be automatically subscribed as followers to the new feed. You can select Yes or No.
  • Suggest To: Select the audience to whom you are making the suggestion. You can select Administrators or Employees.
  • People: Type part of the person's name you want to search for and then place a check mark to select it.
  • Groups: Type part of the group's name you want to search for and then place a check mark to select it.
  • Tags: Type part of the tag's name you want to search for and then place a check mark to select it.
  • Ok: Creates the feed, returns to the Select a Suggestions List to Manage page.
  • Cancel: Does not create the feed, returns to the Select a Suggestions List to Manage page.

Users

Manage Users

In the Select a user to manage box, type part or all of a user name. Place a checkmark next to the user that you want to manage, then click Next.

  • User Details:
    • First Name: The first name of the user.

    • Last Name: The last name of the user.

    • Email Address: A valid email address for the user.

    • Title: A title for the user, such as a professional or job title.

    • Department: A department name for the user, such as Engineering or Sales.

    • Location: A location for the user, such as La Jolla or New York City.

    • Cell: A mobile phone number for the user.

    • Phone: A telephone number for the user.

  • Save: Saves changes and returns to Select a user to manage page.
  • Cancel: Discards changes and returns to Select a user to manage page.
  • Disable/Enable: Toggle between disabling and enabling the user.
  • Delete: Delete the user so they will be unable to sign into Pulse.
Invite a User

 

  • First Name: The first name of the user.
  • Last Name: The last name of the user.
  • Email Address: A valid email address for the user.
  • Title: A title for the user, such as a professional or job title.
  • Department: A department name for the user, such as Engineering or Sales.
  • Location: A location for the user, such as La Jolla or New York City.
  • Cell: A mobile phone number for the user.
  • Phone: A telephone number for the user.
  • Custom Message: Type a message here, it will appear in the email invitation above the standard Pulse Invite content.
  • Group: Select the group(s) for this user to be associated with.
  • Send Invite: Sends the invitation.
  • Cancel: Closes the dialog without saving changes.
Import Users

  • Choose File: Opens the Open dialog to browse for the user data file.
  • Send invitation email to each user: Selected by default, each user will receive an invitation email.
  • Invitation email text input box: Says "Welcome to Pulse" by default, you can edit this text.
  • Import: Enabled once a user data file has been selected, will import the user.

Log

 

Filters:

  • Show All: Select this item to reveal all logged events.

  • Critical: Select this item to reveal a list of critical events that have occurred. Critical errors can lead to system disruption/failure, so it is important to take corrective actions.

  • Error: Select this item to reveal a list of events that can potentially lead to system disruption/failures, so it is import to locate errors and determine if preventative or corrective action is needed.

  • Warning: Select this item to view a list of events that are not significant, but may indicate of a future issue.

  • Information: Select this item to show events that completed successfully.

  • Verbose: Select this item to reveal more detailed information about the issue. This is a helpful option to look at should you need to perform troubleshooting.

  • Query: Type a search term.
  • Search: Submits the search.
  • Reset: Resets the search query.
  • Next page: Loads the next page of log events.
  • Previous page: Loads the previous page of log events.

Columns Shown in the log:

  • EventType: The type of event. For example, critical, error, or warning.
  • CreatedUTC: The date and time steps the event occurred.
  • MachineName: They Pulse web server name.
  • Url: Click to open a URL with more information.
  • ProcessName: Shows the name of the process.
  • ExceptionType: Lists events that required special processing.

Synonyms

Directional

  • Word: Type the word that would be used in the search query (ex: dog).
  • Synonym: Type the synonym that would be included in the search results of the previous term (ex: poodle).
  • Stem: Select to include all forms of the search word (ex: dogs, doggies)
  • New: Creates a new empty row for you to add a new synonym.
  • Delete: Deletes the row.
  • Save Changes: Prompts you to save changes.
    • Ok: Saves changes.
    • Cancel: Closes dialog without saving changes.
  • Discard Changes: Prompts you to discard changes.
    • Ok: Discards changes.
    • Cancel: Closes dialog without discarding changes.
  • Export (.mcsyns): Downloads a copy of the synonym file in the MCSYNS file format which can be opened in Flare.
Groups

  • Group: Type the words in the synonym group (ex: dog=canine,doggy,doggie,hound,pooch,pup,puppy).
  • Stem: Select to include all forms of the search word (ex: dogs).
  • New: Creates a new empty row for you to add a new synonym group.
  • Delete: Deletes the row.
  • Save Changes: Prompts you to save changes.
    • Ok: Saves changes.
    • Cancel: Closes dialog without saving changes.
  • Discard Changes: Prompts you to discard changes.
    • Ok: Discards changes.
    • Cancel: Closes dialog without discarding changes.
  • Export (.mcsyns): Downloads a copy of the synonym file in the MCSYNS file format which can be opened in Flare.
