Pulse Configuration Wizard dialog 2: Select Web Site

UI Controls

  • Select the Website for the Pulse Services: Select the website on the IIS Server that will be storing your Pulse data. By default, Pulse will use the "Default Web Site" in IIS.

Pulse Configuration Wizard dialog 3: Select SQL Server

UI Controls

  • SQL Server: Click the dropdown and select (Get list of SQL Servers...). The progress dialog will appear and look for all available SQL Servers in the Windows domain (must have SQL Server Browser service turned on)

    To turn on SQL Server Browser:

    1. Start SQL Server Configuration Manager.
    2. Select Start > Programs.  Then point to your version of SQL in the menu and select Configuration Tools > SQL Server Configuration Manager.
    3. In the left pane, highlight SQL Server Services.
    4. Right-click SQL Server Browser and select Properties from the context menu.
    5. On the Service tab, set the start mode to Automatic and click OK.
    6. Right-click the SQL Server Browser again, and select Start.
    .  Once the list is generated, the progress dialog will close.  Now when you click the drop down, you will see a list of all available SQL Servers.
  • Connect: If you do not select a SQL Server from the drop-down, but instead type one in, you must click Connect to get the database location from the SQL server.
  • Database Location: If you select a server from the drop-down, this field will populate automatically.  You can also manually type a location or browse for it.
  • Browse...: Opens the Browse For Folder dialog, allows you to browse to a new location for the database to be stored.
  • Manually set SQL authentication settings: When selected, the fields below will be editable. In most cases, you will only select this if you server is located on a different machine on your network.
    • Authentication: Drop-down options are: Windows Authentication and SQL Server Authentication
    • Domain: Type your domain here.  If you have selected SQL Server Authentication, this field will be grayed out.
    • User name: Type your user name here
    • Password: Type your password here (characters will display as asterisks)
  • Back: Proceeds to the previous page of the wizard
  • Next: Proceeds to the next page of the wizard
  • Cancel: Exits the Configuration Wizard

Pulse Configuration Wizard dialog 4: Database Options

UI Controls

  • Create new: This creates a new database instance with Pulse.
  • Use Existing Database: Select this and Retrieving List of Databases progress dialog will appear. Once the scan has completed, the Select Database dialog will appear.
  • Backup path for local SQL database: This field is only enabled if the database has been using a previous version of pulse. You can click the browse button to chose a location to store a copy of the current database.  Note, this is not enabled if you are connecting to a database on a remove server.
  • PulseAdmin Password: Type your password here (characters will display as circle bullets). The password strength will be indicated below in red, changing from Blank - Very Weak - Weak - Medium - Strong - Very Strong - Excellent.  Increase your password strength by increasing the length of password, or adding numbers or special characters.
  • Back: Proceeds to the previous page of the wizard
  • Next: Proceeds to the next page of the wizard
  • Cancel: Exits the Configuration Wizard

Pulse Configuration Wizard dialog 5: Configure SMTP

UI Controls

  • Smtp Server: This is the IP address of the SMTP relay that accepts incoming requests which is the host that will send outbound emails for Pulse. QA- For internal testing, type "exchange07"
  • Smtp Port: This is the port number of the SMTP server handing outbound emails. This is Port 25 (or the port number for your environment).
  • Smtp User Name: This is the username of a valid account on the SMTP relay.

  • Smtp Password: This is the password for the valid account on the SMTP relay.

  • From Address: This is the from address for emails sent by Pulse.

  • From Display Name: This is the name that displays in email client applications for the email address. For example, type: Pulse Registration.  If you leave this blank, emails will not include a name - they will display only the From Address.

  • Enable SSL: A check mark in this box enables Secure Socket Layers (SSL) encryption.

  • Test...: You will see an alert "Test message has been sent" or "Error sending message."  If the test is successful, you should receive the test email (with correct name and email) which has subject line: MadCap Pulse SMTP Test, and message "SMTP server settings are OK." If there is an error sending the message, you will need to correct the SMTP settings you entered.
  • Back: Proceeds to the previous page of the wizard
  • Next: Proceeds to the next page of the wizard
  • Cancel: Exits the Configuration Wizard

Test Cases

  • Should receive error "From address cannot be empty" if you click Next without typing a From Address
  • Should receive error "Invalid email address: "<address you typed>"" if you type an address with an invalid domain
