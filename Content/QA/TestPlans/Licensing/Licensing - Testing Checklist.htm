Licensing - Testing Checklist

  • Test across all supported OS's, 32 and 64 bit versions, and Foreign versions of the OS's. (Win XP, Vista, Win 7)
  • Make sure 66-character keys from previous application versions don't work
  • Test using both single-app and MadPak keys, and make sure MadPak keys don't work for applications they weren't generated for
  • Uninstalling the application should free up an activation
  • Make sure we preserve the license key when reactivating after a deactivation. Note, this is different from activating after uninstalling/reinstalling (license key should not be preserved here)
  • Test that we give you a warning message when trying to deactivate while offline
  • Verify that adding time to a subscription works properly, both before and after the subscription has expired
  • If there are 30 days or less remaining on a subscription, the user should get a warning message once per day which counts down properly
  • Make sure all dialogs are accurate both pre-expire and post-expire, and the buy now links are to the proper SKU
  • Make sure you are able to activate a license key behind a proxy
  • Make sure activations work properly in a Citrix-based environment
  • Make sure old but still valid flare keys work even if the current date is past the expiration date of the original security certificate

Perpetual License Key

Single-User Key

With Non-Expiring Key

  • Can't activate key on more than two machines at a time (one machine per domain or up to two machines off domain)
  • Deactivating the key should free up an activation, which can be used on a new machine
  • Once a key has been freed up and activated on a new machine, should not be able to activate using that key on the original machine
  • Able to activate key on a machine it was previously activated on but never deactivated from even if you are at your activation limit (i.e. if you are running a new parallel image or replaced your hard drive)

With Subscription Key

  • Can't activate key on more than two machines at a time (one machine per domain or up to two machines off domain)
  • Deactivating the key should free up an activation, which can be used on a new machine
  • Once a key has been freed up and activated on a new machine, should not be able to activate using that key on the original machine
  • Key counts down properly and expires at the end of the subscription date
  • Able to activate key on a machine it was previously activated on but never deactivated from even if you are at your activation limit (i.e. if you are running a new parallel image or replaced your hard drive)

With Expired Subscription Key

  • Can't activate using an expired subscription key
  • The application should close after attempting to activate with an expired subscription key
  • Once a subscription key is expired, you cannot uninstall/reinstall or turn back the clock to renew the key
  • If the machine is offline when the subscription expires, and the machine's clock indicates it is past the subscription expiration date, the application should display a warning message and then close
  • If the machine is offline when the subscription expires and the machine's clock does not indicate it is past the subscription expiration date, the application should try to ping the server every 24 hours. The next time the machine is online and successfully pings the server, the application should display a warning message, save all your work and then close

Floating License Key

Single-User Key

With Non-Expiring Key

  • Can't have more than two concurrent users with the same key (one session per domain or up to two sessions off domain)
  • Able to activate key on an unlimited number of machines
  • Can't run application without an active internet connection
  • After starting an active session, the application should try to ping the flare server every 10 minutes. If it is unable to reach the server, the application should display a warning message, save all your work and then close

With Subscription Key

  • Can't have more than two active sessions using the same key (one session per domain or up to two sessions off domain)
  • Key counts down properly and expires at the end of the subscription date
  • Able to activate key on an unlimited number of machines
  • Can't run application without an active internet connection
  • After starting an active session, the application should try to ping the flare server every 10 minutes. If it is unable to reach the server, the application should display a warning message, save all your work and then close

With Expired Subscription Key

  • Can't activate using an expired subscription key
  • The application should close after attempting to activate with an expired subscription key

  • When trying to run an instance of the application with an expired key, the application should display a warning message and then close

  • Once a subscription key is expired, you cannot uninstall/reinstall, deactivate/reactivate or turn back the clock to renew the key

Evaluation Key

  • Key counts down properly and expires at the end of the evaluation period
  • Deactivate/Register options from !help should be greyed out while using an evaluation key

Trial Extension Key

  • Key properly resets the trial period to the appropriate length, depending on the product. For example, Flare has a 45 day trial period while Analyzer, Capture, and Contributor have 40 day trial periods
  • Proper trial behavior (such as scrambled content) continues
  • Key works after the original trial period has expired
  • Key works while the original trial period is still ongoing
  • Key has no expiration date and can be used anytime to reset the trial period
  • Key can be used to extend the trial for any MadCap product, but once used cannot be used to extend the trial for any other MadCap products
  • Can't use the same key to extend the trial multiple times
  • Extended trial properly expires at the end of new trial period

Blacklisted Key

  • Can't activate with a key that has been deactivated. Check for all types of both Perpetual and Floating license keys
  • When trying to run an instance of the application with a deactivated key, the application should display a warning message and then close
  • Make sure you can un-blacklist a blacklisted key, and that it functions as normal afterwards

Proxy Server Activation

  • Note: Check with IT to confirm the proxy address, port, username and password
  •  
  • If already activated, deactivate the product to be tested
  • Properly Set up the use of a proxy server by going into IE>Internet Options>Connections>Lan Settings and select the check box under proxy server and then enter 'linuxbox' as the address and 3128 as the port
  • Apply those settings and try navigating to a web page. The server shall ask for credentials and use 'madcap ' as the username and 'R3m3mb3r!' as the password.
  • Re launch the product to be tested, enter a valid license key. At the activation method screen select Internet Activation and click on the 'Use Proxy Server' check box and then press on the next button.
  • Enter the proxy server address ('linuxbox') the port number ('3128') 
  • Click on Enable Authentication to enable the text boxes for Username and Password
  • Enter 'madcap' for the username and 'R3m3mb3r!' for the password and then hit the next button.
  • The product shall use the proxy server to communicate with MadCap's activation server and successfully receive the proper credentials to continue.
  • Clear the activation and repeat the previous steps this time using an incorrect password to use to proxy server. The lack of proper credentials shall not allow license activation.
