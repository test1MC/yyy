Contributor - UI

Review Package Pane

!File | Open | *.fltrev

 

The review packages should open in the review packages pane. You can have more than one review package open at a time

UI Controls

  • Save: Saves current changes
  • Save all: Saves changes to all topics in all review packages
  • Return to Sender: Create email message with review package attached (only works if originally sent in the same manner)
  • Show/hide columns: Select which columns are displayed
  • Open: Open the selected topic in the XML editor
  • Switch to TOC View: Switches to display the TOC structure of the project

 

Review Package Pane TOC View

UI Controls

  • The TOC structure shown should be the same as it appears in the project
  • You should not be able to interact with topics in the TOC that weren't included in the review package
  • Test functionality with all file-types that can be included in a TOC book or node
  • Topic properties changed while in TOC view should persist when switching back to grid view (and vice versa)

 

Opening Contributor Documents

!File | Open | *.mcco, *.mcdoc

 

contributor documents will open like any normal topic does in the XML editor. test this the same as you would when testing the Flare xml editor but keep in mind that Contributor has more limited functionality. When unsure about a behavior or limitation, compare to the previous version of Contributor (unless it's a new feature)

Creating Contributor Documents

!File | New

 

That will open the following dialog. If creating a document using a custom template:

  • Select New from existing
  • Select a contribution template (*.mccot, *.mcdoct)
  • Click OK
  • Verify all template resources (images, styles, conditions, etc.) are included

OR if using a default template

  • Select New from template
  • Select a factory template
  • Click OK
