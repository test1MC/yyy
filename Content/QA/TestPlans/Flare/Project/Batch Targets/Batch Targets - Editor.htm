Batch Target Editor - UI Test Cases

Local Toolbar

Button (Tooltips in Parentheses):

  • Build and Publish (Build and Publish this Batch Target): Builds and/or publishes all Targets in the target list that have the "Build" and/or "Publish" option toggled on.
  • Build Only (Build this Batch Target): Builds all Targets in the target list that have the "Build" option toggled on.
  • Publish Only (Publish this Batch Target): Publishes all Targets in the target list that have the "Publish" option toggled on.

Targets Tab

UI Controls

  • Batch Targets grid: Displays all Target files in the project and lets you associate them with the Batch Target (for building and/or publishing).

    Columns

    • Target: Lists all Target files in the current Flare or Blaze project.
    • Output Type: Displays the output type of corresponding Target.
    • Build: Select this toggle box to build the corresponding Target according to the Batch Target's schedule.
      • Checkbox: Click to Select/Deselect All targets to build.
    • Publish: Select this toggle box to publish the corresponding Target according to the Batch Target's schedule.
      • Checkbox: Click to Select/Deselect All targets to publish.
    • Last Built: Displays the date and time that the Batch Target last triggered the linked Target to build.
    • Last Published: Displays the date and time that the Batch Target last triggered the linked Target to be published.

Test cases

  • Double-clicking a row opens the selected Target file in the Target Editor.

Schedule Tab

UI Controls

  • Yes: Creates a scheduled task and populates Task Name and Task Action fields in the Schedule tab of the Batch Target Editor
  • No: Does not create a scheduled task and returns to the Schedule tab of the Batch Target Editor

UI Controls

  • Task Scheduler: Displays the status of the Task Scheduler.
  • Task Name: Displays the name of the task, this should be "MadCap Flare Batch Build (#).job"

  • Task Action: Displays the command line command that will be run by the Task Scheduler. It should be something like this:

    C:\Program Files\MadCap Software\MadCap Flare V8\Flare.app\madbuild.exe -project "C:\Users\mferguson\Documents\My Projects\BatchTargets\BatchTargets.flprj" -batch "NewBatchTarget" -minimize true -log true

    If the Batch Target resides inside of a folder use the following commands:

    -batch "Folder Name\NewBatchTarget"

  • Start: ??? Not sure what this does ??? (Bug 50269 to remove or do something about this)
  • Delete Task: Deletes the task (the *.job) file, completely removing it from the Task Scheduler. You will then be prompted to create a new task the next time you open the Batch Target Editor and select the Schedule tab.

  • Triggers grid: Displays all Triggers.

    Columns

    • Trigger:
    • Details:
    • Status:
  • New... : Opens the New Trigger dialog which allows you to create a new trigger for the Batch Target and set
  • Edit... : Opens the Edit Trigger dialog which allows you to edit trigger settings
  • Delete... : Deletes the selected trigger.

New/Edit Trigger dialog:

UI Controls

  • Settings: Lets you set the frequency for the Trigger.
    • One Time: Set the Trigger to fire off the batch build one time only.
      • Start: Sets the date and time that the Trigger should begin to take affect.
    • Daily: Sets the Trigger to fire off the batch build on a daily frequency.
      • Start: Sets the date and time that the Trigger should begin to take affect.
      • Recur every: Lets you set the Trigger to fire off at an interval of a particular number of days.
    • Weekly: Sets the Trigger to fire off the batch build on a weekly frequency.
      • Start: Sets the date and time that the Trigger should begin to take affect.
      • Recur every: Lets you set the Trigger to fire off at an interval of a particular number of weeks.
      • List of Days: Lets you set the day(s) of the week on which the Trigger should fire.
    • Monthly: Sets the Trigger to fire off the batch build on a monthly frequency.
      • Start: Sets the date and time that the Trigger should begin to take affect.
      • Months: Lets you identify in which months the Trigger should fire.
      • Days: Lets you select which number day(s) within each selected month that the trigger should fire.
      • On: Lets you set the trigger to fire off on the 1st, 2nd, 3rd, 4th or last week on any particular weekday within the selected month(s).
  • Advanced Settings: Lets you set additional frequency settings
    • Repeat task every: Lets you set the trigger to repeatedly fire off using a minute or hourly frequency.
    • for a duration of: Lets you set the amount of time the repeated triggering should occur. This option can be set in minutes, hours or days.
    • Expire: Lets you set a date after which the Trigger will stop firing off.
    • Enabled: Deselect this toggle switch to temporarily disable the Trigger.

Note that all options in the New Trigger/Edit Trigger come directly from the Windows Task Scheduler.

Misc

Code

Default Batch Target (no associated targets):

<?xml version="1.0" encoding="utf-8"?>
			<CatapultBatchTarget Version="1" 
			Comment="New Batch Target">
			<Targets>    
			</Targets>  
		</CatapultBatchTarget>
 

Batch Target with two targets (one set to Build and the other to Build and Publish):

<?xml version="1.0" encoding="utf-8"?>
			<CatapultBatchTarget
			Version="1"
			Comment="New Batch Target">
			<Targets>
			<Target
			Name="MyWebHelp"
			Type="WebHelp"
			Build="true"
			Publish="true" />
			<Target
			Name="PDF"
			Type="PDF"
			Build="true" />
			</Targets>
		</CatapultBatchTarget>
