Relationship Tables and Relationship Proxies

Bug Base Product Area:

  • Relationship Tables

 

Description:

An element used in DITA files to link related topics together in online output. It is similar to concept links or Related Topics links. If you import DITA files that already contain relationship tables, those tables are retained in Flare. You can also create new relationship tables in a project and insert proxies into individual topics. In online output, the end result is a link that let users quickly open related topics.

The relationships defined in the Relationships Tables are converted to relationship links in the outputs. In order for these links to appear in a topic, the topic needs to have a Relationships Proxy in it. The Relationships Proxy can be added to a Master Page (and applied to every topic when the project is compiled) or added directly to a topic.

 

In the TOC Relationship tables use the filenames. This is actually by design and was broken in V5. We're not supposed to use TOC titles ever, except in the TOC and breadcrumbs.
