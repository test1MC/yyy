Relationship Tables - Output

  • The relationships defined in the Relationships Tables are converted to relationship links in the outputs. In order for these links to appear in a topic, the topic needs to have a Relationships Proxy in it. The Relationships Proxy can be added to a Master Page (and applied to every topic when the project is compiled) or added directly to a topic.

Target Editor setup:

  • By default, all outputs will use all Relationship Tables that exist in the project. If you have multiple Relationship Table files, you can designate specifically which files should be included in the output in the Target Editor, in the Relationship Table tab:

 

UI Controls

  • (Checkbox): Lets you set whether or not the Relationship Table should be included in the output. Deselecting all Relationship Tables will automatically include all files in the output.
  • (Column) Relationship Table File: Displays a list of all Relationship Table files that exist in the project.
  • Edit Relationship Table: Opens the selected Relationship Table file in the Relationship Table Editor.
  • New Relationship Table: Open the Add File dialog

Support in DITA Output:

  • When including a Relationship Table in DITA output, we are essentially copying the Flare Relationship Table to the output DITA map file. After generating DITA output, open the resulting DITA map in Notepad and you should see the same code that you see in the Flare Relationship Table file.
  • The exception to this would be Conditional Tags. Conditional Tags, set up appropriately in the Flare project will be converted to the corresponding DITA attribute. Conditional Tags that are not set up for DITA output will simply be dropped in DITA output. This is covered more in depth in the DITA output test plan. For the purposes of this test plan, realize that most Conditional Tags will simply be dropped.

 

Example (Flare Relationship Table code):

  <relcell>
    <topicref href="/Content/Concepts/Concept1.htm" MadCap:conditions="Default.PrintOnly"/>
  </relcell>
  <relcell>
    <topicref href="/Content/Tasks/Task1.htm" MadCap:conditions="Audience.Beginner" />
  </relcell>
  <relcell>
    <topicref href="/Content/References/Ref1.htm" MadCap:conditions="Audience.Advanced" />
  </relcell>
 

Will become (DITA output code):

<map>
  <topicref href="Content/StartTopic.dita" navtitle="StartTopic" />
  <topicref href="Content/Example.dita" navtitle="Example" />
  <topichead navtitle="Concepts">
    <topicref href="Content/Concepts/Concept1.dita" navtitle="Concept1" />
  </topichead>
  <topichead navtitle="Tasks">
    <topicref href="Content/Tasks/Task1.dita" navtitle="Task1" />
  </topichead>
  <topichead navtitle="References">
    <topicref href="Content/References/Ref1.dita" navtitle="Ref1" />
  </topichead>
  <reltable>
    <relheader>
      <relcolspec type="concept" />
      <relcolspec type="task" />
      <relcolspec type="reference" />
    </relheader>
    <relrow>
      <relcell>
        <topicref href="Content/Concepts/Concept1.dita" />
      </relcell>
      <relcell>
        <topicref href="Content/Tasks/Task1.dita" audience="Beginner" />
      </relcell>
      <relcell>
        <topicref href="Content/References/Ref1.dita" audience="Advanced" />
      </relcell>
    </relrow>
  </reltable>
</map>
