Relationship Tables - Editor

Opening a Relationship Table File

  • Target Editor (any): In the Relationship Table tab of the Target Editor, select a Relationship Table file and click the "Edit Relationship Table" button.

Relationship Table Editor - UI Test Cases

Local Toolbar:

Button (Tooltips in Parentheses):

  • (Hide/show Conditional Indicators): Toggle on/off Conditional Tag indicators.
  • (Open linked file): Opens the linked Topic file in the XML Editor.
  • (New item): Opens the Open File dialog which lets you select a topic to use for a new Relationship Table item.
  • (Group selected items)/(Ungroup selected items): Creates a Group out of the selected items. If a Group is selected, lets you ungroup the items.
  • (Run Properties Dialog for this item): Opens the Properties dialog for the selected item.
  • (Run the Cell Properties Dialog for this item): Opens the Cell Properties dialog for the current cell.
  • (Run the Row Properties Dialog for this item): Opens the Row Properties dialog for the current row.
  • (Run the Column Properties Dialog for this item):Opens the Column Properties dialog for the current column.
  • (Move item to the left): Moves the selected item to the left
  • (Move item to the right): Moves the selected item to the right
  • (Move item up): Moves the selected item up
  • (Move item down): Moves the selected item down
  • (Insert a new row): Adds a new row to the Relationship Table
  • (Delete selected row(s)): Deletes the row from the Relationship Table
  • (Insert a new column): Adds a new column to the Relationship Table
  • (Delete selected column(s)): Deletes the column from the Relationship Table
  • (Shows the current warning, if any): Displays any warnings (duplicate nodes, etc).

Relationship Tables - General Editing Test Cases

Drag and Drop

Click on a topic from the Content Explorer and drag it into a cell.

  • You should not be able to drag non-Topic files into a Relationship Table.
  • You should not be able to drag Project files into a Relationship Table.

Click on a folder in the Content Explorer and drag it into a cell.

  • If you drag a folder which contains sub-folders into a Relationship Table, only files in the top-level folder will be added.
  • You should not be able to drag a folder into a Relationship Table if it contains at least one non-Topic file.
  • You should not be able to drag a folder from the Project Organizer into a Relationship Table.

 

Undo/Redo works and commands display correctly in the Standard Toolbar.

 

*There is currently no drag/drop editing functionality within the Relationship Table Editor. Only dragging topics from the Content Explorer is allowed.

Row:

  • Click in any cell in the row and click the Delete Row button in the Local Toolbar
  • Right-click in any cell in the row and select Delete Row

Column:

  • Click in any cell in the column and click the Delete Column button in the Local Toolbar
  • Right-click in any cell in the column and select Delete Column

Tooltips

There is currently no tooltip functionality within the Relationship Table Editor.

Conditional Tags

Conditional Tags applied to the Relationship Table file can only be used for the purposes of Global Project Linking (Flare/Blaze project importing). That will be covered in another test plan.

 

Conditional Tags applied to items in a Relationship Table serve to exclude the relationship in the output. It will not condition the linked topic out of the output.

 

Condition Tags cannot currently be applied to Relationship Table cells, rows or columns using Flare or Blaze. However, it is possible for a relationship table created from a DITA import to have condition tags applied to all of these.
