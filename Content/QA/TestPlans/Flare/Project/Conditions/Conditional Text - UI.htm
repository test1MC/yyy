Conditional Text - UI

Applying a Condition Tag

A condition tag can be applied to any file in a Flare project, and folder in the Content Explorer, and any element in a topic. They can also be applied to certain elements in certain files (for example a TOC node or a Relationship Table entry). Each use case for condition tags should be covered in the Test Plan for that module, so it is not necessary to test condition tags in every possible context for the purposes of this test plan. However, a few of them are shown below as a sample.

Applying a Condition Tag through Drag-and-Drop

  • In the Project Organizer, open the Conditional Text folder and expand the appropriate condition tag set
  • Click on the condition tag and drag it to WYSIWYG selections, elements on the block/span bars, or entries in the TOC editor or Browse Sequence editor.

Applying a Condition Tag to a file

  • Select any file in the Content Explorer or Project Organizer and view its Properties.
  • Select the Conditional Text tab

Applying a Condition Tag to an element in the XML Editor

  • Right click an element and select Conditions
  • Right click an element's block bar and select Conditions

Applying a Condition Tag to text in the XML Editor

  • Select some text in the XML Editor and click !Format | Conditions (Standard Toolbar), or go to !Home | Conditions (Ribbon)

Other place where Condition Tags can be applied

Condition Tags can be accessed in the Properties

  • File List
  • Content Explorer
  • TOC Editor
  • Target Editor
