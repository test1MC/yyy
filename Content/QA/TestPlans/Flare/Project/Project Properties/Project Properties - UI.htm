Project Properties - UI

Opening the dialog

  • In the Project Organizer, right-click the Project folder and select Project Properties…
  • Toolstrip interface: !Project | Project Properties…
  • Toolstrip interface: Click the Project Properties… button in the Project Toolbar
  • Ribbon interface: !Project | Project Properties

Any changes made in the Project Properties dialog should automatically get saved to the project file when you click OK to close the dialog. You should never have to save any changes you make in this dialog.

Project Properties dialog - UI Test Cases

General tab

UI Controls

  • Name: Displays the name of the project. This field should not be able to be edited.
  • Location: Displays the location of the project.  This filed should not be able to be edited.
  • Open Folder: Opens the Windows folder holding the project.
  • Read Only: If you do not want the project file (the actual *.flprj) to be edited, click the check box to add a check mark.
  • Size: Displays the size of the project file.
  • Created: Displays the date and time that the project file was created.
  • Modified: Displays the date and time that the project file was last modified.

Defaults tab

UI Controls

  • Master TOC: Displays the Master TOC for the project. If you do not select a TOC, Flare will use the first one (alphabetically) in the project (if there is more than one).
  • Master Page Layout: Displays the Master Page Layout for the project. If you do not select a page layout file, Flare will use the first one (alphabetically) in the project (if there is more than one).
  • Master Stylesheet: Displays the path of the Master Stylesheet for the project. If you do not select a stylesheet, Flare will use the first one (alphabetically) in the project (if there is more than one).
  • Edit: For each item above, the corresponding Edit button opens the selected file in the appropriate editor.

Language tab

UI Controls

  • Hide languages with no spell check support: Reduces the list of visible languages to those which have spell check support.

  • (Language list): Lets you set the spell check language for the project. This language also determines which, if any, language skin should be used when generating web outputs.

    Columns

    • Language: Lists all available languages. Spell check supported languages display in bold.
    • Localized Skin: Indicates whether or not a localized skin exists for the corresponding language.

Source Control tab

UI Controls

Source Control Provider:

  • Provider Name: Displays the name of the source control provider (should be tested in the Source Control Test plan).
  • Database: Displays the name of the database file (should be tested in the Source Control Test plan).
  • Database Folder: Displays the name of the folder holding the database file (should be tested in the Source Control Test plan).
  • Advanced: Open an Advanced options dialog for certain source control providers (should be tested in the Source Control Test plan).
  • Bind Project: Opens the Bind Project dialog which lets you select a source control provider (should be tested in the Source Control Test plan).
  • Unbind Project: Unbinds a project that is bound to source control (should be tested in the Source Control Test plan).

Project File:

  • Source Control: A drop-down menu which lets you perform source control actions (check in, check out, etc) on the actual *.flprj file.
