TOC

Bug Base Product Area:

  • TOC

 

Description:

When your end users need to find specific information in your project, the three most common methods they use are: the search feature, a table of contents (TOC), and an index.

 

You can create a TOC by adding books and items with links (to topics, external files, other Help systems, movies, etc.) in a structure that you think would be useful for the individual. End users then browse through a TOC to find information.

 

In many cases, Flare provides you with an initial TOC, which you further "build" (or create) using the TOC Editor. You can use this TOC as your primary, or "master," TOC. At some point, you may decide to add another TOC to the project. The extra TOCs that you add can then be linked to the master TOC.

 

For print-based output (Adobe PDF, XHTML, Microsoft XPS, Microsoft Word, Adobe FrameMaker), you need to use the TOC Editor to create a TOC just as you would create one for online output (DotNet Help, HTML Help, WebHelp, WebHelp Plus, WebHelp AIR, WebHelp Mobile). However, there is a fundamental difference. Performing this task for online output creates an actual TOC in the output, which people use to navigate from topic to topic. This is not the case for print-based output. Performing this task for print-based output lets you indicate which topics will be included in the output and in what order. In that sense, this TOC functions more as an outline for print-based output. Therefore, for print-based output, you can think of it as an "outline TOC." If you want to include a generated TOC in print-based output, you need to use a TOC proxy in a topic instead
