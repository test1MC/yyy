TOC Entry - UI

(TOC node) Properties - UI Test Cases

Opening the dialog

  • Double-click any TOC node in the TOC Editor (as long as the double-click behavior has not been toggled to open the linked file)
  • Select any TOC node in the TOC Editor and click in the editor's local toolbar
  • Select any TOC node in the TOC Editor and click in the Standard toolbar
  • Right-click any TOC node in the TOC Editor and select Properties from the Content Menu
  • Select any TOC node in the TOC Editor and go !Edit | Properties (Toolstrip)
  • Select any TOC node in the TOC Editor and go !Home | Properties (Ribbon)
  • Select any TOC node in the TOC Editor and press F4 on the keyboard

(TOC node) Properties - General tab

UI Controls:

  • Appearance: This section sets the appearance of the TOC entry in the output
    • Label: Set the textual content of the TOC entry
    • Style Class: Lets you set the TOC node to a style class defined in the Skin
    • Mark as New: Designates the TOC entry to be "new." As a result, the icon for this node will display a extra glyph on it in the TOC Editor and in the output .
  • Link: This section sets the link for the TOC node
    • (Select Link) : Depending on which radio button is selected, this button invokes its respective dialog.
    • (Select Bookmark): Opens the Select Bookmark dialog which lets you select a bookmark or heading from the topic file associated with the selected TOC node. This button is only active for TOC nodes that link to a topic.
    • "Select" radio buttons: Lets you select the type of file to which the TOC node should link
      • Select Topic… : Selecting this button will invoke the Link to Topic dialog which will let you browse for a topic within the project.
      • Select TOC… : Selecting this button will invoke the Link to TOC dialog which will display all TOC files that exist in the project.
      • Select Browse Sequence… : Selecting this button will invoke the Link to Browse Sequence dialog which will display all Browse Sequence files that exist in the project.
      • Select Flare Project and Target… : Selecting this button will invoke the Link to Flare Project and Target dialog which will let you browse for a Flare project. This is covered in greater depth in the Merging test plan.

      • Select HTML Help...:Selecting this button will invoke the Link to HTML Help dialog which will let you browse for a chm.

      • Enter External Help System… : Selecting this button will invoke the Link to Help System dialog which will let you browse for a compiled HTML5, WebHelp, WebHelp Mobile or DotNet. (.mcwebhelp and .mchelp) This is covered in greater depth in the Merging test plan.
      • Mimic Movie or Movie Collection as… Selecting this button will invoke the Open File dialog which allows you to link to a mimic project or movie
    • (Open Linked File): Opens the linked file. This button is not active for a TOC node that has a broken link, is an unlinked node, or links to a Flare project.

(TOC node) Properties - Printed Output tab

UI Controls:

  • Chapter/Page Layout Break:
    • Break Type: Select one of the dropdown options to specify whether you'll be adding a chapter break or page layout break
    • Page Layout: Select the Page Layout file you want to associate with this chapter.
    • Open File: Opens the selected Page Layout file in the Page Layout Editor
    • Page Type: Lets you designate the page type (from the selected Page Layout file) that the chapter should begin with.
    • Chapter File Name: Lets you specify a particular file name for the chapter being created.
    • Auto-end on 'left' page: Lets you disable the use of the "Empty" page. With Flare V9, it is recommended to use the single target option to inject Empty pages. See the Help for more information.
  • Page Number:
    • Reset to: If you want the first page number in the chapter to begin at a specific number, you can select this option and enter the number in the field. If you select the "text" option from the Format field, you can type any kind of text into this field. That text will be used instead of numbering.
    • Format: If you want a particular format of numbering to be used (e.g., decimal, upper Roman, lower alpha, text), select it from this drop-down. If you select the "text" option, you can type any kind of text into the "Reset to" field. That text will be used instead of numbering.
    • Increment number: If you want the first page number in the chapter to continue from the previous chapter, you can select this option.
  • Section Break:
    • Start a new section: Lets you designate the selected TOC node as the beginning of a new chapter. In doing so, lets you begin using a different Master Page file (for Word and Frame only) and lets you set auto-numbering properties for sections (in the Auto-numbers tab).
    • Configure section using this Master Page: Select the Master Page file you want to associate with this section
    • Open File: Opens the selected Master Page file in the XML Editor

(TOC node) Properties - Auto-numbers tab

UI Controls:

  • Chapter Number:
    • Reset to: Select this option if you want the chapter numbers in topics starting at this point in the TOC to begin at a specific number. Enter the number in the blank field. If you select the "text" option from the Format field, you can type any kind of text into this field. That text will be used instead of numbering.
    • Format: Select the type of numbering to be used for the chapter auto-numbers (e.g., decimal, Roman, uppercase, lowercase, text). If you select the "text" option, you can type any kind of text into the "Reset to" field. That text will be used instead of numbering.
    • Same as previous: Select this option if you want the chapter numbers in topics starting at this point in the TOC to begin at the same number as the previous chapter.
    • Increment number: Select this option if you want the chapter numbers in topics starting at this point in the TOC to begin at the next number after the previous chapter.
  • Volume Number:
    • Reset to: Select this option if you want the volume numbers in topics starting at this point in the TOC to begin at a specific number. Enter the number in the blank field. If you select the "text" option from the Format field, you can type any kind of text into this field. That text will be used instead of numbering.
    • Format: Select the type of numbering to be used for the volume auto-numbers (e.g., decimal, Roman, uppercase, lowercase, text). If you select the "text" option, you can type any kind of text into the "Reset to" field. That text will be used instead of numbering.
    • Same as previous: Select this option if you want the volume numbers in topics starting at this point in the TOC to begin at the same number as the previous volume.
    • Increment number: Select this option if you want the volume numbers in topics starting at this point in the TOC to begin at the next number after the previous volume.
  • Section Number:
    • Reset to: Select this option if you want the section numbers in topics starting at this point in the TOC to begin at a specific number. Enter the number in the blank field. If you select the "text" option from the Format field, you can type any kind of text into this field. That text will be used instead of numbering.
    • Format: Select the type of numbering to be used for the section auto-numbers (e.g., decimal, Roman, uppercase, lowercase, text). If you select the "text" option, you can type any kind of text into the "Reset to" field. That text will be used instead of numbering.
    • Same as previous: Select this option if you want the section numbers in topics starting at this point in the TOC to begin at the same number as the previous section.
    • Increment number: Select this option if you want the section numbers in topics starting at this point in the TOC to begin at the next number after the previous section.

(TOC node) Properties - Auto-generate tab

UI Controls:

  • Compute sub-entries from linked file: For web outputs, you can enable this auto-generate feature to have Flare generate a TOC for you in the output based on the heading styles used in your topics.
    • Action: Lets you select the auto-generate behavior.
      • Insert Computed Entries: On output, adds TOC sub-entries directly below the selected node based on h1-h6 styles in the topic.
      • Replace with Computed Entries: On output, removes the current TOC node and replaces it with sub-entries based on the h1-h6 styles in the linked topic.
    • Depth: This number determines the number of levels of headings that should be included in the auto-generation.

(TOC node) Properties - Advanced tab

UI Controls:

  • Open in Browser Frame: Lets you specify the browser frame in which a linked file should open when a user click the TOC entry in the output.
  • Open in Skin: Lets you associate the TOC entry with a particular skin.
  • HTML Help Icon: For HTML Help output only, this option lets you set a specific icon for the TOC node.
  • Auto-set title of linked file: Sets the Topic Title of the linked topic to be the TOC Label in the output, overriding the title that you may have provided in the (Topic) Properties dialog.
  • When merging, replace node with merged TOC: If the TOC node links to another Flare project, another TOC or a Browse Sequence, you can select this option to replace the TOC node with the merged TOC or Browse Sequence. By default, this option is disabled, and such a node will output as a book with the merged TOC or Browse Sequence inside the book.
  • Server-based Auto-Merge: For Webhelp Plus output only, use this option to determine where other Flare project outputs are to be auto-merged into the TOC. This is covered in greater depth in the Output WebHelp Plus test plan.
