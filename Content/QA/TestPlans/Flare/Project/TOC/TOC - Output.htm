TOC Editor Setup for all output types

Topics must be added to the TOC file via the TOC Editor. For all sample screenshots provided in this test plan, I have used a *fltoc file that looks like this to demonstrate the unique behaviors of Web versus Print output types:

 

  • All topics linked from this *.fltoc file use the headings indicated by the TOC label pictured
  • For the Web-based output types, the TOC topic was conditioned out
  • No advanced options were enabled (see Advanced Target Options for print output below)
  • No mc-heading-levels were adjusted (see Other relevant styles: mc-heading-level in the "Styles" test plan)

Support in Web Outputs:

Support in HTML5 Static Outputs:

  • For all Web-based outputs, the TOC must be enabled in the Skin in order for an Table of Contents to appear in the output.
  • For all web outputs, the generated Table of Contents that appears in the output TOC pane should be exactly what you see in the Flare TOC Editor. That is, all TOC nodes that appear as top-level nodes in the TOC Editor will be top-level in the output, second-level nodes will be second-level in the output, etc. The exceptions to this include:
    • A topic in the TOC is conditioned out at the topic level, in which case the TOC nodes linking to that topic should not be present in the output
    • A TOC node is conditioned out, in which case the TOC node will not be present in the output
    • The TOC entry links to another project, TOC or Browse Sequence, that node will be replaced with the linked TOC or Browse Sequence in the output
    • The "Compute sub-entries" option is enabled in the Auto-generate tab of a TOC node, in which case there may be auto-generated TOC entries in the output that do not appear in the TOC Editor.

Support in Print Outputs:

Support in PDF Outputs:

  • For all print outputs, a TOC Proxy must be included in a topic and that topic must be included in the TOC being used by the Print Output Target.
  • For all print outputs, the generated Table of Contents that replaces the TOC Proxy is based entirely on the sequence of linked topics in the TOC Editor and the heading levels used in those topics. Any levels that exist in the TOC Editor are (by default) completely ignored when generating print output. Unlinked books are also ignored by default.
Advanced Target Options for print output

You can opt not to ignore the structure of topics in the TOC Editor by enabling some Advanced options in the Target Editor of a print target. In any print target, select the Advanced tab and find the section labeled "Generated TOC":

UI Controls

  • Use TOC depth for heading levels: Enable this option to use the depth of a TOC node in the TOC Editor to set the heading level of linked topics. This will change your heading levels in you content based on their position in the TOC Editor.
  • Inject headings for unlinked books in TOC: Enable this option to inject a heading into the generated TOC and your content for all unlinked books that exist in the TOC Editor. This will make the generated TOC appear more like the TOC Editor, but it will also add headings to your content.

  • Remove images from paragraphs with heading levels: This option is only relevant if you've inserted an image into any of the headings in your project.

    • If this option is left disabled, Word output will include the image in the generated TOC. All other outputs will not include the image in the generated TOC.
    • If this option is enabled, all outputs will strip the image from the headings completely. The image will not appear in the generated TOC nor will the image appear in the heading as it appears within the content.

Support in DITA Output:

  • When generating DITA output, Flare's TOC source code needs to be converted to a DITAMAP. For example, the TOC used in this Test Plan should output like this:
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE map PUBLIC "-//OASIS//DTD DITA Map//EN" "map.dtd">
<map>
  <topicref href="Content/TOC.dita" navtitle="TOC" />
  <topicref href="Content/Heading 1.dita" navtitle="Heading 1" />
  <topicref href="Content/Heading 2.dita" navtitle="Heading 2">
    <topicref href="Content/Heading 3.dita" navtitle="Heading 3" />
    <topicref href="Content/Heading 4.dita" navtitle="Heading 4" />
  </topicref>
</map>
