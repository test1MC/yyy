Glossary - Editor

Opening a Glossary File

  • Target Editor: In the Glossary tab of the Target Editor, select a Glossary file and click the "Edit Glossary" button.

Glossary Editor - UI Test Cases

Status Bubbles

  • Green bubble: green indicates a unique glossary term.
  • Red bubble: red indicates a duplicate glossary term.

Local Toolbar:

Button (Tooltips in Parentheses):

  • (New item): Creates a new glossary term and opens that term's Properties dialog.
  • (Run Properties Dialog for this item): Opens the Properties dialog for the selected glossary term.
  • Cut: Cuts the selected term(s) to the clipboard.
  • Copy: Copies the selected term(s) to the clipboard.
  • (Paste): Pastes the term(s) from the clipboard.
  • (Delete): Deletes the selected term(s).
  • Warnings Indicates whether there is a broken link or duplicate terms
    • Click the duplicate terms warning to see a list of the duplicate terms.

Glossary Term Tab

UI Controls

  • Terms: Displays (and lets you edit) the Glossary Term. You can enter multiple Glossary Terms (one per line) and...
  • Definition: Lets you set the definition for the Glossary Term.
    • Text: Lets you enter a text definition for the Glossary Term.
    • Topic: Lets you set the definition of the Glossary Term to be a topic in the project. This type of Glossary Term definition is supported in Web output types, Word and FrameMaker only (for Word and Frame, the definition topic must also be included in the TOC). When generating native PDF, XPS or XHTML output types, Glossary Terms with topic definitions will not be included in the generated output.
    • : Lets you browse within the project for a topic file to use as the Glossary Term definition.
    • : Opens the associated topic file in the XML Editor.
Style Tab

UI Controls

  • Style Class: Lets you apply a style class to the glossary term. The area below should display all classes of MadCap|glossaryTerm that exist in the Master Stylesheet.
