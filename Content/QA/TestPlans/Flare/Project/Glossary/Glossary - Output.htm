Glossary and Glossary Term Links - Output

Support in Web Outputs:

Support in HTML5 static Outputs:

  • Test text and topic definitions for Glossary Term Links with each of the available style classes. Test in all web output types.

 

Text definition

Topic definition

default (no class)

Definition should display in output as a text popup.*

Definition topic should display in output as a topic popup.*

Expanding

Definition should display in output as an expanding text.

"Expanding" is not supported for Topic definitions. Definition topic should display in output as a topic popup.*

Hyperlink

Clicking the converted glossary term will open the Glossary pane and scroll to and expand the definition for the selected term. Requires that the Glossary is enabled in the Skin.

Definition topic should open like a hyperlink.

Popup

Definition should display in output as a text popup.*

Definition topic should display in output as a topic popup.*

 

* Popups are not supported by Mobile devices so "popup" glossary term definition must be converted to topics for Mobile output. Verify that these "popup" definitions, when output to Mobile, create topics in the "MyWebHelpMobile\Advanced\Content\MCGeneratedPopups" folder in the output.

  • Verify that any individual glossary terms used to create multiple glossary term links only generate one "popup" topic. Verify that each instance of the glossary term generates a working links.
  • Test with all configurations of Advanced target options (lowercase, use underscores, etc)
  • Test the Glossary Terms that have topic definitions work in all Web output types in all instances where the definition topic is affected by the Advanced Target options:
    • Create a definition topic with mixed case characters and enable "Use lowercase filenames…"
    • Create a definition topic with spaces and other reserved characters in the filename and enable "Replace reserved characters with underscores…"
    • Enable custom file extensions
    • Verify all combinations of the above. Test links from the Glossary Terms within the Glossary Pane. Test links from converted Glossary Terms from within a topic.

Test cases for converted glossary terms set to "Popup"

  • Test the "Popup" definitions work with "Condense JavaScript Files" disabled in the Performance tab of the target
  • Test the "Popup" definitions with "Condense JavaScript Files" enabled in the Performance tab of the target
  • For all web outputs, the generated Glossary that appears in the output Glossary pane should consist of all Glossary Terms included in all Glossary Files that you have selected to have included in the target file.
    • Glossary Terms from unselected Glossary Files should not appear in the generated Glossary in the output.
  • Additionally, Glossary Terms that exist within the content should be converted to Glossary Term Links as defined by the setting in the target file.
  • For all Web-based outputs, the TOC must be enabled in the Skin in order for an Table of Contents to appear in the output.
  • For Web output types only, you can opt to include a Master Page for the generated Glossary.
  • For all output types, you need to select which glossary files you want to include in the output as well as set the behavior for Glossary Term conversion. By default, all Glossary Files are used, so if you do not explicitly indicate that only one file should be included, all will be included.

Support in Print Outputs:

Support in EPUB Outputs:

  • Test text and topic definitions for Glossary Term Links in all Print output types. Term definitions should be converted to footnotes.
  • For all print output types, a Glossary Proxy must be included in a topic and that topic must be included in the TOC being used by the Print Output Target.
  • For all print output types, the generated Glossary that replaces the Glossary Proxy should consist of all Glossary Terms included in all Glossary Files that you have selected to have included in the target file.
    • Glossary Terms from unselected Glossary Files should not appear in the generated Glossary in the output.
    • Glossary Terms that use a linked topic for a definition will NOT appear in the output.
  • Additionally, all Glossary Terms that exist within the content from all Glossaries selected in the Target file should be converted to footnotes.
    • The text definition of a glossary term will become the text of the footnote.
    • Glossary terms from unselected Glossary Files should not generate footnotes.
    • Glossary Terms that use a linked topic for a definition should not generate a footnote. However, up to V6, such terms continue to appear styled in the content with no link.

In addition, the following basic use cases should be added for print:

  • Test using glossary terms in the same topic as other footnotes and verify they generate properly in the proper sequence
  • For all output types, you need to select which glossary files you want to include in the output as well as set the behavior for Glossary Term conversion. By default, all Glossary Files are used, so if you do not explicitly indicate that only one file should be included, all will be included.

  • For all print output types, the generated Glossary that replaces the Glossary Proxy should consist of all Glossary Terms included in all Glossary Files that you have selected to have included in the target file.
    • Glossary Terms from unselected Glossary Files should not appear in the generated Glossary in the output.
    • Glossary Terms that use a linked topic for a definition will NOT appear in the output.
  • Additionally, all Glossary Terms that exist within the content from all Glossaries selected in the Target file should be converted to footnotes.
    • The text definition of a glossary term will become the text of the footnote.
    • Glossary terms from unselected Glossary Files should not generate footnotes.
    • Glossary Terms that use a linked topic for a definition should not generate a footnote. However, up to V6, such terms continue to appear styled in the content with no link.

In addition, the following basic use cases should be added for print:

  • Test using glossary terms in the same topic as other footnotes and verify they generate properly in the proper sequence
  • For all output types, you need to select which glossary files you want to include in the output as well as set the behavior for Glossary Term conversion. By default, all Glossary Files are used, so if you do not explicitly indicate that only one file should be included, all will be included.

Support in DITA Output:

  • Glossary Proxies and (in-topic) Glossary Terms are not supported in DITA output. A compile error should be given when generating DITA output if a Glossary Proxy exists in a topic.
  • Glossary Files are converted to DITA topics and placed in the Content/Resources folder of the generated DITA output.
    • Flare should generate a unique DITA topic for each Glossary file that exists in the Flare project with the filename of the glossary becoming the topic id and the title of the DITA topic.
    • Each glossary term in the Flare project should become a glossentry element in the appropriate DITA topic.
    • Each glossentry element should contain an "id" attribute. The attribute is simply a number. The first glossary processed (alphabetically) will contain glossentry elements with id=1, 2, 3 etc. The second glossary processed should continue incrementing id attributes where the first left off, and any additional glossaries would continue
    • Each glossentry element in the DITA output should contain a glossterm and a glossdef element containing the term and the definition respectively. Glossary Terms in Flare which have topic definitions are not supported in DITA output. Such a term will still generate a glossentry but the glossdef element will contain no text.

DITA Example:

<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE topic PUBLIC "-//OASIS//DTD DITA Composite//EN" "ditabase.dtd">
<topic id="MyGlossary">
  <title>MyGlossary</title>
  <glossentry id="1">
    <glossterm>MyTerm</glossterm>
    <glossdef>Text definition.</glossdef>
  </glossentry>
  <glossentry id="2">
    <glossterm>Topic</glossterm>
    <glossdef>
    </glossdef>
  </glossentry>
</topic>

A subsequent Glossary will begin with id="3" for its first glossterm element.

<topic id="MyOtherGlossary">
  <title>MyOtherGlossary</title>
  <glossentry id="3">
    <glossterm>MyOtherTerm</glossterm>
    <glossdef>Text definition.</glossdef>
  </glossentry>
</topic>

Compiler errors:

  • Attempting to generate any Print output type with a glossary term that uses a Topic link as its defintion:

    "Glossary definition links are not supported in this output type."
  • Attempting to use the same Glossary Term in multiple Glossary Files:

    "Term: [duplicated term], defined in more than one glossary file."
  • Attempting to include a Glossary Proxy in DITA output:
    "MadCap|glossaryProxy found in file: "[topic filename]". It will be removed from the output."
