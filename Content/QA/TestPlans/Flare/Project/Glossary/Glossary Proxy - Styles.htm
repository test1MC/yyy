Glossary Proxy - Styles

The styles detailed below all contribute to the look of the generated glossary in the output (in place of the Glossary Proxy). Of the styles outlined here, only MadCap|glossaryProxy will allow you to create unique classes. All other styles are generated styles used by all Glossary Proxies regardless of class.

The styles applied here should be tested in all outputs, however, keep in mind that these styles only affect the Glossary Proxy placed inside a topic. These styles do not affect the look of the Glossary pane that is generated for Web, DotNet and HTML Help outputs. Styles for the Glossary pane in Web, DotNet, and HTML Help outputs are set in the Skin and will be covered in the Skins test plan.

You can configure a target to convert glossary terms to links automatically when they are found in topics. In addition, there is a way to purposely ignore some terms so that they are not converted to links. This can be done by setting the mc-disable-glossary-terms property on a style. By default, this feature is turned on for <h1> through <h6> styles, as well as hyperlinks (i.e., content with the <a> tag). This means that if a glossary term is found in a heading or text hyperlink, it will not be converted to a link. However, if the same term is found in, say, a regular paragraph, the term will be converted to a link (as long as you do not turn this feature on for that paragraph style).
