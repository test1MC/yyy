Topic Toolbar - Output

  • The Topic Toolbar Proxies used throughout your project are converted to Topic Toolbars in the output. If you added a Topic Toolbar Proxy to a Master Page, you will need to make sure that the Master Page is selected in the Target Editor. If you have added the Topic Toolbar Proxy to only certain topics in which you want the Toolbar to appear, you will not need to do any setup in the Target Editor.
  • Additionally, you may want to use the Skin Editor to create a customized toolbar (a default set of buttons). If you want to create a default customization of the Topic Toolbar for the entire project, you can do this in the Skin Editor.
    • Keep in mind though, that any customizations made directly to the Toolbar Proxy in a topic or Master Page will override any customizations set up in the skin. This would include any style class applied to a Topics Toolbar Proxy where the customized set of toolbar buttons is set in the Stylesheet Editor (See Topic Toolbar - Styles).

Support in Web Outputs:

Target Editor set up:

Target Editor set up is only required if your Topic Toolbar Proxy is used inside a Master Page.

 

 

 

 

 

In the Advanced Tab, set the Master Page to the Master Page in which you've added the Topic Toolbar Proxy.

Skin Editor setup:

Skin Editor set up is only required if you want to use the Skin to define the set of buttons which appear in the toolbar throughout your project.

Alternatively, these buttons can be set using the mc-topic-toolbar-items property in the Stylesheet Editor or locally (in the topic with the Proxy). If mc-topic-toolbar-items is set in either of those locations, that setting will override anything you set here in the Skin.

Additionally, all styling of the toolbar is done in the Styles tab of the Skin. That should have been covered in the Topic Toolbar - Styles test plan.
