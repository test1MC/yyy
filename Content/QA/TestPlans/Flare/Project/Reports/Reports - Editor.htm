Report Editor - UI Test Cases

General Tab

UI Controls

  • Title: Lets you set a title which will appear in the generated Report.
  • Tables: Select from the list of available reports to generate.
  • Select All: Tri-state toggle which selects all, deselects all or returns to previous selections.
  • Skip Empty Reports: Upon generating reports, if this option is enabled, any reports which produce zero results will be skipped.

  • Print report after generation: The report will automatically be printed to your default printer upon completion of a generated report.

  • Report: Displays the generated Report.
    • Project: project path location

    • Generated: displays the date #/##/#### and time 12:12:12 PM

    • By: displays the user name who generated the report.

  • Add To Archive…: Opens the Archive Report dialog which lets you copy the generated Report to the project’s Report Archive. The reports are saved as *.htm files to .../Project/Reports/Archive.
  • Open in Browser: Opens the generated Report in the default browser.
  • Generate: Generates a report for the items selected in the Tables list.

Test cases:

  • Generate each report and test every report item
  • Test to make sure the correct information appears and displayed and formatted in a way that makes sense
  • Test that styles are correctly generated as well

Archive Report dialog:

UI Controls

  • Report Name: Lets you create a new for the report about to be archived.
  • OK: Adds an Archive folder in Windows Explorer| Projects | Reports | Archive with the file [filename].htm.  This will not be visible within Flare.
  • Cancel: Does not create the archived report.

Styles Tab

UI Controls

  • Styles: Displays list of all styles which can be used to customize the appearance of a generated report.
  • Show: Lets you alternate the display of properties by Property Group or Alphabetically
  • Properties: Lists all available properties (alphabetically or by group) for the selected style
  • Preview: Displays a preview of the selected style (if a preview is available; which it never is for any of the Report styles, bug #)

Test Cases

  • See "Reports - Styles "

Archive Tab

UI Controls

  • Remove: Send the selected archived report(s) to the Recycle Bin.
  • Open in Browser: Opens the selected report in your default web browser.

  • Report Archive table: Lists all archived reports.

    Columns

    • Report: Displays the name of each generated report in the project’s report archive.
    • Date: Displays the date and time at which the archived report was generated.

Misc

Code

Default Report (no tables):

<?xml version="1.0" encoding="utf-8"?>
<CatapultReportFile>
</CatapultReportFile>
 

Report with a few tables enabled and Skip Empty Reports enabled:

<?xml version="1.0" encoding="utf-8"?>
<CatapultReportFile
    Tables="broken-bookmarks,broken-links,external-links"
    SkipEmptyReports="true">
</CatapultReportFile>
 

Report with tables and styles:

<?xml version="1.0" encoding="utf-8"?>
<CatapultReportFile
  Tables="broken-bookmarks,broken-links,external-links">
  <Stylesheet>
    <Styles>
      <Style
        Name="ReportText">
        <Classes>
          <StyleClass
            Name="Level1">
            <Properties>
              <Property
                Name="Color">#0000cd</Property>
            </Properties>
          </StyleClass>
        </Classes>
      </Style>
    </Styles>
  </Stylesheet>
</CatapultReportFile>
