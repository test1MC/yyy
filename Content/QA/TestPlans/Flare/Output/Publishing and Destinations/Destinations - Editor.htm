Destinations - Editor

Opening a Destination File

  • Target Editor: In the Publishing tab of the Target Editor, select a Destination and click the "Edit Destination" button.
  • Publish Target dialog: Initiate the publishing process to open the Publish Target dialog, select a Destination and click the "Edit Destination" button.

Destination Editor - UI Test Cases

UI Controls:

  • Type: Select either FTP or File System or SFTP, depending on which method you want to use to publish the output files
  • Host Name: Enter the name of the remote server or the computer where the output files will be published. If you use the File System type, this field is disabled.
  • Port: Select the port that you will use to connect to the remote server.
  • Login Credentials… : Select this button to open the Log In dialog.
  • Anonymous Login: Select this check box if you want to publish to the server without being required to enter a user name or password.
  • Directory: Enter the exact location where the output files will be published.
  • Browse… : If you use the File System type, you can click this button to select a location.
  • Comment: You can use this field to provide a description of the destination so that its purpose is clear.
  • "View" Url: You can enter an http address in this field to be used for viewing the published files.
  • Upload Log File:Select this to upload the log file
    • Update Only Changed Files: Select this option if you want Flare to republish only the files that have changed. This can save significant time.
    • Remove Stale Files: Select this option if you want Flare to identify files that were previously published but are no longer needed. Flare will then remove such files from the destination during the publishing process.

 

Log In dialog:

UI Controls

  • Server: Displays the server where the output is being published.
  • User name: Enter the appropriate user name to access the server.
  • Password: Enter the appropriate password to access the server.
  • Save password: Select to automatically save your password

Conditional Tags

Conditional Tags cannot be applied to anything within a Destination file. Conditional Tags can be applied to the actual Destination file (*.fldes), however, as of Flare V5, this tag can only be used for the purposes of Advanced Global Project importing.

Code

Default Destination:

<CatapultDestination
Version="1"
Comment="This is my publish destination"
Host="ftp.myftpserver.com"
User="MyAccountId"
Password=""
Directory="public_html"
Port=""
ViewUrl="http://www.madcapsoftware.com/"
Type="ftp">
</CatapultDestination>
