Publishing - UI

Setup

In order to publish your compiled output, you must first associate one or more Destinations with a target and then compile your output so you have output to publish.

Associating Destinations with a Target

Baseline tests for UI will be covered in the Target Editor test plan. For this test plan, just test UI Controls.

UI Controls

  • Edit Destination: Opens the selected Destination file in the Destination Editor.
  • New Destination… : Opens the Add Destination… dialog.
  • View Published Help System: Opens the "View URL" of the selected Destination (set in the Destination Editor) in the default browser.

  • Destination List

    Columns

    • Destination: Displays the name of each Destination file in the project.
    • Publish: Lets you associate the Destination with the current target. A checkmark indicates the destination is associated.
    • Target: Displays the Host Name as set in the Destination Editor.
    • View URL: Displays the optional "View URL" as set in the Destination Editor.
    • Comment: Displays the Destination's Comment as set in the Destination Editor.
  • Catalog Name (not pictured): Will be covered in WebHelp Plus output test plan.

Compiling output

Build the Target you want to publish any number of ways.

Publishing

To publish the output of the Primary Target:

  • Click Publish Primary in the Project Toolbar
  • Press CTRL-F6 on the keyboard
  • !Build | Publish Primary

 

To publish the output of any target:

  • Target Editor: Open the target you want to publish. Click the Publish button in the Target Editor's local toolbar.
  • Right-click the Target in the Project Organizer and select Publish '[TargetName]'
  • Select a Target in the Project Organizer and !Build | Publish

Publish Target dialog

UI Controls

  • Destination List

    Columns

    • Destination: Displays the name of each Destination file in the project.
    • Status: Indicates whether the publishing process has been completed.
    • Publish: Lets you select or deselect the Destination with the output to be published. A checkmark indicates the destination will be used. The default state of this option for each Destination is based on any associations set in the Publishing tab of the Target Editor for the generated target.
    • Upload Changes Only: Select this option if you want Flare to republish only the files that have changed. This can save significant time. The default state of this option should be inherited from the associated Destination file.
    • Remove Stale Files: Select this option if you want Flare to identify files that were previously published but are no longer needed. Flare will then remove such files from the destination. The default state of this option should be inherited from the associated Destination file.
    • Url: Identifies the path where the output files will be published.
    • Comment: Displays the Destination's Comment as set in the Destination Editor.
  • View Last Log… : Opens the Publishing Log, displaying details about the files that were included in the most recent publishing process.
  • Edit Destination: Opens the selected Destination file in the Destination Editor.
  • Start Publishing: Initiates the publishing process.
  • Cancel: Closes the dialog without publishing the output.

 

Publish Progress dialog

UI Controls

  • Cancel: Closes the dialog and does not publish the output.

Publishing Log dialog

UI Controls

  • Destination List: Lists all Destinations to which content was published. Select a Destination in this column to display the corresponding log. If the output is published to multiple Destinations, verify that this list is sortable by clicking the column heading.

  • Action List

    Columns

    • Action: Displays the action performed to each published file
    • File: Displays the filename of each published file
    • Folder: Displays the path to which each file is published
  • Save Log File: Saves an xml document ("UploadLog") to the projects Reports folder
  • Log created… : Verify the created date displays correctly
