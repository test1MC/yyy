Output WebHelp Plus - Output Files

The following is an outline of all files and directories that should be generated when building a WebHelp Plus output.

Root-Level Files

  • [Output Filename].htm: This is the landing page for webhelp plus output.
  • [Output Filename].js:
  • [Output Filename].mcwebhelp:
  • [Output Filename]_CSH.htm:
  • [Output Filename]_[Nav position set in skin].htm: This filename will change depending where the pane position is set in the skin editor. By default, the pane position is set left.

AutoMerge Folder

No content should ever be generated in this folder. Additional WebHelp systems are placed here for runtime merging.

AutoSearch Folder

No content should ever be generated in this folder. Non-XHTML files placed in this folder will show up in the search for the output.

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.

Additional Files
  • Glossary.htm
  • Glossary.js
  • Glossary.xml

Resources Folder

All files and folders in the project's Resources folder should appear here. By default, the project's Page Layout folder is created in this folder, but should not appear in the output unless non-.flpgl files are also present. Flpgl files should never appear in the output
