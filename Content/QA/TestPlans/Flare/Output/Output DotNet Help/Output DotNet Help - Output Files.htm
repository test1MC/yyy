Output DotNet Help - Output Files

The following is an outline of all files and directories that should be generated when building a DotNet Help output.

Root-Level Files

  • [Output Filename].mchelp: This file

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.

Additional Files

In addition to the project's Content files, the following files will also appear in the output Content folder:

Search.mcsdbf Folder

If the Target option to "Include SQL Search Database" is enabled, then this folder will appear in the output. It will contain two files: 

  • Search.mcsdb
  • Search.s3db

If this option is not enabled in the Target, the SQL database will get created when the Help System is opened by the Help Viewer. In this case, a folder is created at C:\Users\[username]\AppData\Local\MadCap Software\CachedHelpSystems (Windows Vista). This folder will contain a duplicate [Output Filename].mchelp file, a MadCapSearchCache.xml file, and a Data folder containing the Search.mcsdbf folder detailed above. In this scenario, if you build a merged DotNet output, a unique folder will get created in the CachedHelpSystems folder for the parent project output as well as each merged output.

Skin[SkinName] Folder

The name of this folder is generated by the name of the skin used for the Target. For example, if the filename of the Skin used is MyMobileSkin.flskn, the folder name that appears here will be "SkinMyMobileSkin." It will contain:

  • Custom images added to the skin
  • Factory images used as defaults
  • Skin.xml
  • Stylesheet.xml

Subsystems Folder

This folder will only be present if you merge another DotNet Help system.
