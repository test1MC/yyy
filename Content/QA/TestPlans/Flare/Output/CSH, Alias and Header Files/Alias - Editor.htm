Alias - Editor

Opening an Alias File

  • Target Editor: In the Advanced tab of the Target Editor, in the Alias File section, select an Alias File and click the "Edit" button.

Alias Editor - UI Test Cases

 

Local Toolbar:

Button (Tooltips in Parentheses):

  • (Header files): Displays all header files in the project. By default this is set to (all identifiers).
  • (Hide the assigned identifiers):  Toggle for hiding / showing all identifiers that have topics assigned to them.
  • (Create a new identifier): Will create a new identifier. Value and name will be dependant on which Identifier Options are set. See below.
  • (Delete the selected identifiers): Deletes the selected identifiers.
  • (Generate identifiers for this project): Launches the Generate Identifiers dialog. See below.
  • (Change the identifier options): Launches the Identifier Options dialog. See below.
  • (Shows the current warning, if any): Displays the current warning and current warning count. Clicking this will open up a dialog listing all the current warnings. Currently this only shows unassigned identifiers.
  • (Navigate to the previous warning): Will select the previous identifier, based on the current selection, that has a warning.
  • (Navigate to the next warning): Will select the next identifier, based on the current selection, that has a warning.

UI Controls - Topics section

  • Topics Toolbar (Tooltips in parentheses): 

    • (Show/Hide All Files in Project): Changes the view in the dialog to "Show All Files" view. This view display all topics in the project in one grid. Detailed testing below.
    • (Show/Hide Folders): Changes the view in the dialog to "Show Folders" view. This view displays all topics in the project in a tree view, similar to the Content Explorer. This view can be combined with the "Show Files" view to offer a "Split" view. Detailed testing below.
    • (Show/Hide Files): Changes the view in the dialog to "Show Files" view. This view displays all topics in the project one folder at a time, similar to the "Show Files" view in the Content Explorer. This view can be combined with the "Show Folders" view to offer a "Split" view. Detailed testing below.
    • (Up): Moves up one folder in the hierarchy within the project's Content folder. This button becomes enabled if you use the "Show Folders" or "Split" View and browse into a folder.
    • (Assign topic to the selected identifier): Assigns the selected topic to the currently selected Identifiers.
    • (Assign the selected topic to a new identifier): Assigns the selected topic to a new identifier. Value and name will be dependant on whichIdentifier options are set. See below.
    • (Open the selected topic): Opens the selected topic in the XML Editor.
    • (Locate topic in header files): Will open the Locate Topic dialog. This dialog lists each instance the topic is used in all header files.
      • The dialog will contain the Header file, the Identifier name and the identifier value
      • If the topic is not assigned, this dialog does not open. A warning dialog appears instead.

UI Controls - Identifiers

  • Identifiers Toolbar (Tooltips in parentheses): 

    • (Unassign topic): Unassigns the topic from the currently selected identifiers.
    • (Select a bookmark from the selected topic): Opens the topic currently assigned to the selected identifier in the main XML Editor. A Select Place dialog appears listing all of the headings and bookmarks in that topic.
      • Selecting a bookmark and hitting OK will return you to the Alias Editor and assign the selected bookmark to the selected identifier
      • Clicking Cancel at any point will close the topic and return you to the Alias Editor with no changes made.
    • Assign Skin: Will assign the selected skin to the selected identifiers.
      • Clicking the drop-down arrow, will display all the skins in the project
      • Selecting a skin from the drop down, will set that Skin on the button. Subsequent clicks will now assign that skin to the selected identifiers.

  • Identifiers grid: Displays and lets you edit identifiers.

    Columns

    • Assigned: Will show a green or red icon. This field is not editable.
      • Green denotes the identifier has a topic assigned to it.
      • Red denotes the identifier does not have a topic assigned to it.
    • Identifier: Displays the name of the Identifier. This field is editable.
    • Topic: Displays what Topic, if any, is assigned to the identifier. This field is not editable.
    • Path: Displays the path of the topic file, if one is assigned to the identifier. This field is not editable.
    • Skin: Displays the skin, if any, that is assigned to the identifier. This field is not editable.
    • Value: Displays the Value of the identifier. This field is editable.
    • Header File: Displays the header file the identifier is associated with. Only displays when (all identifiers) is set in the header file filter.

 

Test Cases

  • For Identifier Name, enter single words, multiple words, numbers, words with international characters, symbols, etc.
  • For Identifier Values, enter in negative numbers, decimals and spaces.
  • Selecting a row (or rows) and pressing Delete will remove those identifiers.

Identifier Options

 

UI Controls

  • Starting Value: The value of new identifiers created in the Alias Editor will be based off of this number.
    • Subsequent added identifiers will increment up by one, based on what values have been used already.
  • Use Prefix: The text entered here will be prefixed to newly created identifier names.
  • Include topic name : Inserts the topic name as the identifier
  • Capitalize identifier names: Checking this will force all new identifiers to have capitalized names.
  • Default skin: Sets the skin that is assigned to new identifiers.
  • Primary Header: Sets the header file that new identifiers are created in, when (all identifiers) is selected in the local toolbar of the Alias Editor.
  • Value Format: Let you set the format for the identifier values to Decimal or Hexadecimal.

Generate Identifiers

This tool will create identifiers (based on your settings) for your entire project.

 

UI Controls: Header File area

  • New From Template: Selects a header file based on the selected template file.
    • File Name: Sets the name of the new header file.
    • (Manage Templates): Launches the Manage Templates dialog.
  • Choose Existing: Selects an existing header file.

 

UI Controls: Identifier Options area

  • Starting value: Sets the starting value for the identifiers.
  • Prefix: Sets the Prefix for the identifier names.
  • Include topic name in identifier name: If this is checked, the new identifiers will apply the topic name as the Identifier name.
    • If a prefix is used, the topic name will follow the prefix
  • Capitalize identifier names: All identifiers created, will have the identifier name capitalized.
  • Skins: Sets which skin (if any) will be assigned to the identifiers.
  • Generate Identifiers for: This drop down offers two options
    • Unassigned topics only: Will create identifiers for topics that are currently unassigned to any identifier.
    • All topics: Will create a new identifier for every topic in the project, regardless if it has already been assigned to an identifier.
  • Existing Identifiers: This drop down is disabled if New From template is selected or Unassigned topics is selected in the above dialog.
    • Keep: If this option is set, no identifiers will be removed.
    • Delete: This option will remove all identifiers in the designated header file before the new identifiers are generated.
  • Value Format: Let you set the format for the identifier values to Decimal or Hexadecimal.

Test Cases

  • Verify the combination of choices selected, is accurately represented in the generated identifiers.

Locate Topic

 

UI Controls

  • Topic: Displays the file name of the topic for which header file associations are dispalyed.
  • Header File Grid: Displays all header files which are associated with the selected topic file and also displays the Identifier and Value associated with that topic on that header file.
  • OK: Closes the dialog, opens the selected header file in the Alias Editor, and highlights the identifier associated with the selected topic.
  • Cancel: Closes the dialog and does not open a header file
