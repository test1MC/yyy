Output WebHelp - Output Files

(Recently copied from WebHelp Plus - contents of this test plan need to be verified)

The following is an outline of all files and directories that should be generated when building a WebHelp output.

Root-Level Files

  • [Output Filename].htm: This is the landing page for webhelp output.
  • [Output Filename].js:
  • [Output Filename].mcwebhelp:
  • [Output Filename]_CSH.htm:
  • [Output Filename]_[Nav position set in skin].htm: This filename will change depending where the pane position is set in the skin editor. By default, the pane position is set left.

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.

Additional Files
  • Glossary.htm
  • Glossary.js
  • Glossary.xml

Resources Folder

All files and folders in the project's Resources folder should appear here. By default, the project's Page Layout folder is created in this folder, but should not appear in the output unless non-.flpgl files are also present. Flpgl files should never appear in the output

Data Folder

Root files in this folder will always have the same filename. For example, even if you rename your browse sequence file, the related .js and .xml files will still be named BrowseSequences. Root files:

  • Alias.js
  • Alias.xml
  • BrowseSequences.js:  (only present if Browse Sequence is enabled in the skin and a Browse Sequence exists in the project.)
  • BrowseSequences.xml : (only present if Browse Sequence is enabled in the skin and a Browse Sequence exists in the project.)
  • Concepts.js
  • Concepts.xml
  • Filters.js
  • Filters.xml
  • HelpSystem.js
  • Helpsystem.xml
  • Index.js
  • Index.xml
  • Search.js
  • Search.xml
  • Search_Chunk1.js
  • Search_Chunk1.xml
  • Synonyms.js
  • Synonyms.xml
  • Toc.js
  • Toc.xml

Subsystems Folder (optional)

This folder will contain the output folders of any merged WebHelp projects. This folder will only be present if you have linked to another Flare project or compiled WebHelp output in the parent project's fltoc file.
