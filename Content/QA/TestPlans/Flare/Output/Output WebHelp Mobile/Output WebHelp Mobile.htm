“Advanced” and “Basic” Functionality

  • Topic popups will be converted to hyperlinks since the screen size typically is not large enough to make topic popups useful.
  • Run-time merging works, however all sub projects are pre-merged. This means sub projects cannot be simply updated and dropped into the Subsystems folder of the master project. The master project must be regenerated.
  • Feedback is not supported.
  • It should work in supported desktop browsers.
  • we will not support “local” based help output that can be opened from a disk (file://) or network (\\UNC_Path)

 

“Basic” Functionality

  • Search is not supported.
  • CSH is not supported.
  • Run-time merging works, however all sub projects are pre-merged. This means sub projects cannot be simply updated and dropped into the Subsystems folder of the master project. The master project must be regenerated.
  • Feedback is not supported.
  • Topic effects (expanding text, drop-down text, etc.) are statically displayed in their expanded states. They cannot be collapsed.
  • Only the selected skin in the target will be generated. This means the “Generate all skins” setting is ignored. It also means that if a skin is specified during a CSH call it will be ignored.
  • Only the following topic-toolbar buttons are supported:
    • PreviousTopic
    • NextTopic
    • CurrentTopicIndex
