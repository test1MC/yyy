Output WebHelp Mobile - Output Files

The following is an outline of all files and directories that should be generated when building a WebHelp Plus output.

Root-Level Files

  • [Output Filename].htm: This file determines mobile browser type and forwards to the appropriate Mobile Help system. Basically, if opened on Blackberry v4.5 or earlier, the Basic Help System will open. For any other platform, the Advanced Mobile output will be opened.

Advanced Folder

Contains the Advanced version of the Help System (for all Mobile devices other that Blackberry v4.5 or earlier).

Files

  • [Output Filename].htm: This file serves as the landing page for the Advanced Mobile output.
  • [Output Filename].mcwebhelp: This file
  • Sitemap.xml: This file should appear here only if you have enabled the Target option to generate a Sitemap.

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.

Files

In addition to the project's Content files, the following will also appear in the output Content folder:

  • Glossary.htm:
  • Glossary.js:
MCGeneratedPopups

Each unique "popup" will generate a topic file and place it in this folder. Any of the following will create a new topic when generating Mobile output:

  • Text Popups: A topic file will be generated using the popup body text.
  • Glossary Term Links (if the glossary term uses a text definition): A topic files will be generated using the glossary term's text definition.
  • Image thumbnails: A topic will be generated consisting of a reference to the full-size image.

There should be no duplicate topics in this folder. For example, if a single glossary term is marked for conversion in many different topics, only one "popup" topic should be created in this folder and all marked instances of that term should link to the same "popup" topic. Similarly, if the same popup text is used multiple time in various places in the Flare project, only one "popup" file should be created for each unique popup text string. The same should be true if a single image file is used as a thumbnail multiple times.

Skin[SkinName] Folder

The name of this folder is generated by the name of the skin used for the Target. For example, if the filename of the Skin used is MyMobileSkin.flskn, the folder name that appears here will be "SkinMyMobileSkin." It will contain:

  • Custom images added to the skin
  • Factory images used as defaults
  • Skin.xml
  • Stylesheet.xml

Resources Folder

Files
  • About.htm: This file is the "About" page for the WebHelp Mobile output. It will consist of the image set as the "About" image in the Skin.
  • BrowseSequences_0.htm: This file is the default view of the Browse Sequence (only present if Browse Sequence is enabled in the skin and a Browse Sequence exists in the project.)
  • BrowseSequences_[n].htm: Additional "BrowseSequences_[n]" files will be created for every Browse Sequence book that exists in the output Browse Sequence. When viewing the Mobile Browse Sequence, only one book can be viewed at a time so there will likely be a large number of the files in the output. (only present if Browse Sequence is enabled in the skin and a Browse Sequence exists in the project.)
  • Glossary.htm: This file is the compiled Glossary for the WebHelp Mobile output.
  • Index.htm: This file is the compiled Index for the WebHelp Mobile output.
  • Search.htm
  • Slideshow.css
  • Toc_0.htm: This file is the default view of the TOC.
  • Toc_[n].htm: Additional "Toc_[n]" files will be created for every TOC book that exists in the output TOC. When viewing the Mobile TOC, only one book can be viewed at a time so there will likely be a large number of the files in the output.
  • Topic.css:
Images Folder
  • MCSlider_controls.png
  • MCSlider_loader.gif
Scripts Folder

If the option to "Condense JavaScript Files" is enabled in the Performance tab of the target (which it is by default), this folder will contain the following files:

  • MadCapAll.js (*)

Skins Folder

Default Folder
  • Images Folder
    • header.png: Used as the background image for the main header area of the WebHelp Mobile skin
    • IndexHeadingBG.png: Used as the background image for index and glossary headers (A, B, C, etc) in the generated Index and Glossary
  • Stylesheets Folder
    • Header.css: Contains styles for the header of the WebHelp Mobile output (Home button, etc)
    • StackLayout.css: Contains styles for the navigation of the WebHelp Mobile output (TOC, Index, etc)

Subsystems Folder

This folder will contain the output folders of any merged WebHelp Mobile projects. This folder will only be present if you have linked to another Flare project of compiled WebHelp Mobile output in the parent project's fltoc file.
