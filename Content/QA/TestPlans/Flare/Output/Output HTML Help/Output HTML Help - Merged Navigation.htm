Output HTML Help - Merged Navigation

You can merge HTML Help so that the navigation (table of contents, index, search) for each CHM file is displayed, regardless of which CHM file you open (parent or child). This can be done in a couple of ways, depending on whether you have access to all of the Flare projects or to the CHM files only.

Merged Navigation in HTML Help - Access to All Flare Projects

This method is used when you are merging by linking to other Flare project files from the parent project's TOC.

 

Steps for this method:

  1. The author of the parent project creates links in the parent project's TOC to other Flare projects.

    NOTE: TOC entries in the parent project should have the option "When merging, replace node with merged TOC" enabled. This way the merged navigation will not display a broken link should the child CHM file be removed from the set of merged CHMs.

  2. In the HTML Help target file of the parent project, in the Advanced tab, the author of the parent project enables the option to "Display merged navigation in HTML Help."

  3. The author for the parent project generates the main HTML Help target.

    The parent CHM file and all child CHM files are located in the same output folder, and when you open any of those CHM files, you can see the navigation for any of the other outputs.

Merged Navigation in HTML Help - Access to CHM Output Files Only

This method is used when you do not have access to the source Flare projects to which you need to link, but only the CHM output files. In order for merged navigation to work when you only have access to the CHM files, it is still required that the child CHM files be built with Flare, and that the authors of the child Flare project include a specific TXT file in their Flare project.

NOTE: A sample set of Flare projects that are already set up to test this is available at \\fs1\QA\Test Projects\Test Plan Projects\Output HTML Help\Runtime CHM Merging

Steps for this method:

  1. The author for each project (parent and child) places a simple TXT file named merge.txt in the same folder where the Flare project file (.flprj) is located. This TXT file can be created with Notepad and should simply list the names of all CHM files to be included in the merged output (one on each line), with the name of the parent CHM appearing first.

    Example:

    MyParent.chm
    MyFirstChild.chm
    MySecondChild.chm
    MyThirdChild.chm
  2. The author for each child project generates his output and provides the author of the parent project with the CHM files.

  3. The author for the parent project brings the child CHM files into the parent project and creates links to the CHM files in the parent project's TOC.

    NOTE: TOC entries in the parent project should have the option "When merging, replace node with merged TOC" enabled. This way the merged navigation will not display a broken link should the child CHM file be removed from the set of merged CHMs.

  4. The author for the parent project generates the main HTML Help target.

    The parent CHM file and all child CHM files are located in the same output folder, and when you open any of those CHM files, you can see the navigation for any of the other outputs.

Merged Navigation in HTML Help - Test Cases

Using both methods detailed above, verify the following:

  • Opening any of the CHM files (parent or any of the children) will all open with merged navigation.
  • Remove one or more of the linked child CHM files from the folder containing the CHM files and verify that its content no longer appears when the other CHM files are opened.
  • Add the removed CHM files back to the folder containing all of the CHM files and verify that when they are reintroduced that opening any of the CHM files again displays all of the merged CHM files.

Merged Navigation in HTML Help - Troubleshooting

To verify that everything should be working properly, you can open the temporary hhp file that Flare generates for each CHM file. Next to the output folder for each compiled CHM file is a "Temporary" folder. Browse into "Temporary/[Target Name]/Content" and open _Temp.hhp in Notepad. Scroll to the bottom of the hhp files and you should see a section labeled "[MERGE FILES]." The hhp for every CHM file (parent and all children) should display all linked CHM files in this list.

Example:

[MERGE FILES]
MyParent.chm
MyFirstChild.chm
MySecondChild.chm
MyThirdChild.chm
