Output Eclipse Help - Output Files

The following is an outline of all files and directories that should be generated when building an Eclipse Help output. All output files will exist in a folder that gets its name from the Plug-in ID and Version set in the Eclipse Target Editor ("[Plug-in ID]_[Plug-in Version]").

  • If the Plug-in ID is left at default in the Target Editor, Flare will use "com.[target name in lower case].default_[Plug-in Version]" in the folder name
  • If the Plug-in Version is left at default in the Target Editor, Flare will use "1.0.0.[yyyyddmmhhmmss]" (the current date and time) in the folder name

Note: If the Target option "Archive to JAR" is enabled, the output of an Eclipse target will consist of a single JAR files that uses the same naming scheme as the output folder described above, "[Plug-in ID]_[Plug-in Version].jar"

Root-Level Files

  • csh.xml: The CSH file will be present in the output if an alias file is selected in the target and there is at least one topic associated with a CSH identifier.
  • index.xml: The index file. Project must contain at least one keyword for this file to be present in the output.
  • plugin.mcwebhelp: This is an artifact from the WebHelp part of the Eclipse compiler
  • plugin.xml: This file will be generated by default
  • toc.xml: The TOC file. This will always be present in the output as you must select a TOC to build Eclipse output.

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.

Subsystems Folder

This folder will contain the output folders of any merged Eclipse Help projects. This folder will only be present if you have linked to another Flare project or compiled Eclipse Help output in the parent project's fltoc file.
