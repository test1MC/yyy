Language Skins - Editor

Opening a Language Skin File

Language Skin files are opened from the Language Skins dialog. To open the Language Skins dialog:

  • Tool Strip interface: !Tools | Manage Language Skins…
  • Ribbon interface: !Tools | Manage Language Skins

Language Skins dialog - UI Test Cases

UI Controls

  • Language Skins Grid: Displays a list of all languages supported by Flare. Verify any new languages added appear in this list.

    Columns

    • Language Skins: Displays the language for each potential Language Skin.
    • Path: If a Language Skin has been created for your user, this column will display the path to that file. The path should be "C:\Users\[username]\AppData\Roaming\MadCap Software\Flare\Language Skins\[filename].fllng"
  • Remove User Defined File…: (Only active when the selected Language Skin file exists locally) Send the selected Language Skin file from the user's AppData\Roaming folder to the Recycle Bin. A confirmation is required.
    • OK: Closes the confirmation dialog and sends the Language Skin file to the Recycle Bin.
    • Cancel: Closes the confirmation dialog and does not send the Language Skin file to the Recycle Bin.
  • Open File For Editing: 
    • If the selected Language Skin file already exists for your user (displays in italics): Closes the Language Skins dialog and opens the selected Language Skin file for editing.
    • If the selected Language Skin file only exists as a factory template (displays in bold): Prompts that a Language Skin file for that language has been created in your AppDate\Roaming folder. Upon clicking OK to that confirmation dialog, the Language Skins dialog will close and open the selected Language Skin file for editing.
    • If the selected Language Skin file does not yet exists (displays in plain text): Invokes the Select Template Language Skin dialog which lets you select any existing factory template Language Skin or any user created Language Skin to use as a template to create the new Language Skin file.
  • Close: Closes the dialog and does not open any Language Skin file.

Select Template Language Skin dialog - UI Test Cases

UI Controls

  • Template Language Skins Grid: Displays a list of all Language Skins that are either provided by default or have been created by the user. In the default view, factory provided Language Skins will display in alphabetical order first. User created Language Skins will appear at the end of the list. Verify that user created Language Skins do in fact appear in this list.

    Columns

    • Select Template: Displays the language for each existing Language Skin that could be used as a template.
    • Path: This column has never populated (See bug 12726)
  • OK: Creates a new Language Skin based on the selected template Language Skin and closes both the Select Template Language Skin dialog and the Language Skins dialog.
  • Cancel: Closes the dialog and does not open any Language Skin file.

Language Skin Editor - UI Test Cases

Language Skin Editor: Local Toolbar

Toolbar items (Tooltips in parentheses)

  • (Cut): Cuts the selected style class. Only relevant for classes created for Relationships, Toc Entry and Toolbar Item styles.
  • (Copy): Copies the selected style class. Only relevant for classes created for Relationships, Toc Entry and Toolbar Item styles.
  • (Paste): Pastes the selected style class. Only relevant for classes created for Relationships, Toc Entry and Toolbar Item styles.
  • (Delete): Deletes the selected style class. Only relevant for classes created for Relationships, Toc Entry and Toolbar Item styles.
  • Add Class:Disabled unless a style is selected in the Style Tab, but only relevant for Relationships, Toc Entry and Toolbar Item styles. Click to add a new class to the selected style in the Style Tab.
