Output HTML5 - Output Files

The following is an outline of all files and directories that should be generated when building an HTML5 output.

This list is out of date. Some files are missing.

Root-Level Files

  • csh.js
  • [Output Filename].htm
  • [Output Filename].js
  • [Output Filename].mcwebhelp
  • [Output Filename]_CSH.htm

  • Sitemap.xml: This file should appear here only if you have enabled the Target option to generate a Sitemap.

Automerge Folder (Server-based Output)

No content should ever be generated in this folder. Additional HTML5 systems are placed here for runtime merging. This folder only appears when Server-based Output is enabled in the target options.

AutoSearch Folder (Server-based Output)

No content should ever be generated in this folder. Non-XHTML files placed in this folder will show up in the search for the output. This folder only appears when Server-based Output is enabled in the target options.

Content Folder

This Content folder should contain all of the topic files, stylesheets, and images in the Content folder of your project. Masterpage files, snippet files and page layout files should not appear in this output folder. Other non-Flare file types (*.doc, *.pdf, etc.) included in your Flare project's Content Explorer should appear in this output Content folder as well. The folder structure in this Content folder should mirror the folder structure of your files in the Content Explorer.
