Merging - Baseline

Test Cases

Test that each of the following function in the final output for both the Master and Slave projects in all online outputs. Items marked with an (*) can be tested using the Test Projects available here:

\\files\QA\Test Projects\Test Plan Projects\Merging\Merging

 

TOCs (*)

  • Create multiple TOC in the Slave and assign unique TOC to different Targets. Test that the appropriate TOC is generated for a Slave project.

Browse Sequences

  • Browse Sequences need to be turned on in the Slave project to appear in the Master Project’s output.
  • Merging Browse Sequences is not intuitive and tedious. Must link to Slave BS in Master project’s BS.

Glossary terms (and definitions) (*)

  • Glossary terms from all projects are merged correctly.
  • Glossary terms set in the Master project convert terms in slave projects

Index (Index synching) (*)

  • Index Keywords from Master and Slave projects are present in the output.
  • Index Keywords from all projects appear in the correct order.

Concepts (*)

  • Concept links (Help Control A-Link) from Master to Slave project and vice versa work.
  • Concept links (Help Control A-Link) between two Slave projects work.

Searching (*)

  • Searching finds results in both Master and Slave projects.

Search Filters (*)

  • Search Filters from Slave projects do not appear in any output.

Master Pages (*)

  • Master Pages should be treated independently within each product. Test including Master Pages with the same name in a Master and Slave project and confirm the output.

Variables (*)

  • Variables should be treated independently within each product. Test including variables with the same name in a Master and Slave project and confirm the output.

Snippets (*)

  • Snippets should be treated independently within each product. Test including Snippets with the same name in a Master and Slave project and confirm the output.

Images

  • Images should be treated independently within each product. Test including Images with the same name in a Master and Slave project and confirm the output.

Audio Links (*)

  • Confirm Echo Sound files embedded in slave projects get compiled.

Movie Links

  • Confirm Mimic Movie links embedded in slave projects get compiled/copied into the Master project’s output.

Skins

  • Generally uses the Master skin; some exceptions with Styles with certain outputs

Conditional Text(*)

  • Conditional text is excluded/included based on the Target setting in each Master/Slave Project (ie, Conditional Text settings in the Master Project do not affect content in the Slave Projects and vice versa).
