Merging Flare Projects using Targets

Save your Slave project(s).  Open your Master project and open the TOC in the Master project.  Create a TOC entry which will be used to link to the Slave project (one entry per Slave project).  View the properties dialog for that TOC entry.

TOC Editor – Setup

TOC Entry Properties dialog | Basic tab

If you have access to the Slave project, you can select “Select Flare Project and Target” in the Link Section of the TOC Entry Properties Basic Tab:

 

Relevant UI Controls

  • Link: Displays the path of the Flare project file to which you are linking.
  • Select Flare Project and Target: Clicking on this option will launch the Link to Flare Project and Target dialog.
  • Select Link: Opens the dialog appropriate to the current Link selection.  Once a link to a Flare project is established as indicated in the screenshot, this button will become active. Clicking it will open the Link to Flare Project and Target dialog.

UI Controls

  • Browse for Project: Lets you use the Project File button to browse for a Flare project file (*.flprj) to be the Slave project.
    • Project File: Opens a window which lets you browse for a Flare project file (*.flprj).
  • Select Recent Project: Lets you select the Slave project from the list of Recent Projects.
    • File (bottom view): Displays all Flare projects currently listed in the Recent Projects list.
    • Path (bottom view): Displays the path of all Flare projects currently listed in the Recent Projects list.
  • Select Target: Displays the available qualifying targets (DotNet Help, HTML Help or WebHelp) of the selected Flare project.  Lets you designate which Target the Slave project should generate when the Master is generated.   If “(default)” is selected, Flare will generate the first target of the same type (as the Master’s target) that it finds in the Slave project.

Test Cases

  • Test that all qualifying targets display in the Select Target pull-down for the selected Flare project. 
  • Test that non-qualifying targets do not display in the Select Target pull-down for the selected Flare project.

Target Editor – Setup

Output MOTW setting

If generating WebHelp, the Master project’s Target Mark of the Web setting must be the same as all Child projects being merged. Set the MOTW setting in the Target Editor | Advanced tab:

 

Test Cases

  • Test that inconsistent MOTW settings generate compiler error.
  • Test Master w/MOTW and Slave without

Output file names

If the Parent project links to more than one Child project, each Child project’s Target must have a unique Output Filename.  Set the Output file name for each project in the Target Editor| General tab.

 

Test Cases

  • Test linking to two Child projects using the same output filename.  Confirm error is given for all outputs.
  • Test linking to a Child project which has its own Child project. Test that an error is given when generating a CHM when the sub-Child project has the same Output filename as the Parent or Child project.

NOTE: In the sample project provided, the Child project's CHM Target does not use the Master TOC.  Make sure to add the sub-Child to the appropriate TOC.

Pre-Merging

A user may elect to pre-merge the Index and/or Search Database of the Parent and Child project(s) when compiling the output.  This option is only available when merging projects using Targets (linking to the actual *.flprj file).  The result will be a complete Index and/or Search Database file in the Parent project’s output folder.

If pre-merge is activated for the Index and/or Search Database file, runtime merging is disabled, which means that you cannot compile and publish a "sub-project" later. You must generate and publish all linked Help systems at the same time.

Target Editor | Performance tab

Relevant UI Controls:

  • Pre-merge Index File: Check mark indicates the Index file for the Master and all Slave projects will be pre-merged when generating the output.
  • Pre-merge Search Database File: Check mark indicates the Search Database file for the Master and all Slave projects will be pre-merged when generating the output.
