Merging Compiled WebHelp and DotNet Output

This section focuses specifically on merging WebHelp and DotNet Help outputs.  Merging the output of CHMs is very similar to merging WebHelp and DotNet Help output, so many of the test cases and processes will seem similar, but they are in fact slightly different.

Before your Parent project can be merged with an external Help system, the output folders and files for the external (Child) project need to be copied to a folder called "Subsystems" in the same location where the Parent project is being published.  Flare will do this for you if you know the path to the external Help system.

Example:

Publish Parent project to: …\PublishDestination\ParentProject\(ParentProject.mcwebhelp or ParentProject.mchelp)

Publish Child projects to: …\PublishDestination\ParentProject\Subsystem\ChildProject\(ChildProject.mcwebhelp or ChildProject.mchelp)

TOC Editor – Setup

Open your Parent project and open the TOC in the Parent project.  Create a TOC entry which will be used to link to the Child project.  View the properties dialog for that TOC entry.

TOC Entry Properties dialog / Basic tab

Relevant UI Controls

  • Link: Displays the name or path of the External Help System to which you are linking. (If you enter the path, Flare will copy the output files of the External Help Systems for you.  If you enter only the filename, you will need to copy the output files into the “Subsystem” folder of the Parent project.  Or, you can set the Publishing Destination of the External Help Systems to publish directly to the “Subsystem” folder.  See Target Editor – Setup section below)
  • Enter External Help System: Clicking on this option will launch the Link to Help System dialog.
  • Select Link: Opens the dialog appropriate to the current Link selection.  Once a link to an External Help System is established as indicated in the screenshot, this button will open the Link to Help System dialog.
  • Open Linked File: Opens the selected External Help System file (if the path is provided in the Link field; if the Link field is only populated with the filename of the Help System, this button should be inactive).

Link to Help System dialog - UI

UI Controls

  • Enter the name of the help system…: Enter the path or name of the Help system (If you enter the path, Flare will copy the output files of the External Help Systems for you.  If you enter only the filename, you will need to copy the output files into the “Subsystem” folder of the Parent project.  Or, you can set the Publishing Destination of the External Help Systems to publish directly to the “Subsystem” folder.  See Target Editor – Setup section below)
  • Browse: Opens a window which lets you browse for an External Help System (*.chm, *.mcwebhelp, or *.mchelp).

Appearance in the TOC Editor

The TOC entry icon for an External Help System Link is .  Hovering over the TOC entry should reveal the path to and/or the filename of the External Help System file.

Target Editor – Setup

Output MOTW setting

If generating WebHelp, the Parent project’s Target Mark of the Web setting must be the same as all Child projects being merged. Set the MOTW setting in the Target Editor | Advanced tab:

 

Test Cases

  • Test that inconsistent MOTW settings generate compiler error.
  • Test Parent w/MOTW and Child without
  • Test Child w/MOTW and Parent without

Output file names/folder

If the Parent project links to more than one Child project, each Child project’s Target must have a unique Output Filename. Set the Output file name for each project in the Target Editor | General tab.

 

Set the Output Folder for all Child projects to be:

…\ParentTargetFolder\ParentProjectName\Subsystems\Child1 (Child2, etc)

 

Test Cases

  • Link to an External Help Systems which is not the same output type as the Master Project. Generate the Master Project and test that an error is generated in the compiler.
  • Test linking to two Child projects using the same output filename. Confirm error is given for all outputs.

Output Publishing

If your Parent Project links knows the path of all External Help Systems (the link set in the TOC node's Properties dialog uses the full path and not just the name), the linked Help Systems will automatically be copied to the Parent project’s output when the Parent Project is generated. If this is the case, there is no setup necessary here.

If your Parent project links only to the External Help Systems by filename, you will have to manually copy the Child project’s output into the Parent project’s “Subsystem” folder. Or, you can setup each project to publish accordingly:

  1. Create a Destination in the Parent project. Be aware of the Directory that you set in the Parent project.

  2. Create a Destination in each Child Project. Be sure to set the Directory to be the "Subsystem" folder inside the Parent project's Directory.

  3. In the Target Editor of the Parent project and each Child project, associate each Target with the Destination.

  4. Build and publish all output.
