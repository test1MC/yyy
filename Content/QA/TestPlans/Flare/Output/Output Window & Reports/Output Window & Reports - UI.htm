Output Window & Reports - UI

Generating Output

To generate the output of the Primary Target:

Click Build Primary in the Project Toolbar

Press F6 on the keyboard

!Build | Build Primary

 

To generate the output of any target:

Target Editor: Open the target you want to build. Click the Build button in the Target Editor's local toolbar.

Right-click the Target in the Project Organizer and select Build '[TargetName]'

Select a Target in the Project Organizer and !Build | Build

Build Progress dialog

Messages tab:

Errors tab:

UI Controls

  • Messages tab: Displays messages from the generation of the target that was built.

  • Errors tab: Displays errors that were encountered during the build process.

    Columns

    • Description: A description of each error is given.
    • Subsystem: The Subsystem where the error occurred.
    • File: The file which caused the error.
    • Line: The line number in the file where the error occurred.
    • Char: The character number (within the line) where the error occurred.

  • Save Log: If the build finished and there are errors, this button will become active. Clicking it will create an error report or Log file (*.mclog) and save it to the project's Reports folder, and should produce the following alert:

  • View Output: Opens the generated output in the appropriate viewer.
  • Open Output Folder: Opens the project's Output folder.
  • Close: Closes the dialog.

Publishing Log dialog

UI Controls

  • Destination List: Lists all Destinations to which content was published. Select a Destination in this column to display the corresponding log. If the output is published to multiple Destinations, verify that this list is sortable by clicking the column heading.

  • Action List

    Columns

    • Action: Displays the action performed to each published file
    • File: Displays the filename of each published file
    • Folder: Displays the path to which each file is published
  • Save Log File: Saves an xml document ("UploadLog") to the projects Reports folder
  • Log created… : Verify the created date displays correctly
