Code

No Class:

<MadCap:shortcut path="C:\Windows\System32\notepad.exe" parameters="" message="" wparam=""
lparam="">My shortcut</MadCap:shortcut>

 

With Class:

<MadCap:shortcut path="C:\Windows\System32\notepad.exe" parameters="" message="" wparam=""
lparam="" class="notepad">My shortcut</MadCap:shortcut>

 

Editing Shortcut Controls:

You can edit a Shortcut control that you have inserted into a topic in a couple of different ways.

Changing the settings in a Shortcut control

The first way to edit a Shortcut control is to change the settings (e.g., executable file path, parameters, message ID) associated with the control.

 

  1. Open the topic with the shortcut control that you want to edit.
  2. In the XML Editor, right-click the Shortcut control icon (if "markers" are set to be shown).
  3. In the context menu, select Edit Shortcut Control.
    1. The Insert Shortcut Control dialog opens.
  4. Adjust the settings in the fields as necessary. You may need to consult with your developer for help on completing these fields.
    1. File path to the executable file… Enter the path and name of the program (EXE file) you are calling.
    2. Parameters to be sent…: Enter any parameters you would like to send the program (e.g., the name of a file to open). Separate the parameters with commas.
    3. A message ID of a standard Windows message: Select a standard Windows message from the drop-down menu.
    4. WPARAM: value Enter the WPARAM value.
    5. LPARAM: value Enter the LPARAM value.
  5. Click OK. The Insert Shortcut Control dialog closes, and the link is now changed.
  6. Press CTRL+S or click to save your work.
  7. Edit the style of a Shortcut control

 

The second way to edit a Shortcut control is to edit the style (e.g., font, color) of the link. When you edit the style for a Shortcut control, the style changes for all Shortcut controls in any topics in your project. To do this, follow the steps in the topic Editing Styles, and modify the property settings for the style called <MadCap|shortcut>.

Note: If you are working with HTML Help and import a RoboHelp project containing Shortcut links, the linked files are placed in a special subfolder in the Content Explorer (Resources\CHMSupport). For more information, see Importing Tips
