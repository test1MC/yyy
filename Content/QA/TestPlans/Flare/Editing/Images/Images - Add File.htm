Copy to Project Dialog

If an image that is being inserted into a topic resides outside the project, clicking "OK" in the "Insert Image" dialog will generate the "Copy to Project" Dialog.

 

Columns:

  • Action: Displays the action being performed such as copy.
  • File: Displays the name of the file. If the file is renamed projects containing those files may show a missing file so the user will have to re-add the file.
  • Type: Displays the file type.
  • Path: Displays the path where the file is stored. If the file is moved this path should update automatically.

 

Button/Section Description:

  • The following files will be copied to the project:Displays the files to be copied. If the file being added is dependent on other files to operate correctly (e.g., if it is linked to another topic file), those necessary files are also listed in the dialog (and therefore copied as well).
  • Keep file(s) synchronized (create mapping):Select this check box if you want to create a mapping between the project's copy of the image and the original source image
  • Add Files to source control: If project is bound to source control flare offers to add the file to the repository
  • Cancel: Closes the dialog without adding the new image file.
  • OK: Adds the new image file to the project and opens the image in the Image Viewer.

Note that all added images will by default be located in Content\Resources\Images (Images folder will be automatically created in Resources folder)

Note that captured images using MadCap Capture will be stored in Content\Resources\Images, each image captured will be accompanied with a .props file that stores layer information for the corresponding image. The .props will have the same file name as the captured image

Image Viewer

  • MadCap Capture Icon (Edit this file with in MadCap Capture): Opens the image in MadCap Capture if you have it installed else it will prompt the user that they need MadCap Capture installed. This option should be greyed out if the image is a web image.
  • Regenerate Captured Images: Regenerates the image with the most current version of the source file

Size Tab

UI Controls

  • Size: Set a precise width and height. To preserve the aspect ration of an image, set the width or height to the value you want and set the other to "Automatic."
  • Minimum: Set a minimum width and/or height. Use these options to enlarge a small image and preserve its aspect ratio.
  • Maximum: Set a maximum width and/or height. Use these options to reduce the size of a large image while preserving its aspect ratio.

Note: If you are using percentage for the size of an image, the percentage refers to the block containing that image, not to the image itself. For example, if you set the width to 60%, this does not mean that the image will be reduced to 60% of its size. Instead, it means that the image will be resized so that its width is 60% of the "container" where it is inserted.

Position Tab

UI Controls

  • Float: Use this field to specify where to place the picture on the page.

  • For floats that include frames, you have to be in print layout mode

    • None: Does not place the picture in a specific location.
    • Left: Positions the picture on the left side of the page frame, allowing you to type text to the right of the picture.
    • Right: Positions the picture on the right side of the page frame, allowing you to type text to the left of the picture.
    • Center of Column: Positions the picture in the center of the column on the page.
    • Outside Left Margin: Positions the picture beyond the left margin of the topic text.
    • Outside Right Margin: Positions the picture beyond the right margin of the topic text.
    • Outside Frame Positions: The picture outside of the page frame.
    • Outside Frame, Top Align: Positions the picture outside of the page frame, as well as aligning it with the top of the frame.
    • Left of Frame: Positions the picture to the left of the page frame.
    • Right of Frame: Positions the picture to the right of the page frame.
    • Center of Frame: Positions the picture both vertically and horizontally in the middle of the page frame.
  • Clear: Use this field to position a picture so that it is "clear" of an adjacent picture.
    • None: Does not apply the clear property to the picture.
    • Left Side: The picture will be placed below the bottom outer edge of a previous picture that is floating left.
    • Right Side: The picture will be placed below the bottom outer edge of a previous picture that is floating right.
    • Both Sides: The picture will be placed below the a previous picture, whether it is floating left or right.
  • Vertical Alignment: Use this field to adjust where the item is positioned vertically.
    • Baseline: The baseline of the box will be aligned with the baseline of the parent box.
    • Text Top: The top of the box will be aligned with the top of the parent element's font.
    • Text Bottom: The bottom of the box will be aligned with the bottom of the line box.
    • Top: The top of the box will be aligned with the top of the line box.
    • Middle: The vertical midpoint of the box will be aligned with the baseline of the parent box, plus half the x-height of the parent.
    • Bottom: The bottom of the box will be aligned with the bottom of the line box.

Note: If you want to adjust the space between the object and the text that flows next to it, you can modify the margins on the object (using the Borders & Margins tab in the dialog).

Thumbnails Tab

A feature for Web outputs which lets you display an image as a thumbnail. Upon hovering over or clicking on the thumbnail image, the full-size image appears. See the Thumbnails test plan for more details. Images Thumbnails.htm

Borders & Margins Tab

UI Controls

  • Margin: Margin is the distance from the edge of the document to the edge of the content.

    Click in any of the individual fields (Left, Right, Top, Bottom) to specify the settings for the margins around the image. Click the down arrow to the right of all the fields to apply setting to all of the margin fields.

  • Border: Border is a line drawn around the content.

    Click in any of the individual fields (Left, Right, Top, Bottom) to specify the settings for the image border. Click the down arrow to the right of all the fields to apply settings to all of the border fields.

  • Padding: Adds extra space between an image's border and the image itself.

    Click in any of the individual fields (Left, Right, Top, Bottom) to specify the settings for the padding in the image. If you click the down arrow to the right of all the fields, the settings will be applied to all of padding fields.

  • Preview: Displays a preview of how a image will look as you specify settings.
  • : sets the top, bottom, left and right setting all at the same time with the same values.

Click on this thumbnail to see how you should expect Margin, Border and Padding get applied to an image:

Test: test using the drop down setting selection as well as setting each area manually one at a time and all together. Some combinations may cause a problem. Don't just use the drop down and think that the rest work.

Note: Different browsers may treat margin and padding settings differently. For example, Internet Explorer 8 and Firefox honor padding settings more than they honor margin settings. For example, if you were to set a left margin at 1 inch, Internet Explorer 7 would show it that way. However, in order to get the same results in Internet Explorer 8 or Firefox, you would also need to set the left padding at 1 inch.

Background Tab

UI Control

  • Color: Sets the color of the background. In the Color field, click the down arrow and select a color from the popup.
  • Image: Choose an image for the background.
  • Repeat: Set the repeat pattern for the background image.
  • X: Select or enter the position (in pixels) of the image in the background. This positions the background image horizontally (i.e., on the X axis).
  • Y: Select or enter the position (in pixels) of the image in the background. This positions the background image vertically (i.e., on the Y axis).
  • Preview: Displays the appearance of the background as you make selections in the tab.

UI Controls (General tab)

  • Images in Project

    Toolbar (Tooltips in parentheses): 

    • (Show/Hide All Files in Project): Changes the view in the dialog to "Show All Files" view. This view display all topics in the project in one grid. Detailed testing below.
    • (Show/Hide Folders): Changes the view in the dialog to "Show Folders" view. This view displays all files in the project in a tree view, similar to the Content Explorer. This view can be combined with the "Show Files" view to offer a "Split" view. Detailed testing below.

This view displays all files in the project one folder at a time, similar to the "Show Files" view in the Content Explorer. This view can be combined with the "Show Folders" view to offer a "Split" view. Detailed testing below.
    • (Up): Moves up one folder in the hierarchy within the project's Content folder. This button becomes enabled if you use the "Show Folders" or "Split" View and browse into a folder.
    • [File filter]: Use this filter to limit the view of files to all images in the project (all supported file types) or just web-safe file types (*.png, *.jpg, *.jpeg, *.gif, *.swf). Test that you can also enter text in the filter (and press <Enter>) to filter by filename.
  • [Image Path]: Displays the path to the linked image. Verify that you can use the pulldown to link to recently used image files.
  • : You can click to find and select an image file outside of the project.
  • Preview: Displays a preview of the image
  • Style Class: If you want to apply a specific style class to the image, you can select it from the Style Class field.
  • Screen Tip: Enter a phrase that will appear when the end user hovers over the picture.
  • Alternate Text: Enter alternate text to display when the image is not available, such as when a disabled individual is using a screen reader.

Note: If you the browse button to select an image outside the project, that file is then copied and placed inside the project. The image file is stored in the Resources\Images folder of the Content Explorer.

Note: If you import Microsoft Word documents containing embedded pictures into a project, Flare places copies of those image files in subfolders within the Resources\Images folder. This is done in order to avoid naming conflicts with embedded images from other Word documents that you might import. The subfolders are named after the Word documents being imported.
