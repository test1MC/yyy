Images: Thumbnails - Output

Support in Web Outputs:

Support in Web Help Outputs:

  • Verify Thumbnails work for all web output types as well as topic previews (no java script errors).
    • Test with "Condense JavaScript Files" disabled in the Performance tab of the Target Editor
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the Target Editor
  • Verify that all Thumbnail behaviors work in all supported browsers.
  • Verify that the entire <img> tag and all of its properties are removed when conditionalized out. This includes verifying that neither the full size or the thumbnail of the image is included in the Output.
  • Verify that applied styles are in outputs for both the full-size image and the thumbnail. (There are several "Known Issues" in V5.)
  • Verify that Thumbnails work in conjunction with all "Advanced" options in the Target Editor (ie, "Remove Content folder," "Use lowercase filenames," etc). (Several of these Test Cases are available in the Baseline testing. As more advanced functions are added, be sure to test all of them in all configurations.)
  • Note: If you experience problems in the output with thumbnail images not opening to the full size, this might be caused by "Mark of the Web" being enabled on the Advanced tab of the Target Editor. You can try disabling Mark of the Web from your target to fix the issue. See Adding Mark of the Web.

Support in Print Outputs:

Support in PDF Outputs:

  • Print outputs do not support Image Thumbnails. However, you should verify that full-size images are always generated to print output and not an enlarged version of the thumbnail image (that looks very pixilated). Verify also that the image always displays full-size when viewing a topic in the "print" medium in the XML Editor.

Support in DITA Output:

DITA does not offer an equivalent element to Image Thumbnails. When generating DITA output, confirm that we simply output the original image to the topic. Verify that we do not generate a thumbnail copy of the image.
