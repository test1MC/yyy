Index Links Editor - UI Test Cases

Local Toolbar

Button (Tooltips in Parentheses):

  • (New item): Opens the (Auto-index Phrase) Properties dialog for a new entry. Clicking Cancel in this dialog should not create a new entry.
  • (Run Properties dialog for this item): Opens the Properties dialog for the selected auto-index phrase.
  • Cut: Cuts the selected auto-index phrase(s) to the clipboard.
  • Copy: Copies the selected auto-index phrase(s) to the clipboard.
  • (Paste): Pastes the auto-index phrase(s) from the clipboard.
  • (Delete): Deletes the selected auto-index phrase(s)

Right-click menu

Right-clicking anywhere in the Index Link Editor should produce the following menu:

Right-click menu:

 

Right-click menu items:

  • New: Opens the (Auto-index Phrase) Properties dialog for a new entry. Clicking Cancel in this dialog should not create a new entry.
  • Cut: Cuts the selected auto-index phrase(s) to the clipboard.
  • Copy: Copies the selected auto-index phrase(s) to the clipboard.
  • Paste: Pastes the auto-index phrase(s) from the clipboard.
  • Delete: Deletes the selected auto-index phrase(s)
  • Properties: Opens the Properties dialog for the selected auto-index phrase.

 

(Index Link) Properties dialog

To open:

  • Double-click any row in the editor
  • Select any row in the editor and click in the local toolbar
  • Select any row in the editor and go !Edit | Properties
  • Select any row in the editor and press F4 on the keyboard

 

UI Controls:

  • Term: Displays the term for which you want to create an index link.
  • Link Type: Displays the type of Index Link you want to create.
  • Linked Term: Displays the term to which the “Term” should link.

 

Test Cases

  • For Term, enter single words, multiple words, numbers, words with international characters, etc.; confirm these work in all outputs
  • For Linked Term, enter existing and non-existing keywords; confirm non-existing instances generate errors upon generation and do not appear in output.

Tooltips

If any Term, Linked Term or Link Type is so long that it does not fit within the column, hovering over the cell will display the entire contents of that cell.

Conditional Tags

Conditional Tags cannot be applied to individual Index Links. Conditional Tags can be applied to the actual Index Link Set file (*.flixl), however, as of Flare V5, this tag can only be used for the purposes of Advanced Global Project importing.

Code

<?xml version="1.0" encoding="utf-8"?>
<CatapultIndexLinkSet>
<IndexLink
Term="Flare"
LinkedTerm="MadCap Flare"
LinkType="see" />
<IndexLink
Term="Lingo"
LinkType="see"
LinkedTerm="MadCap Lingo"></IndexLink>
<IndexLink
Term="Mimic"
LinkType="see"
LinkedTerm="MadCap Mimic"></IndexLink>
<IndexLink
Term="MadCap Mimic"
LinkType="seealso"
LinkedTerm="MadCap Movie Sync"></IndexLink>
<IndexLink
Term="MadCap Flare"
LinkType="seealso"
LinkedTerm="MadCap Analyzer"></IndexLink>
<IndexLink
Term="CHM"
LinkType="see"
LinkedTerm="Output:HTML Help"></IndexLink>
</CatapultIndexLinkSet>
