Mode Warning dialog

Opening the dialog

Close Flare when Flare is in Index or Concept Entry Mode. When you re-open Flare, the Mode Warning dialog should display. If it does not, go to !File | Options (!Tools | Option in Toolstrip interface) and reset the Mode Warning dialog option in the Index & Concepts tab. See Restoring the dialog below.

Mode Warning dialog - UI Test Cases

UI Controls

  • (Action) Continue in mode: Upon clicking OK, Flare will continue to be in Index or Concept Entry Mode.
  • (Action) Switch to normal selection mode: Upon clicking OK, Flare will switch from Index or Concept Entry Mode to a normal selection mode.
  • (Options) Check mode and display this dialog when first topic opened: If this option is selected, whenever Flare is opened with the editor in Index Keyword or Concept Entry Mode, the Mode Warning dialog will appear.
  • (Options) Auto-switch to normal selection mode on launch: If this option is selected, and Flare is opened with the editor in Index Keyword or Concept Entry Mode, Flare will automatically switch to normal selection mode. The Mode Warning dialog will not open.
  • (Options) Continue in mode: If this option is selected, and Flare is opened with the editor in Index Keyword or Concept Entry Mode, Flare will do nothing. The Mode Warning dialog will not open and Flare will continue to be in the entry mode it was last in.

Restoring the dialog

If you have selected either the "Auto-switch to normal selection…" or "Continue in mode" options, you will have created a situation where Flare will never produce the Mode Warning dialog because you have set a permanent action. This option can be reset in the Options dialog available by going !File | Options (!Tools | Options in the Toolstrip interface).
