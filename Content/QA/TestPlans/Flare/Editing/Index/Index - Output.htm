Index - Output

Support in Web Outputs:

Support in Web Help Outputs:

  • Create keywords that link to multiple topics
  • Create "compound keywords," that is, use more than one index term in a single <MadCap:keyword> tag, and verify that all keywords output correctly
  • Create multi-level keywords; use the parent term in some topics and the sub-terms in others
  • Create valid and invalid "See" and "See Also" references with an Index Link Set; verify the valid ones output correctly and the invalid ones produce compile errors
  • Create auto-index phrases and verify they create index terms in all outputs
  • Use condition tags to:
    • Condition out keywords and verify those keywords do not appear in the output
    • Condition out topics and verify any keywords contained in those topics do not appear int he output
Skin Setup
  • For all Web-based outputs, the Index must be enabled in the Skin in order for an Index to appear in the output.

Support in Print Outputs:

Support in EPUB Outputs:

  • For all print outputs, an Index Proxy must be included in a topic and that topic must be included in the TOC being used by the Print Output Target.
  • Create keywords that link to multiple topics
  • Create "compound keywords," that is, use more than one index term in a single <MadCap:keyword> tag, and verify that all keywords output correctly
  • Create multi-level keywords; use the parent term in some topics and the sub-terms in others
  • Create valid and invalid "See" and "See Also" references with an Index Link Set; verify the valid ones output correctly and the invalid ones produce compile errors
  • Create auto-index phrases and verify they create index terms in all outputs
  • Use condition tags to:
    • Condition out keywords and verify those keywords do not appear in the output
    • Condition out topics and verify any keywords contained in those topics do not appear int he output
  • Place index keywords in successive topics so that the keyword will produce a range of pages in the output. For example, test that a keyword that occurs on page 9, 10, 11 and 12 is output as 9-12 (except for Word).
  • Test Index Heading variables. In the Page Layout file associated with the topic containing the Index Proxy, insert the heading variables FirstIndexTerm, LastIndexTerm, FirstIndexHeading and LastIndexHeading. The factory template page layout file IndexResizable.flpgl already includes the "term" variables.

Support in DITA Output:

  • When generating DITA output, Flare's index source code needs to be converted to DITA's index source code. For this reason, DITA does not support Index Proxies,

 

Here are some examples of how Flare index code should appears in generated DITA content:

Flare Code: DITA Code:

<MadCap:keyword term="header" />

<indexterm>header</indexterm>

 

 

Compiler errors:

Any attempt to use an Index Link where the linked term does not exist should produce:
"Linked index term "[Linked Term]" is not in the index, skipping the link "[Term]"…
