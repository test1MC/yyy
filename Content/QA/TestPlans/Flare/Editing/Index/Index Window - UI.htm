Index Window - UI

Opening the Index pane

  • !Tools | Index | Index Window ... for the Tool Strip Mode
  • !View | Reference | Index Window ... for the Ribbon Mode
  • !View | Index Window
  • Press <F9> on the keyboard
  • In Index Entry Mode (!Tools | Index | Index Entry Mode), place the insertion point in the XML Editor and begin typing. The Index Window will open when you start typing, and whatever you type will appear in the Index Window.

Index "Entry" (top half of Index window)

The top portion of the Index Window is used for displaying, entering and editing a specific instance of a keyword object. A specific instance of a keyword object may include multiple keywords (one per line).

 

UI Controls

Terms: Lists and lets you edit, add or remove keywords.

Index "Explorer" (bottom half of Index window)

The bottom portion of the Index Window is used to view all Index Keywords that exist throughout the project. Additionally, this "explorer" view will display any Index links ("See" and "See also" references) that exist in any Index Link Sets used in the project. Finally, the "explorer" view will also display any conditional tags applied to a keyword or its parent elements (p, h1, html, etc).

Local Toolbar

Local Toolbar Buttons (Tooltips in parentheses):

  • (Expand all): Expands all of the nodes so that you can see all of the files under each one.
  • (Expand keyword entries): Expands all of the nodes that have keyword sub-nodes, so that you can see all multi-level keywords.
  • (Collapse all): Collapses all of the nodes so that you can no longer see all of the files under each one.
  • (Collapse topic entries): Collapses all of the keyword nodes so that you can no longer see all of the files under each one, but keeps sub-nodes visible.
  • (Open linked file): Opens the selected topic file in the XML Editor. Only active when a topic is selected.

Right-click menus

Root node right-click menu: 

Menu items (tooltips in parenthesis):

  • Add Top-Level Keyword (Add a new top-level index keyword): Adds a new top-level keyword. Until it is added to a topic, this keyword exists only in a temp file in the project's Output/user/Temporary folder.

Keyword (or sub-keyword) right-click menu:

Menu items (tooltips in parenthesis):

  • Add Top-Level Keyword (Add a new top-level index keyword): Adds a new top-level keyword. Until it is added to a topic, this keyword exists only in a temp file in the project's Output/user/Temporary folder.
  • Add Sub-Keyword (Add a new lower-level index keyword): Adds a sub-keyword node to the selected keyword. Until it is added to a topic, this sub-keyword exists only in a temp file in the project's Output/user/Temporary folder.

  • Add Index Link (Add an index link): 

    • If no Index Link Set Exists in the project: Invokes the Add Index Link Set dialog. After creation of the Index Link Set file, a new index link entry is automatically added and it's Properties dialog is invoked with the selected index term pre-populating the "Term" field.
    • If an Index Link Set already exists: Open the Index Link Set Editor, creates a new entry and invokes the Properties dialog for that new entry with the selected index term pre-populating the "Term" field.

    In either case, when the Properties dialog opens, the state of the "See"/"See Also" radio button is dependent on whether or not the selected keyword is associated with any topics.

    • If the selected keyword has associated topics, the Properties dialog should open the with the "See Also" option selected.
    • If the selected keyword does not have associated topics, the Properties dialog should open the with the "See" option selected.
  • Assign Topic (Assign this keyword to an existing topic): Invokes the Open File dialog which lets you select a topic to which the selected keyword will be assigned. Using the "Show Files" view, you should also be able to selected multiple topics which will allow you to assign the selected keyword to multiple topics at once.
  • Rename Keyword (Rename index keyword): Let's you globally rename the keyword for all instances of the keyword throughout the project.
  • Remove Keyword (Remove index keyword): Let's you globally remove the keyword for all instances of the keyword throughout the project.

Topic right-click menu:

Menu items (tooltips in parenthesis):

  • Add Top-Level Keyword (Add a new top-level index keyword): Adds a new top-level keyword. Until it is added to a topic, this keyword exists only in a temp file in the project's Output/user/Temporary folder.
  • Unassign Topic (Remove all "[keyword]" keywords from topic): Deletes all instances of the selected keyword from the selected topic.
  • Open: Opens the selected topic in the XML Editor.
  • Open with: Gives options to open the selected topic in the Internal Text Editor, Internal Browser or another application .

Index Link ("See" or "See Also") right-click menu:

Menu items (tooltips in parenthesis):

  • Add Top-Level Keyword (Add a new top-level index keyword): Adds a new top-level keyword. Until it is added to a topic, this keyword exists only in a temp file in the project's Output/user/Temporary folder.
  • Edit Link (Edit the index link): Opens the appropriate Index Link set and invokes the Properties dialog for the selected index link.
  • Remove Link (Remove the index link): Invokes a prompt indicating which Index Link set file contains the index link and asks for confirmation that the link should be deleted.

 

Drag and Drop from the Index "Explorer"

Should be able to select any Index Keyword or sub-keyword from the lower portion of the Index Window and drag it into a topic in the XML Editor.

Test Cases

  • Try to drag and drop a keyword and multi-level keywords into various elements (Snippets, Drop-downs, Expanding text, etc)
  • Try to drag the keyword to any pane other than the XML Editor (Content Explorer, Project Organizer, Frame Contents Pane, Footnotes Pane, etc.). You should not be able to do this.

 

Keyboard Navigation

Upper portion of the Index Window:

  • <F2> will allow you to rename a selected keyword or sub-keyword
  • <Delete> will delete only the keyword in the selected row if the row is selected by clicking on the area to the left of each keyword, otherwise <Delete> will do nothing
  • <Page Up> will navigate through the list of keywords one "page" at a time up (requires enough keywords to necessitate a vertical scroll)
  • <Page Down> will navigate through the list of keywords one "page" at a time down (requires enough keywords to necessitate a vertical scroll)
  • <Home> and <End> do nothing
  • <Up Arrow> and <Down Arrow> navigate up and down through the keyword list
  • <Left Arrow> and <Right Arrow> do nothing unless you are editing a keyword in which case they are used to assist in editing the keyword text
  • <Enter> and <Tab> move the selection to the next keyword in the list
  • <Insert> does nothing
  • <Backspace> does nothing

 

Lower portion of the Index Window:

  • <F2> will allow you to rename a selected keyword or sub-keyword
  • <Delete> will delete the keyword or keyword link or unassign the topic depending on which type of item is selected
  • <Page Up> will navigate through the list of keywords one "page" at a time up
  • <Page Down> will navigate through the list of keywords one "page" at a time down
  • <Home> will move the selection to the very first node
  • <End> will move the selection to the very last node
  • <Enter> does nothing unless a topic file is selected in which case <Enter> will open that topic
  • <Up Arrow> and <Down Arrow> navigate up and down through the keyword list
  • <Left Arrow> collapses an expanded node
  • <Right Arrow> expands a collapsed node
  • <Insert> does nothing
  • <Backspace> moves the selection to the parent node
