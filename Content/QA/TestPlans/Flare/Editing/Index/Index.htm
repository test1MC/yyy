Index/Index Proxies/Klinks

Bug Base Product Area:

  • Index/Index Proxies/Klinks

 

Description:

An index consists of a multilevel list of keywords and keyword phrases. An Index is built from the keyword entries that you associate with the Content in your project. In Web outputs, the keywords are collected and displayed in the Index pane. For print outputs, keywords will appear alphabetically with page number references wherever the Index Proxy is placed in a topic. A K-Link is a navigation link that lets users open topics that are related to the current topic based on the index entry terms that they share.

This test plan also covers Index Link Sets and the Auto-Index phrase Editor. Index Links are links in a generated index entry that reference another entry. There are two kinds of index links—"See" and "See Also." The Auto-Index Phrase Editor lets you automatically add keywords in your project to a generated index, rather than inserting all of the index markers manually.
