Keyword Link - Output

Support in Web Outputs:

Support in Web Help Outputs:

Default view of clicked Keyword Link Controls in WebHelp and HTML Help output:

HTML 5 / Web Help 2 Output

In WebHelp and HTML Help outputs, if the list of linked topics is very long, a scroll bar should appear.

(The easiest way to replicate this is to decrease the size of the Help system.)

HTML 5 does not have scroll bars in the Keyword Link List

 

Test cases:

  • Verify Keyword Links work for all web output types as well as topic previews (no java script errors).
    • Test with "Condense JavaScript Files" disabled in the Performance tab of the target
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the target
    • Test with "Use lowercase filenames" enabled (where a Keyword Link links to an affected topic)
    • Test with "Replace reserved characters with underscores in filenames" enabled (where a Keyword Link links to an affected topic)
    • Test with "Use custom file extensions for topics" enabled
    • Test creating Keyword Links to various directories: Topic in same directory, topics in child folder, topics in parent folder, topics several folders up an down the tree, etc.
  • Verify that the entire <MadCap:keywordLink> tag and all of its properties are removed when conditionalized out.
  • Verify that applied styles are in outputs. Test MadCap:keywordLink.
  • In Web and HTML Help outputs, verify the X in the linked topics list closes the popup.
  • Clicking on the should bring up a list of topics in alphabetical order by their Topic Title. In a generated output, every topic has a Topic Title whether or not one is set in the Flare project.
    • If a Topic Title is set in the Topic Properties dialog for a topic, that will be the topic's title in the output and the Keyword Link should use that title.
    • If there is no Topic Title set for a topic, the topic's first heading automatically becomes the Topic Title in the output. The list should still appear in alphabetical order by the output Topic Title.
    • If there is no Topic Title or Topic Heading in a topic, the topics File Name is used as the output Topic Title. The list should still appear in alphabetical order by the output Topic Title.

Support in DITA Output:

  • Keyword Link Controls are not included in DITA output. Verify that a compiler error is given indicating each Keyword Link Control will not be present in the output for every instance of a Keyword Link Control in the project.

Compiler errors:

  • Attempting to include a Keyword Link Control in any print or DITA output:
    Keyword Link found in file: "Content/Topic.htm". It will be removed from the output.
