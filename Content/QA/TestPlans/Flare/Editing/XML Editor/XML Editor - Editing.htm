Right-click menu items (Tooltips in parentheses):

  • Cut (Cut): Cuts the selected content to the clipboard.
  • Copy (Copy): Copies the selected content to the clipboard.
  • Copy Xml as Text: 
  • Paste (Paste): Pastes the content from the clipboard into the selected folder.
  • Hyperlink… (Insert a hyperlink): Opens the Insert Hyperlink dialog which lets you create a hyperlink out of the selected text.
  • Quick Link: Invokes the Quick Link submenu (covered in the Hyperlinks test plan).
  • Cross-Reference… (Insert a cross-reference): Opens the Insert Cross-Reference dialog which lets you create a Cross-Reference at the current insertion point.
  • Quick Cross-Reference…: Invokes the Quick Cross-Reference submenu (covered in the Cross-References test plan).
  • Create Auto Suggestion: (Requires a text selection) Produces three different behaviors depending on the existence of an Auto Suggestion List file in the project:
    • If no *.fltbx file exists: Prompts user that no Auto Suggestion List file exists and asks user to create one. Clicking "Yes" invokes the Add File dialog with Auto Suggestion List File preselected as the File Type.
    • If one *.fltbx file exists: Displays a confirmation dialog indicating that the Auto Suggestion was written to the project's *.fltbx file.
    • If multiple *.fltbx files exits: Invokes the Create an Auto Suggestion dialog which lets you select the Auto Suggestion List file to which you want to write the new Auto Suggestion. And then displays a confirmation dialog.
  • Conditions... (Select Conditional Text Tags): Invokes the Condition Tags dialog which lets you apply a condition to the tag around the current insertion point.
  • Span Class: (Requires a text selection) Invokes a sub-menu listing all classes of the span style. Selecting a class from the sub-menu applies a span tag with the selected class to the text selection.
  • Paragraph… (Edit Paragraph Properties): Invokes the Paragraph Properties dialog.
  • Font… (Font Properties): Invokes the Font Properties dialog.
  • Topic Properties…: Opens the Topic Properties dialog for the current topic.

Drag & Drop Text

Select some text and test dragging it to a new location.

  • Select a complete paragraph and drag it to a new location. Verify no malformed XML.
  • Select some text within a paragraph and drag that to a new location. Verify no malformed XML.
  • Select an inline tag (span, madcap:variable, b, etc) and drag it to a new location. Verify no malformed XML.
  • Test dropping dragged content into a variety of locations: Paragraphs, tables, expanding text, drop-downs, etc.

NOTE: If you make a text selection that includes any inline self-closing element (image, keyword, bookmark, etc) and then click on that inline element to initiate the drag, only the inline element will be moved. This has been reported many times as a bug.

Paste from Other Sources (Browser, Word, Frame, Notepad):

 

Pasting content from sources outside of Flare simply paste into the Flare file, based on the default behavior specified (e.g., as paragraphs, as a list). However, a new paste icon displays after the pasted content, and if you click the icon you can choose a different method for bringing the content into Flare.

Following are the different options you can select from the paste icon.

  • Paste Paragraphs: Select this option if you want to paste the text as regular paragraphs. The pasted text will take on the style of the location where its being pasted. New paragraphs will become default paragraphs.
  • Paste Paragraph Block: Select this option if you want to paste the text as paragraph blocks, such as <div>, <blockquote>, or <fieldset>. For more information about these types of styles, see Creating Tag Groups.
  • Paste Inline Text: Select this option if you want to paste the text as inline text, so that it takes on the style of the location where it is being pasted. If you have copied multiple lines of text from the source document and use this option, the text will be pasted in one line, not multiple lines.
  • Paste Table: Select this option if you want to paste the text into a table.
  • Paste List: Select this option if you want to paste the text as a list.
  • Paste HTML: Select this option if you want to paste text from an HTML source, such as a website.
  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

 

UI Controls

  • Text to paste: Displays the text on the clipboard
  • Options
    • Paste as paragraphs: Select this option if you want to paste the text as regular paragraphs.
      • Style: Select any of the paragraph <p> styles in your style sheet.
    • Paste as paragraph block: Select this option if you want to paste the text as paragraph blocks, such as <div>, <blockquote>, or <fieldset>.
      • Style: Select any of the paragraph block styles in your style sheet.
    • Paste as inline text: Select this option if you want to paste the text as inline text, so that it takes on the style of the location where it is being pasted.
    • Paste as a table: Select this option if you want to paste the text into a table.
      • Style: Select any table style that you might have added to your project so that the new table takes on the look from that style sheet.
      • Delimiter: Select the method by which the source text is separated
    • Paste as a list: Select this option if you want to paste the text as a list.
      • Type: Select the kind of list that you want to use (e.g., decimal, bullet square, bullet disc, lowercase Roman).
    • Paste HTML: Select this option if you want to paste text from an HTML source, such as a website.
  • Preview: Displays a preview of how the pasted text will appear in the XML Editor with the selected option.

 

Split-View

Insure that all normal editing works when the split-view is active.

  • Make sure that the separator bar can be docked, moved to different positions, and placed horizontally or vertically.
  • Test that changes made in the XML Editor show up in the Text Editor, and vice-versa.
  • Make sure that changes in one editor are reflected live in the other, and that highlighting one element on one editor simultaneously shows the highlight in the other
