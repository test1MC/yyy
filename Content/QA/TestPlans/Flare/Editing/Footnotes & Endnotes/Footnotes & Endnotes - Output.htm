Support in Web Help Outputs:

Footnotes are converted to popups in Web outputs. Click on the footnote number next to the text on the webpage and the footnote will display in a popup.

 

  • Verify Footnotes & Endnotes work for all web output types as well as topic previews (no java script errors).
    • Test with "Condense JavaScript Files" disabled in the Performance tab of the target
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the target
  • Verify that the entire <MadCap:footnote> tag and all of its properties are removed when conditionalized out.
  • Verify that applied styles are in outputs

Test Cases

  • Create footnotes that are long enough to wrap to the next line and verify they appear correctly.
  • Test with all configurations of Advanced target options (lowercase, use underscores, etc)

Support in Web Help Mobile Outputs:

  • Popups are not supported by Mobile devices so footnotes must be converted to topics for Mobile output. Verify that footnotes output to Mobile create new topics in the "Content\MCGeneratedPopups" folder in the output.

Endnotes

Setup
  1. The position of the Footnote must be set to Endnotes Proxy. The best way to do this is to set the position in the Master Stylesheet.

    2.  

     

  2. You can also set the position locally on every footnote in your project.

    3.  

     

  3. Add the Endnotes Proxy to the TOC in the TOC Editor and set the TOC in the Target Editor.

    4.  

            

     

The order in which the endnotes appear in the proxy should be the order in which they are placed in the TOC and topics. So your 4th footnotes should be the 4th endnote displayed in the Endnotes Proxy.
