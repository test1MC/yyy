Hyperlinks & Popups - Output

Support in Web Outputs:

Support in Web Help Output:

  • Verify that the entire <a> tag and all of its properties are removed when conditionalized out.
  • Verify that the entire <a> tag and all of its properties are unbound when conditionalized out with the "Unbind" behavior.
  • Verify that applied styles are in outputs
    • Verify that styles cascade correctly.
  • Verify that all hyperlinks work.
  • Verify all target frame settings for hyperlinks work for all web output types as well as topic previews.

    • Test the "Popup" option with "Condense JavaScript Files" disabled in the Performance tab of the target

    • Test the "Popup" option with "Condense JavaScript Files" enabled in the Performance tab of the target
  • Refer to the Hyperlinks Test Project and make sure it outputs correctly and all links in it work

Support in Web Help Mobile Output:

  • Although most target frame settings are not supported in Mobile output, verify that regardless of target frame set for a hyperlink that all hyperlinks work in Mobile output.

Support in DITA Output:

  • For DITA output, the <a> tag should simply continue to be an <a> tag. Verify that hyperlinks are unchanged in generated DITA content.
  • Verify that the entire <a> tag and all of its properties are removed when conditionalized out.
  • Verify that the entire <a> tag and all of its properties are unbound when conditionalized out with the "Unbind" behavior.

Compiler errors:

  • If a topic popup links to itself you get this message: Topic contains a topic popup to itself. Topic popup will be converted to a hyperlink.
  • If the topic being linked to is not in the TOC you should get a compiler error: Unresolved link(s) to file: file:///C:Users/.... File may not be included in the output.
