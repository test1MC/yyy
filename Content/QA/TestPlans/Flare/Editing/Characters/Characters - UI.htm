Characters - UI

Opening the UI

To invoke the Character fly-out menu, place the insertion point in the XML Editor or Frame Contents pane, then:

  • XML Editor: Click the triangle portion of the Character button in the XML Editor's local toolbar
  • Frame Contents pane: Click the triangle portion of the Character button in the Frame Contents pane's local toolbar
  • Toolstrip interface: !Insert | Character | (observe fly-out)
  • Ribbon interface: Click the lower (label and triangle) potion of !Insert | Character

To invoke the Character dialog, place the insertion point in the XML Editor or Frame Contents pane, then:

  • Keyboard: Shift+F11
  • XML Editor: Click the icon potion of the Character button in the XML Editor's local toolbar
  • Frame Contents pane: Click the icon portion of the Character button in the Frame Contents pane's local toolbar
  • Toolstrip interface: !Insert | Character | More characters…
  • Ribbon interface: Click the upper (icon) potion of !Insert | Character

Character Menu - UI Test Cases

  • Quick Character
    • Insert Quick Character [the character and associated font] (Insert the 'Quick Character' [the character name and associated font]): Inserts the quick character into the topic at the location of the cursor.
    • Non-breaking Space (Insert a non-breaking space character): Inserts a non-breaking space into the topic at the location of the cursor.
    • Non-breaking Hyphen (Insert a non-breaking hyphen character): Inserts a non-breaking hyphen into the topic at the location of the cursor.
  • Favorites (Insert [the character name and associated font]): This section displays up to 16 special characters that the user has defined as a "Favorite" using the Character dialog. Hovering over a Favorite character should display a tooltip indicating the character name and the associated font. By default, Flare provides the 16 characters pictured on the left as the out-of-the-box Favorite characters.
  • Recently Used (Insert [the character name and associated font]): This section displays up to 16 special characters that the user has recently inserted using the Character dialog. Characters inserted from the Favorites group should not appear as Recently Used (unless that were added to the Favorites from the Recently Used). Hovering over a Recently Used character should display a tooltip indicating the character name and the associated font.
  • More Characters (Insert character): Opens the Character dialog which lets you select a special character to insert at the location of the cursor.

Character dialog - UI Test Cases

UI Controls

  • "Character" table: Displays all unicode characters that exist for the current font.

    • Font: If desired, select a font that you would like to use when inserting the character. If you insert a character with a font association, that character will be inserted wrapped in a span set to use that font. By default, Font is set to "(normal text)" which means the character will be inserted without any specific font association and will therefore use the font set for the element in which it is inserted.

      NOTE: "(normal text)" will display characters in the Character table using the font associated with the current insertion point. That is, if the insertion point is inside a paragraph using the Comic Sans font, and you then insert a character, "(normal text)" will display characters in the Character table in Comic Sans.

    • (Copy to clipboard): Copies the selected character to the clipboard (the associated font is not copied).
    • (Make favorite): Adds the selected character to the "Favorites" list (this button will actually work for any character selected in the Character table or the "Recently used" list, or if the insertion point is in the Quick Character field).
      • If "(normal text)" is selected, verify that only the character is added to the "Favorites" list
      • If any other specific font is selected, verify that the character is added to the "Favorites" list with its associated font
      • If the "Favorites" list is full (16 characters are allowed), Flare will prompt that you must first remove some.
    • (Make quick character): Makes the selected character the Quick Character (this button will actually work for any character selected in the Character table, the "Favorites" list or the "Recently used" list).
      • If "(normal text)" is selected, verify that only the character is used for the Quick Character
      • If any other specific font is selected, verify that the character and its associated font are used for the Quick Character
    • Search: Lets user search for a specific character by name or hexadecimal value. When empty, this field should display Enter name or character code.
      • Press Enter or the button to refresh the search results in the Character table for the latest query.
      • Clearing the search field should automatically refresh the Character table to display all characters.
      • A search with no results should display "No results found" in the character table
  • Quick Character: Displays the current Quick Character.
    • If the Quick Character has no associated font, the character displays using the font of the current element.
    • If the Quick Character has a font association, the character displays with that font.
    • Verify you can make the Quick Character a "favorite" using
  • "Favorites" table: Displays up to 16 characters designated as "favorites" by the user.
    • If the favorite has no associated font, the character displays using the font of the current element.
    • If the favorite has a font association, the character displays with that font.
    • (Remove favorite): Removes the selected character from the "Favorites" list.
    • Verify you can make a character from the "Favorites" list the Quick Character using
  • "Recently used" table: Displays the 16 most recently inserted character.
    • If the recent character has no associated font, the character displays using the font of the current element.
    • If the recent character has a font association, the character displays with that font.
    • Characters inserted from the "Favorites" list should not be added to the "Recently used" list
    • (Remove recent): Removes the selected character from the "Recently used" list.
    • Verify you can make a character from the "Recently used" list a "favorite" using
    • Verify you can make a character from the "Recently used" list the Quick Character using
  • [character name]: Whenever a character is selected in the Character table, "Favorites" list, or "Recently used" list, the name of that character will display at the bottom of the Character dialog. If the selected character has an ALT key code, that will display in front of the name (example: Alt+0225: Latin Small Letter A With Acute for á). Otherwise, the hexadecimal value for the character will display in front of the name (example: 0103: Latin Small Letter A With Breve for ă).
  • Close after inserting a character: By default, the Character dialog will remain open after a character is inserted allowing the user to insert multiple characters if they desire. Enable this option it have the Character dialog close when a character is inserted.
  • Insert: Inserts the currently selected character from the Character table, "Favorites" list, or "Recently used" list. May or may not close the dialog depending on the state of the previous option.
  • Close: Closes the dialog and does not insert any additional characters.

Code

"Favorite" and "Recently Used" characters are saved to a file called CharacterEditorValues.xml which is saved in the Windows user folder (../Users/[username]/AppData/Roaming/MadCap Software/CharacterEditorValues.xml).

Test Case

  • Verify that you can restore the default set of "Favorite" characters and remove all "Recently Used" characters by deleting the CharacterEditorValues.xml file

Because this file is located in the root "MadCap Software" folder, any recently used or favorite characters set in one product should be used by all products (currently Flare, Analyzer and Contributor use the Character dialog).

Test Case

  • Verify that all products share "Favorite" and "Recently Used" characters
