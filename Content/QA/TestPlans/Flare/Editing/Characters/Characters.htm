Characters

Bug Base Product Area:

  • General Editing

 

Description:

Flare allows user to specify characters. Additionally, Flare has the quick character feature: If user uses the character frequently, it allows user to associate the character with the Quick Character button. When clicking on the Quick Character button, the specific character or symbol will be inserted.

Basic Test Plan:

 

 

Dialogue Box

 

• Verify that you can right-click to add a pinned character when appropriate

• Verify that you cannot add any pinned characters after you reach the max number of pinned characters

• Verify you can delete a pinned character and then add a new one

• Verify you can add a pinned character from Recently used or the main symbol area

• Make sure that a clean install of flare (or after removal of the relevant registry key) the default favorite symbols are set to their defaults

• Left-click on the Insert Character icon should open the dialogue

• Verify that searching works, type “euro” for any three fonts and make sure the euro displays

• Verify that the name of the symbol (i.e., the “euro”) appears the in character editor when it is selected

• Verify that you can resize the dialogue, and that changes in the draw are appropriate

• Mouse-over for the icon in the XML editor toolbar should display “Insert Character”

• Verify left-click on font menu should open font menu, verify the scroll bar works, then left-click on a font should load the symbols for that font

• Verify that enter will add a symbol from the recently used, favorite, or main symbol area

• Verify that double-click will add a symbol from the recently used, favorite, or main symbol area

• Verify that the font symbols load within one second of selection, all actions should be responsive

• Verify that the search is responsive, check single letters (‘t’), check whole words (‘euro’), make sure there are no crashes or hangs if there are odd search terms

• Verify that you can insert all characters on the recently used area without losing focus, be sure to test recently used characters that comes from the end and towards the beginning of the symbol display

• Check that functionality matches when loading from !Insert ribbon or local XML Editor toolbar

• Verify that symbols can be added and removed from recently used and favorites

• Verify that copy to clipboard icon is working

• Verify and that enter, Insert button, and double-click all add the symbols directly when highlighting the symbols from the Favorites and recently used area

• Verify that the icons aren’t placed in bad positions when the window is resized

Insertion

• Verify that enter or double-click to add the selected character, be sure to test both methods for the favorites, recently used, and main symbol display area

• Verify you can only insert one space, and that attempting to insert two spaces will not cause a crash

• Inserting quick characters should insert and must match the favorites in the character editor

• Verify that the default favorites ( found at the character button on the !Insert ribbon) exist after a clean install

• Verify that you can insert at least 10 different characters from each font (recommended to an automation strategy, scripting, macros, etc.)

• Verify that symbols are inserted into the WYS via a span tag, symbols that share a font should share a span tag

• Verify that inserting a symbol from the pinned or favorites area does not change the order of this while you are in the dialogue

• Verify that the fly-out can be clicked, and that the indicated symbol is inserted

• Verify that the favorites and recently used characters insert correctly from both the tool strip/ribbon menu and the local XML toolbar fly-out

• Check that all fonts actually display in their font – compare to MS Word or Flare 8. For example, insure that the courier font is actually displaying in the courier characters in the WYS

 

Navigation

 

• Verify you can bring up fonts by typing in the font list

• Verify that you can navigate to a font by typing in letters and hitting enter or clicking the result

• Verify that you can navigate to a font by typing in letters and then scrolling (either by key or with the mouse) in the new sub-list a and then hitting enter or left-clicking the result

• Verify you can navigate left/right/up/down in the main symbol selection area (it should only take one key to navigate of course)

Known Issues

• On some configurations, a “cannot load font” error message appears for certain fonts (for example, Vivaldi). The issues could be corrupted fonts.

• Recently used symbols only display the corresponding Arial symbol. Many symbols will not appear as they should in that area of the dialogue (for example, Wingdings). So, setting a Wingding as your quick character or favorite will never work as the user would like it to (only the corresponding Arial character will be remembered).
