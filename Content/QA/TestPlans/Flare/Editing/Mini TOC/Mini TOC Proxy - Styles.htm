Mini-TOC Proxy - Styles

The styles detailed below all contribute to the look of the generated Mini-TOC in the output (in place of the Mini-TOC Proxy). Of the styles outlined here, only MadCap|miniTocProxy will allow you to create unique classes. All other styles are generated styles used by all Mini-TOC Proxies regardless of class.

When generating print outputs, the generated mini TOC relies completely on the actual heading levels used in your content. For print outputs, "mc-toc-depth" establishes the greatest heading level that can appear in the generated mini TOC. So, if I place a mini-TOC proxy in a topic that begins with an h2 and I set the TOC Depth to 4, the generated mini TOC will display any <h3> and <h4> elements up to the next <h2>. It will not display <h5>, <h6>, etc.

When generating web outputs, the generated mini TOC relies completely on the tree structure of you TOC file (in the TOC Editor). For web outputs, the mini TOC will only appear in topics that are associated with books. So, in the case of web output types, "mc-toc-depth" establishes the actual number of levels that can display in the generated mini TOC. For example, lets say my TOC file is set up such that I have a chain of 10 books, one inside the another. I add a mini-TOC Proxy with TOC Depth set to 2 to every topic in the chain. In the output, every one of the generated mini TOCs will contain two levels (except for the last two which don't have two levels within them to display).
