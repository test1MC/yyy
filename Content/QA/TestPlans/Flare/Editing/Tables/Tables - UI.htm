UI

Insert Table Dialog General Tab

UI Controls:

  • Table size
    • Number of columns: Sets the number of columns in the table
    • Number of rows: Sets the number of standard rows in the table
    • Number of header rows: Sets the number of header rows in the table
    • Number of footer rows: Sets the number of footer rows in the table
  • Table Caption
    • Text: Enter a caption for the table
    • Side: Select if the caption should display above or below the table. Select "(default)" to use the CSS setting.
    • Repeat: For print output, if a table crosses a page break, select the "Repeat" behavior of the caption. Select "(default)" to the use to CSS setting.
    • Continuation: When the caption repeats, you can add continuation content for each repeated instance of the caption. For example, you might enter " (Continued)" here. Select "(default)" to the use to CSS setting.
  • Summary field: Allow you to add a summary to your table. The summary is not displayed in the XML Editor or the output, but is used by screen readers for accessibility.
  • AutoFit Behavior: Select an autofit option
    • AutoFit to contents: Table sizes based on the content in the cells
    • AutoFit to window: Choose what % of the window the table will take
    • Fixed column width: Assign a fixed size of a table
  • Align: Set the alignment of the table on the page
  • Table Style: Allows you to add a table style template, a print style, or a CSS class to your table. This is covered in greater detail in the table style test plan.

    • Table Style: Lets you select from any Table Style contained in the project. Verify that Table Styles contained in different content locations appear in this drop-down.

      NOTE: Selecting "(default)" here implies that you want to use the Table Style that is referenced for this table class from the project stylesheet. The project stylesheet contains a property called mc-table-style, however this feature has never worked. See bug 40802.

Insert Table Dialog Border Tab

UI Controls:

  • Outer Borders: Set the length, color, and type of each border individually (Left, Right, Top, and Bottom) or all borders at the same time using the button.
  • Border Radius: Set how much rounding to be applied to a table border corner (Left, Right, Top, and Bottom) or all border corners at the same time using the button. (**See overflow**)
  • Cell Border Collapse: Select whether you want to collapse the cell borders in the table. If you collapse the cell borders, the row and cell borders of a table are joined in a single border. If you do not collapse the cell borders, the row and cell borders of a table are detached.
  • Cell Border Spacing: Use this area to increase or decrease the amount of spacing for a cell border.
  • Advanced:
    • Hide bottom ruling when table crosses a page break:
    • True: Will hide the bottom border when the table continues on another page
    • False: Default setting
    • Overflow: Determines what happens if the content overflows the table.
    • Visible: The overflow is not clipped. It renders outside the table. This is the default state.
    • Hidden: The overflow is clipped and the rest of the content will be invisible. (**This must be set if using border-radius**)
    • Scroll: The overflow is clipped, but a scroll-bar is added to see the rest of the content.
    • Auto: If overflow is clipped, a scroll-bar shoud be added to see the rest of the content.
    • Inherit: The value of the overflow property is inherited from the parent element.

Simply select one or more cells in a table, select !Home| Paragraph | button on the lower right hand side in the Ribbon view or Format>Cell(s) in the Tool Strip, and click the Background tab. See Editing Table Cell Properties.

 

 

Convert Tables to Text and Text to Tables

You can now convert tables to text and text to tables.

Tables to Text

If you have a table containing text, you can remove the table but keep the text. When doing this, you can choose how the converted text should be represented: paragraphs, separated with commas, separated with text.

Tool strip mode !Table|Convert to text (Drop-down Menu)

 

Ribbon mode !Table|Convert to Text (Ribbon Button)

Text to Tables

If you have a series of paragraphs or some text separated by commas or other strings, you can quickly place all of it into a table. Table merging

If you have two tables in a content file (e.g., topic, snippet) that are positioned one after the other, you can merge those tables so that they become one. You can do this by using the paste icon that may appear to the right of the tables.

 

How to merge tables:

1) Open the content file (e.g., topic, snippet).

2)Perform an action to display the paste icon. Usually this means removing content between two tables so that they become adjacent, or copying and pasting one table before or after another.

Test: select the <p> and delete it using the keyboard as well as by right clicking the structure bar and removing it that way. (currently in V7 using CTRL+X, or dragging the <p> to a new location will not bring of the merge icon.)

 

3) Click the paste icon to the right of the tables.

4) Select the option that best suits your needs, depending on the type of action that you performed.

  • Merge Table Up: Select this option to merge the tables, with the table on top having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Merge Table Down: Select this option to merge the tables, with the table on the bottom having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Don't Merge Tables: Select this option to allow the tables to appear one after the other, without being merged into one table.
  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

 

Paste Options

Paste Icon

A new paste icon appears when you perform certain actions with tables. This icon allows you to specify how you would like to merge tables or paste content. You can also set the default behavior for similar actions in the future.

 

Convert Table to Text

If you use the new "convert table to text" feature, a paste icon displays at the bottom. Following are the different options you can select from the paste icon. See Converting Tables to Text.

  • Convert to Paragraphs: Select this option to convert the content in each table cell into separate paragraphs.
  • Separate with Commas: Select this option to convert the content in each table cell into text separated by commas.
  • Separate with Text: Select this option to convert the content in each table cell into text separated by a text string that you specify (e.g., a semicolon).

  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

Cut Columns

If you cut entire columns in a table, a paste icon displays after the table. Following are the different options you can select from the paste icon. See Cutting Table Columns and Rows.

  • Cut Columns: Select this option to entirely cut the selected columns, removing both the columns and content from the table.
  • Cut Cell Contents: Select this option to cut only the content from the selected columns, leaving the empty columns in the table.

  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

Cut Rows

If you cut entire rows in a table, a paste icon displays after the table. Following are the different options you can select from the paste icon. See Cutting Table Columns and Rows.

  • Cut Rows: Select this option to entirely cut the selected rows, removing both the rows and content from the table.
  • Cut Cell Contents: Select this option to cut only the content from the selected rows, leaving the empty rows in the table.

  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

 

Delete Content or Space Between Tables

If you remove content or empty space between two tables so that the tables are positioned immediately after one another, a paste icon displays to the right. Following are the different options you can select from the paste icon.

  • Merge Table Up: Select this option to merge the tables, with the table on top having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Merge Table Down: Select this option to merge the tables, with the table on the bottom having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Don't Merge Tables: Select this option to allow the tables to appear one after the other, without being merged into one table.
  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

Note: You cannot merge tables if the column count is different in them.

Paste a Table Above or Below Another

If you paste a table immediately above or below another table (or between two tables), a paste icon displays to the right. Following are the different options you can select from the paste icon.

  • Merge Table Up: Select this option to merge the tables, with the table on top having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Merge Table Down: Select this option to merge the tables, with the table on the bottom having precedence. Therefore, the merged table will use the table style sheet from that table.
  • Don't Merge Tables: Select this option to allow the tables to appear one after the other, without being merged into one table.
  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.

Paste Cells Within a Table

If you copy cells from a table, then place your cursor within a table cell and paste the contents, a paste icon displays to the right. Following are the different options you can select from the paste icon. See Pasting Cells Within a table.

  • Paste Cells: Select this option to paste the copied cells, replacing the cells nearest to where you have placed your cursor.
  • Paste Table: Select this option to paste the copied cells as a separate table. It will be placed within the cell where you have placed your cursor.
  • Paste (Above): Select this option to paste the copied cells above the cell where you have placed your cursor.
  • Paste (Below): Select this option to paste the copied cells below the cell where you have placed your cursor.
  • Paste (Left): Select this option to paste the copied cells to the left of the cell where you have placed your cursor.
  • Paste (Right): Select this option to paste the copied cells to the right of the cell where you have placed your cursor.
  • Set Default: Select this option to open the Default Behavior dialog. Then choose the type of behavior you want to use automatically when performing this action in the future, and click OK.
