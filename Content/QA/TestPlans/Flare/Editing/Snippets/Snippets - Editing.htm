Snippets - Editing

Opening a Snippet File

Any Snippet file:

 

A snippet contained in a Topic, Page Layout file, Master Page file or another Snippet file:

Open a file which contains the snippet in the XML Editor (or open the Frame Contents pane if the snippet is in a Page Layout frame):

  • XML Editor: 
    • Right-click the MadCap:snippetText span bar in the XML Editor and select Open Link
    • Right-click the MadCap:snippetBlock block bar in the XML Editor and select Open Link
  • Frame Contents pane: 
    • Right-click the MadCap:snippetText span bar in the Frame Contents pane and select Open Link
    • Right-click the MadCap:snippetBlock block bar in the Frame Contents pane and select Open Link

Snippets - Editing

Snippet files open in the XML Editor. All testing related to the XML Editor should be covered in the XML Editor test plans.

 

Test Cases

  • Recursive snippets (a snippet that links to itself). Confirm that inserting a snippet link to itself displays and error message: “You selected the snippet file you’re currently editing.  Recursive snippet links are not permitted.”
  • Test moving and renaming snippet files in the Content Explorer and confirm that you are prompted to update links in any files that link to the Snippet (test with Topics, Master Pages, Page Layouts and other Snippets).
  • Test moving a Topic, Master Page, Page Layout or other Snippet files that link to a snippet and confirm that you are prompted to update links.
