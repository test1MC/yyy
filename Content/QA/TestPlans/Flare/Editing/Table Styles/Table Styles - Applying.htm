Table Styles - Applying

Table Styles - Applying to a specific table

In order to apply a Table Style to a specific table, you will first need to add some Table Styles to your project. Completing this test plan will require that you have at least three unique Table Styles in your project.

Table Styles - Applying with Insert Table dialog

When inserting a table with the Insert Table dialog (as opposed to using the grid fly-out from the Ribbon), you may choose to apply a table style right then and there. Test that invoking the dialog by each method will let you apply a Table Style.

  • Tool Strip interface: !Table | Insert | Table…
  • Ribbon interface: !Insert | Table (button)
  • Ribbon interface: !Table | Insert Table (button)

 

For each insert method listed above:

  1. Open the Insert Table dialog
  2. In the General tab, find the Table Style section
  3. Select a table style from the drop-down
  4. Click OK and confirm that the Table Style has been applied to the table

Table Styles - Applying and Resetting with Table Properties dialog

For any existing table, you can apply a Table Style using the Table Properties dialog. Test that invoking the dialog by each method will let you apply or change the table style associated with a table.

  • Right-click inside an existing table and select Table Properties…
  • Right-click the table block bar and select Table Properties…
  • Right-click the table span bar (appears above col bars) and select Table Properties…
  • Tool Strip interface: Select or place the insertion inside a table and select !Table | Table Properties…
  • Ribbon interface: Select or place the insertion inside a table and select !Table | Table Properties…

 

For each insert method listed above:

  1. Open the Table Properties dialog
  2. In the General tab, find the Table Style section
  3. Select a Table Style from the drop-down or change the table style in the drop-down
  4. Click OK and confirm that the Table Style has been applied to the table
  5. Open the Table Properties dialog again
  6. In the drop-down, select a different table style
  7. Click OK and confirm that the new table style has been applied to the table
  8. Open the Table Properties dialog again
  9. In the drop-down, change the table-style to "(default)" to reset the table style
  10. Click OK and confirm that the Table Style has been removed from the table

Table Styles - Applying and Resetting with table context menu

For any existing table, you can apply a Table Style using the table's own context menu. Test that invoking the context menu by each method will let you apply or change the table style associated with a table.

  • Right-click inside an existing table and select Table Style | [any project Table Style]
  • Right-click the table block bar and select Table Style | [any project Table Style]
  • Right-click the table span bar (appears above col bars) and select Table Style | [any project Table Style]

 

For each method listed above:

  1. Apply a table style to a clean table using the context menu (Table Style | [any project Table Style])
  2. Verify the table style is applied correctly
  3. Change the table style to another table style using the context menu (Table Style | [any project Table Style])
  4. Verify the table style is applied correctly
  5. Reset the table style using the context menu (Table Style | Reset Table Style)

Table Styles - Applying and Resetting with Table menu (Tool Strip interface)

For any existing table, you can apply a Table Style using the Table menu in the Tool Strip interface. Either select the table, make a selection inside the table or place the insertion point inside the table to activate the menu items.

  • Set the Table Style: !Table | Table Style | [any project Table Style]
  • Change the Table Style: !Table | Table Style | [any other project Table Style]
  • Reset the Table Style: !Table | Table Style | Reset Table Style

Table Styles - Applying and Resetting with Table ribbon

For any existing table, you can apply a Table Style using the Table ribbon in the Ribbon interface. Either select the table, make a selection inside the table or place the insertion point inside the table to activate the ribbon items.

  • Set the Table Style: !Table | Apply Table Style | [any project Table Style]
  • Change the Table Style: !Table | Apply Table Style | [any other project Table Style]
  • Reset the Table Style: !Table | Reset Table Style

Table Styles - Applying a "Print" table style

In Flare V1 through Flare V6, users could not create table style mediums. So, if a user wanted to style a table differently for print output, they would have to use a second table style file and assign two table styles to a table. Although we now encourage users to use Table Style mediums (from V7 forward), we must continue to verify that the old functionality of Print Table Style works.

 

Print Table Styles can only be applied in the Insert Table or Table Properties dialogs. To apply a Print Table Style:

 

TBW

NOTE: Print Table Styles will NOT render in the XML Editor, even when it is set to Print Layout Mode or the print medium. Print Table Styles can only be viewed in "Print" outputs (PDF, XPS, XHTML Book, Word, FrameMaker, EPUB or a "Web" target that uses the print medium.

Table Styles - Applying to multiple tables

To apply a table style to multiple tables (or all tables) throughout a Flare project, use the Apply Style... button in the local toolbar of the Table Style Editor. This will invoke the Apply Table Style dialog which allows you to select which topics or files you want to apply that table style to.

Apply Table Style dialog

UI Controls

  • Select Files: Select the files to apply the table style to
    • Click expand/collapse nodes will expand or collapse the folders.
    • Click inside box to select or deselect topic to apply style to.
    • Selecting a folder should select all folders and files under it.
  • Overwrite existing table styles: Select this option to overwrite the existing table styles in the files
    • Remove Print Table Styles: If you previously created print table styles and want to remove them from tables.
  • Remove Local Formatting: Remove all of the inline table formatting from tables in those files. This option should remove the "style" attribute from the table and descendant tags. (See bug number 78152. Currently does not work for <thead>, <tbody>, <tfoot>).

Test cases

  • Up and down arrow keys can navigate folder structure.
  • Enter, left and right arrows will expand and collapse folders.
