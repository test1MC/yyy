Drop-Down Text - Output

Support in Web Outputs:

Support in Web Help Outputs:

  • Verify Drop-Down Text works for all web output types as well as topic previews (no java script errors).
    • Test with "Condense JavaScript Files" disabled in the Performance tab of the target
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the target
  • Verify that the entire <MadCap:dropDown> tag and all of its properties are removed when conditionalized out.
  • Verify that if the <MadCap:dropDownHotspot> tag is conditioned out, the entire dropDownHead becomes the hotspot.
  • Verify that applied styles are in outputs. Test MadCap:dropDown, MadCap:dropDownHead, MadCap:dropDownBody and MadCap:dropDownHotspot.

Support in Print Outputs:

Support in PDF Outputs:

  • In all print outputs, Drop-Down Text should simply be converted to regular text. Verify that all content in the Drop-Down (including the hotspot, head and body) appears in all print outputs.
  • Verify that applied styles are in outputs. Although drop-downs are converted to regular text in print outputs, you should still be able to use MadCap:dropDown, MadCap:dropDownHead, MadCap:dropDownBody and MadCap:dropDownHotspot styles to styles the textual content in print outputs.

Support in DITA Output:

DITA does offer an equivalent element to Drop-Down Text. When generating DITA output, confirm that we simply unbind the content of the Drop-Down Text tags and preserve all content.
