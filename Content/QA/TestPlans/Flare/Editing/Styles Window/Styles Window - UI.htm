Style Window - UI

Opening the Style Window

  • Keyboard: Press F12
  • Tool Strip interface: !View | Style Window
  • Ribbon interface: !Home | Style Window

Style Window - UI Test Cases

Default view docked in right pane:

UI Controls

Current Style and Local Formatting

  • (Current Style field): Displays the style of the current element.
  • Edit Style: Opens the Stylesheet Editor so that user can modify the property values for the selected style class.
    • If a Master Stylesheet is set for the Primary Target, that stylesheet opens.
    • If no Master Stylesheet is set for the Primary Target, but a Master Stylesheet is set for the project, the project Master Stylesheet opens.
    • If no Master Stylesheet is used by the Primary Target or the project, the stylesheet associated with the current topic opens.
    • If no Master Stylesheet is used by the Primary Target or the project, and the current topic is not associated with any stylesheet, you should be prompted that "The topic is not associated with a stylesheet. Do you want to select one?"
      • Click "Yes" to invoke the Stylesheet Links dialog which lets you link the topic to a stylesheet.
      • Click "No" to simply close the dialog.
  • Create Style…: Opens the Create Style dialog so that you can create a new style class. The new style class will be based upon the same settings as the current style class, although you can adjust these settings as needed.
    • If a Master Stylesheet is set for the Primary Target, the class is written to that stylesheet.
    • If no Master Stylesheet is set for the Primary Target, but a Master Stylesheet is set for the project, the class is written to the project's Master Stylesheet.
    • If no Master Stylesheet is used by the Primary Target or the project, the class is written to the stylesheet associated with the current topic.
    • If no Master Stylesheet is used by the Primary Target or the project, and the current topic is not associated with any stylesheet, you should be prompted that "The topic is not associated with a stylesheet. Do you want to select one?"
      • Click "Yes" to invoke the Stylesheet Links dialog which lets you link the topic to a stylesheet.
      • Click "No" to simply close the dialog.
  • (Local formatting field): Displays any local formatting applied to the current element.

Available Styles: Select an "available style" to apply it to the selected element.

  • (reset class): Resets the element to the basic style (drops the class).
  • (generic style classes): Generic styles should appear at the top of the Available Styles list. Generic Styles can be identified as they display only a class name without any style (for example ".bold" as opposed to "p.bold")
  • (other style classes): Other styles which are available for the selected element are listed below.

 

Test cases

  • If Available Styles list is long, vertical scroll bar appears and is functional.
  • Available Styles should only display style classes from the associated stylesheets and any stylesheets linked to associated stylesheets.
    • If a Master Stylesheet is set for the Primary Target, the Available Styles listed should come from the target's Master Stylesheet as well as any stylesheets to which that stylesheet links.
    • If no Master Stylesheet is set for the Primary Target, but a Master Stylesheet is set for the project, the Available Styles listed should come from the project's Master Stylesheet as well as any stylesheets to which that stylesheet links.
    • If no Master Stylesheet is used by the Primary Target or the project, the Available Styles listed should come from the all stylesheets associated with the current topic as well as any stylesheets to which that stylesheet links.
    • If no Master Stylesheet is used by the Primary Target or the project, and the current topic is not associated with any stylesheet, only parent styles should display in the Available Styles (no style classes).

Stylesheet Links dialog

Stylesheet Links dialog - Opening the dialog

This dialog is only accessible in a Flare project that does not make use of a Master Stylesheet at the Project or Target level. To open the dialog:

  • Attempt to edit or create a style from a topic that does not contain a stylesheet link. When prompted to select one, click "Yes"
  • Tool Strip interface: Open any topic and then !Tools | Stylesheet Links…
  • Ribbon interface: Open any topic and then !Home | Stylesheet Links

 

Attempting to open this dialog in a project that makes use of a Master Stylesheet will prompt: "Local topic stylesheets are disabled since a master style sheet is being used at the project and/or target level"

  • Verify this prompt displays when using a Master Stylesheet at the project level
  • Verify this prompt displays when using a Master Stylesheet at the target level (no project master stylesheet and the primary target uses a master stylesheet)

Stylesheet Links dialog - UI Test Cases

UI Controls

  • Current Links: All columns should be sortable and not resizable.

    Columns

    • Current Links: Displays all stylesheets to which the topic currently links.
      • Verify that associated Table Styles do not appear.
    • Path: Displays the absolute path of the linked stylesheet.
  • [<<]: Associates the selected stylesheets on the right with the topic.

  • [>>]: Removes the selected stylesheets on the left from being associated with the topic.

  • Stylesheets: All columns should be sortable and not resizable.

    Columns

    • Stylesheets: Displays all stylesheets in the project so that you can associate them with the topic
      • Verify that associated Table Styles do not appear.
    • Path: Displays the absolute path of each stylesheet.
  • Open File: Opens the selected stylesheet in the "Current Links" side in the Stylesheet Editor.
  • Add…: Invokes the Add New Stylesheet dialog which lets you add a stylesheet to the project. (See bug 77974; this should invoke the Add File dialog)
  • OK: Closes the dialog and applies the stylesheet link changes to the topic.
  • Cancel: Closes the dialog and does not apply any changes to the topic.
