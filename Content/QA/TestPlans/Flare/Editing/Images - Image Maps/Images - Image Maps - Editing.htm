Image Maps - Editing

Opening the dialog

The Image Map Editor can only be opened from the <img> context menu. You will need to first open a Flare topic which contains an image.

  • Right-click an image and select Image Map... from the context menu
  • Right-click an img block or span bar and select Image Map... from the context menu

Image Map Editor - UI Test Cases

Local Toolbar

Toolbar items (Tooltips in Parentheses):

  • (Fade Out Image): Turn on to fade out the background image in the Image Map Editor. Off by default.
  • (Show shapes): Turn on/off to display/hide objects added to the image map.
  • (Fill shapes): Turn on/off to fill in objects added to the image map.
  • (Run properties dialog for this item): Invokes the Area Properties dialog for the selected object.
  • (Delete point): Deletes the selected point from a polygon object.
  • (Select Mode): The default mode; allows you to select an object in the image map.
  • (New Polygon Mode): Allows you to draw a polygon object on the image map.
  • (New Rectangle Mode): Allows you to draw a rectangle object on the image map.
  • (New Circle Mode): Allows you to draw a circle object on the image map.
  • (Float): Floats the selected objects up one level.
  • (FloatToTop): Floats the selected objects to the top level.
  • (Sink): Sinks the selected objects down one level.
  • (SinkToBottom): Sinks the selected objects to the bottom level.
  • (Set conditional text tags): Allows for conditions to be set on an image map.

Link Properties

  • Link: Displays the hyperlink set in the Area Properties dialog for the object.

    • Note: "Href" requires entering "http://" at the beginning if linking to an external website. Typing or double clicking this field will open the Area Properties dialog.

  • Screen Tip: Displays the Screen Tip set in the Area Properties dialog for the object. Typing or double clicking this field will open the Area Properties dialog.
  • Tab Index: Displays the Tab Index set in the Area Properties dialog for the object.Typing or double clicking this field will open the Area Properties dialog.
  • OK: Applies changes and closes the dialog
  • Cancel: Does not apply changes and closes the dialog.

Test Cases

  • Clicking on an object selects it
  • CTRL+Click allows selecting/deselecting multiple objects
  • CTRL+A selects all objects (if 12417 ever gets fixed; see "Select All" above)
  • Objects can be moved by clicking on them and dragging
  • Objects can be resized by clicking on any of the object's points and dragging
  • Arrow keys move the selected objects in the appropriate direction on the image map
  • Shift+Arrow keys changes the size of Rectangle and Circle objects.
  • Verify that all operations work with one or multiple objects selected.

Area Properties dialog

To open the dialog:

  • Select an object and press F4
  • Select an object and click in the Image Map Editor's Standard Toolbar
  • Right-click an object and select Properties from the menu
  • Select an object and then !Edit | Properties
  • Double-click an object

Image Map Indicator

If an image has an image map applied to it, the Image Map Indicator should appear in the upper-left corner of the image when viewed in the XML Editor.

Image Context Menu test cases

Relevant Menu Items:

  • Copy: Copies the image to the clipboard. (Map reference is copied however it will only work if pasted in the same topic. See bug 34488)
  • Delete: Removes the image from the topic (Image map code from deleted image is not deleted; see bug #7720)
  • Open Link: Open s the image file in a new tab in the XML editor.
  • Open Linked File With: Opens the image file in another application. Any changes made to an image with an image map should not affect the image map.
  • Rest Size: Resets the image to its original size. Currently, the image map does not resize when an image is resized. See bug 7713.
  • Edit Image: Invokes the Insert Picture dialog which allows you to replace the current image file. Doing so should not remove the existing image map. Rather, the existing image map should be placed on the new image.
  • Edit With MadCap Capture: Opens the image file for editing in MadCap Capture. Any changes made to an image with an image map in Capture should not affect the image map.

  • Hyperlink Picture: Launches the Insert Hyperlink dialog which sets the image to be a hyperlink. However, if an image map is applied to an image, you should get the following message:

  • Image Map… : Invokes the Image Map Editor
  • Remove Image Map… : Removes the image map (only active if an image map is applied). The image map icon should disappear from the image in the XML Editor and the map markup should be removed between the head and body tags.
  • Conditions: Applies a condition tag to the image inside of the topic (Displayed in the upper right hand corner of the image).
  • Name: Allows for a named to be assigned to the img element
  • Paragraph: Invokes the Paragraph properties dialog window.
  • Style Class: Invokes the Style Class drop down menu

Code

Map code appears at top of topic between the head and the body tags:

<map id="map1">
     <area shape="rectangle" coords="64,128,140,191" />
</map>

 

Map reference appears in the <img> tag:

<img src="../Resources/Images/Image.png" usemap="#map1">
