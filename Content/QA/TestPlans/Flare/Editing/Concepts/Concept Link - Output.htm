Concept Link - Output

Support in Web Outputs:

Support in Web Help Outputs:

Default view of clicked Concept Link Controls in WebHelp and HTML Help output:

  • Clicking on the should bring up a list of topics in alphabetical order by their Topic Title. In a generated output, every topic has a Topic Title whether or not one is set in the Flare project.

    • If a Topic Title is set in the Topic Properties dialog for a topic, that will be the topic's title in the output and the Concept Link should use that title.
    • If there is no Topic Title set for a topic, the topic's first heading automatically becomes the Topic Title in the output. The list should still appear in alphabetical order by the output Topic Title.
    • If there is no Topic Title or Topic Heading in a topic, the topics File Name is used as the output Topic Title. The list should still appear in alphabetical order by the output Topic Title.

Test Cases

  • Verify Concept Links work for all web output types as well as topic previews (no java script errors).
    • Test with "Condense JavaScript Files" disabled in the Performance tab of the target
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the target
    • Test with "Use lowercase filenames" enabled (where a Concept Link links to an affected topic)
    • Test with "Replace reserved characters with underscores in filenames" enabled (where a Concept Link links to an affected topic)
    • Test with "Use custom file extensions for topics" enabled
    • Test creating Concept Links to various directories: Topic in same directory, topics in child folder, topics in parent folder, topics several folders up an down the tree, etc.
  • Verify that the entire <MadCap:conceptLink> tag and all of its properties are removed when conditionalized out.
  • Verify that applied styles are in outputs. Test MadCap:conceptLink.
  • Verify the X in the linked topics list closes the popup.

Support in XHTML Book Outputs:

  • Concept Link Controls are not used in any print outputs. Verify that a compiler error is given indicating the Concept Link Control will not be present in the output for every instance of a Concept Link Control for each print output.

Support in DITA Output:

  • Concepts and Concept Link Controls are not included in DITA output. Verify that a compiler error is given indicating each Concept and Concept Link Control will not be present in the output for every instance of a Concept or Concept Link Control in the project.

Compiler errors:

  • Attempting to include a Concept Link Control in any print or DITA output:
  • Concept Link control found in file: "Content/Topic.htm". It will be removed from the output.
