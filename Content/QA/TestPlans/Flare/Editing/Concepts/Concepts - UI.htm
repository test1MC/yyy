Concepts - UI

Opening the Concepts pane

!Tools | Concepts | Concepts Window

!View | Concepts Window

Press <Shift>-F9 on the keyboard

In Concept Entry Mode (!Tools | Concepts | Concept Entry Mode), place the insertion point in the XML Editor and begin typing. The Concepts Window will open when you start typing, and whatever you type will be in the Concepts Window.

Concept Entry - UI Test Cases

Default view docked in right pane:

Concept "Entry" (top half of Concepts window)

The top portion of the Concepts Window is used for displaying, entering and editing a specific instance of a concept object. A specific instance of a concept object may include multiple concepts (one per line).

Test Cases

  • Add Some concepts via the concepts pane and some via the XML editor and verify that they are alphabetized in the list correctly

 

UI Controls

Terms: Lists and lets you edit, add or remove concepts.

Concept "Explorer" (bottom half of Concepts window)

The bottom portion of the Concepts Window is used to view all Concepts that exist throughout the project. Additionally, the "explorer" view will also display any conditional tags applied to a concept or its parent elements (p, h1, html, etc).

Local Toolbar

Local Toolbar Buttons (Tooltips in parentheses):

  • (Expand all): Expands all of the nodes so that you can see all of the files under each one.
  • (Collapse all): Collapses all of the nodes so that you can no longer see all of the files under each one.
  • (Open linked file): Opens the selected topic file in the XML Editor. Only active when a topic is selected.

 

Test cases

Apply condition tags to each of the following and confirm the condition tag is reflected in the Concepts Explorer:

  • The concept itself
  • The paragraph or any other parent element which contains the concept
  • The topic containing the concept
  • The folder containing the topic which contains the concept (Known issue: 30533)

 

Right-click menus

Root node right-click menu: 

Menu items (tooltips in parenthesis):

  • Add Concept (Add a new concept): Adds a new top-level concept. Until it is added to a topic, this concept exists only in a temp file in the project's Output/user/Temporary folder.

Concept right-click menu:

Menu items (tooltips in parenthesis):

  • Rename Concept (Rename Concept): Let's you globally rename the concept for all instances of the concept throughout the project.
  • Remove Concept (Remove Concept): Let's you globally remove the concept for all instances of the concept throughout the project.
  • Assign Topic (Assign this concept to an existing topic): Invokes the Open File dialog which lets you select a topic to which the selected concept will be assigned. Using the "Show Files" view, you should also be able to selected multiple topics which will allow you to assign the selected concept to multiple topics at once.

Topic right-click menu:

Menu items (tooltips in parenthesis):

  • Unassign Topic (Remove all "[concept]" concepts from topic): Deletes all instances of the selected concept from the selected topic.
  • Open: Opens the selected topic in the XML Editor.
  • Open with: Gives options to open the selected topic in the Internal Text Editor, Internal Browser or another application .

 

Drag and Drop from the Concept "Explorer"

Should be able to select any Concept from the lower portion of the Concepts Window and drag it into a topic in the XML Editor.

Test Cases

  • Try to drag and drop a concept into various elements (Snippets, Drop-downs, Expanding text, etc)
  • Try to drag the concept to any pane other than the XML Editor (Content Explorer, Project Organizer, Frame Contents Pane, Footnotes Pane, etc.). You should not be able to do this.

 

Keyboard Navigation

Upper portion of the Concepts Window:

  • <F2> will allow you to rename a selected concept
  • <Delete> will delete only the concept in the selected row if the row is selected by clicking on the area to the left of each concept, otherwise <Delete> will do nothing
  • <Page Up> will navigate through the list of concepts one "page" at a time up (requires enough concepts to necessitate a vertical scroll)
  • <Page Down> will navigate through the list of concepts one "page" at a time down (requires enough concepts to necessitate a vertical scroll)
  • <Home> and <End> do nothing
  • <Up Arrow> and <Down Arrow> navigate up and down through the concept list
  • <Left Arrow> and <Right Arrow> do nothing unless you are editing a concept in which case they are used to assist in editing the concept text
  • <Enter> and <Tab> move the selection to the next concept in the list
  • <Insert> does nothing
  • <Backspace> does nothing

 

Lower portion of the Concepts Window:

  • <F2> will allow you to rename a selected concept
  • <Delete> will delete the concept or unassign the topic depending on which type of item is selected
  • <Page Up> will navigate through the list of concepts one "page" at a time up
  • <Page Down> will navigate through the list of concepts one "page" at a time down
  • <Home> will move the selection to the very first node
  • <End> will move the selection to the very last node
  • <Enter> does nothing unless a topic file is selected in which case <Enter> will open that topic
  • <Up Arrow> and <Down Arrow> navigate up and down through the concept list
  • <Left Arrow> collapses an expanded node
  • <Right Arrow> expands a collapsed node
  • <Insert> does nothing
  • <Backspace> moves the selection to the parent node
