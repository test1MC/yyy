X-Refs - UI

Inserting a Cross-Reference without the dialog

Drag and drop method:

  • Content Explorer: Drag and drop a topic from the Content Explorer into another topic that is open in the XML Editor in Print Layout Mode. Note, if a file is dragged onto a text selection, a hyperlink is inserted instead.
    • Drop topic at a an insertion point. The parent madcap|xref style will be used.
    • Test using topics of varying depth within the Content folder and verify paths are written correctly.
  • File List: Drag and drop a topic from the File List into another topic that is open in the XML Editor in Print Layout Mode.
    • Drop topic at a an insertion point. The parent madcap|xref style will be used.
    • Test using topics of varying depth within the Content folder and verify paths are written correctly

Quick Cross-Reference method:

  • XML Editor: Right-click in the XML Editor at an insertion point or with a text selection and select Quick Cross-Reference | (test all sub-menu options):

    • Bookmarks: Invokes a sub-menu that displays all bookmarks in the current topic. Select a bookmark to insert a cross-reference to that bookmark.
    • Same Folder: Invokes a sub-menu that displays all topic files that exist in the same folder as the current topic. Select a topic to insert a cross-reference to that topic.
    • Open Documents: Invokes a sub-menu that displays all topic files that are currently open. Select a topic to insert a cross-reference to that topic.

Inserting a Cross-Reference with the dialog

Insert a cross-reference with the Insert Cross-Reference dialog:

  • Keyboard: Press CTRL+Shift+R
  • XML Editor: Click the Insert Hyperlink button in the XML Editor's local toolbar
  • XML Editor: Right-click in the XML Editor and select Cross-Reference from the context menu
  • Tool Strip interface: !Insert | Cross-Reference
  • Ribbon interface: !Insert | Cross-Reference

Insert Cross-Reference Test Cases

    • Above the cross-reference, below the cross-reference, on a separate page (print layout), etc

UI Controls

  • XRef Class: Displays the class of the current xref.
    • For the New… dialog, this field will allow you to enter the new class name.
    • For the Edit… dialog, this field will be grayed out, but it will display the class name you are editing. It will appear empty and grayed out if you are editing the root style.
  • Stylesheet to modify: Offers a drop-down of all stylesheets associated with the current document. Select which stylesheet to modify if the document is associated with more than one.
  • Enter format (preview below): Lets you enter the format for your cross-reference and displays a sample preview below.
  • Available commands: List of commands that can be used in the xref format.
    • "Show" drop-down: Lets you filter the list of commands to certain types.

    • Available commands grid:

      Columns

      • Command: Displays each command available. Double-click a command to insert it into the format area above.
      • Description: Provides a description of each command.

Updating a Cross-Reference

Although all Cross-References are updated for the output at compile time, they have to be updated manually if you need them to be updated in the XML Editor. You might need to update a cross-reference if you've cross-referenced to a heading and then changed the textual content of the heading. To do this, open the topic that contains the cross-references and select:

  • Tool Strip interface: !Tools | Update Cross-References…
  • Ribbon interface: !Tools | Update Cross-References

Update Cross-Reference dialog: UI Test Cases

UI Controls

  • Update Cross-References Grid

    Columns

    • Update: Lets you select which cross-references you want to update and which you don't. By default, all cross-references will be checked each time this dialog is opened.
    • Cross-Reference: Displays every cross-reference that exists in the current topic.
    • Link: Displays the href of each cross-reference.
    • Paragraph: Displays the textual content of the entire paragraph that contains the cross-reference.
