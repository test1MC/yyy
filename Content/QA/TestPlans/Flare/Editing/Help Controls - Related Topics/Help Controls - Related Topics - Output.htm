Help Controls: Related Topics - Output

Support in Web Outputs:

Support in Web Help Outputs:

Default view of clicked Related Topics Controls in WebHelp, HTML Help, and HTML5 output:

 

In WebHelp and HTML Help outputs, if the list of Related Topics is very long, a scroll bar should appear. No scroll bars will appear in HTML5 output

(The easiest way to replicate this is to decrease the size of the Help system.)

  • Verify the X in the Related Topics list closes the popup.

  • Verify Related Topic Controls work for all web output types as well as topic previews (no java script errors).

    • Test with "Condense JavaScript Files" disabled in the Performance tab of the target
    • Test with "Condense JavaScript Files" enabled in the Performance tab of the target
    • Test with "Use lowercase filenames" enabled (where a Related Topics Control links to an affected topic)
    • Test with "Replace reserved characters with underscores in filenames" enabled (where a Related Topics Control links to an affected topic)
    • Test with "Use custom file extensions for topics" enabled
    • Test creating related topic links to various directories: Topic in same directory, topics in child folder, topics in parent folder, topics several folders up an down the tree, etc.
  • Verify that the entire <MadCap:relatedTopics> tag and all of its properties are removed when conditionalized out.
  • Verify that applied styles are in outputs. Test MadCap:relatedTopics.

Support in DITA Output:

  • DITA will convert a MadCap:relatedTopics tag to a <related-links> tag after the <body> tag, but before the closing </topic> tag. The DITA output should look something like this:
(the bulk of the topic is up here)
    </body>
    <related-links>
        <linklist>
            <link href="NewFolder/NewTopic.dita#NewTopic" />
            <link href="NewFolder/NewTopic1.dita#NewTopic1" />
            <link href="NewFolder/TopicWithBookmark.dita#TopicWithBookmark/bookmark" />
        </linklist>
    </related-links>
</topic>

Compiler errors:

  • Attempting to include a Related Topics Control in any print output.
    Related Topic control found in file: "Content/Topic.htm". It will be removed from the output.
