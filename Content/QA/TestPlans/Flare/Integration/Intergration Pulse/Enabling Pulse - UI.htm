Enabling Pulse - UI

Pulse is enabled for a Flare output in a Flare Target file. In order to enable Pulse in a Flare target, you'll need a Pulse Server and valid login credentials for that server.

Pulse is supported in HTML5 and WebHelp outputs

Target Editor | Community Tab - Default View

UI Controls

Community Log In

UI Controls

  • Server:  Un-editable URL to your Pulse web server (ex: http://ebrown01/pulse/Feedback/Service.FeedbackExplorer/Service.asmx)
  • User name: Type or use the drop down to select your username
  • Password: Type your password here, it should show as asterisks.
  • OK: Closes the dialog and commits the license key into the Pulse Service License Key field in the Target Editor
  • Cancel: Closes the dialog and does not commit a license key to the Target Editor.

Target Editor | Community Tab - Pulse enabled

UI Controls

  • Current User: Displays the current logged in user
    • Login: Launches the login dialog. Button will be grayed out when a user is already logged in.
    • Logout: Clears the existing credentials
  • Communities: Displays "None" or the current community selected. Before you can save the Flare target, you must associate a Flare community with the Pulse system
    • Manage: Launches the Manage Communities dialog
    • Remap Topics: Launches the Remap Community Topics dialog.  If any changes have been made to the TOC structure or topic paths, you will need to remap.
  • Refresh: updates the Current Pulse Configuration info. You will see basic information about the community, such as the status of the server, and the total number of users.

Manage Communities Dialog

UI Controls

  • Add: Launches the Add Community dialog.
    • Enter Name: Type the name of your community
    • Enter Published URL: Type the URL of your published output.  If you do not include the http:// in the url, you will receive an error "published url must be a valid URL" when you click OK.
  • Edit: Launches the Edit Community dialog.  This has the same options as the Add Community dialog.
    • Enter Name: Type the name of your community
    • Enter Published URL: Type the URL of your published output.  If you do not include the http:// in the url, you will receive an error "published url must be a valid URL" when you click OK.
  • Delete: You will get a Yes/No prompt "Deleting this community can not be undone.  Are you sure you want to do this?"
  • OK: The community currently highlighted will be the selected community for this target.
  • Cancel: No changes will be made.

Test Cases

  • Boundary test all community names (Numbers, Text, Various characters and symbols including international). Test creating communities with various characters and confirm they appear correctly in the Pulse Dashboard (log in to the Pulse dashboard using an account that is a member of the Administrators or Employees group. Then select Systems>Find Systems from the menu. The community you created should appear in the list.)

Remap Community Topics dialog

UI Controls

  • New Topics: Displays the Title and Path of any new topics, moved topics, or renamed topics (new name)

  • Remove Topics: Displays the Title and Path of any deleted topics, moved topics, and renamed topics (former name).

  • Update Topics: Displays the Change, Title and Path of any topics that have had reassigned links or changes in their title, etc. *needs more detailed documentation.

  • Exclude Topics: Displays the Title and Path of any topics that are not affected by the topic remapping.
  • OK: Dialog will close, changes will be saved and you will get a message "Community has been updated."
  • Cancel: Dialog will close, changes will not be saved.

Reassign Community Links dialog

If you rename a topic, it will show up as both New and Removed. If you want to link any data from the previous topic to the new topic, right click that topic in the Remap Community Topics dialog, and click "Reassign Link."  This will open the Reassign Community Links dialog and you can select a link from the drop down.
