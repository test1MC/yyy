Project Analysis pane - UI Test Cases

Status view (default view)

UI Controls

  • (View Drop-down): Use this drop-down to change the view of the Project Analysis pane.
  • Last Scan Date: Displays the date and time of the last complete scan of the project.
  • Status: Displays the status of the Analyzer scan (Examples: "Checking…", "Performing Advanced Scan…", "Up-to-date" etc.)
  • Launch MadCap Analyzer: Launches the MadCap Analyzer application and opens the current project simultaneously in Analyzer.

Broken Links view

Project Analysis displaying all columns in the screenshot:

Local Toolbar:

  • (Hide/Show Info Bar): Displays or hides the Info Bar.
  • (Refresh): Refreshes the list of data that appears in the grid.

UI Controls

  • Project Analysis | Broken Links grid: Displays a list of all "broken" (as defined by Analyzer) links that exist in throughout the project. As of Analyzer V3, a broken link is any link that cannot be resolved within the project. Therefore, for the time being, external links will also display in this list.

    Columns

    • File: Displays the filename of the file containing the broken link
    • Folder: Displays the path of the topic file within the project.
    • Link: Display the link that cannot be resolved within the project.
    • Type: Displays the type of "broken" link. This might display "Missing File" or "Missing Bookmark" etc.
    • Link Text: Displays the link text of the broken link.
    • Link Tag: Displays the tag of the broken link.
    • Extension: If the broken link is a link to a file, this column will display the file extension.

  • Right-Click Menu:

    • Refresh: Refreshes the list of data that appears in the grid.
    • Open With: Gives options to open the file in the Internal Text Editor, Internal Browser or another application.
    • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan).

Database Errors view

Project Analysis pane docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

  • (Hide/Show Info Bar): Displays or hides the Info Bar.
  • (Refresh): Refreshes the list of data that appears in the grid.

UI Controls

  • Project Analysis | Database Errors grid: Displays a list of any files that cannot be analyzed due to some kind of error. A couple of the most common reasons for a database errors are invalid XML or too many characters for a particular field in the database.

    Columns

    • File: Displays the filename of the file.
    • Folder: Displays the path of the topic file within the project.
    • Description: Provides a short description of the cause of the database error.

  • Right-Click Menu:

    • Refresh: Refreshes the list of data that appears in the grid.
    • Open With: Gives options to open the file in the Internal Text Editor, Internal Browser or another application.
    • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan).
