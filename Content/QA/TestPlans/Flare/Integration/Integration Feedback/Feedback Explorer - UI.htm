Topic Comments - UI Controls

Feedback Explorer docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

None

 

UI Controls

  • Feedback Explorer | Topic Comments grid: Displays all comments that exist on the Feedback Server for the topic currently open in the MDI.

    Columns

    • User: Displays the username of the user who added the comment.
    • Subject: Displays the subject of the comment.
    • Comment: Displays the comment. Most likely just the beginning of the comment followed by "...". Hovering over the cell will display the full comment in a tooltip.
    • Date: Shows the date and time at which the comment was added to the topic.
    • Status: Displays the review status of the comment.
    • CommentID: Displays the Feedback Server comment ID number for the comment.
    • ParentID: If the comment is a reply to another comment, displays the Feedback Server comment ID number for the parent comment (the comment being replied to).
    • #: Displays a count for every comment in the list.

  • Right-Click Menu:

    • Copy: Copies the selected element to the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Paste: Pastes the content from the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Properties: Invokes the Properties dialog for the selected element. See (Comment) Properties below.

 

 

Topic Ratings - UI Controls

Feedback Explorer docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

  • (Discard selected topic rating(s)): Permanently deletes the selected topic rating from the Feedback Server.

 

UI Controls

  • Feedback Explorer | Topic Ratings grid: Displays all ratings that exist on the Feedback Server for the topic currently open in the MDI.

    Columns

    • Rating: Displays the numerical value of each rating.
    • Feedback: Displays any author-only feedback provided with the rating, if the user provided any. Most likely just the beginning of the comment followed by "...". Hovering over the cell will display the full comment in a tooltip.
    • Date: Shows the date and time at which the rating was added to the topic.

  • Right-Click Menu:

    • Copy: Copies the selected element to the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Paste: Pastes the content from the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Properties: Invokes the Properties dialog for the selected element. See (Rating) Properties below.

 

 

Context-Sensitive Help Calls - UI Controls

Feedback Explorer docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

  • (Reset CSH call counts): Resets the CSH call counts for the selected terms to zero.

 

UI Controls

  • Feedback Explorer | Context-Sensitive Help Calls grid: Displays all CSH Calls tracked by the Feedback Server for any Help system using the Feedback Service key in the project's Primary Target.

    Columns

    • CSH ID: Displays all CSH IDs called by the end user.
    • Path: Displays the path of the topic file within the project.
    • Call Count: Displays the total number of times the CSH ID to the left was called.

  • Right-Click Menu:

    • Copy: Copies the selected element to the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Paste: Pastes the content from the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Properties: Opens the linked topic.

 

 

Pending Comments - UI Controls

Feedback Explorer docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

  • (Discard selected comment(s)): Deletes the comment from the Feedback Server. Clicking this button should prompt for confirmation.
  • (Set comment status): Clicking here invokes a drop-down which lets the reviewer or administrator change the comment status to Accepted, Pending Review or Hidden.

 

UI Controls

  • Feedback Explorer | Pending Comments grid: Displays all comments in the "Pending Review" status on the Feedback Server for any Help system using the Feedback Service key in the project's Primary Target.

    Columns

    • Topic: Displays the path and filename of the topic file that received the comment.
    • User: Displays the username of the user who added the comment.
    • Subject: Displays the subject of the comment.
    • Comment: Displays the comment. Most likely just the beginning of the comment followed by "...". Hovering over the cell will display the full comment in a tooltip.
    • Date: Shows the date and time at which the comment was added to the topic.
    • CommentID: Displays the Feedback Server comment ID number for the comment.
    • ParentID: If the comment is a reply to another comment, displays the Feedback Server comment ID number for the parent comment (the comment being replied to).

  • Right-Click Menu:

    • Copy: Copies the selected element to the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Paste: Pastes the content from the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Properties: Invokes the Properties dialog for the selected element. See (Comment) Properties below.

 

 

Search Phrases - UI Controls

Feedback Explorer docked in MDI in order to display all columns in the screenshot:

Local Toolbar:

  • (Reset search counts): Resets the search counts for the selected term(s) to zero. Clicking this button should prompt for confirmation.

 

UI Controls

  • Feedback Explorer | Search Phrases grid: Displays all search phrases tracked by the Feedback Server for any Help system using the Feedback Service key in the project's Primary Target.

    Columns

    • Search query: Displays all search queries submitted in all published help systems using the Feedback Service key set in the configuration file.
    • Number of searches: Displays the total number of times the search query to the left was submitted.
    • Number of results: Displays the number of search results the end user received when performing the search.

  • Right-Click Menu:

    • Copy: Copies the selected element to the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Paste: Pastes the content from the clipboard (not really applicable in this context; there is a bug to have this item removed)
    • Properties: Invokes the Properties dialog for the selected element. See (Query) Properties below.

(Comment) Properties Dialog - UI Test Cases

To open the dialog:

  • Double-click an item in the Topic Comments or Pending Comments view
  • Select an item in the Topic Comments or Pending Comments view and press <F4>
  • Right-click an item in the Topic Comments or Pending Comments view and select Properties

UI Test Cases

UI Controls

  • Subject: Displays the subject of the comment.
  • User: Displays the username of the user who added the comment.
  • Date: Shows the date and time at which the comment was added to the topic.
  • Comment: Displays the comment. This field should be scrollable.
  • In reply to: If viewing Properties of a comment that is a reply to another comment, the lower fields will be populated as well. If the comment is not a reply, these lower fields should appear empty.
    • User: Displays the username of the user who added the parent comment.
    • Comment: Displays the parent comment. This field should be scrollable.

UI Test Cases
