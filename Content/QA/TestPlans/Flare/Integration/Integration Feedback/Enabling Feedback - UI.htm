Enabling Feedback - UI

Feedback is enabled for a Flare output in a Flare Target file. In order to enable Feedback in a Flare target, you'll need a Feedback Server and a valid license key for that server.

  • If you are testing for a new release of Feedback Server, you will most likely be testing with an installation of the latest build of Feedback Server Admin, in which case you will need to use the name or IP address of the server that is running that latest build.
  • If you are testing Feedback for a new release of Flare (independent from a release of Feedback), you can also use "madcap" as your server if you want to test against the MadCap Software hosted Feedback Service, since the hosted service is always running on the last shipped version of the server.

Target Editor | Feedback Service Tab

 

UI Controls

Feedback Service End User License Agreement dialog

UI Controls

  • (EULA): Displays the End User License Agreement. Verify this is the current version.
  • Feedback Service License Key: Paste your Feedback Service License Key in this field. Verify that only a license key established to work with the MadCap hosted Feedback Service will work in this field.Standard service keys that would be used with an installation of Feedback Server Admin (not the MadCap hosted service) should not work in this field.
  • Accept: Closes the dialog and commits the license key into the Feedback Service License Key field in the Target Editor
  • Cancel: Closes the dialog and does not commit a license key to the Target Editor.
