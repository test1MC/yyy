Import HTML - Editor

About the HTML Import Editor

The HTML Import Editor can be used to view and edit settings of a set of HTML files that was imported into the project. Or, you can add a new HTML Import file to import HTML files into the existing project. In both cases, the HTML Import Editor also lets you change import settings and re-import content.

Opening the HTML Import Editor

HTML Import Editor - UI Test Cases

HTML Import Editor: Local Toolbar

Controls:

Import/Reimport...: Imports or re-imports the set of *.html, *html and *.xhtml files added to the HTML Import file. (Reimport will be tested at the end of this test plan)

HTML Import Editor: Source Files tab

UI Controls

  • HTML Files grid:

    Columns

    • HTML Files: Displays each *.doc, *.docx or *.rtf file added to the import
    • Status: ???
  • Open Selected: Opens the selected file in the default application. Also opens a selected folder location via Windows Explorer. Button will gray out with a multi-selection.
  • Preview Conversion: Creates a preview of what the conversion will look like. Not available when a mapped folder is selected.
  • Link Generate Files To Source Files: This option creates a connection between the original HTML files and the files that are created as a result of the import.
  • Move Up: This moves the selected file/folder higher in the list (if you have more than one document to import). The file/folder at the top is used for the name of the content folder holding the imported topics in Flare. Also, the order determines how the imported documents are arranged in the Flare TOC that is created as a result.
  • Move Down: This moves the selected file/folder lower in the list (if you have more than one document to import).
  • Remove: This removes the selected files/folders from the list.
  • Add Folder: Produces the Add Folder dialog which lets you browse for folders that contain *.htm, *.html or *.xhtml files to add to the import.
  • Add Files...: Produces the Open File dialog which lets you browse for *.htm, *.html or *.xhtml files to add to the import.

HTML Import Editor: Options tab

UI Controls

  • Import to project content folder: Select this option if you want the html files to import to the root of the Content folder. If mapping to a folder, that will come in as a subfolder of the Content folder.
  • Import to folder: Select this option if you want the html files to import to a custom folder under the Content folder. If mapping to a folder, that will come in as a subfolder to the custom folder (There is a next version bug to map directly to the root of the custom folder).
  • Import linked HTML files: This option will bring in any linked htm files that may or may not be in the same folder as mapped files/files in mapped folders.
  • Import resources: Select this option if you want to import any resource that is linked to from the mapped files or folders
    • Keep existing structure: Select this option if you want to import any resource as they exist in structure in relation to the mapped files or folders
    • To project resources folder: Select this option if you want to import any resource to the appropriate subfolders that we normally create under the Resources folder
    • To folder: Select this option if you want the resource files to import to a custom folder under the Content folder.
  • Auto-reimport before 'Generate Output': Also called "Easy Sync," this option will tell Flare to automatically re-import HTML files when you attempt to build output.

HTML Import Editor: Imported Files tab

UI Controls

  • Imported Files grid:

    Columns

    • Imported File: Displays each project file created from the HTML import.
    • Folder: Displays the location of the imported file within the project.
  • Open: Opens the selected files

Import Progress

During import as well as reimport, before the import complete, Flare will display the Import Progress dialog if files associated with the mapped files fail to import, i.e., missing resources.

 

UI Controls

  • Generated documents grid

    Columns

    • File: Displays each new Flare file that will be created as a result of the import.
    • Type: Displays the file extension of each file.
    • Folder: Displays the location where the file will exist in the Flare project.
  • Save Log:  Option to save the import progress log to the Project Reports folder
  • Close: Closes the dialog, then the Accept Imported Documents appears (There is a Next version bug to have this say "Continue")

Accept Imported Documents

During import as well as reimport, before the import complete, Flare will display the Accept Import Documents dialog providing the user with a preview of the files that will be added to the project with the import.

 

UI Controls

  • Generated documents grid

    Columns

    • File: Displays each new Flare file that will be created as a result of the import.
    • Type: Displays the file extension of each file.
    • Folder: Displays the location where the file will exist in the Flare project.
  • Show Source Code: Displays a source code preview (if available) of the file selected.
  • Accept: Closes the dialog and returns focus to the HTML Import File Editor.
    • The imported files are now present in the existing project.
  • Cancel: Closes the dialog, does not complete the import and returns focus to the HTML Import File Editor
  • Converted (tab): Displays a HTML or Source code preview (if available) of the imported version of the file selected.
  • Original (tab): Displays a HTML or Source code preview (if available) of the source version of the file selected.
