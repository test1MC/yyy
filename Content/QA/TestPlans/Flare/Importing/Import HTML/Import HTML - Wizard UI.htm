Importing HTML Documents into an existing project

To import HTML files into an existing project, launch the Import HTML Files Wizard by:

  • Tool Strip interface: !Project | Import HTML Files...
  • Ribbon interface: !Project | Import | HTML Files

UI Test Cases:

Import HTML Files Wizard - Dialog 1a (Scenario 1): Import Options

UI Controls

  • Project Name: Lets you enter a name for the project that will be created from the imported documents.
  • Project Folder: Lets you set the location of the project
    • Browse: Invokes the Browse For Folder dialog to select the folder where project will be saved
  • Output Type: Lets you select the output type for the primary target for the new project
  • Back: Returns to the previous dialog in the Import HTML Files Wizard. (Should be grayed out at this point)
  • Next: Proceeds to the next dialog in the Import HTML Files Wizard.
  • Finish: Closes the Import HTML Files Wizard, invokes the Import Progress dialog and proceeds to import the HTML Files. (Should be grayed out at this point)
  • Cancel: Closes the Import HTML Files Wizard and does not import anything.

Import Microsoft Word Wizard dialog 1(a): HTML-> XHTML Conversion

UI Controls

  • Show Source Code: If this option is toggled on, the HTML-> XHTML Conversion dialog will display the original file and the (to be) converted file in code view. If this option is toggled off, the topics will display in WYSIWYG view.
  • Cancel: Cancels the conversion operation and closes the dialog.

Import HTML Files Wizard dialog 2: Options

UI Controls

  • Topics: Displays the folder in Content Explorer where the imported files will be stored.
    • Import to project content folder: Select this option if you want to store the files at the root level of the Content Explorer
    • Import to folder: Select this option if you want to store the files in a specific folder (NOT AVAILABLE FOR SCENARIO 1...into a new project)
    • Browse: Opens the Select Folder dialog which lets you browse for (or create) a target folder to which all linked resources for all imported HTML files will be placed upon import. (NOT AVAILABLE FOR SCENARIO 1...into a new project)
    • Import linked HTML files: This option will bring in any linked htm files that may or may not be in the same folder as mapped files/files in mapped folders.
  • Import Resources: Enable this option to automatically import any "resource" files (linked files) along with all imported HTML files.
    • Keep existing structure: Select this option to import resources in such a way as to maintain the existing structure of all links in the imported HTML file(s).
    • To project resources folder: Select this option to import resource files into the Flare or Blaze project's "Resources" folder
    • To folder: Select this option to import resource files into a specific folder in the Flare or Blaze project. (NOT AVAILABLE FOR SCENARIO 1...into a new project)
      • (the specific folder): Displays the folder to which all resources for imported HTML files will be placed upon import. (NOT AVAILABLE FOR SCENARIO 1...into a new project)
      • Browse: Opens the Select Folder dialog which lets you browse for (or create) a target folder to which all linked resources for all imported HTML files will be placed upon import. (NOT AVAILABLE FOR SCENARIO 1...into a new project)

Import HTML Files Wizard dialog 2(a): Select Folder dialog

UI Controls

  • (Content Tree): Displays a tree view of your Content folder. Click on the folder you want to use for imported HTML files or their linked resources.
  • Make New Folder: Creates a new folder inside the currently selected folder.
  • OK: Closes the dialog and sets the selected folder as the Target Folder for imported HTML files or their linked resources.
  • Cancel: Closes the dialog and does not set any Target Folder.

Import Progress

During import as well as reimport, before the import complete, Flare will display the Import Progress dialog if files associated with the mapped files fail to import, i.e., missing resources.

 

UI Controls

  • Generated documents grid

    Columns

    • File: Displays each new Flare file that will be created as a result of the import.
    • Type: Displays the file extension of each file.
    • Folder: Displays the location where the file will exist in the Flare project.
  • Save Log:  Option to save the import progress log to the Project Reports folder
  • Close: Closes the dialog, then the Accept Imported Documents appears (There is a Next version bug to have this say "Continue")

Accept Imported Documents

During import as well as reimport, before the import complete, Flare will display the Accept Import Documents dialog providing the user with a preview of the files that will be added to the project with the import.

 

UI Controls

  • Generated documents grid

    Columns

    • File: Displays each new Flare file that will be created as a result of the import.
    • Type: Displays the file extension of each file.
    • Folder: Displays the location where the file will exist in the Flare project.
  • Show Source Code: Displays a source code preview (if available) of the file selected.
  • Accept: Closes the dialog and returns focus to the HTML Import Files Wizard.
    • The imported files are now present in the existing project.
  • Cancel: Closes the dialog, does not complete the import and returns focus to the HTML Import Files Wizard
  • Converted (tab): Displays a HTML or Source code preview (if available) of the imported version of the file selected.
  • Original (tab): Displays a HTML or Source code preview (if available) of the source version of the file selected.
