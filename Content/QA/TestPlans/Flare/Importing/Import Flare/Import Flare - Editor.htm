Project Import Editor - UI Test Cases Part I

Project Import Editor: Local Toolbar

UI Controls

  • Import/Reimport... : Imports or re-imports the files designated in the Flare Project Import file.

Project Import Editor: Source Project tab

UI Controls

  • Source Project: Lets you set the path of the Flare project from which you want to import files.
  • Auto-reimport before "Generate Output": Also called "Easy Sync," this option will tell Flare to automatically re-import Flare project files associated with this import file when you attempt to build output.
  • Delete Stale Files: This feature will delete stale files when you re-import either manually or automatically when you generate output (if you have selected Auto-reimport before generate output in the Project Import Editor).
  • Browse… : Invokes an Open dialog which lets you browse for a Flare (*.flprj) or Blaze (*.blprj) project file.
  • Open: Opens the Flare or Blaze project set in the Source Project field in a new instance of Flare.
  • Include Files: A semi-colon delimited filter which allows you to limit the imported files to a particular set of files based on filename or extension. Use the asterisk for Wildcards. Example: If you enter "*TestCases*;*.htm;*.html;" in this field, the Flare Project Import file will only import Topic Files that contain "TestCases" in their filename. You can enter the filter rule directly by typing in this field or click Edit… to use the Import File Filter dialog.
  • Auto-include linked files: Select this option to automatically import all files to which the imported files link, even if they are not specifically included in the Include Files field. These might include any linked resource files (images, snippets, etc) or linked topics that may have been filtered out by the Include Files field or by Conditions (see below).
  • Edit… : Invokes the Import File Filter dialog which assists in defining the Include Files filter set..
  • Exclude Files: A semi-colon delimited filter which allows you to exclude certain files from being imported based on filename or extension. Use the asterisk for Wildcards. Example: If you entered "*.htm; *.html;" in the Include Files filter, and then enter "*TestCases*;" in this field, the Flare Project Import file will only import Topic Files but exclude those that contain "TestCases" in their filename. You can enter the filter rule directly by typing in this field or click Edit… to use the Import File Filter dialog..
  • Edit… : Invokes the Import File Filter dialog which assists in defining the Exclude Files filter set...
  • Import Conditions: This filter lets you use the Condition Tags applied to the Flare Project files to determine which files should be imported. The condition logic used here is essentially the same as the logic used when building output.
  • Auto-Exclude Non-Tagged Files: Use this option to exclude from import any files that do not contain any condition tags.
  • Edit… : Invokes the Import Conditions dialog which displays all condition tags in the linked project and lets you select which tags should be included or excluded in the import.
Import File Filter - UI Test Cases

UI Controls

  • Select patterns grid:

    Columns

    • File Type: Displays the file types to be included in the filter.
    • Pattern: Displays the actual string (the "pattern") that is used by the filter to determine if a file should be included in the filter. In most cases, this will be a file extension, however it could also be any string that might be used in a filename.
  • Add: Invokes the Import File Filter Designer dialog which lets you add a pattern to the file filter
  • Remove: Removes the selected pattern from the file filter.
Import File Filter Designer - UI Test Cases

UI Controls

  • Select patterns grid:

    Columns

    • File Type: Displays all Flare project file types that can be included in the filter. Verify all Flare project file types are listed here, particularly any new ones.
    • Pattern: Displays the actual string (the "pattern") that is used by the filter to determine if a file should be included in the filter.
  • Pattern: Displays the pattern that you want to add to the file filter. Selecting any row in the grid will populate this field with the pattern. You may also type directly into this field, particularly if you want to use a text filter for filenames.
  • OK: Closes the dialog, returns you to the Import File Filter dialog and adds the selected file to the filter.
  • Cancel: Closes the dialog, returns you to the Import File Filter dialog and does not add the selected file to the filter.
Import Conditions - UI Test Cases

UI Controls

  • Import Conditions grid:

    Columns

    • Tags: Displays all condition tags that exist in the linked Flare project.
    • Include: Select the checkbox for the condition tags you want to include in the import. By default, all tagged content is included in the import. However, you can also use this Include option to include files that are also tagged with an excluded tag.
    • Exclude: Select the checkbox for the condition tags you want to exclude from the import.
  • Edit Set: Opens the selected Condition Tag Set from the linked project in the MDI of the current project. (There is a bug, 28652, to have this button removed in this context)
  • OK: Closes the dialog, and applies your selected Include/Exclude expressions to the Flare Project Import file.
  • Cancel: Closes the dialog, and does not apply changes made to the Include/Exclude expressions to the Flare Project Import file.

Importing the Files

Once the Project link, file filters and import conditions are established, click Import… in the local toolbar to import the files from the other Flare project. Before files are actually copied to the project, you will be presented a list of all files that will be imported and have an opportunity to deselect them using the Accept Imported Documents dialog.

Accept Imported Documents - UI Test Cases

UI Controls

  • Imported Documents grid:

    Columns

    • Import: A check mark in this column indicates that the file will be imported. Deselect any items that you do not want to import. In the event that a local copy of that file already exists with the same filename, by default this box should not be checked. Select any unchecked items if you want to go ahead and import, overwriting the local copy.
    • File: Displays the file that is to be copied to the project.
    • Folder: Displays the folder in the current project where the imported files will be copied.
    • Status: May display a message indicating why the file might not need to be imported. For example, you might see "File exists" or "Local is newer" in this field. Im most cases, this filed will be empty.
    • Type: Displays the actual string (the "pattern") that is used by the filter to determine if a file should be included in the filter.
    • Source Folder: Displays the folder in the linked project where the imported files originate.
  • Preview: Displays a preview of the selected file (if available). Some file types may not preview. Import all file types and verify that those that should preview do preview correctly.
  • Select All: Selects all files (places a checkmark in the Import box for all files).
  • Clear All: Deselects all files (removes the checkmark for all files).
  • Accept: Proceeds to import the selected files into the existing project.
  • Cancel: Cancels out of the import wizard.

Editing Imported Documents

Once you click Accept, the files are copied in to the Flare project. Files imported into the Flare project through the Flare Project Import file are linked to the original file. This is indicated in the Content Explorer and Project Organizer with a small link icon:

 

In general, when files are imported via a Flare Project Import file, the idea is that the document will only be edited in the original project and changes will be regularly imported. For that reason, if you attempt to edit a linked file (any file that displays "linked" icon), you should see the Linked Document dialog

Linked Document - UI Test Cases

UI Controls

  • Link: Displays the path of the original document in the original Flare project
  • Continue with operation: Proceeds to apply the change locally but does not remove the link to the original document. This option may result in changes being overwritten the next time a Re-Import occurs.
  • Remove the link and continue the operation: Proceeds to apply the change locally and removes the link to the original document. Selecting this option will also add this file to the Removed Links list in the Project Import Editor.
  • Remove the link and cancel the operation: Does not apply any changes but removes the link to the original document. Selecting this option will also add this file to the Removed Links list in the Project Import Editor.

Project Import Editor: Removed Links tab

UI Controls

  • Imported Files grid:

    Columns

    • File: Displays any project file created from the Flare Project Import that have had their link removed.
    • Folder: Displays the location of the file within the project.
  • Restore Link: Restores the link between the file in the project and the Flare Project Import file.
  • Open: Opens the selected files
