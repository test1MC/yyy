Hyphenation pane - UI Test Cases

Default view docked in right pane:

UI Controls

  • Word: Displays the word that is currently selected in the editor.
  • Current Rule: Displays the current hyphenation rule for the currently select word (if one exists). If no rule exists, "(no hyphenation info found)" should be displayed.
  • Rule Source: Displays the name of the Hyphenation Dictionary file that is supplying the current rule. This field will be empty if the source is Flare's factory English Hyphenation Dictionary.
  • Add Custom Rule…: Opens the Add Custom Hyphenation Rule dialog which lets you create a custom rule for the current word. The rule is added to a dictionary file based on the language of the element which contains the selected word.
  • Manage Custom Rules…: Opens the Open Hyphenation Dictionary dialog which lets you open any Hyphenation Dictionary file that you have already created in the project.

Add Custom Hyphenation Rule - UI Test Cases

UI Controls

  • Word: Displays the word for which you are going to create a new custom hyphenation rule.
  • Rule: Lets the user define the hyphenation rule for the word.

UI Controls
