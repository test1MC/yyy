Link Viewer - UI

Opening the Link Viewer

  • Content Explorer: Right-click a file in the Content Explorer and select View Links
  • Project Organizer: Right-click a file in the Project Organizer and select View Links
  • File List: Right-click a file in the File List and select View Links
  • Pending Check-Ins: Right-click a file in the Pending Check-Ins window and select View Links
  • Tool Strip interface: !View | Link Viewer
  • Ribbon interface: !View | Link Viewer

Link Viewer - UI Test Cases

Local Toolbar

  • (Lock the Link Viewer): Locks the current file in the Link Viewer. This allows you to open different files and continue to view links for the original file. (There's an issue where the Link Viewer stays locked when you close and re-open Flare. Bug #31227 has been entered to consider changing this behavior).
  • (Refresh): Refreshes the list of links.
  • (Show links for previous file): Changed the file displayed in the Link Viewer to the previous file that was open in the Link Viewer.
  • (Show links for next file): If you have clicked "Show links for previous file," this option will let you browse forward in the sequence of files you have viewed in the Link Viewer.

UI Controls

File Information

  • (filename): Current file's filename and the appropriate icon should display.
  • File Path: Displays the path of the current file
  • Associated Targets: Displays all targets that the current file is associated with. If there are none, the field should display "(no associated targets)."

Links

  • Linked From: Displays all files which link to the current file. If there are none, the field should display "(no incoming links)." Clicking the column heading should sort files alphabetically and reverse alphabetically.
  • Linked To: Displays all files which are linked to by the current file. If there are none, the field should display "(no outgoing links)." Clicking the column heading should sort files alphabetically and reverse alphabetically.

The "Links" section of the Link Viewer requires an explicit link to display a link. The Associated Target section will allow for implied links when determining if a target should be associated with a file. For example, all Web-based target will always be "associated" for all content files since all content files are included in Web-based output. The Link Viewer does not consider Conditional arguments set in the Target.

Right-Click Menus

Right-clicking any file in the Associated Targets, Linked From or Linked To fields should produce the following menu:

 

 

Right-click menu items:

  • Open: Opens the selected file. In turn, unless the Link Viewer is locked, the Link Viewer should update to display links for the newly opened file.
  • View Links: Changes the Link Viewer view to display links for the selected file. This option will not open the file.

Test Cases

  • Open every type of file that can be opened in Flare and confirm the Link Viewer displays links appropriately.
  • Link to every type of file that can be linked to and confirm the Link Viewer displays links and icons appropriately. This includes but should not be limited to external links of all kinds, broken links, e-mail links, media links, etc.
  • Confirm that the Link Viewer does not display links for any files that can be opened in the MDI for which it would be inappropriate to display links (Help topics, files opened in the Differences Viewer, incoming review or contribution files, etc).
  • Hovering over any file in the Associated Targets, Linked From or Linked To fields should produce a tooltip displaying the file's path within the Flare project (beginning with the Content or Project folder)
  • Double-clicking any file in the Associated Targets, Linked From or Linked To fields should open that file.
