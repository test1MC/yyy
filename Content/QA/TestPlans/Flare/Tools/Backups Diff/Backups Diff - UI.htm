Backups/Diff - UI

Opening the Backups Pane

  • Tool Strip interface: !View | Backups
  • Tool Strip interface: !File | Backups | Backups
  • Ribbon interface: !View | Backups

Sending a file to the Backups folder

The Backups pane can also be opened in the process of sending a copy of a file to the Backups folder. To do this, open any project file in the MDI and give it focus. Then:

  • Tool Strip interface: !File | Backups | Send copy of "[filename]" to the Backups folder
  • Ribbon interface: !File | Send to Backups | Send to Backups

 

This will bring up the following: 

 

  • Clicking "Yes" should copy the file to Backups folder and open the Backups pane.
  • Clicking "No" will still copy the file to the Backups folder however the Backups pane will not open.

Backups Folder location

The Backups Folder is created in the "Project" folder of each individual project. To accommodate for multiple users accessing a single project over a network, unique Backups folders are created per User. From the project file, it can be found here:

\Project\Users\[your username]\Backups

Test Cases

Access a project from \\files\QA or \\dc2 on two unique machines as two completely different users. Have both users send documents to the Backups Folder and confirm that each user has their own folder and that Backup files are created in the user-appropriate folder.

Clearing the Backups Folder

  • Tool Strip interface: !File | Backups | Clear the "Backups Folder"…
  • Ribbon interface: !File | Send to Backups | Clear the "Backups Folder"

 

Selecting this menu option will prompt for confirmation first, and then delete all files in the Backups Folder. If the Backups Folder is empty, you should be prompted as such.

Test Cases

Access a project from \\files\QA or \\dc2 on two unique machines as two completely different users. Have both users send documents to the Backups Folder and confirm that clearing the Backups Folder only clears the folder for the appropriate user.

Backups Pane - UI Test Cases

Default view docked in the left pane:

Local Toolbar

Local Toolbar Buttons (Tooltips in parentheses):

  • Restore... (Restore selected files(s)): Prompts for confirmation first, and then restores the active copy of the file to the backup copy and removes the files from the Backups Folder.
  • Differences (Show differences between backup version and current version): Opens the selected file and its backup copy in the Differences Viewer.
  • (Refresh): Refreshes the list of Backup files.
  • Clear the "Backups Folder"... (Delete all files in the Backups Folder): Prompts for confirmation first, and then deletes all files in the Backups Folder. If the Backups Folder is empty, you should be prompted as such.

Backup Files grid

Columns

  • Name: Shows the name of the file in the Backups folder.
  • Original Path: Shows the location of the original file in the project.
  • Extension: Shows the extension of the file
  • Backup Date: Shows the backup date and time

Local Toolbar

Local Toolbar Buttons (Tooltips in parentheses):

  • (Refresh): Refreshes the view of the file and the backup file.
  • Options: Opens the Files Differences dialog which lets you set color options for how differences are displayed in the Differences Viewer.
  • Show WYSIWYG: Lets you toggle between Text and WYSIWYG views of the file. Should only be active for Topic, Master Page and Snippet files. All other files should only open in Text view.
  • Ignore Case: If enabled, the Difference Viewer will ignore differences in letter casing.
  • Ignore Whitespace: If enabled, the Difference Viewer will ignore differences which only involve whitespace.

 

Test Cases

You should not be able to edit the backup or the current copy of the file in the Differences Viewer. You should be able to select text and copy it, but in no way should you be able to edit the content.

You should not be able to drag and drop anything into the backup or current copy of the file in the Differences Viewer.

When the Differences Viewer is in focus, the option to "Send for Review" should be disabled.
