Options Dialog - UI Test Cases:

General tab

UI Controls

  • Auto-Reload Documents: Select to reload documents automatically. In other words, if you have topics or editor pages open and then close the project, this option ensures that those same documents are reopened the next time you open the project.
  • Auto-Save Documents: Select to periodically auto-save any open files.
    • "#" minutes: You can specify the number of minutes that should elapse between each auto-save.
  • Close others when opening new document: An option which tells Flare to close other documents as more documents are opened based on the behavior set here:
    • No action: The default option. Opening new files will have no affect on files already open. All files will remain open.
    • Close others: This option automatically closes all other open documents. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with no changes: This option automatically closes all other open documents that do not have unsaved changes. If you have files open where changes have been made but not saved, those files will remain open until you save and close them.
    • Close others with same extension: This option automatically closes all other open documents that share the same file extension. For example, let's say you have a topic, a style sheet, and a target open. If you open another topic, the topic file that is currently open will be closed, but the style sheet and target will remain open. You should be prompted to save changes to any unsaved files that will be closed.
    • Close others with same extension and no changes: This option is the same as the previous option, except that files with unsaved changes will not automatically be closed. Those files remain open until you save and close them.
  • Show Select UI Language Dialog on Startup: Select this box to show the Select UI Language dialog the next time you launch Flare.
  • Enable Instant Messaging: TBW
  • Reset "Show Once" Dialogs…: There are some dialogs in Flare that open just once to give you a message or warning when something changes or you switch modes. An example of this is the dialog that opens when you first switch between layout modes in the XML Editor. These dialogs have a check box labeled "Show this dialog." If you deselect that checkbox, you will not see the dialog again. However, you can use this reset feature to display that dialog (and all other "Show Once" dialogs) again the next time.
  • Import/Export Word Files Without MS Office: Select to import and export Microsoft Word documents without having Word installed locally, for example, if you use Microsoft Office 365.
  • Automatically resize low DPI images:  Selected by default, this newer, more precise resizing of low DPI images is used. If you do not select this option, the behavior of Version 9 and earlier is used.

Source Control tab

This tab will only be present if you are working in a project that is bound to Source Control. The functionality identified below will be tested in greater depth in the Source Control test plan.

 

UI Controls

  • Automatically check file out from source control when saving file changes: When you save changes to a file that is not checked out, we will check that file out for you.
    • Lock files when automatically checking out(files cannot ve checked in by other users): When a file is automatically checked out of source control such as by editing and saving a topic, we will automatically lock the file (only available in TFS and SVN).
  • Show prompt to add auto-created files to source control: Select to make sure you are prompted when files are automatically created in Flare and may be added to source control.
  • Enable background status checks: Enable the automatic background checks that Flare performs at the specified interval to determine the status of files bound to a source control application (cannot set this value to 0min and 0secs and disabled by default)
  • Do not prompt to get latest when opening source control bound projects: TBW

XML Editor tab

UI Controls

  • Options
    • Enable Drag and Drop Text: Select to enable the drag-and-drop feature for text. If this feature is enabled, you can highlight and drag text to move it within topics.
    • Show web images: Select to enable showing web images in the XML Editor.
    • Insert Empty Alt Text: Select to automatically add an empty alt tag to images when you insert them into a topic or snippet (unless you specifically add an alt tag to an image).
    • Replace straight quotes with smart quotes: Select to use single and double curly smart quotes (”) instead of straight quotes (") when you type in the XML Editor. This option is disabled by default

        Note: Selecting this option does not automatically replace all of the straight quotes that you already have throughout your project. In order to do that, you should use the Find and Replace in Files window pane to find all of the files in the project containing straight quotes and then enter smart quotes in their place.

  • Text Rendering
    • Enable ClearType Font Smoothing:Select to enable Microsoft's "ClearType" technology to improve the readability of text on LCD screens
    • Enable GDI+ Text Renderer: Toggles between Microsoft Windows Graphics Device Interface (GDI) and GDI+ in terms of how content is rendered on your monitor.

Analyzer tab

Note: In order for these options to be effective, you must set the options before opening your project. If you have your project open and a scan is in process, setting these options will not improve your system performance.

UI Controls

  • Advanced Scan Options
    • Collect Segments: Select if you want Analyzer to collect segments of text while scanning your project. Segment collection allows you to view information about your project such as frequent segments found and index keyword suggestions.
    • Collect Variable prospects: Select if you want Analyzer to look for text that you might want to convert to variables.
    • Collect Snippet prospects: Select if you want Analyzer to look for text that you might want to convert to snippets.
    • Collect Local style prospects: Select if you want Analyzer to look for places where you have applied local formatting and might want to apply styles to those areas instead.

    • Collect Embedded stylesheet prospects: Select if you want Analyzer to look for style information (embedded styles in <style> elements) in your topic files. Disabling this scan limits the results found in the style reports.

      Note: You should select this option if you have imported topics that have embedded styles. Otherwise, it is not usually necessary to select this option.

    • Collect Accessibility prospects: Select if you want Analyzer to look for text where you might want to add Web accessibility features (e.g., captions, alternate text). See About Accessibility.
    • Collect Markup prospects: Select if you want Analyzer to look for a variety of XHTML markup issues in your content (e.g., extra spaces, empty attributes). You can open the affected files and "clean up" the markup in them. You may be able to fix some issues in the XML Editor, but other issues may require you to open the behind-the-scenes code in the Internal Text Editor. See Internal Text Editor.
  • Performance Options
    • Search Limits: Select a value (All | 5,000 | 1,000 | 500 | 100) from this dropdown to help increase the performance for the following Analyzer scan options: Undefined Styles, Index Keyword Suggestions, New Style Suggestions, Replace Local Style Suggestions

Auto Suggestion tab

UI Controls

  • Enable Auto Suggestion: Enables Auto Suggestion functionality when authoring in the XML Editor. By default, all Auto Suggestion List Files and all Variable Sets in the project will produce Auto Suggestions.
  • Enable Snippet Auto Suggestion: All snippets will now be included in the suggestions that display.
  • Max Suggestions: The maximum number of suggestions that will display in the suggestion window.
  • Min Character Length: The minimum number of characters a user must type to bring up suggestions. Default is 3 and can not go lower than 3.

  • Enable/disable files: Allows the user to enable or disable specific files that will generate Auto Suggestions. This grid view should display all Auto Suggestion List Files and all Variable Sets that exist in the project.

    Columns

    • (icon): Displays either a Variable Set icon or an Auto Suggestion List File icon to easily identify the file type
    • Enabled: Check the box to include this file for Auto Suggestions
    • File: Displays the name of the file
    • Path: Displays the path of the file within the project

Spelling tab

UI Controls

Select Global Dictionary Locations

  • Use default location: Saves the dictionaries to the default location which is C:\Users\[username]\AppData\Roaming\MadCap Software\Flare\Spelling Dictionaries
  • Use custom location: Saves the dictionary to the path provided.
    • (Browse): Opens the Select Folder dialog to browse for a dictionary location.

Installed Dictionaries Grid

  • Columns
    • Language: This column lists the names of all languages with dictionaries that are installed on your computer. Dictionaries are stored in your AppData Windows folder.
    • Code: Displays the code for each language.
    • Spell: Green dots indicate which dictionaries support spell checking.
    • Hyphen: Green dots indicate which dictionaries support hyphenators. A hyphenator is used to automatically hyphenate words in an editor based on particular word patterns, as outlined in the dictionary file. (Hyphenators are important only for certain languages.)
    • Description: This column provides a short description of each dictionary
    • Import Dictionaries...: Invokes the Select dictionary files dialog which is a browse dialog that lets you find Hunspell dictionary files (*.oxt or *.zip). You can download more dictionaries from OpenOffice.org.

Index & Concepts tab

UI Controls

  • Auto-complete Index and Concept Entries:As you enter index keywords in the Index window pane, the application recognizes if you have created a similar index keyword. The similar keywords appear in a list just below the entry field. You can quickly select one of the keywords from the list instead of typing the entire word manually.
  • Mode Warnings for Index and Concept Entries:
    • Check mode and display the dialog when first topic opened: Select to avoid seeing the Mode Warning dialog
    • Auto-swith to normal selection mode on launch: Select this option if always want the application to disable the mode automatically when you open the program in the future.
    • Continue in mode: The "Index Entry Mode" or "Concept Entry Mode" will continue to be enabled after this dialog is closed.

Build tab

UI Controls

  •  Options
    • Save log on file: . Select to automatically save the log file after an output has been generated at a global level for all targets in all projects (command line build too).

    • View output after successful build: Select to automatically view the output after a successful build at a global level for all targets in all projects.

    • Build target (if needed) after selecting 'View': This feature lets you avoid seeing the message “There is no generated output for the [name] target./The output for the [name] target is not up-to-date. Do you want to generate it?” for all targets in all projects, the target will automatically build when needed after selecting ‘View.’

  • Warnings Changes here are applied globally for all targets in all projects, unless targets have the options “Use target build warnings instead of global settings” selected. Deselect certain warnings so that they do not display in the Build Progress dialog when generating output. (Preferences saved to ...\AppData\Roaming\MadCap Software\Flare\CompilerSettings\V10)
    • Expand/Collapse warning categories by clicking the + or - sign.
    • Select/Deselect all warnings in a category by toggling the check mark in the root category
    • Search warnings by ID allows numerical searching of warnings
  • Reset Warnings: By default, Accessibility and Web Accessibility categories are not selected.

Review tab

UI Controls

  • User
    • User Name: Field to enter the user's name
    • Initials: Field to enter the user's initials as they want them to appear in annotations and tracked changes
  • Tracked Change Display
    • Display all changes in balloons: All tracked changes will be displayed in balloons.
    • Display only format changes in balloons: Only format changes will be displayed in balloons. The other changes will be displayed inline.
    • Display all changes inline: All tracked changes will be displayed inline and none will be displayed in balloons.
  • User Colors
    • Use a different color for each user: This will allow you to specify specific colors for each different user that makes tracked changes.
    • Use the following color for all users: This option allows you to select one color that will be used for all users who make tracked changes.
