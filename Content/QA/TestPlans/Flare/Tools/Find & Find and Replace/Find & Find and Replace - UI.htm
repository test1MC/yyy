Find & Find and Replace - UI

Opening the Find and Replace widget

The Find and Replace widget is supported in the following editors: Internal Text editor, XML Editor, Condition TagSet Editor, Auto-Index Editor, Auto Suggestion List Editor, FileTagSet Editor, Index Links Editor, SearchFilterSet Editor, Glossary Editor, VariableSet Editor, and Frame Contents Editor

If an attempt is made to launch the Find and Replace widget while focus is not in one of the supported editors, then the Find and Replace in Files pane will open instead.

  • To open the Quick Find Widget - Replace option is collapsed by default in the widget
    • Press CTRL + F on the keyboard
    • Toolstrip interface: !Edit | Find and Replace | Quick Find
    • Ribbon interface: !Home |Quick Find
  • To open the Quick Replace Widget - Replace option is expanded by default in the widget
    • Press CTRL + H on the keyboard
    • Toolstrip interface: !Edit | Find and Replace | Quick Replace
    • Ribbon interface: !Home |Quick Replace

Opening the Find and Replace in Files Pane

  • Press CTRL + SHIFT + F on the keyboard
  • Toolstrip interface: !Edit | Find and Replace | Find and Replace in Files
  • Ribbon interface: !Home | Find and Replace in Files

Load Find Selection

Load Find Selection functionality allows you to select text in the XML Editor and load that selection into either the Find and Replace Widget pane or the Find and Replace in Files pane.

Load Find Selection functionality occurs by default when using the default commands to open the Find and Replace widget and the Find and Replace in Files pane.

  • To test this function in the Find and Replace widget: select some text in any of the supported editors and use any of the commands to launch the widget
  • To test this function in the Find and Replace in Files pane: select some text in any of the supported editors and use any of the commands to launch the pane

Opening the Find Results panes

  • !Edit | Find and Replace | Find Results 1 or Find Results 2
  • !Home | Find and Replace in Files | Find Results 1 or Find Results 2
  • Expand “Result Options” in the Find and Replace in Files pane when performing a find (see below).

Find and Replace in Files Pane- UI Test Cases

Default view is the top right of the editor from what it was opened

UI Controls

Find

  • Find:Type the text that you want to find.
  • Replace with: Type the text that will replace the found text.

  • Find in: Click the down arrow to select the parameters of the search:

    • (current document): Flare looks for the text in file that is currently open
    • (documents in same folder): Flare looks for the text in all files located in the same folder as the current one.
    • (all open documents): Flare looks for the text in all files currently open in the interface.
    • (content folder): Flare looks for the text in all files within the "Content" folder.
    • (pick a folder): Flare looks for the text in all files in the selected folder (and its subfolders).
    • (whole project): Flare looks for the text in every file in the project.
  • File types: Select the kind of files that you want to include in the search (topics, stylesheets, snippets, page layouts, glossaries).

Options

  • Match case: Finds only occurrences of the text that match the case (e.g., uppercase, lowercase) of the text entered in the "Find what" field.
  • Use: An advanced feature for users who are familiar with wildcard and regular expressions in searches. If you select this check box, you can specify whether the search will include wildcards or regular expressions.
  • Find in source code: Includes the source code of the topics or style sheets in the search.
  • Output in Find Results 1: Displays the search results (list of files) in the Find Results 1 window.
  • Output in Find Results 2: Displays the search results (list of files) in the Find Results 2 window. This is useful if you have already performed a search using the Find Results 1 window and want to keep those results open while you do another search.
  • Find All: Initiates the search, opens the Find Results window that is selected and returns all the results in the Find Results window

Find Results 1 and 2 - UI Test Cases

Default view docked in bottom pane:

UI Controls

Local Toolbar Buttons (Tooltips in parentheses):

  • (Open selected): Opens the selected file in the Find Results pane and highlights the found text.
  • (Open previous): Opens the file previous to (above) the selected file in the Find Results pane and highlights the found text.
  • (Open next): Opens the file following (below) the selected file in the Find Results pane and highlights the found text.
  • (Clear table): Clears the Find Results pane.

Find Results grid

Columns

  • File: Shows the name of the file.
  • Context: Shows the context of the found text within the file.
  • Folder: Shows the location of the file within the project.
  • Title: Shows the Topic Title as set in the Topic Properties dialog.
