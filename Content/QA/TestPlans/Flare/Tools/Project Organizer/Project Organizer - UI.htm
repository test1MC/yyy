Project Organizer - UI

Opening the Project Organizer

The Project Organizer should be open and docked along the left side using Flare's default layout. If the Project Organizer is closed or not currently in focus, it can be reopened or given focus by:

  • Keyboard: Press CTRL+[
  • Tool Strip interface: !View | Project Organizer
  • Ribbon interface: !View | Project Organizer

Local Toolbar

Local Toolbar Buttons (Tooltips in parentheses):

  • (Show/Hide Folders): Toggles between showing or hiding the folders in the Project Organizer.
  • (Show/Hide Files): Splits the Project Organizer into two halves. The right half shows additional details for the files. When you click a file on the right side, a red circle shows the location in the Project Organizer. See a movie about this feature.
  • (Expand All): Expands all of the folders so that you can see all of the files under each one.
  • (Collapse All): Collapses all of the folders so that you can no longer see all of the files under each one.
  • (Hide/show Conditional Indicators): Lets you toggle between showing and hiding conditional indicators in the Project Organizer.
  • (Up): Moves up one folder in the hierarchy in the Project Organizer. This button becomes enabled if you use the Split View button and select a file or folder on the right side of the split view.
  • (New Folder): Creates a new folder inside the selected folder. If a file is selected, the folder is created in the same location as the file.

Show Files grid

Columns

  • Name: Shows the name of the file.
  • Size: Shows the size of the file.
  • Date Modified: Shows the date and time the file was last modified.

Right-Click Menus

Right-clicking a file (except an *.mclog file) in the Project Organizer should produce the following menu:

Right-clicking an *.mclog file in the Project Organizer should produce the following menu:

Right-click menu items:

  • Open: Opens the selected Project file in the MDI
  • Open with: Gives options to open the Project file in the Internal Text Editor, Internal Browser or another application
  • Delete: Removes the selected Project file from the project
  • Rename: Allows you to rename the Project file

      Note: Bug 8840:Log file is saved with the time displayed as H.MM so an attempt to rename the log file only highlights the file name up to the period.

Right-clicking the Project folder in the Project Organizer should produce the following menu:

Menu Items (Tooltips in parentheses):

  • Source Control: Invokes the Source Control sub-menu (covered in the Source Control test plan).
  • Add: Invokes the Add sub-menu which presents menu options to add Content and Project files.
  • Clean Project: Sends all files in the project's Output folder to the Recycle Bin.
  • Open Output Folder: Opens the project's Output folder in a Windows browser.
  • Project Properties…: Opens the Project Properties dialog.

Keyboard Navigation

  • Enter Open/Closes the selected item (Folders open/close to display/hide their contents – Files open in the correct editor)
  • Up Arrow and Down Arrow move the selection to the next displayed node
  • Left Arrow and Right Arrow select and open\close folders will go as far as selecting the first non-folder node in a folder
  • Page Up and Page Down will move the Organizer a page at a time up or down
  • Home or End will move the selection to the very first node or the very last displayed node
  • Insert does nothing
  • Delete Only works for files inside of the folders. Shouldn’t do anything for folders.
