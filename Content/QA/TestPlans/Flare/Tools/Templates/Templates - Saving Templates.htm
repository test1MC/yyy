Templates - Saving as a Template

Saving a Flare Project as a Template

!Project | Save Project As Template...

  • Disabled when there is no project open.
  • Launches a Wizard to save the opened project as a template.
  • After saving the template you should get a dialog confirming the save.

 

Project Templates are stored under the Template Folder | Projects | "Project Template Name" folder. The flprj file will match the folder name.

 

Test Creating templates to:

  • Local systems
  • UNC paths
  • Mapped Network drives
  • Secondary Drives

 

Test creating templates with spaces, symbols and non-standard characters in their names.

 

Test template files in:

  • Read-only folders
  • Network folders where you do not have permission to write
UI Test Cases:

 

 

 

Test the "Wizardness" of the UI.

  • Go forward and back to make sure buttons do not get activated etc.
  • Make selections go forward then back, make sure the selections are still there.

 

UI Controls

Wizard Page #1: Select template location...

  •  Template Folder: Allows you to select a Template folder to add your project to.
    • Manage Templates... Launches the Manage templates dialog.
  • Template Description: Allows you to type a description for the template project.
  • Template Name: Allows you to Name the template:
    • Names the folder that the template project will be stored under.
    • Names the FLPRJ/BLPRJ to the typed name.
  • < Back: Inactive.
  • Next >: Active. Proceeds to the next wizard page.
  • Finish: Inactive.
  • Cancel: Active. Cancels the operation.

 

Wizard Page #2: Select content files...

  • Displays the Contents folder in the project and allows you to choose what topics and folders to save in the template.
    • Try different selections. You should not be able to make selections that do not make sense like Files selected but the containing folder is not selected.
  • < Back: Active. Allows you to go back to the first page and make changes. Selections on Page #2 should be retained.
  • Next >: Proceeds to the next wizard page.
  • Finish: Inactive.
  • Cancel: Active. Cancels the operation.

 

Wizard Page #3: Select project files...

  • Displays the Project folder in the project and allows you to choose what topics and folders to save in the template.
    • Try different selections. You should not be able to make selections that do not make sense like Files selected but the containing folder is not selected.
  • < Back: Active. Allows you to go back to the first page and make changes. Selections on Page #2 should be retained.
  • Next >: Inactive.
  • Finish: Active. Finishes the operation copying the files and applying all of the settings in the wizard to the selected folder/files.
  • Cancel: Active. Cancels the operation.

 

After saving the template you should get a dialog confirming the save.

Saving Flare Project Files as Templates

You can save any file type to a template folder or you can save an entire project.

Files as Templates

!File | Save As Template....

  • This is only active when the Topic/File is open, not when it is selected in the Content Explorer or Project Organizer
  • After saving the template you should get a dialog confirming the save.

 

Test Creating templates to:

  • Local systems
  • UNC paths
  • Mapped Network drives
  • Secondary Drives

 

Test creating templates with spaces, symbols and non-standard characters in their names.

 

Test template files in:

  • Read-only folders
  • Network folders where you do not have permission to write
UI Test Cases:

UI Controls

Save As Template

  •  Template Folder: Allows you to select a Template folder to add your project to.
    • Manage Templates... Launches the Manage templates dialog.
  • Template Description: Allows you to type a description for the template project (Disabled or enabled based on file type)
  • Template Name: Allows you to Name the template:
    • Names the file and appends the correct suffix to it.
    • Copies it to the correct folder in the template folder directory

After saving the template you should get a dialog confirming the save.
