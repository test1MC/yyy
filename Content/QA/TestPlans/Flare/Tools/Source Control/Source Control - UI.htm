Project Properties Dialog - UI Test Cases

Opening this Dialog:

  • !Project | Project Properties | Source Control Tab

 

 

UI Controls

  • Advanced: Opens the advanced options. You might be able to click the Advanced button to enter more options. This depends on whether the provider has enabled advanced options; if not, the button is disabled. The advanced options available to you are different for each provider.
  • Bind Project: Bind the current project to a Source Control.
  • Unbind Project: Unbind a bound project from its current Source Control provider.
  • Project File: Use this button to perform Source Control tasks on the project file
    • Check In...:Check's in the selected project file into Source Control (only active when you have the project file checked out).
    • Check Out...:Check's out the project file into Source Control.
    • Get Latest Version...:Get's the latest version of the project file from Source Control.
    • Undo Check Out...:Undo's your checkout of the project file (only active if you have it checked out already). This undoes any changes you have made of the local copy that have not been checked in.
    • Show Differences:Shows the differences between your local copy and the newest Source Control copy.
    • View History:You can view the history (change sets) of the project file.
    • Request Check In...:If you discover that another user already has that file checked out, you can automatically send a request to that user to check in the project file.

Right-Click Menus

Source Control Context Menu:

  • !File| Source Control
  • Right-Click a file/folder in the Content Explorer and select the Source Control Menu
  • Click the Source Control button on the standard toolbar

 

 

Right-click menu items:

  • Project:Invokes the Project submenu.
  • Add...: Add's the selected file/folder to source control (only active for new files/folders that have never been checked in).
  • Check In...:Check's in the selected file/folder into Source Control (only active when you have the file/folder checked out).
  • Check Out...:Check's out the selected file/folder into Source Control.
  • Get Latest Version...:Get's the latest version of the selected file/folder from Source Control.
  • Undo Check Out...:Undo's your checkout of the selected file/folder (only active if you have it checked out already). This undoes any changes you have made of the local copy that have not been checked in.
  • Show Differences:Shows the differences between your local copy and the newest Source Control copy.
  • View History:You can view the history (change sets) of a particular source control file.
  • Request Check In...:If you discover that another user already has that file checked out, you can automatically send a request to that user to check in the file.
  • Request View File:If you want another author to review a file and provide comments, this can be done by performing a request to view files.
  • Undelete: Undeletes the selected file/folder (This is for TFS only. See Jay to test this)
  • Refresh: Refreshes the selected file/folder.
  • Pending Check-Ins:Opens the Pending Check-Ins Pane.

Project Submenu:

 

 

Submenu items:

  • Check In All...:Checks in all files/folders you have checked out from Source Control.
  • Check Out All...: Checks out all files/folders from Source Control.
  • Get Latest Version All...:Gets the latest copy of all files/folders from Source Control.
  • Undo Check Out All...:Undo's the check out of all files/folders that you have checked out from Source Control.

Source Control tab in the Flare Options (Pertains to Locking Files and Background Worker)

This tab will only be present if you are working in a project that is bound to Source Control.

UI Controls

  • Automatically check file out from source control when saving file changes: When you save changes to a file that is not checked out, we will check that file out for you.
    • Lock files when automatically checking out(files cannot ve checked in by other users): When a file is automatically checked out of source control such as by editing and saving a topic, we will automatically lock the file (only available in TFS and SVN).
  • Show prompt to add auto-created files to source control: Select this check box to make sure you are prompted when files are automatically created in Flare and may be added to source control.
  • Enable background status checks: Enable the automatic background checks that Flare performs at the specified interval to determine the status of files bound to a source control application (cannot set this value to 0min and 0secs and disabled by default)(Also known as the "background worker" or "pinger".)
