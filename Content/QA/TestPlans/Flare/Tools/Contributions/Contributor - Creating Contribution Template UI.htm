Creating Contribution content from a Flare project - UI

There are 3 types of content Flare users can send to a Contributor user: Review Packages, Contributor Documents, and Contribution Templates. This topic shows the UI for creating Contributor Documents and Contribution Templates.

Creating a Contribution Document from Flare

!File | Save | Save To

 

In the Save As dialogue, select Flare Contribution File (*.mcco) or MadCap Document File (*.mcdoc). These file formats are effectively the same and interchangeable.

 

Creating a Contribution Template from Flare

!File | Save | Save As Contribution Template

 

That will open up the following wizard dialog:

                                   

                                  

 

 

 

Test the "Wizardness" of the UI.

  • Go forward and back to make sure buttons do not get activated etc.
  • Make selections go forward then back, make sure the selections are still there.

 

UI Controls

Wizard Page #1: Select template location...

  • Template Name: Allows you to Name the .mccot file:
  • Select template location: Allows you to select the location the .mccot file should be saved.
    • Names the full path the .mccot file will be stored (by default it will be: "../Documents/My Contribution Templates/").
    • Browse: Allows you to browse to an alternate location
  • < Back: Inactive.
  • Next >: Active. Proceeds to the next wizard page.
  • Finish: Active. Finishes the operation copying the files and applying all of the default settings in the wizard to the selected file.
  • Cancel: Active. Cancels the operation.

 

Wizard Page #2: Select content files...

  • Displays the Contents folder in the project and allows you to choose what content files and folders to save in the template.
    • By default, only files directly referenced from the topic should be pre-selected, but you can select more to include.
    • Try different selections. You should not be able to make selections that do not make sense like Files selected but the containing folder is not selected.
  • < Back: Active. Allows you to go back to the first page and make changes. Selections on Page #2 should be retained.
  • Next >: Proceeds to the next wizard page.
  • Finish: Active. Finishes the operation copying the files and applying all of the settings (default or custom from this screen or the third screen if you hit <Back) in the wizard to the selected file.
  • Cancel: Active. Cancels the operation.

 

Wizard Page #3: Select project files...

  • Displays the Project folder in the project and allows you to choose what project files and folders to save in the template.
    • By default, all project files should be selected, but you can choose to exclude files.
    • Try different selections. You should not be able to make selections that do not make sense like Files selected but the containing folder is not selected.
  • < Back: Active. Allows you to go back to the second page and make changes. Selections on Page #3 should be retained.
  • Next >: Inactive.
  • Finish: Active. Finishes the operation copying the files and applying all of the settings (default or custom from this screen or the second screen) in the wizard to the selected file.
  • Cancel: Active. Cancels the operation.

 

After saving the template you should get a dialog confirming the save and reminding you where the file was saved.

  • In addition the dialog asks: “Would you like to email the Contribution Template now?”
    • 'Yes' will invoke an outgoing Outlook email with the newly created Contribution template file attached, then it will dismiss the dialog and the wizard
    • 'No' will dismiss the dialog and the wizard
