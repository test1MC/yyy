Topic Properties - UI

Opening the UI

  • Content Explorer
    • Right-click a topic in the Content Explorer and select Properties
    • Select a topic in the Content Explorer and press F4
    • Select a topic in the Content Explorer and click the Properties button in the Content Explorer's local toolbar
  • File List
    • Right-click a topic in the File List and select Properties
    • Select a topic in the File List and press F4
    • Select a topic in the File List and click the Properties button in the File List's local toolbar
  • Pending Check-Ins
    • Right-click a topic in the Pending Check-Ins pane and select Properties
    • Select a topic in the Pending Check-Ins pane and press F4
    • Select a topic in the Pending Check-Ins pane and click the Properties button in the Pending Check-Ins pane's local toolbar
  • XML Editor
    • Open a topic in the MDI, give it the focus, and press CTRL+Shift+P
    • Open a topic in the MDI, right-click anywhere in an open topic and select Topic Properties
  • Tools Strip interface:
    • Open a topic in the MDI, give it the focus, and go !File | Properties
    • Select a topic in the File List or Pending Check-Ins pane and go !File | Properties
    • Select a topic in the Content Explorer, File List or Pending Check-Ins pane and click the Properties button in the Standard Toolbar
    • Select a topic in the Content Explorer, File List or Pending Check-Ins pane and go !Edit | Properties
  • Ribbon interface:
    • Open a topic in the MDI, give it the focus, and go !File | Properties
    • Select a topic in the File List or Pending Check-Ins pane and go !File | Properties
    • Select a topic in the Content Explorer, File List or Pending Check-Ins pane and go !Home | Properties

Test Cases

Using File view of the Content Explorer, or using the File List or Pending Check-Ins pane, select multiple topics and open the Properties dialog. In doing so, you should be able to set Properties for multiple topics at once. Test that "(as is)" options exist for all Properties.

(Topic) Properties - UI Test Cases

General tab

UI Controls

  • Name: Displays the name of the topic file. This field is currently (as of V4) read only. In early versions of Flare, you could modify the name in this field, however it would not update links properly, so it was disabled. User bug #26938 may re-enable this field at some point.
  • Location: Displays the location of the topic file.
  • Open Folder: Opens the Windows folder holding the topic file.
  • Read Only: If you do not want the file to be edited, click the check box to add a check mark.
  • Size: Displays the size of the topic file.
  • Created: Displays the date and time that the topic file was created.
  • Modified: Displays the date and time that the topic file was last modified.

Topic Properties tab

UI Controls

  • Topic Title: You can enter a display title for the topic in this field. If no title is entered, Flare will automatically populate the title element with content from the topic's first heading at compile time.
  • Topic Style Class: You can select a topic style class to associate with the topic. All css classes of the html style should appear in the dropdown.
  • Stylesheet: If the project or primary target is not using a Master Stylesheet, select the stylesheet for the topic.
  • Description: Lets the user enter a description for the topic. The description is not visible when viewing the topic, but description content is used when determining search results. A description is also used by external search engine results pages. Additionally, HTML5 output displays the topic description in its search results.
  • Searchable: If this check box displays a check mark, the topic will be included in a search performed by your users in the output. If you remove the check mark, the topic will be excluded from the search, but it will still be included in the generated output.

Snippet Conditions tab

The Include/Exclude expressions that you set in this dialog allow the user to include/exclude conditionally tagged content that exists inside of a snippet that is included in the topic.

 

UI Controls

  • Tags: Displays a list of all Condition Tag Sets in the project.
  • Include: If the "Include" box for a particular tag is selected, any content associated with that tag will be included even if it is excluded by another conditional tag.
  • Exclude: If the "Exclude" box for a particular tag is selected, any content associated with that tag will be excluded.
  • Edit Set: Open the selected Condition Tag Set in the Condition Tag Set Editor.

Language tab

UI Controls

  • Hide languages with no spell check support: Reduces the list of visible languages to those which have spell check support.
  • (Language list): Lets you set the spell check language for the topic. Selecting "(default)" will use the project language.
