File List - UI

Opening the File List

  • Keyboard: Press CTRL + Shift + J
  • Tool Strip interface: !View | File List
  • Ribbon interface: !View | File List

File List - UI Test Cases

Local Toolbar and Columns

 

Local Toolbar Buttons (Tooltips in parentheses):

  • Columns… (Show/Hide Columns): Opens the Columns dialog which lets you customize the column order and whether or not each column is shown.
  • (Open): Opens the selected file.
  • (Run Properties dialog for this item): Opens the Properties dialog for the selected file.
  • (Refresh): Refreshes the list of files in the File List.
  • (Export the data to a CSV file): Invokes a Save As dialog which lets you save the full File List (all columns for all files which are currently displayed) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).
  • (Export the selected data to a CSV file): Invokes a Save As dialog which lets you save the currently selected set of files from the File List (all columns for all files which are currently selected) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).

  • Filter: Lets you filter the visible files in the File List to a particular file type. Verify all files listed are correct and that any new Content file types are added.

File List grid

Columns

  • Name: Shows the name of the file.
  • Path: Shows the location of the content file within the project.
  • Size: Shows the size of the file.
  • Date Modified: Shows the date and time the file was last modified.
  • Status: If the project is bound to source control, this column displays the current status of the content file.
  • User: If the project is bound to source control, and the file is checked out, this column will display the user who has the file checked out.
  • Title: (Topic files only) Shows the Topic Title as set in the Topic Properties dialog.
  • Description: (Topic files only) Shows the Topic Description as set in the Topic Properties dialog.
  • Class: Shows the html class for a topic file.
  • Stylesheet: (Topic files only) Shows the linked stylesheet for a topic file.
  • Conditions: Shows any conditions applied to the file.
  • Language: (Topic files only) Shows the language of the file.
  • Searchable: Shows the "Searchable" status of a topic file as set in the Topic Properties dialog.

Additional Columns

If you create File Tag Sets in your project, a unique column will appear in the File List for each File Tag Set in your project, so there could be several of these.

  • [Tag Type]:[File Tag Set Name]: Displays any File Tags applied to the file for the File Tag Set specified.

 

Right-Click Menu

Right-clicking a file in the File List should produce the following menu:

File List right-click menu (single file):

  • Menu items are correct
  • Spelling and Grammar are correct
  • No duplicate accelerators
  • All accelerators function
  • Keyboard can navigate and select options
  • Menu is easy to use and understand

 

Right-click menu items:

  • Preview: Opens a Preview of the selected content file (Only active for topic files)
  • Open: Opens the selected Content file in the MDI
  • Open with: Gives options to open the Content file in the Internal Text Editor, Internal Browser or another application
  • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan)
  • Delete: Removes the selected Content file from the project
  • Rename: Allows you to rename the Content file
  • Export to CSV: Invokes a Save As dialog which lets you save the full File List (all columns for all files which are currently displayed) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).
  • Export selection to CSV: Invokes a Save As dialog which lets you save the currently selected set of files from the File List (all columns for all files which are currently selected) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).
  • Edit With MadCap Capture: Opens the selected image file in MadCap Capture (Menu option only present for image files)
  • Convert to XML...: (Only active for non-XML topic files) Invokes the HTML -> XHTML Conversion dialog which lets you convert non-XML topic files to XML.
  • Properties: Opens the Properties dialog window for the selected file
  • View Links: Opens the file in the Link Viewer
  • Locate in TOC: Locates the Content file in the TOC
  • Locate in Explorer: Locates the Content file in the Content Explorer

 

 

File List right-click menu (multiple files selected):

  • Menu items are correct
  • Spelling and Grammar are correct
  • No duplicate accelerators
  • All accelerators function
  • Keyboard can navigate and select options
  • Menu is easy to use and understand

 

Right-click menu items:

  • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan)
  • Cut, Copy and Paste: Options should always be disabled (Someday should be removed; see bug 77258).
  • Delete: Removes the selected Content file from the project.
  • Export to CSV: Invokes a Save As dialog which lets you save the full File List (all columns for all files which are currently displayed) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).
  • Export selection to CSV: Invokes a Save As dialog which lets you save the currently selected set of files from the File List (all columns for all files which are currently selected) to a comma delimited text file (*.csv) or a tab delimited text file (*.txt).
  • Convert to XML...: (Only active if all selected files are non-XML topic files) Invokes the Convert Files to XML Wizard which lets you convert non-XML topic files to XML.
  • Properties: Opens the Properties dialog window for the selected files

 

 

Columns dialog - UI Test Cases

UI Controls

  • Move Up: Moves the selected row(s) up in the sequence which in turn moves the corresponding column to the left in the File List.
  • Move Down: Moves the selected row(s) down in the sequence which in turn moves the corresponding column to the right in the File List.
  • Reset to Factory Settings... : Removes all custom column settings and restores factory defaults.

Columns dialog grid

Columns

  • Show: Lets user show/hide the associated column in the File List.
  • Column: Displays the name of each column that can be displayed in the File List.
  • Width: Lets user set the width in pixels of each column in the File List.
  • Color: Lets user open the Color Picker dialog to set the font color for each column in the File List.
  • Background: Lets user open the Color Picker dialog to set a background color for each column in the File List.

Grid Test Cases

  • Columns dialog columns should not be sortable
  • Columns dialog columns should not be resizable
  • Columns dialog columns cannot be moved by clicking on a column heading and dragging
  • Horizontal and vertical scrollbars should be working correctly
  • Columns dialog rows should not be able to be multi-selected with Shift- or CTRL-click
  • If file tag Sets are used in the project, verify that those files appear as rows in the Columns dialog for the File List

Link Update dialog - UI Test Cases

Any renaming or deletion of content files that affects links (either in the affected content files or files that link to them) should open the Link Update dialog.

UI Controls

  • Update Links: Completes the changes and updates all of the links in the affected files.
  • Do Not Update Links: Completes the changes, but does not update all of the links in the files in question. If you select this option, you must manually fix or remove the links in affected files.
  • Cancel: Closes this dialog and does not make the requested changes to the files.

Link Update grid

Columns

  • Writable/Checked Out: A check mark appears in the "Checked Out" box if the project is bound to a source control application and you currently have the file checked out. A check mark appears in the "Writable" box if the project is not bound to a source control application and the file is not marked as "Read Only" in its properties dialog. A check mark in either box indicates that you are allowed to make changes to the file.
  • File: Displays the name of each file containing a link to the file in question.

  • Path: Displays the location of each file in the project.

 

Keyboard Navigation

  • Enter opens the selected file in the correct editor
  • Up Arrow and Down Arrow move the selection to the next displayed node
  • Left Arrow and Right Arrow move the focus to the adjacent cells within a row
  • Page Up and Page Down will move the selection within the File List a page at a time up or down
  • Home or End will move the selection to the very first cell or the very last cell in a row
  • CTRL+Home and CTRL+End will move the selection to the top or bottom of the File List
  • Insert does nothing
  • Delete removes the selected file or folder from the project
  • Shift+Up Arrow expands the selection up one row
  • Shift+Down Arrow expands the selection down one row
  • CTRL+Up Arrow moves the selection to the top of the File List
  • CTRL+Down Arrow moves the selection to the end of the File List
  • CTRL+Shift+Up Arrow expands the selection to include the current file up to the top of the File List
  • Shift+Page Up expands the selection up one page
  • Shift+Page Down expands the selection down one page
  • CTRL+Shift+Down Arrow expands the selection to include the current file to the end of the File List
  • CTRL+A selects all files in the File List
  • Typing (A-Z, 0-9) will scroll to and select the first file that begins with the character you type. Typing multiple characters quickly will scroll further. For example, typing 'M' might scroll to MadCap.htm. If you then type 'I' quickly enough, the file list will scroll to Mimic.htm as opposed to scrolling to a topic that begins with an 'I'.

Drag & Drop

All Drag & Drop functionality out of the File List requires that you click on and drag the file(s) from the Icon column.

Single topic selection into XML Editor

  • Into white space: Launches the Hyperlink Dialog with selected file in the Topic field – Inserts Hyperlink using file name as the text
  • Onto a text selection: Launches the Hyperlink Dialog with selected file in the Topic field – Inserts Hyperlink using selection as the text
  • Test dragging into and onto various elements (Lists, tables, cells, etc…) – the results should be desirable

Multiple topic selection into XML Editor

  • Into white space: Launches the Related Topics Dialog with selected files in the Topic field – Inserts the control
  • Onto a selection: Launches the Related Topics Dialog with selected files in the Topic field – Splits the text and inserts the control
  • Test dragging into and onto various elements (Lists, tables, cells, etc…) – the results should be desirable

Single file selection (Image, Snippet, Echo file) into XML Editor

  • Into white space: Launches the Insert Image/Snippet/Audio Link Dialog with selected file in the linked file field
  • Onto a text selection: Launches the Insert Image/Snippet/Audio Link Dialog with selected file in the linked file field – replaces text selection
  • Test dragging into and onto various elements (Lists, tables, cells, etc…) – the results should be desirable

Single or Multiple topic selection into TOC Editor or Browse Sequence Editor

  • Creates a page node to the selected item – Uses the Topic Title or the First Heading in the topic as the node label

Cut, Copy and Paste

  • There is currently no Cut, Copy or Paste functionality implemented for the File List.
