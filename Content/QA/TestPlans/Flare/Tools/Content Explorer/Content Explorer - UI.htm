Content Explorer - UI

Opening the Content Explorer

The Content Explorer should be open and docked along the left side using Flare's default layout. If the Content Explorer is closed or not currently in focus, it can be reopened doing any of the following:

  • Press CTRL + J on the keyboard
  • Tool Strip interface:
    • !View | Content Explorer
    • Standard Toolbar: With any content file open and in focus, click the Locate in Content Explorer button
  • Ribbon interface:
    • !View | Content Explorer
    • With any content file open and in focus, !Project | Locate in Explorer
  • TOC Editor:
    • Right-click any TOC node that links to a content file (Topic or Image) and select Locate in Explorer
    • Select any TOC node that links to a content file (Topic or Image) and click the Locate in Content Explorer button in the local toolbar

Local Toolbar Buttons (Tooltips in parentheses)

  • (Show/Hide Folders): Toggles between showing or hiding the Tree View in the Content Explorer.
  • (Show/Hide Files):  Toggles between showing and hiding the Files View in the Content Explorer.
    • Drop down allows you to choose to view files as a List or Tiles

 

  • If both options above are enabled, the Content Explorer displays is what is commonly referred to as "Split View." The left half displays folders only. The right half shows folders and files in the currently selected folder with additional details for the files. As you navigate through folders on the right side, the containing folder should highlight in the left half of the Content Explorer.
  • Verify that you can never disable both views. If only one view is enabled (you are not in "Split View"), attempting to disable the current view will disable that view and enable the other view.
  • (Expand All): Expands all of the folders so that you can see all of the files under each one.
  • (Collapse All): Collapses all of the folders so that you can no longer see all of the files under each one.
  • (Hide/show Conditional Indicators): Lets you toggle between showing and hiding conditional indicators in the Content Explorer.
  • (Up): Moves up one folder in the hierarchy in the Content Explorer. This button becomes enabled if you use the Show Folders or Split View and browse into a folder.
  • (Refresh): Refreshes the selected file/folder.
  • (Open): Opens the selected file.
  • (Run Properties dialog for this item): Opens the Properties dialog for the selected file.
  • (New Folder): Creates a new folder
    • If clicked when a folder is selected, the new folder is created inside the selected folder.
    • If clicked when a file is selected, the folder is created in the same location as the file.
  • (Create a new topic file): Launches the Add File dialog.
    • If clicked when a folder is selected in the Content Explorer, in the Add File dialog, confirm that the default Folder is the folder which was selected when the Add Topic button was clicked.
    • If a file was selected when the Add Topic button is clicked, the default Folder should be the folder containing the selected file.

  • Filter: Lets you filter the visible files in the Content Explorer to a particular file type. Verify all files listed are correct and that any new Content file types are added.

Show Files grid

Columns

  • Name: Shows the name of the file or folder.
  • Size: Shows the size of the file (empty for folders).
  • Date Modified: Shows the date and time the file or folder was last modified.
  • Title: Shows the Topic Title as set in the Topic Properties dialog.
  • Class: Shows the html class for a topic file.
  • Stylesheet: Shows the linked stylesheet for a topic file.
  • Conditions: Shows any conditions applied to the file.
  • Description: Shows the Topic description as set in the Topic Properties dialog.
  • Language: Shows the language of a topic file.
  • Searchable: Shows the "Searchable" status of a topic file as set in the Topic Properties dialog.

Right-click menu items:

  • Open Folder in Windows: Opens the selected folder in a Windows browser.
  • Source Control: Invokes the Source Control sub-menu (covered in the Source Control Test Plan).
  • Cut: Cuts the selected folder and all Content files it contains to the clipboard.
  • Copy: Copies the selected folder and all Content files it contains to the clipboard.
  • Paste: Pastes the folder or Content files from the clipboard into the selected folder.
  • Delete: Removes the selected folder and all Content files it contains from the project.
  • Rename: Allows you to rename the folder.
  • Send to Folder… (Send to Folder): Invokes the Send to Folder dialog which lets you copy the file out of the project to any location,
  • New: Invokes a sub-menu which lets you add a new folder or content file inside the selected folder. If you select one the content file types, the Add File dialog will open with the selected folder and file type preselected to the current folder and selected file type.
  • Properties: Opens the Properties dialog window for the selected folder.

Link Update dialog - UI Test Cases

Any reorganization of content files or folders that affects links (either in the affected content files or files that link to them) should open the Link Update dialog. This would include but may not be limited to renaming a file or folder, moving a file or folder or deleting a file or folder.

UI Controls

  • Update Links: Completes the changes and updates all of the links in the affected files.
  • Do Not Update Links: Completes the changes, but does not update all of the links in the files in question. If you select this option, you must manually fix or remove the links in affected files.
  • Cancel: Closes this dialog and does not make the requested changes to the files.

Link Update grid

Columns

  • Writable/Checked Out: A check mark appears in the "Checked Out" box if the project is bound to a source control application and you currently have the file checked out. A check mark appears in the "Writable" box if the project is not bound to a source control application and the file is not marked as "Read Only" in its properties dialog. A check mark in either box indicates that you are allowed to make changes to the file.
  • File: Displays the name of each file containing a link to the file in question.

  • Path: Displays the location of each file in the project.

Keyboard Navigation

  • Enter opens/closes the selected folder to display/hide their contents – opens the selected file in the correct editor. (See bug #12778, same as ever)
  • Up Arrow and Down Arrow move the selection to the next displayed node.
  • Left Arrow and Right Arrowselect and open/close folders will go as far as selecting the first non-folder node in a folder.
  • Page Up and Page Down will move the selection within the Explorer a page at a time up or down.
  • Home or End will move the selection to the very first node or the very last displayed node.
  • Insert does nothing.
  • Delete removes the selected file or folder from the project.
  • Keying the first character of a folder or file will move the selection to that file or folder.

Drag & Drop/Cut, Copy and Paste (within the Content Explorer)

  • You should be able to drag and drop folders and files within the Content Explorer as you would in any Windows Explorer Window. As indicated above, any changes should result in the Link Update dialog opening.
