Annotations/Reviewing

Bug Base Product Area:

  • Annotations/Reviewing

 

Description:

Annotations allow you (or another user) to apply "comments" to selected content within a topic (or snippet, etc). Annotations will most commonly be inserted in Contributor when a Flare topic is sent for review. However, the functionality to insert an annotation is present in Flare to allow the Flare author to add annotations for their own benefit.

Reviewing allows an author to send a topic to another person, who may then open the topic in Contributor. From there, the reviewer may add annotations (and possibly edit content) and send the topic back to the Flare or Blaze author. Annotations added by a reviewer can be accepted or rejected when a topic is returned from a reviewer.

Annotations

Annotations are now edited directly in the XML Editor. Therefore, the Edit Annotations and Annotations window panes have been removed. See Inserting Annotations and Editing Annotations.

 

In addition, annotations now use the same user initials that are part of the new track changes feature. You can change the initials (and user name) from Tools>Options>Review tab. See Changing Your User Name and Initials.

 

Changes Tracked

Tracked Changes are covered in this text plan: Track Changes

Note: Because of the addition of the track changes feature, when you send a topic for review, there is no longer an option to allow reviewers to edit the topic or not. However, you can always lock certain portions of a topic so that they cannot be edited. See Locking Elements.
