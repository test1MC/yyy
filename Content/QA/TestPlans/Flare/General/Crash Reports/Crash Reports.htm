Crash Reports

Bug Base Product Area:

Framework/MDI

 

Description:

When one of our applications encounters an unhandled exception, it will crash. Depending on the type of crash, the application may continue to run, it may continue to run but require a restart to run properly, or the application might be forced to close. In each case, the crash dialog should appear and provide the user the option of reporting the crash to .

 

Test Plans:

Dependencies:

In order to test the Crash Report system, you have to identify a crash. Since we usually try to fix crashes, this may become difficult to test. Search the BugBase for outstanding crash issues that have not been fixed.
