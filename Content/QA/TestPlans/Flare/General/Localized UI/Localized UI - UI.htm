Localized UI - UI

When Flare is first launched, following the product splash image, you should see the Select UI Language dialog. If you have already disabled this dialog from showing on launch, you may need to reset the "Show" option for this dialog. To reset this "Show" option:

  1. In Flare, open the Options dialog (Tool Strip interface: !Tools | Options; Ribbon interface: !File | Options)
  2. Select the General tab
  3. Check the box next to "Show Select UI Language Dialog on Startup" and click OK
  4. Close and reopen Flare

Select Language dialog - UI Test Cases

UI Controls

  • Select User Interface Language Order: From top to bottom, sets the order of priority for UI language. Flare will display all UI in the language at the top of this list. If, for some reason, a string does not exist for the preferred language, the string will appear in the language which appears next on the list.
  • Select: Moves the selected language to the top of the list.
  • Up: Moves the selected language up one spot in the list.
  • Down: Moves the selected language down one spot in the list.
  • Show this dialog on startup: Disable this option to stop Flare from showing this dialog every time Flare is launched.

Test Cases

  • Verify all languages work

  • Verify language priority works. (English will always work since it does not pull from the strings files.)

    Since most strings will exist in all languages, you may need to alter the strings files to test this. For example, in the MadCap Flare install directory, browse to Flare.app\Resources\Strings and find the B3.StartPageKit.mcstrings file. Find string ID number 6. This is the string "Open an Existing Project" which appears on the Start Page. In the mcstrings file, delete the French string for ID number 6. Then launch Flare with French UI. Try using German or Japanese as the second language of priority and verify that the string in the Start Page uses the appropriate backup language and does not always default to English.
