Localized UI - Baseline

Launch Flare in each of the supported Localized UI languages and:

  • Verify all menus items are translated and that they do not appear cutoff, particularly any new or changed menu items.
  • Verify all pane titles appear translated, particularly any new or changed panes.
  • Verify the Start Page is translated and that links in all languages are correct.
  • Verify that you can create a new project and add all file types, particularly any new file types.
    • Create a new project from all sample projects (Getting Started Wizard)
    • Create a new project using the New Project Wizard
    • Create new files with factory templates (verify they are translated if appropriate)
    • Create new files with user templates
  • Verify all HTML pages that are included in Flare appear in the correct language (that files are typically found here: Flare.app\Resources\French (or German or Japanese)). For example:
    • Use !Tools | Launch to launch a MadCap application that is not installed; verify the dialog that opens is in the correct language.
    • Use !Tools | Launch to launch a MadCap application that is installed without a valid license (expired trial period); verify the dialog that opens is in the correct language.
    • Go through the Getting Started Wizard in each language, verify that all descriptions appear in the correct language.
    • View Properties of a TOC node. In the Basic tab, select the option to "Enter External Help System." Verify the message in the dialog appears in the correct language.
  • Carefully test any feature that references a folder external to the Content folder. For example:
    • Test the Review workflow
    • Test the Contribution workflow
    • Test the Backups workflow
    • Test adding a picture from the "My Pictures" folder
  • As much as possible, click on all drop-downs in all UI, particularly new or changed UI, and verify all options are completely visible in the space allotted to the drop-down in all languages.
  • As much as possible, look for consistency in the translation. Especially when viewing new UI, verify that new features are translated consistently throughout the various UI in which that feature is referred to.
  • Open projects, add topics, add content, and build outputs. Look for any text that appears to be missing punctuation or spaces. Many of the strings have an extra space at the end of them so that they can be inserted into another string. If one of these is translated without the extra space, the result can look like this "WordOutput" as opposed to "Word Output."
  • Verify that references to "MadCap Software" and MadCap products are not translated. For example, "MadCap Analyzer" should not be "Analyzeur" in French. Likewise, references to Word and Frame Maker should not be translated.
  • Additional test cases added to the baseline below.

 

Common problems with sting translations that you should look out for:

  • It is often the case that a string translated to another language will be longer than the original English string. That is, it will take up more space. As best as you can, analyze all UI, particularly new UI and look for overlapping or hidden text. Resize dialogs to reveal text that might be hidden.
  • Symbols used in English are often affected by the translation process. For example, the "Next >" button in some of our Wizards uses the greater than character. If you invoke the Build menu, you will find menu items that use quotation marks. Verify these UI and particularly any new UI, appear correctly in all translated language.
