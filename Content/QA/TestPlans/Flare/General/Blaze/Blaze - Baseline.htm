Blaze - Baseline

When testing Blaze, keep in mind that Blaze is limited to print outputs. Any features added to Flare need to be considered in the context of Blaze to ensure they make sense for a print-exclusive application:

 

  • Confirm whether or not a new Flare feature belongs in Blaze

    If the feature DOES NOT belong in Blaze:

    • Verify the UI is not accessible via menu, toolbar or hotkey
    • Verify that references to the new feature in other UI are removed (EXAMPLE: The Target Editor in Blaze should not reference Master Pages)
    • Verify the feature is not included in the documentation (TOC, Index, Search, Glossary)

    If the feature IS going to be included in Blaze:

    • Verify the full set of functionality is Blaze-appropriate ("Web"-only options are excluded).
    • Verify the documentation is Blaze-appropriate (ie, does not use "Web" examples)
    • Verify any template files created to support the feature are Blaze-appropriate (EXAMPLE: The default condition tag set in Blaze is different from Flare's since Flare's references Web output with its "ScreenOnly" tag.)

  • Test all new features fully in Blaze as well as Flare

    • Test Blaze on a machine that does not have Flare installed to confirm any new filetypes register correctly
