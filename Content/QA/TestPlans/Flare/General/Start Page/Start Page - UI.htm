Start Page - UI

Opening the Start Page

The Start Page should be open and docked in the MDI using Flare's default layout. If the Start Page is closed or not currently in focus, it can be reopened or given focus by:

  • !View | Start Page
  • Click the Open Start Page button in the Standard Toolbar

Start Page - UI Test Cases

By default, the Start Page will appear open in the MDI. It is still, essentially, a pane that can be floated and docked anywhere. Right-click the Start Page tab and select Floating to dock it somewhere else and test the following:

Trial Mode

If you launch a product in trial mode, this will be indicated on the start page with Trial banner displaying “You are currently using a Trial Version, contact MadCap Sales for this month’s promotions. -Sales@MadCapSoftware.com | +1 888.623.2271”

  • The color of the Trial banner should match the UI theme
    • Black Theme: Green (Flare), Orange (Analyzer), Green (Contributor)
    • Classic Theme: Blue (All)
    • Silver Theme: Green (Flare), Orange (Analyzer), Green (Contributor)
  • Clicking the Trial banner will open your browser to this page: http://www.madcapsoftware.com/products/flare/

Getting Started

  • Open (Open File): Opens the Open dialog.
  • New Project (Starts the New Project Wizard): Opens the Start New Project Wizard.

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does not resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • This area can be resized by dragging the middle vertical bar

Recent Projects

Recent Projects: Lists recently opened Flare projects. Clicking on any listed project will open that project in the current instance of Flare

Test Cases:

  • Title is correct
  • Spelling and Grammar are correct
  • This area does not resize horizontally when the window is resized
  • This area does not resize vertically when the window is resized
  • This area can be resized by dragging the middle vertical bar
  • Files are listed with the Flare or Blaze icon, “FileName” then “Path”
  • Only Flare or Blaze project files should appear in this list
  • Tooltips with full paths will display when hovering over the links
  • Clicking the link will open the project and turn the Title bar orange for a few seconds.
  • The most recently used project should be listed on top
  • Projects removed from !File | Recent Project | Manage Recent Projects will be removed from the list
  • If a project is clicked that no longer exists, Flare will present an error.

    Note: Bug 86536:We should be more informative in our Start page's "Recent projects/Files" when mousing over a project/file that has been deleted
