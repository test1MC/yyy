Slideshow

Users can now add a slideshow to their HTML5 (WebHelp support TBD) output. Slideshows are contained in <MadCap:slider> tags which contain slides represented with <MadCap:slide> tags. Any content within the slide tag will be converted into a slide. By default, the slider contains a ‘pager’ (dots representing each slide) and navigation controls (prev/next slide). Users have the ability to customize these controls, along with the slider’s size, transitions, auto play, among others.

 

To insert a slide, go to Insert->Slideshow(location in the ribbon may change). -New page in Insert/Edit Slideshow dialog (‘Slides’) allows users to rearrange the order of the slide, insert new blank slides, and remove slides. This page also includes a preview of the slide’s content. At the bottom, users can set a caption and/or thumbnail to a slide. (To enable thumbnails, go to the ‘Advanced’ page and check ‘Thumbnails’.)

 

XML Editor: Users can only view one slide at a time. To view a different slide, users can use the Next/Previous slide controls (also as menu items when right-clicking on the ‘MadCap:slideshow’ block) or chose the slide’s block on the left. Adding and removing a slide can be done through the slide controls (also as menu items when right-clicking on the ‘MadCap:slideshow’ block). Slides can also be rearranged using the blocks.

 

The slideshow source was obtained from here: http://bxslider.com/

WebHelp Toolbar and Navigation Pane scrolling styles (Standard Skin)

Section 508 § 1194.22 (d) states "Documents shall be organized so they are readable without requiring an associated style sheet."

 

What this means is that some users will view our output with all CSS disabled. In order to be 508 compliant, our output must be usable when CSS is disabled. This has been problematic in WebHelp because the frame containing the WebHelp toolbar, when stripped of all CSS, may not be not fully visible, and no scrollbar is provided because it was explicitly set not to display.

 

Now, users requiring WebHelp output to be 508 compliant can set scrolling values for the toolbar and navigation frames when stylesheets are disabled so all content can be accessed.

 

There are 3 options:

  • Auto – Scrollbars appear if needed
  • Yes – Scrollbars are always shown
  • No – Scrollbars are never shown

 

Note: This is ONLY takes effect when stylesheets are disabled in the web browser.

 

This can be found under:

  • Skin | Styles | Frame
  • Skin | Styles | Frame | Toolbar

Eclipse Output

Eclipse Help is an embedded format that is exclusively used within the Eclipse framework, requiring either a local or remote install of the Eclipse Help plug-in in order to host the content. This type of help is particularly useful for creating dynamic, searchable help content for Eclipse Workbench plug-ins.

Following are some of the key features of the Eclipse Help format.

  • WebHelp features: This format lets you take advantage of several of the features and benefits available with the WebHelp output type.
  • Dynamic Help: This format allows content developers to provide context help that updates automatically in the Eclipse Dynamic Help view within the Eclipse IDE.
  • Help Index: This format will publish your glossary terms to the searchable index provided by the Eclipse Help system.
  • Local output: This format is also a good choice if you want users to store and open the output locally, rather than from a remote Infocenter server.
  • Single file: This format gives the content developer the option to archive output to a single file (.JAR).

Eclipse Help is viewed utilizing an embedded Jetty server instance that ships with the Eclipse IDE, making it possible for the help developer to either utilize the local Eclipse install as a viewer, or to set up dedicated local or remote help server.

The Eclipse Help output consists of several XML files necessary to support hosting the Eclipse Help content via the Eclipse plug-in loader, using help extension plug-ins for Table of Contents, Glossaries and Context Sensitive Help as necessary. Locally installed content is typically distributed via Eclipse software "sites", which allow the user to download software through the "Install New Software" item in the Eclipse Help menu. The resulting output can either be distributed as-is, or bundled side-by-side with associated software plug-ins.

Eclipse Target: Eclipse Options Tab

 

Major Features NOT supported by Eclipse Output

  • Anything related to Search. Eclipse handles it's own search functionality.
    • Search Filters are not supported in Eclipse output.
    • Synonym Sets are not supported in Eclipse output.
    • "Exclude from Search" topic property is not supported in Eclipse Help
    • Additionally, all search options that exist in the Performance tab of a WebHelp target do not apply to Eclipse output.
  • All "Performance" options (TOC chunking, index chunking, etc) traditionally used for WebHelp and HTML5
  • Index Link Sets
  • Skins
  • Browse Sequences (there is no BS view in the output, but you could use a TOC which links to a BS, but in this case, the BS entries would become part of the TOC)
  • Topic Toolbar Proxy (see bug 78926)
  • Site Maps
  • Feedback/Pulse
  • CSH is supported (you can associate an Alias file), but the CSH Tester is currently not supported (see bug 79093)

Project Export

Flare can now create new project files from an existing project using file tags, file tag expressions, conditions, conditional expressions or from a target file.

 

Export using Conditions

Exporting using conditions/conditional expressions will create a new project with all the files tagged with the selected condition or that are included in the conditional expression. All files referenced by the tagged content will also be copied over. Additionally, conditioned content in topic files will be included or removed based on the selected condition/conditional expression. A default target, toc, stylesheet, and pagelayout file will be added to the exported project if they are not present in the new project.

 

Export using File Tags

Exporting using file tags/file tag expressions will create a new project with all the files tagged with the selected file tag or that are included in the file tag expression. All files referenced by the tagged content will be copied over. A default target, toc, stylesheet, and pagelayout file will be added to the exported project if they are not present in the new project.

 

Export using Target

Exporting using a target file will attempt to create a new project that mirrors the output when building the selected target. There are options to convert snippets and variables to text in the referenced topics and toc file.

 

Export using Select Files

This option will allow users to select which files from the project to export. You will get two dialogs, one for Content explorer and one for Project Explorer.

 

Export to project template

Will create a Project Template for template use.

 

Export from Madbuild

The following command option will run the project exporter from madbuild:

madbuild -export <export project file>

 

<export project file> - the location of an export project file

 

The following error codes have been added to Madbuild:

3001 – The entered export project file does not exist

3002 – There was an error parsing the entered export project file

3003 – Export failed

 

Save Export Settings

You can save your settings of any export you do.This will create a file in the Project/Export folder.You can re-run these exports by right-clicking them and choosing Run Export Project File. You can open this file up and make changes to what the export will accomplish in the main editor.

Custom List Formats and CSS Support

Custom List Formats

Users can now add custom list formats through the Flare UI. List dropdown menus now include an option for a “Custom List Format” which will prompt a dialog where the user can create a custom list format.

 

Select Create Custom Format to invoke the Define New List Format dialog to create your own list formats.

 

Define New List Format dialog

Enter a class name for your new custom list format. Use the list style type drop-down to select the counter type (only one counter allowed per list format). The format will display in the Content field as it does for auto number formats (brackets around n, r, R, a, A). The ‘Add Character’ and ‘Edit Font’ buttons will invoke the previously known ‘Insert Character’ and ‘Edit Font’ dialogs, respectively allowing you to add characters and formatting to the list format.

Clicking OK in this dialog creates a new ol or ul style class with the list-style-type set as well as a complex selector for the list items, and adds them to the selected stylesheet. It then applies the class to the newly created list.

 

Once a Custom List Format is created, it will display in the Custom Format flyout menu underneath the Create Custom Format option, so you can easily reuse your defined custom list formats.

Crash Report System

All of the products will now have the ability to create a crash report when an unhandled exception crashes the application. Users can then send that report to our new product exception web service or can save the file and send it to customer support. When a user encounters a crash, they will see the Unhandled Exception dialog.

"Needs to close" variant:

"Other actions" invokes a drop-down which lets you select the following options:

  • Provide additional information: Invokes the User information dialog which lets users add personal information and a description of what they were doing when the exception occurred.
  • Report Details: Invokes the View Exception Log dialog which lets the user see what is being sent.
  • Save Log: Invokes the Save Exception Log dialog which lets you save the report locally as an XML file.

"No force close" variant:

  

 

Selecting "Provide additional information" will open the User Information dialog:

  • First Name: Enter your first name
  • Last Name: Enter your last name
  • Email: Enter your email address
  • Description: Enter up to 1,024 characters to describe what was happening when the exception occurred

Selecting "Report Details" opens the View Exception Log dialog so you can see what will be sent to MadCap Software.

 

Clicking "Send Report" will send the report and provide a confirmation message and then continue to crash.

NOTE: If you receive the confirmation that the log was successfully submitted to MadCap, all is well. According to Marcus if you are able to figure out a way to create an exception in the SQL table, you will get an alternate message that says the report failed to submit and a log file will automatically be saved to the users machine. Marcus says "Good luck" trying to make that happen.

Clicking "Don't send" will simply dismiss the dialog and continue to crash.

Hanging Punctuation

We've added support for "hanging-punctuation" in our Stylesheet Editor. At this time, it is only available in the Advanced View. Hanging punctuation allows you to have certain punctuation symbols "hang" outside the block element which allows the text to align.

 

Property values for hanging-punctuation are:

  • none: no character hangs
  • first: An opening bracket or quote at the start of the first formatted line of an element hangs. This applies to all characters in the Unicode categories Ps, Pf, Pi.
  • last: A closing bracket or quote at the end of the last formatted line of an element hangs. This applies to all characters in the Unicode categories Pe, Pf, Pi.
  • force-end: A stop or comma at the end of a line hangs.
  • allow-end: A stop or comma at the end of a line hangs if it does not otherwise fit prior to justification.
  • inherit: Inherits the parent element's property

 

It is significant to note that the "first" and "last" property values apply to one specific set of punctuation symbols and "force-end" and "allow-end" apply to a separate set. Therefore, combinations of these properties are also allowed (excluding "none" and "inherit"; and "force-end" and "allow-end" would not be used in tandem). Valid combinations would be: "first last," "first force-end," "first allow-end," "first force-end last," "first allow-end last," "force-end last," and "allow-end last."

 

For full information and a list of stop and comma characters, see http://www.w3.org/TR/css3-text/#hanging-punctuation

 

Examples

"first" and "last" allow characters such as brackets, parentheses and quotations to hang outside the line box.

 

"force-end" and "allow-end" allow characters such as periods and commas to hang outside the line box ("allow-end" works in conjunction with text that is justified).

Customer Date/Time Variables

Users can now create their own DateTime variable with any desired format. Strings that follow the ‘Custom Date and Time Format Strings’ (specified here: http://msdn.microsoft.com/en-us/library/8kb3ddd4.aspx) are translated to the appropriate value.

 

To create this variable, open a Variable Set, click on the new button in the local toolbar to Add DateTime Variable. DateTime variables can also be given alternate definition like standard variables (for example, you could provide different date formats for American and European audiences). Additionally, DateTime variable formats can be overridden in the target. When overriding a DateTime variable in the target, the target should also invoke the Edit Format of [name] dialog.

Multiple Variable Definitions

Variables may now have multiple definitions. To add a definition to a variable, the user must click the variable row within the Variable Set Editor and either click the ‘add definition’ button on the tool strip bar or the right-click menu. When a variable has multiple definitions, the row cells of the non-default definitions will be grayed out (except for the definition cell). The default definition is determined by the order of creation, with the oldest definition being the default. Within the Insert Variable dialog, the default definition is displayed in the Definitions column. In the Variables tab within the Target Editor, users can choose from a list of the previously added definitions or type in another definition value, when determining what value to be displayed for the variable within the target.
