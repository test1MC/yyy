Finished Scripts

 

Script Name Function
Autobake When the path to a baseline file is input to the script the script will download each bug folder from DC2, build it using MadBuild and record the time it takes to build the project along with the number of files and the file size. All of the metrics will be output to a date stamped csv file so that they can be analyzed in excel.
Flare 8 vs Flare 9 Will do the same as above but the script will repeat the process with Flare 9. All of the data will be output to two separate csv files so they can be imported into excel and trends can be established.
Mimic Object tester Will create a user specified number of objects with trajectories per frame (i.e. slide 1 will have 1 object, slide 24 will have 24 objects). The script will also record memory information after creating a new frame so memory consumption of Mimc can be monitored.
Mimic Live recording Will create a user specified number of clicks while Mimc is in live recording mode and sample the amount of memory that Mimic uses.
Topic Creation This script will hand write a Flare project with a user specified number of topics and a HTML 5 output. Each topic has a keyword and a concept term in it
WinDiff patch Will install a version of a product, copy the .app folder, remove the product install a release version of the product and the update patch and then compare the old and new .app folders using windiff
Bad Email parallel This script will perform a few sanity checks on Flare. It will de-activate flare, get rid of the registry entries and other items, re-register it with a mixed case email address, check to see if it crashes and then do a batch build and a scheduled build
Build Log Comparison This script will read the build log off of build2 and then compare the items mentioned in the log to items in the Flare Folder
Commenting Chrome This script will generate comments on a pulse enabled help system. It will also up vote / favorite things, add pictures and other actions
Dll printout This script will ask the user for a major version, minor version, revision, copyright date and company and then it will go through all of the dll’s in the .app folder and will print out a list of ones that are wrong
Install testing quick checklist This script will automate ~66% of the Quick checklist for Flare and Lingo
JS Version Checker This script will open up the build log, read the build log and look for files with .js extensions to add to a list. Once its finished reading the file the script will then open up each mentioned .js file and output the version number to a .csv file
Memstats This program will run in the background over night and collect memory samples at a user defined rate. It will also output this information to a csv file for later analysis
Uninstall_Install This program will uninstall the version of flare currently installed on the local machine and it will the install and register the latest version on \\Build2
Dot Net Issue

This script will simulate a users working on and switching between three separate projects.The program will also log each action and monitor to see if flare crashes during an action. It will randomly choose from the following actions:

  • Wait
  • Switch Ribbons
  • Switch Projects
  • Minimize Flare
  • Maximize Flare
  • Move Flare
  • Spam Characters
  • Close and Reopen Flare
  • Undock and Redock a pane
  • Add a Table
  • Switch from print to xml layout
  • Create a table with ~150 rows and content in some of the cells
Smoke Project

This program will

  • Uninstall Flare
  • Download a new build of Flare
  • Register Flare
  • Creates a new project
  • Adds Each type of file type and makes a slight modification to it
  • Exercises most of the actions available in the Home, Insert and Table ribbon. Each item gets its own topic
  • Reads the raw xml of each topic and compares it to a known good topic
